本文只用于设置页链路闭环测试，不用于主页 `cc.ngc` 运动黄金链路。主页测试另按 `待做工作/测试主页.md` 执行；全自动流程只能在设置页单步闭合和主页单步闭合后再运行。

## 当前代码事实

以 VM 源头 `/root/Desktop/v5` 当前代码为准：

- 设置页 UI 在 `app/src/v5_settings_page.c`，按钮固定为 `登记本机码`、`下载授权`、`服务器下载`、`扫描从站`、`复位驱动`、`读取驱动`、`清除故障`、`设置驱动`，右上角按钮为两态：进入设置页且无参数变更时显示 `返回主页面`，任意设置页参数成功更改后切换为 `保存并重启`。
- 设置页按钮通过 `app/src/v5_settings_actions.c` 请求 `/run/8ax_v5_product_ui/settings_actiond.sock`，常驻服务是 `services/drive_profile/v5_settings_actiond.py`。
- 授权和下载动作已接入 `services/auth_download/`：`device_dna_register`、`device_authorization_download`、`drive_profile_server_download`。
- 驱动动作已接入 `services/drive_profile/v5_drive_bus_action.py`：`scan` 和 `read` 有真实实现；`factory-reset`、`fault-reset`、`set-drive` 当前明确 fail-closed，返回未接入 canonical 写入/读回 gate，且 `write_executed=false`、`motion_executed=false`。
- 轴表和 G53 编辑在 `app/src/v5_settings_axis_table.c`，当前是软键盘/下拉控件写 `config/settings/*.tsv` 后读回；这只能证明字段本地写入和 UI 回读，不能证明 BUS count-domain 设 0、Pulse HOME_OFFSET、驱动写入或运动闭环。
- 右上角 `返回主页面` 只证明本地页面切换；参数更改后出现的 `保存并重启` 当前是 UI local handoff，尚不能单独证明后端重启、LinuxCNC/native 运行态已重新装载。

因此本文的通过口径必须分级：字段编辑可做 `local_verified_only`；授权/下载/扫描/读取可做板端 operator path 单步验证；`复位驱动`、`清除故障`、`设置驱动` 当前预期是 fail-closed 阻塞，不能写成通过步骤。

## 操作顺序

### A. 身份与下载链

1. 如果顶部 `本机码` 不是合法 6 位已登记 ID，先执行 `登记本机码`。
2. 执行 `下载授权`。
3. 执行 `服务器下载`。
4. 确认 `drive_profile_resident_snapshot.json` 已生成或启动阶段已加载 profile 快照；若没有，先按服务启动/受控刷新链路修复，不直接跳到驱动动作。

通过条件：

- `登记本机码` 成功时，顶部只显示 6 位 `vpsDistributionId`，不追加 `已登记` 状态后缀，不显示 raw DNA、DNA hash 或 id-dna folder；缺少合法 ID 时同一位置显示 `未登记`。
- `下载授权` 结果必须是 `DEVICE_AUTH_DOWNLOAD_OK`，并证明 `/etc/6x-cnc/device_authorization.json` 已写入且通过 `/etc/6x-cnc/device_auth_public.pem` 验签。
- `服务器下载` 必须证明 public/private profile 下载、授权校验、hash 校验和板端缓存写入结果；private 缺失只能记录 absent/skipped reason，不能写成 private 成功。
- 弹窗最终正文必须是 `提示 / 原因 / 下一步` 三项；成功/失败结果保留到操作员点击关闭。

### B. 驱动只读链

5. 执行 `扫描从站`。
6. 执行 `读取驱动`。

通过条件：

- `扫描从站` 结果为 `DRIVE_SCAN_OK`，`slaves` 非空，并写入硬盘自建参数表 `config/settings/self_parameter_table.tsv` 的 `SETTINGS/slave_options`。
- UI 轴表 `从站` 下拉候选必须在打开下拉前重读硬盘自建参数表的 `SETTINGS/slave_options`；候选显示为 `轴号 驱动名型号`，例如 `0 SV630_1Axis_03713`，提交写入 self 和 microkernel 参数表时仍只能写纯轴号；不能用固定默认列表、`/run` 结果、旧缓存、轴当前分配行或驱动映射表伪造候选。
- 默认 X/Y/Z/A/B/C/龙门/刀库的 `从站` 列必须是 `0-7`，不得从 `10` 开始。
- 扫描只刷新 `从站` 下拉候选，不自动改写现有轴-从站绑定；只有用户在轴参数表 `从站` 下拉中提交选择后，才写入该轴的 self 和 microkernel 参数表。
- `扫描从站` 成功弹窗仍保持 `提示 / 原因 / 下一步` 三项；`原因` 中必须显示本次扫描驱动列表，固定两列四行，每格带从站编号和驱动名，少于 8 个用 `--` 占位。
- `读取驱动` 结果为 `DRIVE_READ_OK` 才能继续声明驱动只读链通过；如果是 `DRIVE_READ_PARTIAL`，必须记录缺失的从站、profile 或必读 SDO 项后停止。
- 读取动作必须选择 resident snapshot 中匹配真实从站身份的 profile，并记录 `selected_map_sha256`、`map_source` 和必读命令读回。

### C. 驱动写入链

7. 执行 `复位驱动`。
8. 执行 `清除故障`。
9. 执行 `设置驱动`。

当前代码预期：

- 这三步当前不是通过项，而是 fail-closed 检查项。
- 预期结果分别是 `DRIVE_RESET_PRECHECK_FAILED`、`DRIVE_FAULT_RESET_PARTIAL`、`DRIVE_SET_PARTIAL` 或同族 canonical write gate 缺失错误。
- 结果必须包含 `write_executed=false`、`motion_executed=false`、`failed_stage=canonical_write_gate` 或等价字段。

关闭规则：

- 如果上述动作意外返回 `ok=true`，必须立即停止并审查代码；当前源码没有 canonical 写入/读回 gate，不允许把意外成功当作通过。
- 在这三步未实现真实写入/读回前，设置页完整黄金链状态只能是 `blocked_by_drive_write_gate_missing`，不能继续声明设置驱动闭环通过。

### D. 字段编辑局部链

10. 逐类测试一个代表字段：轴参数普通输入、下拉字段、`bit` 字段、G53 可编辑字段。
11. 每个字段点击前抓屏，点击后确认软键盘或下拉出现；提交后读取 `config/settings/*.tsv` 和 UI 单元格回读。

当前代码字段范围：

- 下拉字段：`轴模式`、`方向`、`从站`、`回零次序`、`回零方向`、`bit`。
- 普通输入字段：`目标精度`、`螺距`、`电机`、`负载`、软限位、`零位`、速度、加速、反向间隙。
- G53 中 `A中心 X`、`B中心 Y`、`C中心 Z` 为 WCS/native 只读；其它 G53 单元格可编辑。
- `A中心 Y/Z`、`B中心 X/Z`、`C中心 X/Y` 的编辑只能证明 `microkernel_parameter_table.tsv` pending/持久请求写入；UI 单元格必须保持 runtime INI/native readback，不能被硬盘表覆盖。只有受控重载或运行态写入后 native readback 与请求值一致，才能声明 G53 native 几何写入成功。

通过条件：

- 软件盘必须是当前 LVGL 自绘键盘，提交后立刻写对应 TSV 并读回一致。
- `从站` 写入必须同时进入 self 和 microkernel 参数表；`bit` 写入必须同时进入 drive 和 microkernel 参数表。
- 其它字段按 `v5_settings_axis_table.c` 的当前 owner 写入 self、drive 或 microkernel TSV，并读回一致；G53 native geometry 例外，硬盘 TSV 读回只算 pending 持久化证据，不算 UI actual/readback 成功。
- 事件日志必须出现 `settings_axis_param_commit`，且 `ok=true`。

限制：

- 字段编辑通过只说明本地参数表读写和 UI 回读通过，不说明 LinuxCNC/native runtime 已生效。
- G53 native geometry 字段编辑的局部状态只能写 `pending_persisted_local_verified`；未取得受控重载后的 native/runtime readback 前，不得写成 `local readback passed`、`native 生效` 或 `board_verified`。
- 当前 `零位` 单元格不是旧 `settings_axis_zero` 产品动作，不得用它证明 BUS 设 0、raw 限位同次写入、Pulse HOME_OFFSET、A/C count-domain 或任何运动安全闭环。

### E. 返回主页面 / 保存并重启

12. 进入设置页后先确认右上角显示 `返回主页面`；完成一个字段有效提交后，确认右上角切换为 `保存并重启`，再点击 `保存并重启`。

当前代码通过条件：

- 未改参数时点击 `返回主页面` 只执行页面切换，不得触发 settings apply 或重启 handoff。
- 参数更改后按钮必须显示 `保存并重启`；点击后 UI 事件显示对应 save/restart handoff，状态文本显示已提交重启/重载 handoff。
- `保存并重启` 这一步只能记录为当前 UI local handoff 已触发。

限制：

- 不得把这一步写成后端已重启、LinuxCNC/native 已重新装载或驱动 Operation Enabled 已恢复。
- 要声明重启闭环通过，必须另有受控 backend/UI restart 证据、重启后重新进入主页面证据、参数 owner readback 证据和必要的 native/runtime 回读。

## 弹窗与反应时间

- 最终弹窗正文固定三项：`提示:`、`原因:`、`下一步:`。不得新增 `详情`、`操作轴`、`现状` 等第四项。
- 成功结果使用绿色文字，失败/阻塞结果使用红色文字。
- 软键盘出现和数字输入的目标反应时间仍按 `0.2s` 检查；字段提交到 TSV 写入/读回的目标反应时间按 `0.5s` 检查。
- 长流程按钮按下后 `0.2s` 内必须出现进度窗口或明确已提交反馈，随后必须持续显示阶段或等待状态。

## 停步规则

- 页面不对、按钮不可见、目标被遮挡、截图不清楚时，不点击。
- 点击未命中、事件日志无对应动作、settings_actiond 未接收、结果文件未更新或弹窗长期无阶段推进时，不继续下一步。
- 授权/DNA、服务器下载、profile resident snapshot、从站扫描或读取驱动任一失败，必须先修该单点。
- `复位驱动`、`清除故障`、`设置驱动` 当前 fail-closed 是已知实现缺口；严格黄金链在这里停止并记录 blocker。
- 任何步骤不得用 SHM 显示投影、旧 `/run` 结果、UI dirty、本地默认值或旧 JSON 作为参数保存、驱动写入、运动安全或授权成功真源。
