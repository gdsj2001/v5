# 参数归类

引用需求真源：`REQ-PARAM-MEMORY-LIGHTWEIGHT-SAVE`、`REQ-NATIVE-OWNER-FIRST`、`REQ-SETTINGS-RUNTIME-DRIVE-ONLY`、`REQ-RTCP-G53-NATIVE-ACTUAL`、`REQ-BUS-PULSE-MODE-DIFFERENCE`、`REQ-SCALE-CHAIN-HISTORY-POLLUTION`。

## 归类原则

设置页参数只按三类 owner 归类：

1. 微内核参数：LinuxCNC runtime INI、HAL/native、kinematics/native、WCS/native、已登记 native memory/API。凡是微内核已有或应该下沉到微内核的参数，必须使用微内核/native readback，不得进自建参数表。
2. 自建参数：微内核参数和驱动映射表参数都没有承载的设置页参数。自建参数必须开机进入 RAM，UI 自动读取，不得显示 `NAT`、空占位、UI 默认值、旧 JSON、`/run` result 或 SHM 投影。
3. 驱动映射表参数：驱动参数映射表/drive provider，只服务驱动私域对象字典、SDO/PDO、驱动 profile、驱动读写证据和授权下载链路。驱动映射表参数不能反向成为 LinuxCNC 运动参数、坐标基、RTCP/G53、WCS/G92、刀补、模态或安全真源。

读取来源和写入目标必须分开记录：读取来源决定 UI 初始显示和默认值来源；写入目标必须落到对应硬盘持久参数表，并且可以同时包含微内核运行态 owner。双写不得形成双真值，写入成功必须以全部硬盘目标写入成功、受控重载/运行态写入成功和微内核/native readback 一致为准；任一目标失败时 `写入状态` 不能显示成功。

UI 中任何可修改参数控件发生改变时，必须立即写入该字段对应的硬盘参数表，并从同一 owner/readback 链路读回后刷新 UI 单元格；不得只修改 LVGL 控件文本、RAM 临时变量或屏幕显示。只要硬盘 owner 写入、受控重载或 readback 不一致，就必须保持/刷新为 owner 读回值并把 `写入状态` 标为未成功。

设置页右上角按钮文案和语义为 `保存并重启`，不得再显示为 `返回`。该按钮只用于把当前已变更参数的硬盘参数表落盘状态确认后执行受控重启/重载；不得把未写入硬盘参数表的 UI 临时值当成已保存参数。

设置页参数 UI 的视觉布局必须直接继承 v3 设置页源码坐标，不得另造一套布局。v3 基准坐标为：顶部三块从 `y=42` 开始，G53 标题 `264,55`，G53 表头 `X/Y/Z=395/479/563,y=86`，G53 行从 `y=109` 起按 28px 行距排列，单元格为 `366/450/534,y,76x24`；驱动动作按钮行为 `x=410/494/.../914,y=236`；轴参数大面板为 `21,225,982,343`，轴表标题 `y=230`，横向滚动区 `95,274,908,300`，轴名行从 `y=318` 起按 31px 行距排列。右上机械坐标小面板必须按 v3 `create_settings_machine_coordinate_widget(page, 696, 42)`：外层仍保留 `696,42,312` 背景，内部坐标面板为 `696,42,216`，只显示 X/Y/Z/A/C 五个机械坐标和 `MCS`，数值区保持 v3 固定宽度/数码字视觉；龙门、刀库是下方轴参数表第 7/8 轴，不得塞进右上机械坐标小面板形成自造 8 行普通 label 表。新增 `B中心` 行必须按 v3 28px 行距插在 `A中心` 与 `C中心` 中间，不能压缩成非 v3 行距、不能改成 `20px` 高度、不能隐藏表头或插入另一套自造表格；为容纳第 5 行，顶部三块面板高度按同一增量加高，驱动动作按钮行、轴参数大面板、轴表标题、横向滚动区、轴名行和轴参数行按同一增量整体下移，仍保持 v3 的相对距离、颜色、尺寸和行距，并保证底部第 8 轴不重叠屏幕边界。轴参数标题文案沿用 v3 `轴参数(总线)`，不得把当前新增轴数写成标题视觉元素。轴参数表新增 `方向` 列时，只允许压缩 v3 原 `轴模式`/`从站` 段内部宽度来插入，`目标精度` 及其右侧所有列必须继续使用 v3 原始 x 坐标锚点（`272/390/472/...`），不得因为新增列把后续字段整体右移。

轴参数表控件类型固定：`轴模式`、`方向`、`从站`、`回零次序`、`回零方向`、`bit` 使用下拉；`分子`、`分母`、`写入状态` 为自动计算/动作结果只读显示；除此之外的轴参数单元格都必须点击弹出软件盘输入。软件盘必须保持 v3 设置页同款数字键盘布局和视觉风格，用 LVGL 面板/按键/标签自绘实现，不得用 LVGL 默认 `lv_keyboard` 成品键盘替代。软件盘确认后立即写入该字段对应硬盘参数表，并按 owner/readback 回读刷新。

G53 机床坐标目标位置表中，只有图示红线 owner 格 `A中心 X`、`B中心 Y`、`C中心 Z` 保持对应 WCS/native owner 只读，不从设置页软件盘写入。除此之外的 G53 区单元格，包括 `A中心 Y/Z`、`B中心 X/Z`、`C中心 X/Y`、`对刀仪 X/Y/Z`、`5方向监测仪 X/Y/Z`，点击必须弹出同一套 v3 样式 LVGL 自绘软件盘输入；确认后按字段归属写入对应硬盘参数表并回读刷新。

## 微内核参数

| 设置页位置 | UI 参数 | 微内核来源 | 读取要求 |
| --- | --- | --- | --- |
| 顶部 | 运动模型 | kinematics/native 或 runtime INI `[RTCP]` | 例如 AC 摇篮；不得进自建参数。 |
| G53 机床坐标目标位置表 | A 中心 X | 当前模态加工坐标系 X 偏移值，WCS/native owner readback | 不是固定 G53 几何常量；不得写入自建参数。 |
| G53 机床坐标目标位置表 | A 中心 Y | `G53_A_Y`，native geometry/runtime INI readback | 从微内核/native 读取；不得用 UI 默认值补。 |
| G53 机床坐标目标位置表 | A 中心 Z | `G53_A_Z`，native geometry/runtime INI readback | 从微内核/native 读取；不得用 UI 默认值补。 |
| G53 机床坐标目标位置表 | B 中心 X | `G53_B_X` 或等价 G53/native geometry readback | B 中心位于 A/C 中间，X 是 G53 几何坐标；不得写入自建参数。 |
| G53 机床坐标目标位置表 | B 中心 Y | 当前模态加工坐标系 Y 偏移值，WCS/native owner readback | B 中心位于 A/C 中间，Y 不是固定 G53 几何常量；不得写入自建参数。 |
| G53 机床坐标目标位置表 | B 中心 Z | `G53_B_Z` 或等价 G53/native geometry readback | B 中心位于 A/C 中间，Z 是 G53 几何坐标；不得写入自建参数。 |
| G53 机床坐标目标位置表 | C 中心 X | `G53_C_X`，native geometry/runtime INI readback | 从微内核/native 读取；不得用 UI 默认值补。 |
| G53 机床坐标目标位置表 | C 中心 Y | `G53_C_Y`，native geometry/runtime INI readback | 从微内核/native 读取；不得用 UI 默认值补。 |
| G53 机床坐标目标位置表 | C 中心 Z | 当前模态加工坐标系 Z 偏移值，WCS/native owner readback | 不是固定 G53 几何常量；不得写入自建参数。 |
| 机械坐标显示 | X/Y/Z/A/B/C/龙门/刀库 MCS | native 运行态坐标/显示投影链路 | 运行态坐标显示来自微内核/native，不属于自建参数。 |

| 轴参数表 | 目标精度 | runtime INI/native `SCALE` 相关 readback | 不能用 UI 默认值补。 |
| 轴参数表 | 螺距/每转角度 | runtime INI/native 比例链 readback | 影响 `SCALE` 和电子齿轮计算。 |

| 轴参数表 | 回零次序 | LinuxCNC homing/native owner | 从微内核/native 读取。 |
| 轴参数表 | 回零方向 | LinuxCNC homing/native owner | 从微内核/native 读取。 |
| 轴参数表 | -软限位 | runtime INI/native limit owner | 从微内核/native 读取。 |

| 轴参数表 | 速度 | runtime INI/native motion owner | 从微内核/native 读取。 |

## 自建参数

| 设置页位置 | UI 参数 | 自建参数用途 | 读取要求 |
| --- | --- | --- | --- |
| 顶部 | 脉冲/总线 | 当前机型设置项 | UI 从自建参数读取；当前默认显示为 `总线`，不用硬件探测、驱动映射表或默认文案补值。 |
| G53 机床坐标目标位置表 | 对刀仪 X/Y/Z | 设置页测量/标定显示参数 | 微内核和驱动映射表没有该设置页参数，放入自建参数并由 UI 读取。 |
| G53 机床坐标目标位置表 | 5 方向检测仪 X/Y/Z | 设置页测量/标定显示参数 | 微内核和驱动映射表没有该设置页参数，放入自建参数并由 UI 读取。 |
| 轴参数表 | 从站 | 设置页轴从站编号显示参数 | 读取来自硬盘自建参数表；`扫描从站` 扫到的从站编号必须写入硬盘自建参数表并作为该列下拉候选，UI 只能从扫描到的从站里选择。用户改变从站后必须同时写入硬盘自建参数表和微内核参数表；自建参数表保存 UI 选择/候选，微内核参数表保存运行态轴-从站绑定，不使用驱动映射表伪造。 |
| 轴参数表 | 分子 | 设置页稳定显示参数 | UI 稳定显示归自建参数；写驱动/读回证据不等于 UI 稳定显示 owner。 |
| 轴参数表 | 分母 | 设置页稳定显示参数 | UI 稳定显示归自建参数；写驱动/读回证据不等于 UI 稳定显示 owner。 |
| 轴参数表 | 写入状态 | 设置页稳定显示参数 | UI 稳定显示归自建参数；驱动动作结果只作为刷新/证据来源。 |

## 驱动映射表参数

| 设置页位置 | 参数/动作 | 驱动映射表来源 | 边界 |
| --- | --- | --- | --- |
| 设置驱动按钮/下载授权 | 驱动 profile | drive provider / 驱动参数映射表 | 只服务驱动私域选择、下载和证据链；不是轴运动参数 owner。 |
| 设置驱动按钮/下载授权 | SDO/PDO 对象字典 | 驱动参数映射表 | 只用于真实驱动读写；不得反向成为 UI 稳定显示参数真源。 |
| 设置驱动按钮/下载授权 | 授权下载参数 | drive provider 授权/下载链路 | 只服务驱动写入流程；不得接管微内核参数或自建参数。 |
| 设置驱动按钮/下载授权 | 驱动读写证据 | drive provider readback/evidence | 只能作为设置驱动动作闭环证据；不能替代 LinuxCNC/native readback。 |
| 轴参数表 | bit | 驱动参数表 / 驱动映射表 `encoder_bits` | 默认读取硬盘驱动参数表；允许人工更改，人工值仍归驱动参数表/驱动映射表参数并优先于默认值。用户改变 bit 后必须同时写入硬盘驱动参数表和微内核参数表；驱动参数表保存驱动侧人工覆盖，微内核参数表保存运行态比例链/编码器分辨率使用值；不得进自建参数。 |
| 轴参数表驱动动作 | 分子/分母写驱动证据 | 驱动参数映射表 | 分子/分母的 UI 稳定显示仍归自建参数；驱动映射表只记录真实驱动读写证据。 |

## 禁止项

- `轴模式`、`方向`、`从站`、`回零次序`、`回零方向`、`bit` 必须渲染为实际下拉控件；其中 `从站` 的下拉候选只能来自 `扫描从站` 写入自建参数表的候选。
- 除下拉列以及自动计算/只读的 `分子`、`分母`、`写入状态` 外，轴参数表其他参数不得做成静态文本；点击必须弹出 v3 同款 LVGL 自绘数字软件盘，并按硬盘参数表写入/回读链路保存。
- 不得把“读一个地方、写两个地方”误写成双真值；`从站` 改变后写硬盘自建参数表和微内核参数表，`bit` 改变后写硬盘驱动参数表和微内核参数表，最终成功以微内核/native readback 与硬盘持久 owner 一致为准。
- 不得把右上角 `保存并重启` 做成普通返回、只切主页面或只改按钮文字；它必须对应参数落盘确认后的受控重启/重载语义。
- 不得把微内核/native 已有或应该拥有的参数放进自建参数。
- 不得为了消除 `NAT` 把 UI 默认值、旧 `settings_runtime.json`、`config_cache`、SHM、`/run` result 或驱动映射表中不存在的伪字段当真值。
- 不得把驱动参数映射表扩展成运动参数、G53/RTCP、WCS/G92、刀补、模态或安全 owner。
- 龙门和刀库是第 7、8 轴，不是占位行；有 owner 的字段必须和其他轴一样显示，不得灰色禁用。
