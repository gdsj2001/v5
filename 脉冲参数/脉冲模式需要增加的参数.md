# 脉冲模式轴参数设计

## 1. 结论

- 设置页继续使用现有一张轴参数表，不创建第二张表或第二套轴行。
- BUS 模式必须保持当前代码的 `19` 列、顺序、标题和行为不变。
- Pulse 模式复用现有公共字段，隐藏 BUS 私有字段，并显示 Pulse 专属字段。
- Pulse 模式只新增一个普通用户输入：`脉冲宽度`，默认及当前最小值暂定为 `2500 ns`。
- 脉冲间隔、DIR 建立时间和 DIR 保持时间按固定比例自动生成，不允许用户分别填写。
- 当前整机 Pulse 输出上限暂定为 `200 kHz`；取得新的 FPGA、adapter、板级接口和驱动器联合波形证据前不得提高。
- 现有“目标精度（单位/count）”就是脉冲当量，不新增第二个脉冲当量或可编辑 `SCALE`。
- BUS 的从站、编码器 bit、电子齿轮、写入状态、profile 和 SDO/PDO 不得被 Pulse 引用。

本文区分“当前代码现状”和“Pulse 待实现设计”。未标为现状的内容不能视为已经实现。

## 3. Pulse 模式轴参数表设计

### 3.1 显示原则

- Pulse 标题显示“轴参数(脉冲)”。
- 继续复用现有表格、轴行、横向翻页、公共字段 owner 和保存入口。
- 不显示、不读取、不依赖 BUS 私有的 `从站、bit、分子、分母、写入状态`。
- 切回 BUS 后必须原样恢复第 2 节的现有 `19` 列。
- 模式列切换只能由 fresh native active mode 决定；请求值、UI 文案、dirty 状态或旧缓存不能决定。
- 隐藏模式的持久配置不得被清空或覆盖，但不能参与当前模式 runtime。

### 3.3 普通用户实际新增的输入

只有“脉冲宽度”是 Pulse 新增的可编辑列。普通用户通常保留默认 `2500 ns`，只有驱动器手册明确要求更长脉冲时才修改。

下列数据不新增编辑框：

- 脉冲当量：复用“目标精度”。
- `SCALE`：由目标精度生成并从 runtime/native owner 回读。
- 脉冲间隔、DIR 建立时间、DIR 保持时间：由脉冲宽度自动计算。
- 频率上限、实际生效速度、时序结果、门控 actual 和 ENA actual：只读。

## 4. Pulse 参数及来源

| 参数                        | 人工填写 | 默认值或来源                     | runtime/readback              |
| --------------------------- | -------- | -------------------------------- | ----------------------------- |
| 脉冲宽度                    | 是       | 默认 `2500 ns`                   | `step-width-ns`               |
| 脉冲间隔                    | 否       | `1 × 脉冲宽度`                   | `step-space-ns`               |
| DIR 建立时间                | 否       | `2 × 脉冲宽度`                   | `dir-setup-ns`                |
| DIR 保持时间                | 否       | `4 × 脉冲宽度`                   | `dir-hold-ns`                 |
| DIR 反向                    | 否       | 现有“方向”字段                   | FPGA `CONTROL[1] dir invert`  |
| 脉冲当量                    | 否新增   | 现有“目标精度”                   | runtime/native owner          |
| `SCALE`                     | 否       | `1 / 目标精度`                   | runtime `SCALE`               |
| 最大脉冲频率                | 否       | 暂定不高于 `200 kHz`             | capability readback           |
| 实际生效速度                | 否       | native 校验、限幅和实际回读       | native motion readback        |
| Pulse 发生器门控 actual     | 否       | 安全链实际状态                   | native/FPGA readback          |
| 驱动器 ENA actual           | 否       | IO owner 经急停门控后的实际状态  | native/FPGA readback          |

## 5. 计算和校验

### 5.1 固定计算公式

```text
SCALE = 1 / target_precision
step_space_ns = step_width_ns
dir_setup_ns = 2 * step_width_ns
dir_hold_ns = 4 * step_width_ns
timing_frequency_limit_hz = 1e9 / (2 * step_width_ns)
required_frequency_hz = abs(requested_velocity_units_per_s * SCALE)
effective_frequency_limit_hz = min(
    200000,
    timing_frequency_limit_hz,
    fpga_limit_hz,
    adapter_limit_hz,
    board_interface_limit_hz
)
pulse_capability_max_velocity = effective_frequency_limit_hz / abs(SCALE)
```

直线轴目标精度单位为 `mm/count`，旋转轴为 `deg/count`。例如：

```text
0.0001 mm/count -> SCALE 10000 count/mm
0.001 deg/count -> SCALE 1000 count/deg
```

螺距、电机转数和负载转数用于机械比例一致性校验。若这些字段与目标精度生成的 `SCALE` 冲突，必须提示并拒绝生效，不能生成第二套 SCALE。

### 5.2 脉冲宽度强校验

保存及 runtime 生效前必须同时满足：

1. 输入是大于 `0` 的整数纳秒值；空值、负数、小数、NaN 和越界值均拒绝。
2. 当前阶段不得小于 `2500 ns`，避免配置需求突破暂定 `200 kHz` 上限。
3. 必须按 FPGA `CLK_FREQ` 换算为安全整数 tick。不能整除时向上取整，并显示输入值与实际生效值。
4. 派生的 width/space/setup/hold 必须全部落在 FPGA 寄存器和 runtime 字段范围内。
5. 当前最大速度所需频率不得超过新计算的有效频率上限；冲突时拒绝保存，不能用静默降速掩盖错误配置。
6. controlled reload 后必须回读实际 tick、换算纳秒值和 native/FPGA actual；请求值与 readback 不一致时失败。

配置阶段采用“冲突拒绝保存”。运行期仍必须保留安全限幅，防止异常请求超过已生效能力，但运行期限幅不能替代配置校验。

### 5.3 默认能力示例

默认 `step_width=2500 ns` 时：

```text
step_space = 2500 ns
dir_setup = 5000 ns
dir_hold = 10000 ns
完整脉冲周期 = 5000 ns
理论时序上限 = 200 kHz
```

当 `SCALE=10000 pulse/mm` 时，最大能力为：

```text
200000 * 60 / 10000 = 1200 mm/min
```

若设置的最大速度需要超过 `200 kHz`，保存时必须提示所需频率、允许频率和允许速度，并拒绝生效。

## 6. Pulse“设零”既定契约

本节引用 `功能/2-5轴设0软件偏置方案.md` 和 `功能/2-1总线脉冲区别.md`，不另造零位语义。

1. 在当前轴行点击“设零”，弹出数字键盘。
2. 用户输入当前轴0位坐标；直线轴单位为 `mm`，旋转轴单位为 `deg`。
3. UI 提交 `settings_axis_zero`：
   - `driver_mode=pulse`
   - `target_scope=pulse_mechanical_home_offset`
   - `apply_mode=persist_home_offset`
4. settings/native apply owner 校验输入 finite，且位于当前有效软限位区间。
5. owner 写入目标 joint 的 runtime INI/native `HOME_OFFSET`，并完成源位置回读。
6. 成功只表示持久 owner 已写入并进入 canonical clean restart pending；不能立即把 UI 坐标改成0。
7. clean restart 后还需通过 LinuxCNC homing/native readback 证明坐标建立。

“回零方向”由独立下拉列写自己的 owner，不随“设零”键盘提交。Pulse 设零不读取 BUS `zero_model`，不发送 EtherCAT SDO，也不写驱动 preset。

若 Pulse contract 尚未成为 `runtime_selectable`，或 fresh native active mode 尚未证明为 Pulse，UI 必须禁用该动作，后台强制请求必须 fail-closed。

## 7. 最终 FPGA 工程事实和安全边界

最终硬件工程为 `new-vivado/z20_v1_5_hw_project`。

- `step_o`固定输出高电平脉冲，当前没有可配置 STEP 极性。
- 现有“方向”映射 FPGA `CONTROL[1] dir invert`；改变它使电机反向属于正确行为，不能命名为 STEP 有效极性。
- `CONTROL[0] enable`是 Pulse 发生器内部门控。
- 工程具有独立 `ENA1~ENA8` 输出，正常请求来自 IO owner 的 `AXIS_ENA`。
- 急停时顶层硬件门把 ENA 引脚强制为低。
- ENA 板级有效极性尚未完成验证，因此不得宣称高有效、低有效或默认使能。
- Pulse 发生器门控 actual 与驱动器 ENA actual 是两个不同状态，必须分别回读。

安全默认状态：

```text
开机、重启、服务未就绪、模式切换、急停或安全状态无效：
Pulse generator enable actual = disabled
STEP output = no pulse
ENA request = disabled
```

只有 LinuxCNC、HAL、FPGA、急停和必要安全状态全部有效后，系统才可打开 Pulse 发生器门控并请求驱动器 ENA。若现场驱动器需要可配置 ENA 极性，必须先完成原理图、接口电路和实机验证，再增加维护人员级字段；普通用户不填写。

## 9. 软件层验收要求

- BUS 模式的19列、顺序、标题、控件行为和 owner 与当前代码一致。
- Pulse 与 BUS 复用同一张表和轴行，模式切换不清空另一模式持久配置。
- Pulse 不显示、不读取、不保存 BUS 私有字段到 Pulse runtime。
- Pulse 只新增一个普通用户输入“脉冲宽度”，其余新增数据自动计算或只读回读。
- 覆盖默认值、修改值、tick 量化、零/负值、小数、溢出、比例冲突和频率超限。
- “方向”只映射 `dir-invert`，不存在可配置 STEP 极性或第二方向字段。
- Pulse 发生器门控 actual、ENA actual 和方向配置相互独立。
- Pulse 参数链不读取 BUS profile、驱动映射、SDO/PDO 或电子齿轮证据。
- Pulse“设零”请求、软限位校验、持久 owner 回读和 cold-stage fail-closed 符合既有功能文档。
- 保存成功必须证明唯一持久 owner 写入和源位置回读一致；controlled reload 后 runtime/native actual 一致。
- 软件测试覆盖开机、disable、急停、限位、服务未就绪、模式切换、异常退出和重启时的安全状态逻辑，不能用 UI 请求值冒充 actual。
- FPGA ABI 寄存器值、tick 换算、emitted count 和 LinuxCNC commanded position的软件链路按同一 generation 对齐。

本阶段不要求示波器、逻辑分析仪、真实驱动器波形、板级 ENA 极性或外部编码器验收。这些属于后续硬件联调阶段，不能作为本阶段软件实现的阻塞条件，也不能因未执行而宣称硬件波形、机械位置或丢步检测已经通过。
