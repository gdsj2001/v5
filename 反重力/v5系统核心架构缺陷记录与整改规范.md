本文只保留当前仍需整改的架构缺陷。已由源码闭合的事项不再作为“未改事实”保留，避免后续重复按旧问题返工。

| --- | --- | --- |
| `v5_settings_apply.c` 写入事务未收口 | 部分存在，原描述已过时 | 保留为活动缺陷，但收窄为“全量字段 owner 事务未完全收口”。比例链 raw limits 已进入 `settings_apply` 辅助事务，不再描述为仅有 prepare/gate。 |
| 普通比例链修改缺少 raw limits 同步重算 | 主路径已闭合 | 从活动缺陷删除。保留回归要求：比例链字段保存后必须继续覆盖 raw limits 重算、源位置回读、UI 单元格刷新和 clean restart 后 native readback 验收。 |

## 活动缺陷 1：`settings_apply` 全量 owner 事务仍未完全收口

**现状批复**：`D:\v5\board\services\command_gate\v5_settings_apply.c` 已承担比例链事务的一部分，包含比例链字段识别、raw limit 重算和结果返回；因此“仍仅起校验 prepare/gate 作用”的旧描述不再准确。

**仍存在的问题**：设置页普通 runtime INI、自建参数表、drive-only 参数等字段的写入、源位置回读、restart pending、运行态 reload/native readback 仍没有全部统一收口到 `settings_apply` 或已登记 native owner。UI 侧仍保留部分字段级写入和回读逻辑，不能把它等同于完整 owner/native 事务闭环。

**整改要求**：

1. UI 只构造 typed save request，不直接承担 owner 写入事务的最终语义。
2. `settings_apply` 或已登记 native owner 返回分层结果：已写入持久 owner、已源位置回读、待重启、已 runtime/native fresh readback。
3. 需要 LinuxCNC/HAL/EtherCAT 生效的字段必须保留 restart pending，最终以 `保存并重启` 后的 native/runtime readback 证明。
4. 不得用 UI dirty、本地显示值、Broker result、SHM、`/run` 快照或 JSON 证明保存成功。

## 活动缺陷 2：UI/root 权限沙箱化降权缺设备节点 Capability 清单

**现状批复**：UI、command gate、State Publisher、settings actiond 等板端服务仍主要由 root init 链路启动，现阶段只有 CPU affinity、`LD_BIND_NOW`、`mlockall` 等运行约束，不等于完成了非 root 沙箱化。

**仍存在的问题**：尚未形成可执行的设备节点、socket、共享内存和 LinuxCNC/HAL 访问 allow-list。直接把服务改成非 root 运行，可能导致首屏、触摸、远程输入、SHM、command gate 或 LinuxCNC 后端启动失败。

**整改要求**：

1. 先审计每个常驻服务实际需要的设备和路径：`/dev/fb0`、`/dev/input/event*`、`/dev/shm/v3_status_shm`、`/run/8ax_v5_product_ui/*.sock`、`/run/8ax/*.pid`、LinuxCNC/HAL/EtherCAT 相关访问路径。
2. 明确服务用户、用户组、目录权限、socket owner、SHM owner、必要 capability 和禁止 capability。
3. 降权必须分阶段落地：先审计输出，再调整权限，再单服务验证，最后全栈 clean restart 验证。
4. 在 allow-list 和板端验证完成前，不得把“root 运行”改成硬失败；只能作为审计告警和整改入口。

## 已关闭但需要回归保护：比例链 raw limits 同步重算

**关闭口径**：设置页修改 `precision`、`pitch`、`motor_rev`、`load_rev` 等影响 `SCALE` 的比例链字段时，若该轴存在 BUS count-domain `zero_model`，保存事务必须按新 `SCALE` 重算 `raw_zero_position`、`raw_min_limit`、`raw_max_limit` 并写回 runtime INI。当前主路径已具备该能力。

**回归要求**：

1. smoke 覆盖比例链字段保存、runtime INI 源位置回读、raw limits 重算和 UI 单元格刷新。
2. `保存并重启` 后必须由 LinuxCNC/HAL/EtherCAT/native owner readback 证明 runtime 生效；旧运行态中的 SHM、result JSON 或 UI 缓存不能作为完成条件。
3. 该项以后若重新失败，按 `REQ-SCALE-CHAIN-HISTORY-POLLUTION` 回到 `功能/0-1开机参数入内存.md` owner 修正，不再恢复旧 V3 JSON/cache 旁路。
