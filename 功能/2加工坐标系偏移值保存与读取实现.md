# 加工坐标系偏移值保存与读取实现

引用需求真源：`REQ-DOC-SINGLE-SOURCE`、`REQ-WCS-NATIVE-OWNER`、`REQ-NATIVE-OWNER-FIRST`、`REQ-LINUXCNC-COMMAND-GATE`、`REQ-PARAM-MEMORY-LIGHTWEIGHT-SAVE`、`REQ-SETTINGS-RUNTIME-DRIVE-ONLY`、`REQ-G92-NATIVE-RUNTIME`、`REQ-RTCP-G53-NATIVE-ACTUAL`。

## AI 阅读入口

- 启动内存/热路径通用规则：见 `REQ-PARAM-MEMORY-LIGHTWEIGHT-SAVE` / `功能/0开机参数入内存.md`，本文只保留本功能特有边界。
- 本文定位：WCS/G54-G59.3、Work Zero、当前活动 WCS 和全表显示的 owner。
- 先读 `REQ-WCS-NATIVE-OWNER`，涉及 G92、RTCP/G53 或参数真源时再跳到对应 REQ owner。
- WCS 持久化真源是 LinuxCNC `PARAMETER_FILE`，运行真源是解释器/native memory/API；UI/Broker/product action 不得自造 WCS JSON、缓存、SHM 字段或显示真源。SHM 只允许承载坐标/模态/速度倍率显示投影，不承载 WCS owner。


## 1. 结论

WCS / G54-G59.3 偏移属于 LinuxCNC/native owner 参数域。

| 范围 | 真源 | V3 允许用途 | 禁止 |
| --- | --- | --- | --- |
| 当前 active WCS | LinuxCNC interpreter runtime `stat.g5x_index` | UI/provider 直连 native/interpreter 回读显示 | UI 按钮选中态、Broker result、SHM 字段或私有缓存当真源 |
| 当前 WCS offset | LinuxCNC runtime `stat.g5x_offset` | UI/provider 直连 native/runtime 回读显示 | UI 本地计算、SHM 字段或旧 JSON 补洞 |
| G54-G59.3 全表 | LinuxCNC `PARAMETER_FILE` / `linuxcnc.var` 冷路径证据 | 低频读取、诊断、偏置页显示 | UI 热路径直接解析文件 |
| Work Zero 写入 | 登记 native gate/helper 或 linuxcncrsh gate 执行 G10/等价接口 | product action 做预检、计算、组帧、等待 native/runtime/API 回读 | 直接手改参数文件或 Python product action 直接 `linuxcnc.command()` |

WCS 成功标准不是“命令已发送”或“文件 mtime 变化”，而是 LinuxCNC 写入后经：

```text
LinuxCNC/runtime or PARAMETER_FILE -> native/interpreter API -> UI/provider
```

回读确认。

## 2. 三条链路

### 2.1 热路径：当前活动 WCS

热路径只服务高频 UI 显示：

- UI/provider 直连读取当前 `active_wcs` 和 `wcs_offset`。
- UI 坐标显示可使用 SHM `mcs/cmd_mcs` 投影；WCS 名称、偏移值和刀路坐标基必须来自同 generation 的 native/interpreter 回读。
- 机械坐标列和刀路 actual 机械点必须来自同一帧 `mcs` 内存投影。
- `cmd_mcs` 可用于红色 commanded 点和跟随误差，但不能回填机械坐标。

### 2.2 冷路径：G54-G59.3 全表

冷路径只服务偏置页、刀路全表显示、诊断和写入后证据：

- 全表读取不得进入 UI 热循环。
- 当前产品 ABI 使用 `wcs_offsets[9][5]`；扩展原生 9 轴必须升 ABI version 或新增字段组。
- 不得通过连续切换 G54/G55/G56 来探测全表，因为 WCS 切换本身是控制动作。
- `linuxcnc.var` 的 mtime/hash/key/epoch 是持久化证据，不是同帧 UI 成功证据。
- `wcs_offsets_epoch` 只在全表刷新成功后递增。

### 2.3 写入路径：Work Zero

写入只保留一条路：

```text
UI Work Zero
  -> C UI native/linuxcncrsh gate
  -> native/LinuxCNC/HAL owner 校验 fresh 状态 / WCS / axis / idle / RTCP OFF / 同源刀补模态并执行 LinuxCNC G10/等价接口
  -> native/interpreter API 从 runtime/stat/冷路径读回
  -> UI/provider 连续确认
```

UI 不得直接写参数文件、V3 私有持久文件或临时 JSON 来伪造成功。

## 3. 字段 owner 卡

| 字段/动作 | 禁止的第二真源 | 目标 owner | native/API 回读 | 退休条件 |
| --- | --- | --- | --- | --- |
| active WCS | UI 按钮态、Broker result、私有 active WCS cache、SHM 字段 | LinuxCNC interpreter runtime | `stat.g5x_index`、`stat.g5x_offset` 同源回读 | 产品路径不读写 active WCS 私有文件 |
| G54-G59.3 offset | V3 WCS JSON、UI dirty、`/run` snapshot、SHM config cache | LinuxCNC `PARAMETER_FILE` 与 runtime `stat.g5x_offset` | `stat.g5x_offset`、参数文件全表 key/epoch | 旧写入 owner、读取入口、allowlist 和示例删除 |
| Work Zero 结果 | MDI/G10 accepted、Broker JSON、参数文件 mtime 单点变化、SHM 显示投影 | LinuxCNC 写入后 runtime 与参数文件一致 | active WCS、offset、tool-tip/WCS 语义同源连续确认 | 未回读前不得清 dirty 或显示成功 |

## 4. UI 消费

UI 消费分两类：坐标显示只消费允许的 SHM 显示投影；WCS/offset 直连 native/interpreter owner：

- 坐标面板、偏置页机械坐标、设置页机械坐标、刀路绿色机械点、实际刀尖/刀柄反馈点和 MCS 基础投影使用 `mcs`。
- WCS 名称和 offset 使用同 generation 的 native/interpreter 回读，不从 SHM 字段消费。
- 非活动 WCS 缺全表时不得用 0、旧 offset 或 active offset 猜位置，应显示 `wcs-ofs--` 或 degraded。
- 显示层不做抗抖、死区保持、迟滞、连续样本确认或运动窗口缓存。
- 坐标面板显示不做数值滤波，但必须 fail-hold：`snapshot_stale`、`status_epoch=0`、非有限数或占位坐标帧不得覆盖上一帧有效 MCS/WCS 显示值。
- 主页面坐标数字层必须按变化单元格重画；不得在 30Hz 热路径中整块清空坐标 canvas，避免单帧无效数据或无变化刷新造成坐标区域全量闪烁。
- 小于 `0.0005` 的显示归零只属于三位小数格式化，不是稳定滤波。

## 5. WCS 切换

WCS 切换只改变 LinuxCNC runtime active WCS，不修改 G54/G55 偏移值。

要求：

- 通过 Broker/product action/native gate 发起。
- 等 native/interpreter API 回读 `stat.g5x_index` 和 `stat.g5x_offset`。
- 不另存 active WCS 文件。
- 不把按钮态、UI dirty、Broker result 或旧诊断当真源。
- product action 不得直接 `import linuxcnc`、创建 `linuxcnc.command()` 或 `stat().poll()`；若有兼容 adapter，只能薄转发到登记 gate。

## 6. Work Zero 门禁

Work Zero 必须 fail-closed：

- 缺 fresh native/microkernel 状态。
- 机器非 idle、MDI 忙、Jog/Home/Start 活跃。
- 未选择 WCS、轴或目标字段。
- RTCP actual enabled。
- 同帧刀补/刀尖语义缺失。
- native gate/linuxcncrsh gate 不可用。
- G10/等价写入后 native/interpreter/API 未回读确认。

RTCP 开启时必须拒绝 Work Zero，不能在 RTCP 几何、G53、刀长、WCS 混合状态下写偏移。

gate 执行后必须恢复/确认 MANUAL 或原约定模式；模式恢复失败时不得把 Work Zero 标为成功。

推荐响应字段：

```text
ok
code
active_wcs
axis
target_offset
wcs_offsets_epoch
readback_confirmed
pending_readback
details
```

## 7. 代码落点

| 层级 | owner |
| --- | --- |
| UI 按钮和显示 | `lvgl_app/src/v3_ui_command*.c`、WCS/坐标显示 owner |
| C UI native gate | `lvgl_app/src/v3_product_native_controls.c` 中 WCS/Work Zero owner |

| State Publisher | `v3_state_publisher.py` 仅作为 UI 坐标显示投影同步，禁止将 WCS 状态或偏移量写入 SHM |

| Toolpath 消费 | `功能/UI刀路绘图顺序与状态稳定方案.md` 对应 owner |

新增解析器、状态机、表驱动或跨层决策不得塞进超过 500 行的大文件；大文件只保留 glue/转发。

## 8. 验收

本地至少证明：

- active WCS 来自 LinuxCNC interpreter/native/API。
- WCS 全表冷路径不进 UI 热循环。
- Work Zero 写入后必须等 runtime/native/API 回读。
- RTCP ON 时 Work Zero 拒绝。
- WCS 切换不修改 offset。
- 旧 JSON/私有文件/Broker result 不作为 WCS 真源。
- SHM ABI 字段和 C/Python pack/unpack 对齐。

板端声明通过时必须有：

- 原始 UI/operator 触发路径。
- UDS request/response。
- LinuxCNC runtime 或参数文件回读。
- native/interpreter/API 连续确认；SHM 只可作为坐标显示截图证据。
- UI 截图或状态证据。
- 若影响运动，按 `AGENTS.md` 跑 `nc/cc.ngc` 黄金运动闭环。

## 9. 禁止事项

- UI 热路径直接读参数文件或私有 JSON。
- product action 直接拥有 LinuxCNC command/stat owner。
- G10 已发送就显示成功。
- 只看文件 mtime/hash 就清 UI dirty。
- RTCP 开启时写 Work Zero。
- 用 WCS 偏移修 MCS/编码器/设零显示问题。
- 把 Work Zero 当机器回零、设0、rebase、G92 或 RTCP 几何修改。

- 将 WCS 偏移、活动 WCS 状态或 Work Zero 结果同步到共享内存（SHM）中，UI 消费必须直连 UDS。
## 10. 关闭条件

关闭 WCS/Work Zero 问题时必须说明：

- active WCS、offset、全表、Work Zero 分别使用哪个 owner。
- 写入路径是否通过 native/linuxcncrsh gate。
- native/interpreter/API 回读字段和 generation；如引用 SHM，只能说明坐标显示投影证据。
- 本地测试、板端证据、截图或黄金运动结果。
- 未闭合事项只写真实缺口，不写过程日志。
