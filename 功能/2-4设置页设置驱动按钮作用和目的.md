- 启动内存/热路径通用规则：见 `REQ-PARAM-MEMORY-LIGHTWEIGHT-SAVE` / `功能/0-1开机参数入内存.md`，本文只保留本功能特有边界。

- 先读 `REQ-BUS-PULSE-MODE-DIFFERENCE`、`REQ-SCALE-CHAIN-HISTORY-POLLUTION` 和驱动 profile 文档，再改任何驱动动作。
- 设置驱动动作不得写成通用参数真源、运动安全真源或 Pulse HOME owner；缺 profile/readback/模式证明时 fail-closed。

本文只定义 BUS/EtherCAT 模式下设置页 `设置驱动` / `settings_set_drive` 的动作。`计算bit` / `settings_calc_bit` 是禁止产品路径：产品 UI、Broker 和板端脚本不得再通过微动、样本或 `log2(count)` 计算编码器 bit。

物理发现与驱动目标是两个严格分离的阶段。`扫描从站` 始终执行无 `-p` 过滤的 `ethercat slaves`，完整返回所有物理从站并只更新候选；它不受 active-model 过滤，也不建立或改写轴映射。`设置驱动` 的目标轴集合只能来自同一 resident active-model descriptor，并同时校验 runtime INI `[RTCP].MODEL`、`[RTCP].KINS_COORDINATES` 与 `[TRAJ].COORDINATES` 一致；随后每个 active 轴只能读取轴参数 resident self owner 中该轴的 `slave` 值，到本次全量扫描结果中按 position 精确查找。AC/`XYZAC_TRT` 的唯一 active set 为 `X/Y/Z/A/C`，status slot 为 `0/1/2/3/4`，B 必须为 `NAT`；BC/`XYZBC_TRT` 的唯一 active set 为 `X/Y/Z/B/C`，status slot 为 `0/1/2/3/4`，A 必须为 `NAT`。设置表可以继续保存 inactive A/B、龙门和刀库自己的参数，但这些行不因存在、非空或历史绑定而成为本次驱动目标；扫描到的额外从站也只作为候选存在。descriptor 与 KINS/TRAJ、resident 轴到从站绑定不一致，或任一 active 轴缺绑定、为 `NAT`、重复绑定、绑定位置不在本次扫描中时，必须在第一条 identity/SDO 前 fail-closed；不得按模型名字拆字、按扫描顺序分配轴、扫描全部非 `NAT` 行或按设置表行号猜目标。

设置页驱动动作按钮按 `REQ-BUS-PULSE-MODE-DIFFERENCE` / `功能/2-1总线脉冲区别.md` 的能力边界处理，不再把 Pulse/StepDir 模式下静态禁用/预检拦截组合作为默认闭合方式。`服务器下载` 按授权/VPS owner 执行；驱动硬件读写类动作由后台按当前硬件能力、目标从站、profile、授权和安全状态判断。能跑通真实 action/profile/readback 的路径必须优先跑通；只有硬安全、身份授权、资源确实不存在或目标模式真实不适用时才结构化失败。本文后续规则描述 BUS/EtherCAT 目标可用时的完整驱动读写闭环。

设置页轴参数表显示 `bit` 列，位置在电子齿轮 `分子` 列左侧。bit 属于驱动参数表/驱动映射表参数：默认 bit 从硬盘驱动参数表读取；用户可通过三角下拉人工更改 bit 位，人工选择必须写入硬盘驱动参数表/驱动映射表私域参数，并从 resident drive owner/drive provider readback 一致后才算成功。驱动参数表保存驱动侧人工覆盖来源；canonical clean restart 后由 drive/runtime owner 消费该值。不得恢复 `microkernel_parameter_table.tsv` 或微内核参数表第二套运行 owner。`驱动count/圈`、`多圈周期` 仍不在轴参数表显示，只作为后端直接参数用于电子齿轮计算、绝对多圈诊断、旋转 count-domain 和 fail-closed 校验。
`bit` 列不得提供或显示“自动”选项；驱动映射表、驱动参数表或本机人工覆盖缺有效 bit 时显示 unavailable/degraded，不得显示 `NAT`、默认值或旧值。人工选择只能选择数字 bit。人工选择数字 bit 后，硬盘驱动参数表/驱动映射表私域必须长期保存 `encoder_resolution_bits_source=ui_manual_override` / `encoder_bits_source=ui_manual_override`；`设置驱动`、开机加载和状态发布都必须优先使用该人工值，不得被 profile 默认 `encoder_bits` 覆盖。
设置页轴参数表按 8 轴显示：`X/Y/Z/A/B/C/龙门/刀库`，其中龙门和刀库就是第 7、8 轴，不是占位行、影子行或强制禁用行。所有行都可按各自唯一 owner 保存配置，供模型切换或独立设备功能使用；但“表格可编辑”不等于“当前 active model 生效”。只有 descriptor 选中的五轴可进入本次 JOINT、kinematics、Home 和 drive target，inactive A/B 以及龙门/刀库不得自动进入五轴运行集合。缺微内核/native 或驱动映射表 owner 的普通字段进入自建参数表并显示自建表读回值，不得用 UI 空值或固定默认冒充。`bit` 列例外：驱动映射表、驱动参数表或本机人工覆盖缺有效 bit 时按本文件 bit 规则显示 unavailable/degraded。轴名列固定在左侧，参数列位于右侧可横向平移区域；列较多时不得压缩轴名或拆掉 `bit`、电子齿轮、回零、软限位、零位等关键列。
轴参数表 `轴模式` 列本身必须以下拉方式选择 `直线`、`旋转`、`虚拟`，只读写 runtime INI/native owner。`轴模式` 右侧必须增加微内核参数列 `方向`，以下拉方式选择 `cw`、`ccw`；该列只读写 runtime INI/native owner，不得进入自建参数表或驱动映射表。
轴参数表 `从站` 列读取归硬盘自建参数表；点击 `扫描从站` 后，本次全量物理扫描到的每一个 EtherCAT 从站编号都必须进入硬盘自建参数表 `SETTINGS/slave_options` 并成为 `从站` 列真实下拉候选，显示格式为 `轴号 驱动名型号`。候选列表不得按 active-model、当前轴集合或已有绑定裁剪，扫描本身不得改写任何轴的 `slave` 行。唯一自动写映射的例外是运动模型实际切换事务：它只按已登记 descriptor 的 active axis/status slot 写默认编号并把离开 active set 的转换轴写 `NAT`，不得消费扫描顺序或驱动名称。`从站` 下拉必须提供固定 `NAT`，并允许用户在模型默认值基础上人工覆盖；真实从站值仍只能是纯编号。每个轴行打开下拉时，UI 从固定 `NAT` 加扫描候选选择，并排除已被其他轴当前选定的真实从站；当前轴自己的从站保留，`NAT` 不参与唯一性排除。用户改变 `从站` 后写入同一自建参数表并从 resident self owner 回读；canonical clean restart 后 INI/HAL/native/drive owner 必须消费最终 resident 绑定，使运动、Home 和 drive target 跟随物理从站。不得恢复第二套映射 owner。
从站选择改变并经硬盘自建参数表/resident self owner 回读一致后，设置页当前会话中的轴到从站关系必须立即按该 resident 绑定刷新，后续设置页请求只能使用该行当前 `从站` 值；最终 live 轴绑定仍以 `保存并重启` 后的运行 owner readback 为准。若某行 `从站` 选择为 `NAT`，该行除 `从站` 列外全部灰色禁用，不允许打开其它下拉、软件盘、设0/设零或提交参数改动，`从站` 列保持可用以便重新选择真实从站。
轴参数表 `轴模式`、`方向`、`从站`、`回零次序`、`回零方向`、`bit` 必须是实际下拉控件，不得只用红色文字或普通输入框模拟。下拉候选分别为：`轴模式=直线/旋转/虚拟`，`方向=cw/ccw`，`从站=NAT + 扫描从站写入自建参数表的未占用真实候选`，`回零次序=禁用/0..7`，`回零方向=0/+/-`，`bit=有效数字 bit 值或 24..16 数字候选，缺有效 bit 时单元格显示 unavailable/degraded`；下拉触发命中区必须和该单元格一样大，点击单元格任意位置都应展开当前下拉；候选来源仍必须服从各字段 owner，不能为了显示下拉而改变参数归属。
轴参数表中，除上述下拉列以及自动计算/动作结果列 `分子`、`分母`、`写入状态` 外，其他轴参数单元格都必须点击弹出软件盘输入。软件盘必须保持 v3 设置页同款数字键盘布局和视觉风格，用 LVGL 面板/按键/标签自绘实现，不得用 LVGL 默认 `lv_keyboard` 成品键盘替代。软件盘确认后 UI 只能向唯一 settings/native apply owner 提交该字段保存请求，由 owner 写入对应硬盘/native 持久 owner，并立即按 owner/readback 链路回读刷新单元格；不得让 UI 进程直接调用写盘函数，不得把这些可编辑参数做成静态文本、只改本地缓存或等待 `保存并重启` 才首次提交 owner。轴参数表数字显示和输入只允许 `目标精度` 使用小数；除 `目标精度` 外，所有数字参数必须显示为整数，软件盘也只能提交整数，带小数点、分数、乘号或非整数格式的输入必须拒绝。
读取和写入必须服从字段唯一 owner：`从站` 改变后写硬盘自建参数表并由 resident self owner 回读，`bit` 改变后写硬盘驱动参数表并由 resident drive owner/drive provider 回读。二者都不得恢复 `microkernel_parameter_table.tsv`；最终运行态生效以 canonical clean restart 后的真实 owner readback 与硬盘持久 owner 一致为准。任一对应 owner 失败时 `写入状态` 不能显示成功。

任一轴参数改变后的运行态生效只能走 `保存并重启` 的 canonical clean restart handoff；从站、bit、电子齿轮、轴当量及其它轴参数不得只靠局部服务重启或当前进程热状态宣称生效。重启期间相关驱动必须进入 Disabled / Safe-Op 或等价安全去使能，停止并重建 UI、Broker、Action Dispatcher、State Publisher、LinuxCNC/微内核、HAL/EtherCAT/drive provider 等运行闭包，清理或重建 `/run`、UDS socket、SHM/native 状态块、resident snapshot、命令队列和进程内 cache，再按开机链路重新装载 RAM，重建 HAL、EtherCAT master、SDO/PDO、轴到从站绑定、encoder bit 和轴当量换算。只有硬盘持久 owner、重启后的 resident snapshot、物理总线从站 evidence、SDO/PDO/encoder bit readback 和 native/runtime generation 全部一致时，才允许声明运行态生效；任一失败必须终止重启完成声明并 fail-closed，不得带半更新状态上线。动态 Native Gate live 注入只在未来底层提供 P0 原子事务和失败回滚后才允许。

设置页任何可修改参数控件发生改变时，事件处理只能构造字段保存请求并通过 IPC 发送给唯一 settings/native apply owner；写入、源位置 readback、resident self/drive refresh 或待受控重载标记必须由该 owner 串行完成后返回 UI。UI 不得直接写硬盘参数表、runtime INI 或本地 resident 镜像，不得只改 LVGL 控件文本、RAM 临时变量或本地显示缓存。字段必须按自己的唯一 owner 闭环，不能用双写微内核参数表、SHM、旧 JSON 或 `/run` result 补成功。
设置页右上角按钮初始显示 `返回主页`；本次会话任一轴参数已写入对应唯一 owner 并完成源位置回读，或 `设置驱动` 最终返回 fresh `DRIVE_SET_OK` 后，必须原位、单向切换为 `保存并重启`。`DRIVE_SET_OK` 必须同时返回 `restart_required=true`、`restart_deferred=true`；部分失败、窗口恢复失败、取消或异常不得制造 restart pending，已存在的 pending 也不得被后续动作清除。`保存并重启` 的语义是确认当前参数改动已经写入对应持久 owner，并进入开发板断电重启等价的 canonical clean restart handoff；必须停止并重建完整运行闭包、清理运行期 IPC/SHM/resident cache 并执行开发板完整重启，完成证明来自重启后的 resident owner/native/runtime/drive provider 和必要物理总线 readback 一致。不得把退出设置页、返回主页面、仅重启 actiond/UI/若干服务、machine-on 或 Operation Enabled 当成完成。无有效变更时允许本地返回主页，但有有效变更后不得继续保留普通返回语义，也不得让未落盘的 UI 临时值通过重启丢失。
设置页 G53 区显示顺序为 `A中心`、`B中心`、`C中心`、`对刀仪`、`5方向监测仪`。为容纳 B 中心行，设置页可把左下角状态提示移动到左上角，整体加高顶部参数区域，并把轴参数表整体移动到底部；不得压缩掉 B 中心、龙门、刀库或轴参数关键列。
G53 机床坐标目标位置表中，图示红线 owner 格 `A中心 X`、`B中心 Y`、`C中心 Z` 保持 WCS/native owner 只读，不从设置页软件盘写入；其余 G53 区单元格必须像轴参数普通输入格一样点击弹出 v3 同款 LVGL 自绘数字软件盘，确认后通过唯一 settings/native apply owner 写入对应硬盘/native 持久 owner 并回读刷新。UI 不得直接写 G53 自建参数表或 `[RTCP]` runtime INI，不得把这些可编辑 G53 单元格做成静态文本、只改本地显示缓存或等待 `保存并重启` 才首次提交 owner。
设置页 `登记本机码`、`下载授权`、`服务器下载`、`扫描从站`、`复位驱动`、`读取驱动`、`清除故障`、`设置驱动` 按当前 v5 长动作交互处理：点击后先显示同一个进度弹窗，后台返回最终结果后把原弹窗更新为成功/失败结果并保留到操作员点击 `关闭`。`下载授权`、`服务器下载`、`扫描从站`、`复位驱动`、`读取驱动`、`清除故障`、`设置驱动` 的执行中弹窗必须显示 `预计: MM:SS` 倒计时；最终结果不得保留倒计时。`扫描从站` 成功结果的弹窗正文仍保持 `提示 / 原因 / 下一步` 三项，其中 `原因` 必须显示本次扫描到的驱动/从站列表，固定为两列四行；每格显示序号、从站编号和驱动名，少于 8 个用 `--` 占位，多于 8 个在同一 `原因` 项内提示剩余数量。该列表只能来自本次扫描返回的 `slaves`、本次 `ethercat slaves` 行解析出的驱动名，或本次扫描写入自建参数表的 `slave_options`，不得使用固定默认列表、固定驱动型号、驱动映射表或旧结果伪造。当前后端尚未闭合真实写入/读回的驱动写动作不得伪造成成功；代码解决方向是补齐真实写入、读回和恢复流程。除硬安全、身份授权、资源确实不存在或目标模式真实不适用外，不得为了让 UI 看起来安全而新增静态禁用、空跑分支或长期 fail-closed 分支。

`登记本机码` 成功后，设置页顶部 `本机码` 必须立即从板端硬盘登记状态 `/opt/8ax/drive-profiles/device_register_status.json` 回读并显示 VPS 原样返回的 6 位 `vpsDistributionId`，不得继续显示 `未登记`。该显示只使用已持久化登记状态中的 canonical `vpsDistributionId`；不得追加 `已登记` 等状态后缀，不得显示、保存或从 UI 推导 raw PL DNA、DNA hash、`pl_dna_hash` 或本地猜测 ID，也不得从 `device_id`、`vps_distribution_id`、旧状态文件或授权文件回退推断。缺少合法 6 位 `vpsDistributionId` 时，原 ID 位置显示 `未登记`。设置页顶部不得再显示 `请求:NAT 实际:NAT` 这类旧状态占位。若本次登记返回成功但硬盘登记状态写入失败、缺少合法 canonical ID，登记弹窗必须显示失败，顶部 `本机码` 保持 `未登记`。

## 当前规则

最新闭环修正规则（2026-07-03）：`设置驱动` 写入电子齿轮和 `drive.write_mode` 后仍必须读回一致才算成功；若同一设置驱动事务内的恢复出厂/必读 SDO 读回已经证明 `mode_of_operation` 等于目标模式且驱动无 fault，后端必须把 mode 写入步骤记录为 `pre_write_readback_already_at_target` 并跳过重复 `drive.write_mode` SDO 下载。若执行 `drive.write_mode` 时出现 `Input/output error` 或同类下载失败，只允许在同一事务内执行 OP recovery/必读读回证明；只有目标 mode、电子齿轮和 fault-free 状态全部读回一致时，才可把该步骤记录为 `download_sdo_error_but_readback_matched`。读回缺失、mode 不一致或 fault 未清时必须返回 `DRIVE_MODE_READBACK_MISMATCH` / `DRIVE_SET_PARTIAL`，不能无条件吞掉写失败，也不能增加备用脚本、旧路径或兜底成功分支。如果电子齿轮分子/分母已经读回一致、驱动无 fault、EtherCAT 已恢复 OP，而单次 `mode_of_operation`/`drive.read_mode` 读回为空或瞬时不一致，后端必须在同一设置驱动事务内做有限短间隔重读/必要时重写 mode 后再判定。有限重试仍读不到目标模式时才返回 `DRIVE_MODE_READBACK_MISMATCH`，不能用一次 SDO upload 瞬时失败直接中断整个闭环。

1. 驱动映射表和本机驱动参数表必须按轴显式提供以下参数：
   - `profile_id` / `drive_profile_id`
   - `encoder_bits`
   - `encoder_resolution_bits`
   - `drive_internal_counts_per_motor_rev`（由最终 bit 派生）
   - `encoder_single_turn_counts`（由最终 bit 派生）
   - `encoder_multiturn_limit`
   - `encoder_multiturn_raw_modulus`
2. 代码只能直接读取硬盘驱动参数表/驱动映射表私域和运行内存中的最终 bit 参数。启动、保存或准备 `设置驱动` 时，后端先按轴配置的 `profile_id` 从 `driver_profile_map.json` 选择命令 profile；profile 只提供默认 `encoder_bits`。当前汇川 SV630N 默认 `encoder_bits=18`，对应 `drive_internal_counts_per_motor_rev=262144`；不得沿用旧 23 位 / 8388608 作为默认。若轴参数表已在硬盘驱动参数表/驱动映射表私域保存人工 bit 覆盖（`encoder_resolution_bits_source=ui_manual_override`），必须长期使用人工 bit，不得回退映射表默认；该人工 bit 必须从 resident drive owner/drive provider readback 确认，不得同步写入微内核参数表。否则使用 profile 默认 bit。`设置驱动` 按钮引用的驱动内部一圈 count/raw count 必须由这个最终 bit 派生，禁止引用已删除的 `驱动count/圈` 表列、旧 profile count、旧参数表 count、UI 默认 count，也禁止通过旧 `计算bit` 微动流程生成 bit。没有人工覆盖且所选 profile 缺少 `encoder_bits`，或人工覆盖标记存在但 bit 值无效时，后端必须清空/忽略本轴旧的 bit/count 派生字段并 fail-closed，不能沿用上一次 `设置驱动` 或旧映射表残留。
3. 电子齿轮目标只由本次 descriptor active axes 的轴参数 owner 和驱动参数表计算。`target_precision`、螺距/旋转每转角度、`motor_rev`、`load_rev` 属于 active runtime INI/native 比例链 owner，必须从与本次 active axis/status slot 同 generation 的 runtime/native readback 读取；inactive 轴可以保存自己的 AXIS 参数，但不得借旧 JOINT、设置表行号或历史 drive readback进入本次计算。驱动映射表已有的字段从驱动参数映射表或 drive provider 读取；只有微内核/native 和驱动映射表都没有承载的设置页轴参数才进入自建参数表。第 7、8 轴若某字段确实没有 native/runtime owner，可按字段进入自建参数表，但不得自动成为本次目标，也不得把已有 runtime/native owner 的比例链字段迁入自建参数表或用 UI 默认值、旧 JSON、`NAT`、空占位补值：

```text
ratio = motor_rev / load_rev
target_command_counts_per_motor_rev = pitch / ratio / target_precision
egear = drive_internal_counts_per_motor_rev / round(target_command_counts_per_motor_rev)
```

目标精度输入归一化：
- `target_precision` 是优先输入；如果轴级字段为空，后端必须使用 `electronic_gear.target_precision`。
- 如果 `复位驱动` / `读取驱动` 后只保留了 `electronic_gear.target_unit_per_count` 或 `electronic_gear.actual_unit_per_count`，`设置驱动` 必须把该值投影为本次事务的 `target_precision` 后再计算电子齿轮，不得因轴级字段为空直接失败。
- `target_precision`、螺距/旋转每转角度、`motor_rev/load_rev` / `motor_revs_per_load_rev` 三类输入必须来自 active runtime INI/native 比例链 owner 或同 generation owner readback；驱动侧派生值从驱动读回或 drive provider 读取。不得从旧 `settings_runtime.json`、旧 `target_counts/egear`、`pulse_equivalent_counts_per_unit` / `cia402_pos_scale_counts_per_unit` 投影、自建参数表、UI 默认值或缺省 `1/1` ratio 补出本次电子齿轮目标。
- 相同 active runtime INI/native 比例链输入和相同最终 `encoder_bits` 必须计算出相同电子齿轮目标；descriptor 当前选中的任意旋转轴在每转角度、目标精度、`motor_rev/load_rev` 和 bit 均一致时，目标分子/分母必须一致。旧 drive-only `egear_numerator/egear_denominator`、旧 readback 或旧 `electronic_gear.computed_*` 只能作为历史读回证据，不得优先作为本次写驱动目标。
- 当前已登记的 AC/BC 模型都必须逐 active rotary axis 使用各自 `MOTOR_REV/LOAD_REV`：AC 使用 A/C，BC 使用 B/C。任意两根旋转轴减速比不一致时必须分别计算，不能沿用另一模型、另一轴或旧 readback 的分子/分母。
- 任一输入缺少可信来源时，`设置驱动` 必须 fail-closed，并在错误详情列出缺失字段；不得在内存轴对象中临时写入默认 `0.0001/0.001`、`4.0/360.0` 或 `motor_rev/load_rev=1/1` 后继续计算。若这些值是实际产品参数，必须先进入 active runtime INI/native 比例链 owner 并作为该字段 owner 被 UI/后端自动读取。`target_precision`、螺距/旋转每转角度、`motor_rev/load_rev`、`motor_revs_per_load_rev`、电子齿轮分子和分母都必须大于 0；任何 0、负数、NaN、无穷或非数字输入必须返回结构化业务错误，不能触发未捕获的 Python 除零或异常闪退。该规则归属 `REQ-SCALE-CHAIN-HISTORY-POLLUTION`，本文只定义设置驱动本地需求边界。

4. 电子齿轮写入并读回一致后，驱动参数表/设置驱动私域同步保存实际 readback；电子齿轮证据归属物理从站，drive-only 参数表必须保存 `SLAVE_<从站号>` 行作为该从站的显示/readback 快照，允许同时写当前轴名行作为兼容显示快照。轴参数表 `分子`、`分母`、`写入状态` 三列只读显示本行当前 `从站` 对应的 drive-only owner；`从站` 改变后这三列必须立即跟随新从站刷新，`从站=NAT` 或新从站无可信读回时显示 unavailable/`--`，不得继续显示该轴旧从站、轴名旧值、自建参数表或 UI 默认值：
   - `egear_numerator`
   - `egear_denominator`
   - `write_status`
   - `feedback_counts_per_motor_rev = drive_internal_counts_per_motor_rev * egear_denominator / egear_numerator`
   - `actual_precision`
   - `rotary_load_counts_per_rev`
5. active-model descriptor 选中的旋转轴（AC 为 A/C、BC 为 B/C）的启动、设零、等效角、unwrap、checkpoint 和 Rebase 只读取各自 owner 的这些字段，不再等待 `计算bit`；inactive 旋转轴不得进入对应运行链。
6. 当前源码的 `设置驱动` 只支持普通 CSP 驱动设置路径。Broker canonical request 会为每个目标轴生成 `mode_int8=8` / `mode_of_operation=8`，后端请求过滤按电子齿轮目标和 CSP `8` 一起校验；请求里模式字段缺失、非数字、负数或不等于当前 canonical CSP 目标时 fail-closed。主轴、速度环轴、特殊同步轴或非 CSP 轴不得复用当前普通设置驱动路径，必须先新增对应 profile/轴 owner 和模式目标规则后再接入。

## 设置驱动动作

`设置驱动` 用于把当前 active-model descriptor 的生效轴参数、电子齿轮和模式参数写入其实际从站，并统一读回验证；它不是“扫描设置表全部行”的动作。

执行顺序：

1. 保存当前 dirty 轴参数；inactive 轴只完成自己的持久 AXIS owner/readback，不进入当前 JOINT 或驱动事务。随后冻结同 generation 的 active model、active axes/status slots 与实际 slave binding，只对这组目标按轴配置 `profile_id` 选择映射表 profile，把默认 bit 或人工覆盖 bit 刷新到硬盘驱动参数表/驱动映射表私域和 resident drive owner；内部一圈 count、raw count 由最终 bit 派生。用户可编辑的螺距、减速比、目标精度按各自 owner 保存；本次写驱动只读取 active runtime INI/native readback。人工 bit 选择必须已写入硬盘驱动参数表/驱动映射表私域并经 resident drive owner/drive provider readback。bit 默认读取硬盘驱动参数表，允许人工更改且人工选择长期优先；内部一圈 count、多圈周期不在轴参数表中显示或编辑。
2. 自动执行安全前置：由 microkernel/native drive-write safety gate 先读取 fresh Machine Enable、急停和全部扫描目标的 statusword/实际速度，再按本次 profile 命令登记的使能要求对全部目标执行一次协调状态切换。任一命令 `requires_disabled=true` 时，当前初始 Operation Enabled 是自动下使能的输入，不是 `DRIVE_WRITE_REQUIRES_DISABLED` 业务失败；必须先由 native owner 请求 Machine Off，等 fresh `machine_enabled=false`、全部目标 Operation Enabled 位清除且无运动 actual 后才允许第一条 SDO。不得逐轴反复上下使能，不得用 PREOP/OP 代替普通 CiA402 使能切换。`requires_disabled=false` 只表示本命令不要求下使能，不能推导成必须上使能；只有同一 profile/阶段明确登记 `requires_enabled=true` 时才允许自动上使能，两个字段同时为 true 必须在写入前按 profile 合同错误拒绝。UI/Broker/action 不得用速度为 0、解释器 idle、SHM 速度、JSON 状态、`status_epoch`、`motion_stillness_gate` 或私有阈值自行判断停稳。
3. 扫描/匹配驱动 profile，确认所需标准命令存在。
4. 计算电子齿轮分子/分母。
5. 先完成全部目标轴的电子齿轮、模式等驱动参数写入。EtherCAT mailbox 报文允许按总线顺序串行发送，但不得在目标集合尚未全部写完时按轴交错执行 activation、PREOP/OP recovery 或即时完整闭环。
6. 全部目标写完后按 profile 决定是否需要保存参数、重启或其它 activation。`requires_save_parameters=true` 只要求调用同 profile 登记的 `drive.save_parameters`，不能推导或替代 `drive.software_reset`；只有另有明确、字段级 restart/activation 依赖时才允许调用独立 `drive.software_reset`。`drive.set_egear.requires_save_parameters=false` 且 profile 未声明 restart/activation 依赖时，必须跳过 `drive.save_parameters`、`drive.software_reset`、PREOP 和 OP 状态切换，直接进入统一 fresh readback。`drive.software_reset` 作为独立 P1 trigger 存在，不代表普通 `设置驱动` 自动调用它。
7. 在写入所需使能态保持不变时，统一读取全部目标的分子/分母、CSP 模式、statusword/error_code 和关键参数。普通 `设置驱动` 的 profile 未声明 AL activation 时，本次 fresh readback 显示从站未回 OP、必读 SDO 缺失或 mailbox 不可用，只允许在原状态下对本次批次做有界 fresh 重读；不得为了补读回而发送全总线 `ethercat states`，也不得逐从站发送 `ethercat state -p` PREOP/OP。重读后仍缺失时立即按失败目标证据 fail-closed；正常目标不得被切换 PREOP/OP。正常的协调 Machine Off/CiA402 状态过渡不得作为报警或失败返回；过程不得产生 EtherCAT 同步周期类报警，最终读回不一致时 `设置驱动` 不得返回 `DRIVE_SET_OK`，也不得把轴参数表 `写入状态` 显示为已写入。
   - profile 明确要求 `drive.software_reset` 时，该命令可能因驱动主动重启导致本次 SDO download 返回 `Input/output error`；只有同一事务随后完成必要的 OP/mailbox recovery，并用 post-reset 必读 SDO readback 证明分子/分母、mode、statusword/error_code 全部一致时，才允许记录为 `download_sdo_error_but_post_reset_readback_matched`。非 reset 断连型错误、读回缺失或读回不一致仍必须失败，不得用旧 readback、旧 result 或普通 OP 状态补成功。
8. profile 要求的驱动侧 activation 已完成（若无要求则明确记录跳过）且全局 fresh readback 一致后，才能保存 drive-only 设置驱动私域证据并刷新 resident drive owner/drive provider；同时把实际分子、实际分母和显示用写入状态按物理从站同步到硬盘 drive-only 参数表 `SLAVE_<从站号>` 行，当前轴名行只可作为同一事务的兼容显示快照。当前设置页按本行 `从站` 对应的 `SLAVE_<从站号>` owner 刷新表格；用户随后改变 `从站` 时，`分子`、`分母`、`写入状态` 必须切换到新从站证据。native drive-write window 必须按入口 fresh 状态恢复，入口已使能且急停/安全 actual 未变化时协调 Machine On 并确认，入口未使能则保持 Machine Off。只有全部写入、统一 readback 和窗口终态都成功，最终 `DRIVE_SET_OK` 才同时置 `restart_required=true`、`restart_deferred=true`，让右上角单向切换为 `保存并重启`；任一写入、readback、恢复使能或安全状态确认失败必须保持 Machine Off、返回具体失败且不得置 restart pending。操作员确认 `保存并重启` 后必须完成开发板完整 clean restart，不能用当前会话恢复 Machine On 冒充轴参数运行闭包已经重载。

等待与超时：

- `设置驱动` 是全部目标批量写参数、按 profile 执行必要 activation、全局读回校验和刷新参数表的长流程；UI 不得沿用普通驱动写动作的 70 秒等待上限。
- 写电子齿轮、写 mode 的 SDO 通信超时只约束单条 SDO 报文。profile 明确登记 activation 时只执行该登记动作；普通 fresh readback 失败只进入有界重读窗口，不进入 OP recovery，不得把 SDO/readback 失败转成未经 profile 登记的 PREOP/OP。重读窗口耗尽后保留失败目标 actual 并返回 `DRIVE_SET_PARTIAL`。
- UI 等待期间必须持续刷新设置驱动 progress owner 的阶段进度；当前代码可用 UI 本地阶段、`job_status` 快查询和 `/run/8ax_v5_drive/drive_set_result.json` 作为本次弹窗/action evidence 与诊断输入，但不得把某个 `/run` 进度文件写成唯一进度真源，也不得把 `/run` result 当作设置页稳定刷新、保存成功或驱动已生效真源。只有 Broker `job_status` 明确返回失败、连续通信失败达到阈值，或超过 `drive_set_parameters` 专用长流程上限后，才允许弹失败窗口。
- 后台最终返回 `DRIVE_SET_OK` 时，UI 必须按成功处理并刷新轴参数表，不得因为普通 job 等待上限先到而把已成功动作显示成 `DRIVE_WRITE_JOB_TIMEOUT`。
- 后台最终返回 `DRIVE_SET_OK` 且 profile-driven activation（若需要）及 drive/native provider fresh readback 校验通过、表格显示 `已写入` 后，必须同时返回 `restart_required=true`、`restart_deferred=true`，UI 在同一 fresh run_id 结果上把右上角从 `返回主页` 单向切换为 `保存并重启`。若入口 fresh actual 为已使能且本次安全状态未变化，native drive-write window 仍需先恢复并确认 Machine On；该恢复只关闭本次写窗口，不能清除 restart pending，也不能替代完整 clean restart。失败、取消、窗口关闭或使能恢复未确认时不得置 restart pending。

失败规则：

- 缺最终 `encoder_bits`、由 bit 派生出的 `drive_internal_counts_per_motor_rev`、`encoder_multiturn_limit` 或 `encoder_multiturn_raw_modulus` 时 fail-closed。
- 当前普通设置驱动路径只接受 canonical CSP `8`；请求模式字段缺失、非法或不等于 canonical 目标时 fail-closed。
- 电子齿轮派生输入为 0、负数、NaN、无穷或非数字时 fail-closed，并返回结构化错误；不得出现未捕获 `ZeroDivisionError` 或后台 job 无 JSON 闪退。
- profile 明确要求 activation 而登记动作未完成，或本次有界 fresh readback 后仍存在失败目标时必须 fail-closed；不得临时发明 OP recovery，也不得让正常目标进入任何 AL 状态切换窗口。
- 自动下使能的 native request、Machine Off actual、目标 statusword 或无运动 actual 任一未确认时必须在第一条 SDO 前 fail-closed；自动状态切换超时、真实 fault/error_code、`SAFEOP E` 或同步故障必须保留结构化原因。不得把正常 CiA402 过渡报成设置驱动失败，也不得以“不报错”为由吞掉真实硬件/总线故障。
- 恢复出厂或清除故障后的成功判定必须同时满足必读 SDO 读回完整、EtherCAT OP、`error_code=0`、statusword 无 fault、`drive.read_egear` 读回可用；`statusword`、`drive.read_egear` 等必读项读回失败时，只能在同一事务内有限重试/OP recovery，不能把缺失读回当作 `no_fault` 或成功。
- 恢复出厂或清除故障中的 OP recovery 不能只检查 `ethercat slaves` 显示 OP；其具体全目标恢复按各自动作 owner 执行，并用必读 SDO readback probe 证明邮箱恢复。探针未通过只能继续有限恢复或 fail-closed，不能把 OP 状态本身当作成功。普通 `设置驱动` 不进入这条 AL recovery。
- `读取驱动` 按钮的必读项读回失败时，后端必须对失败目标轴执行有限 PREOP/OP 状态循环，并以同一个 `read_drive_for_target` 探针重新证明读回完整；若读回显示真实 fault，才允许执行标准 `drive.reset_fault` 后再读一次。有限恢复后仍缺失 `statusword`、电子齿轮或其他必读项时，结果只能保持 `DRIVE_READ_PARTIAL` 并保留失败证据，不得补默认值、跳过该轴或把后续 UI 等待改成兜底成功。
- `设置驱动` 写入电子齿轮和 `drive.write_mode` 后，最终刷新 `settings_runtime` 的汇总读回必须使用必读 SDO readback；profile 未声明 activation 时不得调用 `drive.software_reset`、全总线 `ethercat states` 或逐从站 `ethercat state -p` PREOP/OP。`statusword`、电子齿轮分子/分母或 mode 等必读项缺失、从站未回 OP 或 mailbox 不可用时，只允许对同一冻结批次做有界 fresh 重读；重读后仍缺必读项、无法证明 mode 与目标一致，或 profile 明确要求的 activation 命令缺失/失败，顶层结果必须转为 `DRIVE_SET_PARTIAL`。不得出现 `DRIVE_SET_OK` 已返回但 `settings_runtime.axes[].drive_parameters_valid=false`、`drive_mode_readback_mismatch`、profile activation 未证明或必读 SDO 缺失仍残留的分裂状态。
- `复位驱动` / `读取驱动` 写回设置表时，驱动实际读回的出厂 `1/1` 只能进入 readback 证据字段和 `readback_mismatch` 状态，不能覆盖已配置的目标电子齿轮，也不能让空的顶层 `egear_numerator` / `egear_denominator` 遮蔽 nested `electronic_gear` 中的 canonical 目标值后变成 `0/0`。
- 缺 `drive.set_egear`、`drive.read_egear` 或读回不一致时 fail-closed。
- 任何弹窗必须显示具体原因，不允许提示“查看日志”。
- 成功窗口不自动关闭。

`settings_axis_zero` / 设置页 BUS `设0` 不在本文维护，完整 owner、通过条件和禁止项以 `功能/2-5轴设0软件偏置方案.md` 为准。本文只保留设置驱动动作边界：设置驱动不得复用设0的 SHM/display 投影、`status_epoch`、V3 私有阈值或旧 JSON 作为安全窗口、动作 proof 或保存证明。
