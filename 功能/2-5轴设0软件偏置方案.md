## P0 BUS/EtherCAT 单轴设 0 最终规则

- BUS/EtherCAT 单轴设0的保存通过条件只有五项：驱动 count-domain 位置读数成功、LinuxCNC raw 机械坐标读数成功、active runtime INI `SCALE` 可用、`zero_model` 成功写入硬盘、按新 0 位和 UI 限位距离计算出的 raw limit 成功写入硬盘。除此之外的 RTCP/G53 geometry、静止窗口、UI/SHM 显示归零等待、live raw limit 读回、live runtime raw 归零、memory/UI 归零、自动上使能/idle 判定都不得作为本次 `设0` 按钮保存通过门禁。
- BUS/EtherCAT 单轴设0时，0 位一变，硬盘 runtime INI/native owner 中的 raw 软限位必须在同一次产品动作里跟随新 0 位计算并写入；否则最终重启后底层仍可能按旧 raw 限位判定越界。
- UI 软限位字段仍表示相对 0 位的物理距离；State Publisher/设置页必须继续显示这个距离，不得把已同步到 native/runtime 的 raw 限位坐标反向显示成 UI 距离。
- raw 限位落盘公式固定为 `raw_min_limit = raw_zero_position + ui_min_limit_distance`、`raw_max_limit = raw_zero_position + ui_max_limit_distance`。其中 `raw_zero_position` 对 BUS count-domain 零点等于 `zero_counts / active runtime INI SCALE`；螺距、减速比、目标精度、轴方向等比例链字段改变并导致 `SCALE` 改变时，必须用新 `SCALE` 同事务重算硬盘 raw 限位。
- 单次设0按钮的通过条件是当前 raw 机械位置/count-domain 零点证据和按 UI 限位距离计算出的 raw 限位值已成功写入硬盘 owner，且公式合法、写盘无错误；不要求同次校验 LinuxCNC/HAL/EtherCAT live raw 限位，也不要求等待 SHM/UI 已按新零点投影为 0。
- 操作员可见的设0结果弹窗不得只表示“请求已收到”，也不得拿当前编码器去比旧硬盘零位。后端必须先读取本轴编码器/驱动 count-domain 当前位置，把该当前位置作为本次新 0 位写入硬盘 `zero_model`，并按新 0 位同次写入硬盘 raw limit；随后重新读取硬盘新 `zero_model` 和编码器当前位置，两者使用同一 active runtime INI/native `SCALE` 或已登记 `unit_per_count` 换算成物理单位后比较。直线轴单位为 `mm`，旋转轴单位为 `deg`；`abs(current_encoder_physical - saved_zero_physical) <= 0.1` 时才提示设0成功，超过 `0.1 mm/deg` 或读数/换算/硬盘写入/硬盘读回缺失时必须提示设0失败。
- 设置页单轴 `设0/设零` 的进程、成功、失败弹窗顶部标题必须显示具体轴，例如 `X 设0`、`A 设0`、`B 设0` 或 `X 设零`；不得显示泛化的 `单轴设0` 作为最终操作标题。轴号来自本次 UI 轴行请求或后端结构化 `axis` 字段，缺失时必须 fail-closed 或在正文说明轴号缺失，不得伪造成某一轴。BUS/EtherCAT 设0的驱动读取目标来自当前轴参数表该行 `从站` 列选中的从站号，UI 请求必须携带该从站号；该行 `从站=NAT` 表示没有物理驱动绑定，UI 必须禁用本行 `设0/设零` 且后台收到缺失/`NAT` 从站时必须 fail-closed；后端不得按 X/Y/Z/A/C 固定顺序、固定 AC 模型、轴号默认映射或 `settings_runtime.json` 旧映射猜测要读的驱动，旋转轴必须来自 fresh Active Model descriptor/readback。
- 设0或比例链保存只完成硬盘写入时，响应证明必须标明 live raw limit 未验证、`backend_restart_required=true`、`canonical_clean_restart_required=true`、`restart_deferred=true`；`backend_restart_required` 只是旧字段兼容名，实际语义必须是 `功能/0-1开机参数入内存.md` 定义的 canonical clean restart handoff，达到开发板断电重启同等干净度后，才能验证 LinuxCNC/HAL/EtherCAT live raw 限位、允许的 SHM 坐标/UI 投影归零和运动闭环。
- 编码器 raw 位置允许轻微跳动；BUS 设0不得用前后 raw 双采样静止窗口作为阻塞门禁。可保留 raw delta 诊断，但不得因为微小编码器跳动返回 `SETTINGS_ZERO_AXIS_MOTION_ACTIVE`。

本文只维护设置页单轴 `设0/零位` 的本地请求契约、owner 和验收边界。BUS/Pulse 模式差异、模式切换闭环、Pulse 轴参数 `设零` 键盘和 `回零方向` 列的总口径以 `REQ-BUS-PULSE-MODE-DIFFERENCE` 及 `功能/2-1总线脉冲区别.md` 为准。

## 1. 接手边界

先确认三件事：

- 当前 active mode 是 BUS/EtherCAT 还是 Pulse/StepDir；`motion_driver.conf` 只能作为 requested/持久 owner，真实 active mode 必须按 `功能/2-1总线脉冲区别.md` 的 SelectLink/运行 INI/LinuxCNC/RT-HAL 四项并联证明进入 native/microkernel 运行内存。
- 本次动作是设置页单轴 `设0`，不是主页面 `回零`、Work Zero、驱动硬件 preset 或 position rebase。
- 当前字段是否会触碰 native/microkernel truth；若触碰，先按 `功能/需求真源索引.md` 找唯一 owner。

控制链固定为：

```text
UI C -> Command Broker UDS framed socket -> 已登记 settings/native apply owner -> LinuxCNC/HAL/EtherCAT/native owner
```

UI 不直接读 raw、不写 `settings_runtime.json`、不改 HAL/INI/驱动器对象。

## 2. 核心语义

`设0/零位` 在两种模式下不是同一个动作。

| 模式 | UI 文案 | 用户输入 | 后端动作 | 当前坐标是否立刻清零 | 是否写驱动 preset |
| --- | --- | --- | --- | --- | --- |
| BUS/EtherCAT | `设0` / `当前位置设为0` | 不输入数值 | 读取当前 LinuxCNC/HAL/EtherCAT/native raw 机械坐标和驱动 count-domain 位置，保存 Settings Drive 私域零点证据；按新 0 位和 UI 限位距离计算硬盘 raw 限位并写入 runtime INI/native owner | 零点证据和硬盘 raw 限位写入成功即为本次设0保存通过；最终 live 坐标归零、raw 限位装载和允许的 SHM 坐标/UI 投影显示闭环等 canonical clean restart handoff 完成后闭合 | 禁止普通设0直接写驱动 preset；raw 限位不是 preset，必须走受控 runtime/native owner 落盘 |
| Pulse/StepDir | `设零` / `当前轴0位` | 弹出键盘，输入坐标值作为当前轴 0 位；轴参数表另有 `回零方向` 列下拉选择回零方向 | 写 runtime INI/native owner 的当前轴 `HOME_OFFSET`；`回零方向` 列写独立回零参数 owner | 禁止只靠 UI 立即改坐标；必须经 owner 写入和后续回零/native 证明 | 禁止 |

BUS 模式设置页轴参数表里的单轴 `设0` 不是回零动作，也不生成 Pulse homing 参数；Pulse 模式轴参数表里的单轴 `设零` 是把键盘输入的坐标值写成当前行当前轴 0 位对应的 `HOME_OFFSET`，不读取或复用 BUS count-domain 零点证据。两种模式的请求字段、后端 owner 和验收闭环必须分开。

当前阶段 Pulse/StepDir 仍受 `功能/2-1总线脉冲区别.md` 的 `cold_staged_not_runtime_selectable` 合同约束：Pulse 轴参数表 `设零` 路线只在 native active mode 已经是 Pulse 且 Pulse contract 被 owner 更新为 `runtime_selectable` 后才允许进入运行 apply。contract 仍为冷态预配置时，UI 目标应 disabled，任何强制提交都必须由 C/Broker dispatch fail-closed；不得借本文的 Pulse HOME_OFFSET 说明绕过 `2-1`。

危险动作分类（2026-07-03，2026-07-07 修订）：设置页 BUS 单轴设0是硬盘零点证据和 raw 限位保存动作，不执行运动、不执行驱动 preset、不要求自动上使能，也不得把上使能、idle、stillness 或 SHM ready 当成本次保存通过门禁。设置页 BUS 单轴设0不作为危险确认动作；A/C 轴坐标写入、会导致旋转轴实际运动或相位重定的动作仍必须二次确认。X/Y/Z 直线轴设0不是危险动作，不得因“可能需要上使能”而弹危险确认。

硬规则：

- BUS 设0先保存 count-domain 证据，例如 `zero_counts`、A/C `zero_abs_turns`、`zero_abs_single_turn_counts`、`zero_abs_counts64`、驱动读回来源和 profile/hash 证据；同一次产品动作还必须按当前零点和 UI 限位距离写入硬盘 raw 限位。允许的 SHM 坐标/UI 等效坐标归零属于 canonical clean restart handoff 完成后的 live 显示闭环，不作为单次设0按钮保存通过门禁。
- `旧浮点零位字段`、`旧倒数比例字段` 只能作为内存诊断表达式，不得重新落盘，不得作为 UI/State Publisher/settings/native apply 的产品输入。
- BUS 坐标显示和 BUS Home 必须使用 count-domain 证据 + active runtime INI/native `SCALE` 回读 + native/API 回读。UI 坐标值不是独立真相，必须由 native/resident owner 基于 LinuxCNC/HAL/EtherCAT 的真实 raw 机械坐标、当前零点、`SCALE` 和 A/C 等效相位规则计算，再经 State Publisher 发布允许的坐标显示投影；只改 UI 本地状态、旧快照或 SHM 字段不能算通过。
- Pulse 不把 `home_offset` 或 `pulse_home_model` 写入 Settings Drive 私域持久文件；它们属于 runtime INI/native owner 的一次性 apply 请求。
- Pulse 禁止普通 `设零` 路径写 EtherCAT SDO `0x607C`、保存对象 `0x1010`、驱动侧零位改写、position preset 或 drive rebase。BUS 普通 `设0` 不写驱动 preset、不改写驱动 raw 位置；但 0 位改变后硬盘 raw 软限位必须由受控 runtime/native owner 同次计算写入，禁止 UI、Broker、Python 临时脚本、旧兼容路径或静默 preset/静默限位写入。

## 4. BUS/EtherCAT 设0

BUS 模式普通 `设0` 是“保存零点证据 + 硬盘 raw 限位同次写入”的保存动作，不是 Pulse 回零，也不得把当前 UI 等效坐标立即归零作为按钮通过门禁。X/Y/Z/A/C 按下单轴设0后直接提交正式 settings/native apply 请求，不弹危险确认。系统必须读取当前 raw 机械坐标和驱动 count-domain 位置写入硬盘，并按新零点和 UI 限位距离计算写入硬盘 raw 限位；LinuxCNC/HAL/EtherCAT live raw 限位装载、允许的 SHM 坐标/UI 等效坐标 `0.000` 显示闭环在 canonical clean restart handoff 完成后验证。

### 4.1 操作流程

BUS/EtherCAT 设置页单轴 `设0` 的过程固定如下：

1. UI 在当前轴行点击 `设0` 后，只发送 `settings_axis_zero` 请求；请求字段必须包含 `axis/joint`、当前行 `从站` 列提取出的 `slave_index`、`driver_mode=bus|ethercat`、`target_scope=bus_count_domain_zero`、`apply_mode=count_domain_zero`，不得携带 `home_offset`，不得打开软件盘，不得直接写驱动 preset。
2. 后端按本次请求的 `slave_index` 读取当前 active BUS 从站和 profile；缺轴、缺本次从站号、模式不是 BUS/EtherCAT、profile 未加载、从站身份不匹配或 `drive.read_actual_position` 不可用时 fail-closed。`settings_runtime.json` 中的历史 `axis -> slave_index` 只能作为离线证据/诊断，不得在本次请求缺少从站列值时替代 UI 当前选择。
3. 后端读取本次 `slave_index` 对应驱动的当前编码器/驱动 count-domain 当前位置。X/Y/Z 等直线轴最小读数是 `drive.read_actual_position`；A/C 等旋转轴还必须读取已登记的 absolute basis（例如 direct64 或 multi-turn + single-turn 合成证据），缺 basis 时旋转轴 fail-closed。SDO/PDO 读数必须按 profile 标注的有符号/无符号类型解析；`int32` 等有符号对象不得把 `0xffff...` 负数计数保存成 32 位无符号大正数。
4. 后端读取当前 LinuxCNC/HAL/EtherCAT/native raw 机械坐标和 active runtime INI/native `SCALE`，或读取已登记的 `unit_per_count`。若 `SCALE/unit_per_count` 未直接登记，必须由同一轴的比例链推导：直线轴至少要考虑螺距 `pitch`、减速比/电机负载比 `motor_rev/load_rev` 或 `reducer_ratio`、目标精度 `precision`、驱动/编码器 bit 或每转计数；旋转轴至少要考虑 `360 deg`、减速比/电机负载比、bit 或每转计数。缺任一必要比例链字段、字段为 unavailable/degraded、非 finite 或来源不一致时必须失败，不得用 `raw_zero_position / actual_counts` 反推替代真实比例链。
5. 后端把设0瞬间的 count-domain 读数保存为硬盘 `zero_model`，并用新零位和 UI 软限位距离计算硬盘 raw limit 写入 runtime INI/native owner。旧硬盘零位只能用于从当前 raw limit 反推出 UI 相对限位距离，不能参与最终成功/失败比较；若旧硬盘零位已落在当前 raw 限位区间外，必须判为旧证据污染并 fail-closed，不得继续用该值滚动重算限位；只保存 `zero_model` 而未同次写硬盘 raw limit 是失败。
6. 写盘完成后，后端必须重新读取用于弹窗判定的两项数据：当前编码器/驱动 count-domain 位置，以及硬盘 owner 里本次刚保存的本轴新 `zero_model` 零位。弹窗成功/失败只按这两项的物理单位差值判断，不能按“请求已接收”判断，也不能按旧硬盘零位判断。
7. 换算单位规则固定为：直线轴 `mm`，旋转轴 `deg`。优先用 active runtime INI/native `SCALE` 做 `physical = counts / SCALE`；若本轴 owner 已登记 `unit_per_count`，可等价使用 `physical = counts * unit_per_count`，但必须和保存 `zero_model` 时的比例链一致。比例链计算必须把螺距、减速比/电机负载比和 bit/每转计数纳入结果，并把采用的字段和值写入 evidence。
8. 容差固定为 `0.1 mm/deg`：`abs(current_encoder_physical - saved_zero_physical) <= 0.1` 返回成功弹窗；超过容差、无法换算、本次新硬盘零位缺失、硬盘读回不一致、编码器读回失败或 raw limit 写盘失败均返回失败弹窗。
9. 成功或失败弹窗顶部必须显示具体轴名和动作名；成功弹窗仍必须声明 live raw limit、UI/SHM 等效归零和运动闭环尚未完成，下一步是执行 canonical clean restart handoff 后做 native/runtime readback；失败弹窗必须显示当前编码器值、硬盘零位、差值和容差，要求重新读取驱动或重新设0，不得提示“已成功”。

该流程中的“成功”只表示硬盘零位与当前编码器位置在 `0.1 mm/deg` 内一致，且本次硬盘零位/raw limit 保存链路闭合；它不表示 LinuxCNC/HAL/EtherCAT live raw limit 已装载，也不表示 UI/SHM 坐标已经实时归零。

基本模型：

```text
pre_zero_machine_position = 设0瞬间 LinuxCNC/HAL/EtherCAT/native 产品可见机械坐标
zero_shift                = pre_zero_machine_position
current_counts            = 当前驱动/编码器反馈计数
zero_counts               = 设0瞬间保存的 current_counts
active_runtime_SCALE      = active runtime INI/native owner 回读的 count/单位
post_raw_machine_position  = pre_zero_machine_position
post_zero_memory_position  = canonical clean restart handoff 后由 native/owner 证明的 0.000 等效投影值
ui_equivalent_position     = native/resident owner equivalent_projection(post_raw_machine_position, zero_model, SCALE, rotary_phase_rule)，按钮保存阶段不作为通过门禁
ui_min_limit_distance      = UI 设置的负向限位物理距离，坐标值相对 0 位
ui_max_limit_distance      = UI 设置的正向限位物理距离，坐标值相对 0 位
post_min_limit_raw         = zero_shift + ui_min_limit_distance
post_max_limit_raw         = zero_shift + ui_max_limit_distance
```

UI 显示值必须是 `ui_equivalent_position`，不是 UI 本地状态、旧快照、硬盘字段、SHM 伪字段或弹窗状态。对 A/C 旋转轴，UI 显示的是 native/resident owner 从真实 raw 机械坐标/count-domain 位置投影得到的等效角；按钮保存阶段只证明新零点和硬盘 raw 限位已写入，等效角归零显示闭环必须在 canonical clean restart handoff 后由 native 事实经允许的 SHM 坐标/UI 投影闭合。

UI 里的限位设置只是相对于 0 位的物理距离，不是 LinuxCNC/HAL/EtherCAT 的绝对 raw 限位坐标。零位改变时，UI 限位距离值不自动改写；后端必须按新 0 位和这些距离计算 raw 限位，并在同一次设0动作内由受控 runtime/native owner 写入硬盘 `MIN_LIMIT/MAX_LIMIT`、`soft_minus/soft_plus` 或等价 raw limit 字段。只改 UI 显示限位、只改旧快照、只记录计划但不改硬盘 raw 限位，或让最终重启后的底层限位仍停留在旧 raw 坐标系，都不是闭合。

A/C 绝对多圈轴还必须保存：

```text
zero_abs_turns
zero_abs_single_turn_counts
zero_abs_counts64
raw_bits/raw_modulus/raw_value_type/raw_modulus_trust
```

驱动 profile 必须提供：

- `drive.read_actual_position`
- A/C 绝对多圈轴还必须提供 `drive.read_abs_multi_turn`
- A/C 绝对多圈轴还必须提供 `drive.read_abs_single_turn`
- A/C 绝对多圈轴还必须提供 `drive.read_abs_position64`，可由 Adapter 用 split 读数合成

X/Y/Z 线性轴设 0 的驱动读回只能读取本轴 `drive.read_actual_position` 这一条 count-domain 锚点，不得为了普通线性设 0 串行读取 statusword、mode、绝对编码器、电齿轮等全量诊断字段；缺 profile、缺轴-从站绑定、缺 `drive.read_actual_position` 或单条读回失败时直接 fail-closed，不得退回全量读回、扫描全站或旧兼容路径。A/C 旋转轴设 0 仍必须读取绝对多圈 basis；RTCP/G53 geometry 可在可用时保存为诊断证据，但缺失、不新鲜或不可证明时不得阻塞本次设0保存。

split 合成不得把 `uint16` 圈数直接强转有符号值；运行时必须使用相对零点差分取模公式。缺 profile、缺轴-从站绑定、缺读回命令、读回失败、`SCALE` 未经 active owner 回读、native/microkernel readback stale 或机器忙时，必须 fail-closed。

BUS 设0不得把前后 raw 双采样静止窗口作为阻塞门禁。编码器位置允许轻微跳动，后端可以记录设0前后的 raw delta 作为诊断证据，但不得因为微小跳动返回 `SETTINGS_ZERO_AXIS_MOTION_ACTIVE` 或拒绝保存 `zero_model`。请求契约校验通过后，本次保存是否成功只看驱动 count-domain 读数、LinuxCNC raw 读数、active runtime INI `SCALE`、`zero_model` 写盘和 raw limit 写盘这五项。

A/C 绝对旋转轴的 `zero_model` 不以 RTCP/G53 回转几何身份作为设0保存门禁。若设0瞬间同一帧状态已经提供可信 geometry，可把 `G53_A_Y/G53_A_Z/G53_C_X/G53_C_Y` 所在 owner 的源 generation/hash 或规范化 geometry hash 作为可选诊断证据保存；若缺失、不新鲜或来源不可证明，本次设0保存仍按五项通过条件继续。后续 unwrap、checkpoint 恢复、runtime rebase 或 BUS Home 若自身需要 geometry 证明，必须在使用该能力前单独比对当前 RTCP/G53 geometry hash；该后续能力可以 fail-closed，但不得反向阻塞设置页本次设0保存。

BUS 保存成功不能只表示零点源数据已保存；同次产品动作必须完成硬盘 raw 限位写入。按钮成功响应必须诚实标明 live raw 限位和 UI/SHM 等效归零尚未验证，并要求 canonical clean restart handoff 后补齐 live 证明：

```text
code=SETTINGS_COUNT_DOMAIN_ZERO_SAVED_RESTART_REQUIRED
zero_applied=true
zero_display_verified=false
raw_limit_disk_saved=true
raw_limit_live_verified=false
raw_runtime_zero_verified=false
memory_zero_verified=false
pending_runtime_proof=true
backend_restart_required=true
canonical_clean_restart_required=true
restart_deferred=true
```

本次单轴设0保存通过必须来自同一次 settings_axis_zero 产品动作内的硬盘零点证据写入和硬盘 raw 限位写入。只改 UI 本地值、只写 `/run` result、只保存零点证据但未写硬盘 raw 限位，或缺 raw 读回、缺 `SCALE`、缺驱动 count-domain 读回、native/microkernel readback stale、机器忙时仍必须 fail-closed。允许的 SHM 坐标/UI 等效坐标不是 `0.000` 不得阻塞本次按钮保存，但必须作为 canonical clean restart handoff 后的 live 显示闭环条件。

## 5. Pulse/StepDir 设0

Pulse 模式按轴参数表当前行的单轴 `设零` 时必须弹出键盘；用户输入坐标值，这个值作为当前行当前轴 0 位写入 native/runtime owner：

```text
home_offset      = 键盘输入的当前轴 0 位坐标值，直线轴 mm，旋转轴 deg
```

规则：

- 当前 `step_ip_v1_5.contract.json` 或等价 Pulse contract 仍为 `cold_staged_not_runtime_selectable` 时，本文 Pulse 路线只作为冷态预配置/未来运行闭环要求，不能进入运行 apply；UI 强制提交和 Broker/C dispatch 必须 fail-closed。
- 只有 native active mode 已经由 `REQ-BUS-PULSE-MODE-DIFFERENCE` 证明为 Pulse，且 Pulse contract 已由 owner 更新为 `runtime_selectable` 后，下面的 Pulse `设零` 运行路径才允许执行。
- Pulse 普通设置页轴参数表 `设零` 请求必须带 finite `home_offset`，该值来自本次键盘输入，表示当前行当前轴 0 位坐标值。
- Pulse 普通设置页轴参数表 `设零` 不再要求用户输入 `home_direction`；回零方向、搜索速度、锁存速度和回零顺序属于独立回零参数 owner，不能由本次设零键盘临时改写。
- Pulse 轴参数表必须有独立 `回零方向` 列，通过下拉选择回零方向；该列写入 `home_direction` 对应 owner，供后续 LinuxCNC/HAL Home 使用。
- Pulse 写入后只可返回 pending restart/homing 或 owner write/readback 状态，不得声称当前 LinuxCNC 坐标已被 UI 立即改成 0。
- Pulse 不发送 EtherCAT SDO read/write，不读取 BUS zero_model，不写 `motion_driver.conf`。
- Pulse `home_offset` 必须和当前 runtime/native 软限位区间一起校验。后端提交前要读取目标 joint 当前 `MIN_LIMIT/MAX_LIMIT` 或对应 `soft_minus/soft_plus` owner 投影；当该轴软限位有效时，`home_offset` 必须落在允许区间内。若软限位 owner 缺失、区间非法、单位不可信，或 `home_offset` 超出有效软限位范围，必须返回 `SETTINGS_PULSE_HOME_OFFSET_OUT_OF_LIMITS`，不得写 runtime INI。软限位被明确配置为该方向不生效时，才能按 owner 语义跳过该方向限制，不能用 `±1_000_000` 宽限值替代真实软限位。

成功响应口径：

```text
code=SETTINGS_PULSE_HOME_OFFSET_RUNTIME_INI_WRITTEN
home_offset_saved=true
zero_captured=false
zero_applied=false
backend_restart_required=true
canonical_clean_restart_required=true
```

## 6. 请求契约

两种模式复用 `settings_axis_zero` action，但必须用不同 scope。

| 字段 | BUS/EtherCAT | Pulse/StepDir |
| --- | --- | --- |
| `action/native_action` | `settings_axis_zero` | `settings_axis_zero` |
| `request_id` / `idempotency_key` | 必填 | 必填 |
| `status_epoch` | 不提交；SHM/display epoch 不得作为设0保存准入或证明 | 不提交；SHM/display epoch 不得作为设0保存准入或证明 |
| `driver_mode` | `ethercat` / `bus` | `pulse`，但仅在 native active mode 已证明为 Pulse 且 contract 为 `runtime_selectable` 后允许进入运行 apply |
| `axis` / `joint` | 必须匹配 | 必须匹配 |
| `slave_index` / `slave` | 必须来自当前行 `从站` 列选中的从站号；后端按该从站读驱动，不按轴序猜测 | 不参与 Pulse 设零 |
| `target_scope` | `bus_count_domain_zero` | `pulse_mechanical_home_offset` |
| `apply_mode` | `count_domain_zero` | `persist_home_offset` |
| `home_offset` | 禁止作为请求源 | 必填 finite；来自键盘输入，表示当前轴 0 位坐标值 |
| `home_direction` | 不参与 BUS 设0或 BUS Home | 不由普通设置页 `设零` 键盘提交；由 Pulse 轴参数表 `回零方向` 列下拉选择并写独立回零参数 owner |

模式、scope 或字段冲突必须 fail-closed；不得尝试“兼容猜测”。

## 7. Apply 路线

### 7.1 BUS

BUS apply 必须在同一次设0动作里保存 count-domain 零点证据，并以 UI 限位距离为输入，同次计算写入新 0 位对应的硬盘 raw 限位坐标。UI 的 `0.000` 不能由 UI 独立置零；最终是否已经由 native/resident owner 基于 raw 机械坐标回读计算，并经 State Publisher 白名单显示投影得到 `0.000`，属于 canonical clean restart handoff 后的 live 闭环证明，不作为单次设0按钮保存通过条件。

允许状态：

- `disk_zero_saved_restart_required`：硬盘零点证据和硬盘 raw 限位已写入；需要 canonical clean restart handoff 后由 native 事实经允许的 SHM 坐标/UI 投影证明当前等效坐标归零。
- `zero_saved_but_not_applied`：只保存证据或显示诊断，必须作为失败/阻塞返回，不得作为成功。

禁止把 Route A 当生产路线：

```text
UI 本地私自置零，或 State Publisher/SHM 绕过 native/resident owner，直接用旧快照、磁盘字段或显示状态造 0
```

当前单轴设0生产路线是保存 count-domain 零点证据后，同次计算并写入硬盘 raw 限位坐标；canonical clean restart handoff 负责让 LinuxCNC/HAL/EtherCAT 装载这些硬盘 raw 限位，并由 native 事实经允许的 SHM 坐标/UI 投影证明等效坐标归零。不能只保存零点证据而把 raw 限位硬盘更新遗漏到最后。

### 7.2 Pulse

Pulse apply 固定为 homing 参数生成：

```text
settings_axis_zero request
  -> settings_apply/native owner
  -> runtime INI/native HOME owner
  -> [JOINT_N] HOME_OFFSET
  -> canonical clean restart handoff
  -> LinuxCNC homing
  -> homed 后建立机械坐标
```

当前 Pulse contract 仍为 `cold_staged_not_runtime_selectable` 时，上述路线只能作为未来运行闭环，不得由 UI/Broker 强行进入。Pulse 关闭条件不是字段写入成功，而是 runtime INI/native HOME owner、clean restart 后 un-homed、homing 后坐标、native active mode 与 UI 显示都闭合。

### 7.3 硬件 preset

驱动侧 position-basis mutation 不属于普通 BUS 单轴 `设0` 成功条件。若产品需要真正改写 EtherCAT 产品可见 raw 机械坐标，该能力必须有明确 profile/对象字典 allowlist、机器安全空闲、写入/保存/回读/必要重启校验闭环；不得作为独立隐藏维护 preset、临时脚本、旧路径或失败兜底保留。

## 8. Owner 文件

| 层级 | owner/锚点 | 责任 |
| --- | --- | --- |
| UI | 当前 v5 设置页 UI command owner 和键盘 owner | BUS 不输入值；Pulse 在 contract 允许时弹软键盘输入当前轴 0 位坐标值；请求带强类型字段 |
| Command Broker UDS/native dispatcher | 已登记 `settings_axis_zero` 请求入口 | 校验 action、mode/scope、axis/joint、Pulse 字段、幂等 TTL/LRU；不得把 SHM/display `status_epoch` 作为设0准入或证明 |
| settings/native apply owner | 已登记 BUS/Pulse axis-zero apply owner | BUS 读当前 raw 机械坐标、驱动 actual/split absolute，写 count-domain 证据，计算并写入硬盘 raw 限位，返回待统一重启；Pulse 构造 HOME apply 请求 |
| Settings apply | 已登记 settings apply / axis INI generation owner | BUS 不恢复旧字段；Pulse 在 contract 允许时写入当前轴 `HOME_OFFSET`；Pulse 轴参数表 `回零方向` 列写 `home_direction` owner，速度/顺序走各自 owner |
| SHM/UI 显示投影 | native/resident display projection owner -> State Publisher typed SHM | 只发布允许的坐标/轨迹、转速、进给速度和主轴/进给倍率显示投影；当前模态、active mode、count-domain、raw zero applied proof、homed、安全和 degraded control state 必须直连 native/microkernel owner，不得同步到 SHM |

大文件只保留调度/glue/兼容转发；新增业务 helper、状态机、解析器、表驱动或参数迁移写路径必须放到合适 owner 或新 sub-owner。

## 9. Fail-closed

以下情况必须拒绝：

- native/microkernel owner 状态不可用、动作准入状态来自 SHM/display epoch、或把 SHM 显示投影 stale/seq 校验失败当作设0保存准入。
- 缺 `axis`、`joint` 或轴关节不匹配。
- 缺 `driver_mode`、`motion_driver.conf` 损坏、模式与 scope 冲突。
- 机器运行、MDI 忙、Jog 活跃、Start 活跃、Homing 活跃；这些必须来自 native/microkernel fresh readback，不得来自 UI 文案、JSON、`/run` result 或 SHM。
- BUS 设置页设0路径出现上使能、idle、stillness、UI/SHM 显示归零等待、live raw limit 读回或 RTCP/G53 geometry fresh 要求等额外保存门禁。
- BUS 缺 public/private profile、轴-从站绑定、读回命令、驱动 actual/split absolute 读回或 active `SCALE`。
- BUS native/microkernel readback stale、机器正在运行、MDI/Jog/Start/Homing 活跃、`SCALE` 不可信、驱动 count-domain 读回失败，或 raw 读回完全不可用；前后 raw 微小 delta 只能作为诊断，不得单独作为阻塞条件。
- 后续 unwrap、checkpoint 恢复、runtime rebase 或 BUS Home 使用旋转 geometry 能力时，若当前 RTCP/G53 geometry hash 与可用零点诊断证据不一致，可在该后续能力内 fail-closed；设置页本次 BUS 设0保存不得因为 geometry 缺失、不新鲜或不一致而失败。
- BUS 请求试图写 `home_offset`、旧 `旧浮点零位字段/旧倒数比例字段`，或绕过 canonical native/microkernel gate 直接执行 hardware preset、`0x607C`、`0x1010`、drive rebase。
- BUS 设0后没有按新 0 位和 UI 限位距离同次计算并写入硬盘 raw 限位，或 `MIN_LIMIT/MAX_LIMIT`、`soft_minus/soft_plus` 区间非法、方向错误、距离被扩大/缩小。
- Pulse 缺 `home_offset`、非 finite，或试图把回零方向塞进普通设置页 `设零` 键盘路径。
- Pulse contract 仍为 `cold_staged_not_runtime_selectable`、native active mode 未证明为 Pulse，或强制绕过 `功能/2-1总线脉冲区别.md` 的 UI disabled / Broker fail-closed 约束。
- Pulse `home_offset` 超出当前有效 runtime/native 软限位区间，或软限位 owner/readback 不可信导致无法证明该 offset 安全。
- BUS Settings Drive 私域持久文件写入、fsync、rename、读回失败。
- Pulse runtime INI/native owner 写入、生成或读回失败。

建议错误码维持现有命名族：`SETTINGS_ZERO_*`、`SETTINGS_COUNT_DOMAIN_ZERO_*`、`SETTINGS_PULSE_*`。
