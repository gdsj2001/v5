## P0 BUS/EtherCAT 单轴设 0 最终规则

- BUS/EtherCAT 单轴设0是“保存新零位 + 设置页轻量预览 + 彻底重启生效”。后端读取当前映射从站的 count-domain 位置和 active `SCALE`，写入本次新 `zero_model`，并在同一写盘事务内按 `raw_min = raw_zero + 负软限位距离`、`raw_max = raw_zero + 正软限位距离` 重算硬盘软限位。随后重读编码器、本次零位和两项raw软限位，验证 `abs(current_encoder_physical - saved_zero_physical) <= 0.1 mm/deg` 且raw公式一致。设置页本轴显示值由 `(captured_counts - saved_zero_counts) / SCALE` 重算为本次设零值；第二次读取只用于漂移验证。本次不热切换native Router/Home/checkpoint/命令目标，不进入Command Gate写窗口；主页和控制链保持当前基准，直到“保存并重启”后才统一消费新零位。
- BUS `zero_model` 的唯一身份是本次真实读取的物理 `slave_position`，逻辑轴名只是当前映射和操作入口。运动模型切换事务写 descriptor 默认映射、轴参数 `从站` 下拉人工覆盖映射时，同一物理从站的零点证据都只随映射由新逻辑轴消费；不得读取目标逻辑轴历史旧零点、同时保留两个同从站零点或把另一个从站零点复制过来。一次设0成功写入某 `slave_position` 时，同一持久事务必须撤销其它逻辑轴上占用该从站的旧 `zero_model`，保证物理从站零点唯一。脉冲当量、软限位距离、速度等仍归目标逻辑轴自己的 AXIS owner；模型事务把它们同步装入目标 JOINT，人工从站覆盖不迁移这些参数。clean restart 后 HAL/native movement 必须同时消费最终映射和对应物理从站零点。
- 冷启动装载零点必须按物理从站逐轴独立投影：已设0且证据唯一、完整的轴必须进入 HAL/native `home_zero_counts`；其它未设0、旧映射或重复从站证据的轴保持 `home_ready=false`，只禁止该轴使用零点相关 Home，不得把已验证目标轴的零点也整表清零。轴-从站映射本身仍必须五轴完整、唯一；这条只放宽“尚未设0的轴”，不放宽映射错误。
- BUS/EtherCAT 单轴设0时，0位一变，硬盘 runtime INI/native owner 的软限位必须在同一次产品动作里跟随新0位计算并写入。LinuxCNC AXIS/JOINT live软限位可以在随后彻底clean restart装载；在此之前响应必须明确`raw_limit_live_verified=false`和restart pending，不得声称全部live生效。
- UI 软限位字段仍表示相对 0 位的物理距离；State Publisher/设置页必须继续显示这个距离，不得把已同步到 native/runtime 的 raw 限位坐标反向显示成 UI 距离。
- raw 限位落盘公式固定为 `raw_min_limit = raw_zero_position + ui_min_limit_distance`、`raw_max_limit = raw_zero_position + ui_max_limit_distance`。其中 `raw_zero_position` 对 BUS count-domain 零点等于 `zero_counts / active runtime INI SCALE`；螺距、减速比、目标精度、轴方向等比例链字段改变并导致 `SCALE` 改变时，必须用新 `SCALE` 同事务重算硬盘 raw 限位。
- 单次设0按钮只返回“零位与raw软限位已保存，设置页本轴已按新零位重算，全局待保存并重启”。当场通过条件是硬盘新零点/软限位及回读、`<=0.1 mm/deg`证据，以及由本次捕获计数和已回读零位计算为0的`settings_mcs_position`；不包含native commit或fresh native MCS。UI只缓存本次后端成功结果中的设置页投影，并在当前设置页该轴行持续显示；不主动刷新主页、WCS、轨迹或其它页面。
- 操作员可见的设0结果弹窗不得只表示“请求已收到”，也不得拿当前编码器去比旧硬盘零位。后端必须先读取本轴编码器/驱动 count-domain 当前位置，把该当前位置作为本次新 0 位记录到硬盘 `zero_model`；同一事务按 `raw_min = raw_zero + 负软限位距离`、`raw_max = raw_zero + 正软限位距离` 写入硬盘 raw limit。随后重新读取硬盘新 `zero_model`、`raw_min/raw_max` 和编码器当前位置，使用同一 active runtime INI/native `SCALE` 或已登记 `unit_per_count` 换算成物理单位后比较。直线轴单位为 `mm`，旋转轴单位为 `deg`；只有 `abs(current_encoder_physical - saved_zero_physical) <= 0.1`、两项 raw limit 公式和回读一致，且设置页本轴 `settings_mcs_position=(captured_counts-saved_zero_counts)/SCALE` 已由本次写盘证据重算为0时才提示保存成功。第二次编码器读数只验证本次保存期间没有超过容差的运动/漂移，不改变设置页本次零点显示。超过 `0.1 mm/deg`，或读数、换算、零位/软限位写入和回读任一缺失时，必须回滚本次持久文件并提示失败。
- 设置页单轴 `设0/设零` 的进程、成功、失败弹窗顶部标题必须显示具体轴，例如 `X 设0`、`A 设0`、`B 设0` 或 `X 设零`；不得显示泛化的 `单轴设0` 作为最终操作标题。轴号来自本次 UI 轴行请求或后端结构化 `axis` 字段，缺失时必须 fail-closed 或在正文说明轴号缺失，不得伪造成某一轴。BUS/EtherCAT 设0的驱动读取目标来自当前轴参数表该行 `从站` 列选中的从站号，UI 请求必须携带该从站号；该行 `从站=NAT` 表示没有物理驱动绑定，UI 必须禁用本行 `设0/设零` 且后台收到缺失/`NAT` 从站时必须 fail-closed；后端不得按 X/Y/Z/A/C 固定顺序、固定 AC 模型、轴号默认映射或 `settings_runtime.json` 旧映射猜测要读的驱动，旋转轴必须来自 fresh Active Model descriptor/readback。
- 从站下拉、比例链和BUS单轴设0都形成 restart pending。若映射或SCALE同时 dirty，设0必须拒绝并要求先保存重启，不能用旧映射/比例捕获新零位。
- BUS设0成功响应必须标明`settings_zero_display_verified=true`、`raw_limit_disk_saved=true`、`backend_restart_required=true`、`canonical_clean_restart_required=true`、`restart_deferred=true`和`raw_limit_live_verified=false`。`commit_seq`固定为0，`raw_runtime_zero_verified/memory_zero_verified/zero_display_verified`必须为false，避免把设置页预览写成native已生效。
- `settings_axis_zero`轻量保存路径不依赖`v5_command_gate_drive_window`或`v5_command_gate_server`，不得因Command Gate socket/ABI问题阻塞设0。Command Gate仍保留给设置驱动、运动和其它native写事务。
- 编码器 raw 位置允许轻微跳动；BUS 设0不得用前后 raw 双采样静止窗口作为阻塞门禁。可保留 raw delta 诊断，但不得因为微小编码器跳动返回 `SETTINGS_ZERO_AXIS_MOTION_ACTIVE`。

本文只维护设置页单轴 `设0/零位` 的本地请求契约、owner 和验收边界。BUS/Pulse 模式差异、模式切换闭环、Pulse 轴参数 `设零` 键盘和 `回零方向` 列的总口径以 `REQ-BUS-PULSE-MODE-DIFFERENCE` 及 `功能/2-1总线脉冲区别.md` 为准。

## 1. 接手边界

先确认三件事：

- 当前 active mode 是 BUS/EtherCAT 还是 Pulse/StepDir；`self_parameter_table.tsv` 中唯一的 `SETTINGS/bus_pulse_setting` 只能作为 requested/持久 owner，真实 active mode 必须按 `功能/2-1总线脉冲区别.md` 的 SelectLink/运行 INI/LinuxCNC/RT-HAL 四项并联证明进入 native/microkernel 运行内存。
- 本次动作是设置页单轴 `设0`，不是主页面 `回零`、Work Zero、驱动硬件 preset 或 position rebase。
- 当前字段是否会触碰 native/microkernel truth；若触碰，先按 `功能/需求真源索引.md` 找唯一 owner。

控制链固定为：

```text
UI C -> Command Broker UDS framed socket -> 已登记 settings/native apply owner -> LinuxCNC/HAL/EtherCAT/native owner
```

UI 不直接读 raw、不写 `settings_runtime.json`、不改 HAL/INI/驱动器对象。

## 2. 核心语义

`设0/零位` 在两种模式下不是同一个动作。

| 模式 | UI 文案 | 用户输入 | 后端动作 | 当前坐标是否立刻清零 | 是否写驱动 preset |
| --- | --- | --- | --- | --- | --- |
| BUS/EtherCAT | `设0` / `当前位置设为0` | 不输入数值 | 读取当前驱动count-domain位置，保存零点；同次按`raw_min=raw_zero+负软限位距离`、`raw_max=raw_zero+正软限位距离`更新硬盘软限位；重读验证后标记保存并重启 | 仅设置页本轴按新零位立即重算为0；主页和控制链待彻底重启生效 | 禁止普通设0直接写驱动preset；零位和raw软限位必须同次落盘并回读 |
| Pulse/StepDir | `设零` / `当前轴0位` | 弹出键盘，输入坐标值作为当前轴 0 位；轴参数表另有 `回零方向` 列下拉选择回零方向 | 写 runtime INI/native owner 的当前轴 `HOME_OFFSET`；`回零方向` 列写独立回零参数 owner | 禁止只靠 UI 立即改坐标；必须经 owner 写入和后续回零/native 证明 | 禁止 |

BUS 模式设置页轴参数表里的单轴 `设0` 不是回零动作，也不生成 Pulse homing 参数；Pulse 模式轴参数表里的单轴 `设零` 是把键盘输入的坐标值写成当前行当前轴 0 位对应的 `HOME_OFFSET`，不读取或复用 BUS count-domain 零点证据。两种模式的请求字段、后端 owner 和验收闭环必须分开。

当前阶段 Pulse/StepDir 仍受 `功能/2-1总线脉冲区别.md` 的 `cold_staged_not_runtime_selectable` 合同约束：Pulse 轴参数表 `设零` 路线只在 native active mode 已经是 Pulse 且 Pulse contract 被 owner 更新为 `runtime_selectable` 后才允许进入运行 apply。contract 仍为冷态预配置时，UI 目标应 disabled，任何强制提交都必须由 C/Broker dispatch fail-closed；不得借本文的 Pulse HOME_OFFSET 说明绕过 `2-1`。

危险动作分类（2026-07-20 修订）：BUS单轴设0不执行运动、不写驱动preset、不切换native运行零点，只写持久零位与raw软限位并更新设置页预览。因此本路径不自动下使能、不打开Command Gate写窗口、不恢复或改变使能状态。若本次编码器读取与写盘后复核超出`0.1 mm/deg`，则回滚持久文件并返回失败。该动作不弹危险二次确认。

硬规则：

- BUS设0先保存count-domain证据和硬盘raw软限位；UI只能用后端成功结果携带的`settings_mcs_position`更新设置页本轴，不得把该预览同步到主页或当作native运行事实。
- `旧浮点零位字段`、`旧倒数比例字段` 只能作为内存诊断表达式，不得重新落盘，不得作为 UI/State Publisher/settings/native apply 的产品输入。
- BUS 坐标显示和 BUS Home 必须使用 count-domain 证据 + active runtime INI/native `SCALE` 回读 + native/API 回读。UI 坐标值不是独立真相，必须由 native/resident owner 基于 LinuxCNC/HAL/EtherCAT 的真实 raw 机械坐标、当前零点、`SCALE` 和 A/C 等效相位规则计算，再经 State Publisher 发布允许的坐标显示投影；只改 UI 本地状态、旧快照或 SHM 字段不能算通过。
- Pulse 不把 `home_offset` 或 `pulse_home_model` 写入 Settings Drive 私域持久文件；它们属于 runtime INI/native owner 的一次性 apply 请求。
- Pulse禁止普通`设零`路径写EtherCAT SDO `0x607C`、`0x1010`、驱动preset或rebase。BUS普通`设0`也不改驱动raw位置；只更新持久零位和与raw软限位，native反馈/目标双向换算等彻底重启后由owner统一装载。

## 4. BUS/EtherCAT 设0

BUS模式普通`设0`是“持久零位与raw软限位 + 设置页本轴预览 + 保存并重启”的单一轻量事务，不是Pulse回零。本次不进入Command Gate写窗口，不改变Machine enable，不热切换native零位消费者。只有后端落盘并重读验证成功后，UI才可在设置页局部显示该轴的新零位结果。LinuxCNC/HAL Router、Home、checkpoint、主页坐标和live软限位由随后的彻底重启统一装载。

### 4.1 操作流程

BUS/EtherCAT 设置页单轴 `设0` 的过程固定如下：

1. UI 在当前轴行点击 `设0` 后，只发送 `settings_axis_zero` 请求；请求字段必须包含 `axis/joint`、当前行 `从站` 列提取出的 `slave_index`、`driver_mode=bus|ethercat`、`target_scope=bus_count_domain_zero`、`apply_mode=count_domain_zero`，不得携带 `home_offset`，不得打开软件盘，不得直接写驱动 preset。
2. 后端按本次请求的 `slave_index` 读取当前 active BUS 从站和 profile；缺轴、缺从站号、模式不是 BUS/EtherCAT、profile 未加载、从站身份不匹配或 `drive.read_actual_position` 不可用时 fail-closed。不得用历史轴序映射代替UI当前从站选择。
3. 后端读取目标从站的当前 count-domain 位置和对应 active `SCALE/unit_per_count`。有符号对象必须按 profile 类型解析；A/C 等旋转轴仍保留绝对多圈basis证据要求。
4. 后端把设0瞬间的count-domain读数保存为硬盘`zero_model`，并在同一持久事务内撤销其它逻辑轴上同`slave_position`的旧零点，再计算与写入`raw_min = raw_zero + 负软限位距离`、`raw_max = raw_zero + 正软限位距离`。旧零位只用于还原原相对限位距离；只保存零点、未重算两项raw软限位或留下同从站双零点是失败。
5. 写盘完成后重读当前编码器、本次新`zero_model`、`raw_min/raw_max`。直线轴用mm，旋转轴用deg；按同一SCALE验证零位差值不超过`0.1 mm/deg`，并验证两项raw软限位与公式一致。失败时只回滚本次已触及的持久文件。
6. 成功结果计算`settings_mcs_position = (captured_counts - saved_zero_counts) / SCALE`并验证为本次设零值，携带轴号、从站号、新零位、两项raw软限位及`restart_deferred=true`。第二次`current_counts`只用于`<=0.1 mm/deg`漂移复核。UI只保留当前设置页对应轴的该值，不写SHM、不更新主页。
7. 本次不调用Command Gate live axis-zero，不要求Machine Off，不切换Router/Home/checkpoint或native命令目标；弹窗只能写“已保存，设置页已重算，待保存并重启”。
8. 操作员执行“保存并重启”后，native/Router/Home/checkpoint/命令目标和live软限位才加载新零位；启动 owner 必须把本次已验证轴独立投影为 `home_ready=true`，其它缺零点证据的轴保持 `home_ready=false` 而不阻断本轴。重启后再用native MCS、HAL `home_zero_counts`和raw限位回读验收本轴全局生效。

该流程中的“设0成功”只表示硬盘零位/raw软限位已闭合，且设置页本轴预览已按后端计算值更新。重启完成前不得声称native零点表、Router反馈/目标、Home/checkpoint或live软限位已生效。

基本模型：

```text
pre_zero_machine_position = 设0瞬间 LinuxCNC/HAL/EtherCAT/native 产品可见机械坐标
zero_shift                = pre_zero_machine_position
captured_counts           = 设0瞬间捕获并保存的驱动/编码器反馈计数
current_counts            = 写盘后第二次驱动/编码器反馈计数，仅用于漂移复核
zero_counts               = 设0瞬间保存的 captured_counts
active_runtime_SCALE      = active runtime INI/native owner 回读的 count/单位
post_raw_machine_position  = pre_zero_machine_position
settings_mcs_position      = (captured_counts - zero_counts) / active_runtime_SCALE
runtime_mcs_before_restart = 保持旧native基准，不被本次设置页预览修改
ui_min_limit_distance      = UI 设置的负向限位物理距离，坐标值相对 0 位
ui_max_limit_distance      = UI 设置的正向限位物理距离，坐标值相对 0 位
post_min_limit_raw         = zero_shift + ui_min_limit_distance
post_max_limit_raw         = zero_shift + ui_max_limit_distance
```

设置页本轴的即时显示值必须是后端用本次捕获值和已回读零位计算出的`settings_mcs_position`，不得由UI脱离后端成功结果自行将数字写死为0。主页和其它页面继续显示native/State Publisher事实；旋转轴等效角和checkpoint等彻底重启后由native重建。

UI 里的限位设置只是相对于 0 位的物理距离，不是 LinuxCNC/HAL/EtherCAT 的绝对 raw 限位坐标。零位改变时，UI 限位距离值不自动改写；后端必须按新 0 位和这些距离计算 raw 限位，并在同一次设0动作内由受控 runtime/native owner 写入硬盘 `MIN_LIMIT/MAX_LIMIT`、`soft_minus/soft_plus` 或等价 raw limit 字段。只改 UI 显示限位、只改旧快照或只记录计划但不改硬盘 raw 限位不是闭合；live底层限位允许在响应明确要求的彻底clean restart后装载和回读。

A/C 绝对多圈轴还必须保存：

```text
zero_abs_turns
zero_abs_single_turn_counts
zero_abs_counts
raw_bits/raw_modulus/raw_value_type/raw_modulus_trust
```

驱动 profile 必须提供：

- `drive.read_actual_position`
- A/C 绝对多圈轴还必须提供 `drive.read_abs_multi_turn`
- A/C 绝对多圈轴还必须提供 `drive.read_abs_single_turn`
- active model 绝对多圈旋转轴还必须提供 `drive.read_abs_position`，可由 Adapter 用 split 读数按映射位宽无损合成

X/Y/Z 线性轴设 0 的驱动读回只能读取本轴 `drive.read_actual_position` 这一条 count-domain 锚点，不得为了普通线性设 0 串行读取 statusword、mode、绝对编码器、电齿轮等全量诊断字段；缺 profile、缺轴-从站绑定、缺 `drive.read_actual_position` 或单条读回失败时直接 fail-closed，不得退回全量读回、扫描全站或旧兼容路径。A/C 旋转轴设 0 仍必须读取绝对多圈 basis；RTCP/G53 geometry 可在可用时保存为诊断证据，但缺失、不新鲜或不可证明时不得阻塞本次设0保存。

split 合成不得把 `uint16` 圈数直接强转有符号值；运行时必须使用相对零点差分取模公式。缺 profile、缺轴-从站绑定、缺读回命令、读回失败、`SCALE` 未经 active owner 回读、native/microkernel readback stale 或机器忙时，必须 fail-closed。

BUS 设0不以Machine Off或Command Gate写窗口为前置，但后端仍必须做写盘前后两次编码器读回和`0.1 mm/deg`物理容差校验；这不是静止窗口，只是确保保存零位与设置页预览对应同一位置。编码器轻微跳动只要未超容差就不拒绝。本次成功路径必须完整经过“记录硬盘零位 -> 同次计算与写入raw_min/raw_max -> 重读编码器/零位/raw软限位 -> 同一SCALE验证 -> 返回settings_mcs_position -> UI局部刷新 -> 保存并重启待办”。

A/C 绝对旋转轴的 `zero_model` 不以 RTCP/G53 回转几何身份作为设0保存门禁。若设0瞬间同一帧状态已经提供可信 geometry，可把 `G53_A_Y/G53_A_Z/G53_C_X/G53_C_Y` 所在 owner 的源 generation/hash 或规范化 geometry hash 作为可选诊断证据保存；若缺失、不新鲜或来源不可证明，本次设0保存仍按本文的零位/raw软限位写入、回读、重算和容差验证闭环继续。后续 unwrap、checkpoint 恢复、runtime rebase 或 BUS Home 若自身需要 geometry 证明，必须在使用该能力前单独比对当前 RTCP/G53 geometry hash；该后续能力可以 fail-closed，但不得反向阻塞设置页本次设0保存。

BUS设0成功响应必须来自同一动作内的零位/raw软限位写入与回读、编码器复核和设置页位置计算：

```text
code=SETTINGS_COUNT_DOMAIN_ZERO_SAVED_RESTART_REQUIRED
zero_saved=true
settings_zero_display_verified=true
settings_mcs_position=0.0
raw_limit_disk_saved=true
raw_limit_live_verified=false
raw_runtime_zero_verified=false
memory_zero_verified=false
zero_display_verified=false
backend_restart_required=true
canonical_clean_restart_required=true
restart_deferred=true
commit_seq=0
```

只改UI、只写`/run`结果、只保存零位而未同步写入与回读raw_min/raw_max，均必须fail-closed并回滚持久文件。native和live软限位未装载是预期状态，必须保持restart pending且不得标为verified。

## 5. Pulse/StepDir 设0

Pulse 模式按轴参数表当前行的单轴 `设零` 时必须弹出键盘；用户输入坐标值，这个值作为当前行当前轴 0 位写入 native/runtime owner：

```text
home_offset      = 键盘输入的当前轴 0 位坐标值，直线轴 mm，旋转轴 deg
```

规则：

- 当前 `step_ip_v1_5.contract.json` 或等价 Pulse contract 仍为 `cold_staged_not_runtime_selectable` 时，本文 Pulse 路线只作为冷态预配置/未来运行闭环要求，不能进入运行 apply；UI 强制提交和 Broker/C dispatch 必须 fail-closed。
- 只有 native active mode 已经由 `REQ-BUS-PULSE-MODE-DIFFERENCE` 证明为 Pulse，且 Pulse contract 已由 owner 更新为 `runtime_selectable` 后，下面的 Pulse `设零` 运行路径才允许执行。
- Pulse 普通设置页轴参数表 `设零` 请求必须带 finite `home_offset`，该值来自本次键盘输入，表示当前行当前轴 0 位坐标值。
- Pulse 普通设置页轴参数表 `设零` 不再要求用户输入 `home_direction`；回零方向、搜索速度、锁存速度和回零顺序属于独立回零参数 owner，不能由本次设零键盘临时改写。
- Pulse 轴参数表必须有独立 `回零方向` 列，通过下拉选择回零方向；该列写入 `home_direction` 对应 owner，供后续 LinuxCNC/HAL Home 使用。
- Pulse 写入后只可返回 pending restart/homing 或 owner write/readback 状态，不得声称当前 LinuxCNC 坐标已被 UI 立即改成 0。
- Pulse 不发送 EtherCAT SDO read/write，不读取 BUS zero_model，不改 `SETTINGS/bus_pulse_setting`。
- Pulse `home_offset` 必须和当前 runtime/native 软限位区间一起校验。后端提交前要读取目标 joint 当前 `MIN_LIMIT/MAX_LIMIT` 或对应 `soft_minus/soft_plus` owner 投影；当该轴软限位有效时，`home_offset` 必须落在允许区间内。若软限位 owner 缺失、区间非法、单位不可信，或 `home_offset` 超出有效软限位范围，必须返回 `SETTINGS_PULSE_HOME_OFFSET_OUT_OF_LIMITS`，不得写 runtime INI。软限位被明确配置为该方向不生效时，才能按 owner 语义跳过该方向限制，不能用 `±1_000_000` 宽限值替代真实软限位。

成功响应口径：

```text
code=SETTINGS_PULSE_HOME_OFFSET_RUNTIME_INI_WRITTEN
home_offset_saved=true
zero_captured=false
zero_applied=false
backend_restart_required=true
canonical_clean_restart_required=true
```

## 6. 请求契约

两种模式复用 `settings_axis_zero` action，但必须用不同 scope。

| 字段 | BUS/EtherCAT | Pulse/StepDir |
| --- | --- | --- |
| `action/native_action` | `settings_axis_zero` | `settings_axis_zero` |
| `request_id` / `idempotency_key` | 必填 | 必填 |
| `status_epoch` | 不提交；SHM/display epoch 不得作为设0保存准入或证明 | 不提交；SHM/display epoch 不得作为设0保存准入或证明 |
| `driver_mode` | `ethercat` / `bus` | `pulse`，但仅在 native active mode 已证明为 Pulse 且 contract 为 `runtime_selectable` 后允许进入运行 apply |
| `axis` / `joint` | 必须匹配 | 必须匹配 |
| `slave_index` / `slave` | 必须来自当前行 `从站` 列选中的从站号；后端按该从站读驱动，不按轴序猜测 | 不参与 Pulse 设零 |
| `target_scope` | `bus_count_domain_zero` | `pulse_mechanical_home_offset` |
| `apply_mode` | `count_domain_zero` | `persist_home_offset` |

模式、scope 或字段冲突必须 fail-closed；不得尝试“兼容猜测”。

## 7. Apply 路线

### 7.1 BUS

BUS apply必须保存count-domain零点，并同次按公式更新与回读硬盘`raw_min/raw_max`。UI的`0.000`不能独立置零，只能来自后端已验证的`settings_mcs_position`；该值仅覆盖设置页本轴，全部live零点消费者等待保存并重启。

允许状态：

- `zero_saved_restart_pending`：零位和raw软限位已写入/回读，设置页本轴已重算，native待重启。
- `zero_save_failed`：编码器、SCALE、写盘、零位回读、raw软限位公式/回读任一失败，回滚持久文件并返回原错误。

禁止把 Route A 当生产路线：

```text
UI 本地私自置零，或 State Publisher/SHM 绕过 native/resident owner，直接用旧快照、磁盘字段或显示状态造 0
```

当前生产路线是硬盘零位/raw软限位与设置页局部预览；canonical clean restart是正常设0全局生效步骤。

### 7.2 Pulse

Pulse apply 固定为 homing 参数生成：

```text
settings_axis_zero request
  -> settings_apply/native owner
  -> runtime INI/native HOME owner
  -> [JOINT_N] HOME_OFFSET
  -> canonical clean restart handoff
  -> LinuxCNC homing
  -> homed 后建立机械坐标
```

当前 Pulse contract 仍为 `cold_staged_not_runtime_selectable` 时，上述路线只能作为未来运行闭环，不得由 UI/Broker 强行进入。Pulse 关闭条件不是字段写入成功，而是 runtime INI/native HOME owner、clean restart 后 un-homed、homing 后坐标、native active mode 与 UI 显示都闭合。

### 7.3 硬件 preset

驱动侧 position-basis mutation 不属于普通 BUS 单轴 `设0` 成功条件。若产品需要真正改写 EtherCAT 产品可见 raw 机械坐标，该能力必须有明确 profile/对象字典 allowlist、机器安全空闲、写入/保存/回读/必要重启校验闭环；不得作为独立隐藏维护 preset、临时脚本、旧路径或失败兜底保留。

| settings/native apply owner | 已登记 BUS/Pulse axis-zero apply owner | BUS读驱动actual，写与回读count-domain零位及按公式重算的硬盘raw软限位，返回设置页本轴计算值与restart pending；Pulse构造HOME apply请求 |
| Settings apply | 已登记 settings apply / axis INI generation owner | BUS 不恢复旧字段；Pulse 在 contract 允许时写入当前轴 `HOME_OFFSET`；Pulse 轴参数表 `回零方向` 列写 `home_direction` owner，速度/顺序走各自 owner |

## 9. Fail-closed

以下情况必须拒绝：

- native/microkernel owner 状态不可用、动作准入状态来自 SHM/display epoch、或把 SHM 显示投影 stale/seq 校验失败当作设0保存准入。
- 缺 `axis`、`joint` 或轴关节不匹配。
- 缺 `driver_mode`、`SETTINGS/bus_pulse_setting` 缺失/重复/未知、模式与 scope 冲突。
- BUS设置页设0路径出现自动上/下使能、Command Gate live axis-zero、Router/Home/checkpoint热切换、UI/Broker自造后端成功结果或RTCP/G53 geometry门禁。
- BUS 缺 public/private profile、轴-从站绑定、读回命令、驱动 actual/split absolute 读回或 active `SCALE`。
- BUS `SCALE`不可信、驱动 count-domain 读回失败，或 raw 读回完全不可用；前后 raw 差值未超`0.1 mm/deg`时不得因微小跳动拒绝。
- 后续 unwrap、checkpoint 恢复、runtime rebase 或 BUS Home 使用旋转 geometry 能力时，若当前 RTCP/G53 geometry hash 与可用零点诊断证据不一致，可在该后续能力内 fail-closed；设置页本次 BUS 设0保存不得因为 geometry 缺失、不新鲜或不一致而失败。
- BUS 请求试图写 `home_offset`、旧 `旧浮点零位字段/旧倒数比例字段`，或绕过 canonical native/microkernel gate 直接执行 hardware preset、`0x607C`、`0x1010`、drive rebase。
- BUS 设0后没有按新 0 位和 UI 限位距离同次计算并写入硬盘 raw 限位，或 `MIN_LIMIT/MAX_LIMIT`、`soft_minus/soft_plus` 区间非法、方向错误、距离被扩大/缩小。
- BUS设0后没有重读编码器/硬盘零位并验证`<=0.1 mm/deg`，没有按公式同次写入与回读`raw_min/raw_max`，没有返回已验证`settings_mcs_position`，或没有明确保存并重启要求。
- Pulse 缺 `home_offset`、非 finite，或试图把回零方向塞进普通设置页 `设零` 键盘路径。
- Pulse contract 仍为 `cold_staged_not_runtime_selectable`、native active mode 未证明为 Pulse，或强制绕过 `功能/2-1总线脉冲区别.md` 的 UI disabled / Broker fail-closed 约束。
- Pulse `home_offset` 超出当前有效 runtime/native 软限位区间，或软限位 owner/readback 不可信导致无法证明该 offset 安全。
- BUS Settings Drive 私域持久文件写入、fsync、rename、读回失败。
- Pulse runtime INI/native owner 写入、生成或读回失败。

建议错误码维持现有命名族：`SETTINGS_ZERO_*`、`SETTINGS_COUNT_DOMAIN_ZERO_*`、`SETTINGS_PULSE_*`。

| 模式 | 板端关闭条件 |
| --- | --- |
| BUS/EtherCAT | 完整受影响产物上板；active mode为BUS；记录目标轴当前编码器计数、旧zero_model和软限位距离；点击`设0`后同次UDS结果证明新zero_model与编码器差值`<=0.1 mm/deg`，同物理从站旧零点已撤销，`raw_min=raw_zero+负软限位距离`、`raw_max=raw_zero+正软限位距离`已落盘与回读；设置页只有该轴立即显示`0.000`，主页/WCS/轨迹不被本次预览刷新；响应为`SETTINGS_COUNT_DOMAIN_ZERO_SAVED_RESTART_REQUIRED`且`commit_seq=0`；随后执行“保存并重启”，重启后目标轴 HAL `home_zero_counts`与native MCS必须消费新零位，其它未设0轴只保持`home_ready=false`而不得整表阻断，Router/Home/checkpoint和live软限位回读一致 |
| Pulse/StepDir | contract 已由 owner 更新为 `runtime_selectable` 且 native active mode 为 Pulse；设置页轴参数表点击当前行当前轴 `设零` 后弹出键盘；键盘输入坐标值作为当前轴 0 位；轴参数表 `回零方向` 列可下拉选择回零方向并写 `home_direction` owner；后端证明 `home_offset` 在当前有效 `MIN_LIMIT/MAX_LIMIT` 或对应 soft limit owner 区间内；越界输入返回 `SETTINGS_PULSE_HOME_OFFSET_OUT_OF_LIMITS`；canonical clean restart handoff 后 runtime INI/native HOME owner 目标 joint 的 `HOME_OFFSET` 和 `home_direction` 字段正确；LinuxCNC homing 后坐标建立符合该 0 位；普通设置页轴参数表 `设零` 不提交回零方向；无 EtherCAT SDO read/write。若 contract 仍为 `cold_staged_not_runtime_selectable`，验收只允许证明 UI disabled 与 Broker/C fail-closed，不允许声称 Pulse 设零运行闭环通过 |

涉及真实运动、小增量 MDI/G-code、Home/Jog/Start/E-stop 或运动 gate 放行时，还必须按 `REQ-AUTO-STATE-DRIVEN` 完成 native active model 对应的 `cc-ac.ngc` 或 `cc-bc.ngc` 黄金运动闭环。

## 12. 关闭条件

关闭前必须说明：

- 当前模式、请求payload、响应码和saved/fail-closed状态；BUS设0当场成功码为`SETTINGS_COUNT_DOMAIN_ZERO_SAVED_RESTART_REQUIRED`，明确只有设置页预览已重算。
- BUS驱动读回、profile hash、zero count-domain、active `SCALE`、硬盘`raw_zero/raw_min/raw_max`公式与回读、设置页本轴`settings_mcs_position`和UI新鲜帧证明；彻底重启后另补fresh native MCS、Router/Home/checkpoint与live软限位读回。
- Pulse 的当前 contract、active mode、当前轴 0 位坐标值、`HOME_OFFSET` 写入、`回零方向` 下拉选择写入、canonical clean restart handoff 和 homing 证明。
- 本地测试、板端截图/日志/状态证据和最终安全态。
- 未闭合事项只写真实缺口，不保留过程日志。
