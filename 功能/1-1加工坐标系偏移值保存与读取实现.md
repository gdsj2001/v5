# 加工坐标系偏移值保存与读取实现

引用需求真源：`REQ-DOC-SINGLE-SOURCE`、`REQ-WCS-NATIVE-OWNER`、`REQ-NATIVE-OWNER-FIRST`、`REQ-LINUXCNC-COMMAND-GATE`、`REQ-PARAM-MEMORY-LIGHTWEIGHT-SAVE`、`REQ-SHM-DISPLAY-PROJECTION`、`REQ-SETTINGS-RUNTIME-DRIVE-ONLY`、`REQ-G92-NATIVE-RUNTIME`、`REQ-RTCP-G53-NATIVE-ACTUAL`。

## AI 阅读入口

- 启动内存/热路径通用规则：见 `REQ-PARAM-MEMORY-LIGHTWEIGHT-SAVE` / `功能/0-1开机参数入内存.md`，本文只保留本功能特有边界。
- 本文定位：WCS/G54-G59.3、Work Zero、当前活动 WCS 和全表显示的 owner。
- 先读 `REQ-WCS-NATIVE-OWNER`，涉及 G92、RTCP/G53 或参数真源时再跳到对应 REQ owner。
- WCS 持久化真源是 LinuxCNC `PARAMETER_FILE`，但它必须在开机或受控重载时一次性进入微内核/native resident memory；运行真源是解释器/native memory/API。偏置页、主页面、刀路和 Work Zero 回读只消费 resident memory/API，不得运行期解析 `PARAMETER_FILE`、旧 `/run` 快照、JSON、SHM 字段或 UI 缓存作为显示真源。SHM 只允许承载坐标、转速、进给速度和主轴/进给倍率等显示投影；当前模态、当前刀号和刀长直连 native modal/tool 状态块或等价 API，不承载 WCS owner。


## 1. 结论

WCS / G54-G59.3 偏移属于 LinuxCNC/native owner 参数域。

| 范围 | 真源 | v5 允许用途 | 禁止 |
| --- | --- | --- | --- |
| 当前 active WCS | LinuxCNC interpreter runtime `stat.g5x_index` | WCS provider 刷新 resident WCS snapshot/API，UI 只合并 fresh snapshot 显示 | UI 按钮选中态、Broker result、SHM 字段或私有缓存当真源 |
| 当前 WCS offset | LinuxCNC runtime `stat.g5x_offset` | WCS provider 刷新 resident WCS snapshot/API，UI 只合并 fresh snapshot 显示 | UI 本地计算、SHM 字段或旧 JSON 补洞 |
| G54-G59.3 全表 | 开机/受控重载装载后的微内核/native resident WCS 参数表；持久 owner 为 LinuxCNC `PARAMETER_FILE` | `PARAMETER_FILE` 只用于开机装载、保存写回和显式诊断核对；偏置页稳定显示必须消费 resident WCS snapshot/native/interpreter/API fresh 回读 | UI/provider 运行期解析文件、读取旧 `/run` 快照，或把参数文件冷证据当 UI 显示真源 |
| Work Zero 写入 | 登记 native/linuxcncrsh gate 执行 G10/等价接口 | UI/native dispatcher 只做请求组帧、等待 native/runtime/API 回读 | 直接手改参数文件或 Python/脚本中介直接 `linuxcnc.command()` |

WCS 成功标准不是“命令已发送”或“文件 mtime 变化”，而是 LinuxCNC 写入后经：

```text
LinuxCNC/runtime -> native/interpreter API -> UI/provider
```

回读确认。

## 2. 三条链路

### 2.1 热路径：当前活动 WCS

热路径只服务高频 UI 显示：

- WCS provider 必须维护当前 `active_wcs`、`active_wcs_offset`、`wcs_generation` 和 freshness；UI 30Hz 刷新不得每帧直连 LinuxCNC/native poll，也不得运行期解析 `PARAMETER_FILE`。
- UI 坐标显示可使用 SHM `mcs/cmd_mcs` 投影；WCS 名称、偏移值和刀路坐标基必须来自 fresh native/interpreter WCS snapshot。主页面加工坐标列必须在 UI scene 收敛阶段配对 `position_epoch` 与 `wcs_generation` 后计算：`wcs_pos[i] = mcs[i] - active_wcs_offset[i]`。MCS 来自 SHM 显示帧，WCS offset 来自 native WCS snapshot；两者不能写成同一 SHM 发布者。若任一侧 stale、generation 缺失或超过 freshness 窗口，只允许 degraded/fail-hold，不得用 `cmd_mcs`、旧 offset、UI 缓存、默认 0 或非同源快照参与计算。
- 机械坐标列和刀路 actual 机械点必须来自同一帧 `mcs` 内存投影。
- `cmd_mcs` 可用于红色 commanded 点和跟随误差，但不能回填机械坐标。

### 2.2 冷路径：G54-G59.3 全表

冷路径只服务开机装载、受控重载、诊断、刀路全表准备和写入后证据；偏置页稳定显示必须消费开机已装载的 resident WCS 参数表和 native/interpreter/API fresh 回读，不能在运行期直接把 `linuxcnc.var` 当显示真源：

- 全表读取不得进入 UI 热循环。
- WCS provider 启动时必须把 G54-G59.3 全表装入 resident memory；后续刷新由 WCS owner 按 generation、Work Zero/WCS 切换事件、受控重载或低频健康刷新触发。UI 读取只访问 resident WCS snapshot/API，不能把全表刷新塞进 30Hz 坐标显示循环。
- Work Zero 或等价 G10 写入后，必须由 native/interpreter readback 刷新 resident memory 中对应 WCS 行，再确认 UI 显示。
- 当前 WCS native/provider ABI 使用 `wcs_offsets[9][5]`；这不是 `/dev/shm/v3_status_shm` ABI。扩展原生 9 轴必须升 WCS provider ABI version 或新增字段组，仍不得把 WCS 全表写入 SHM。
- 不得通过连续切换 G54/G55/G56 来探测全表，因为 WCS 切换本身是控制动作。
- `linuxcnc.var` 的 mtime/hash/key/epoch 是持久化证据，不是同帧 UI 成功证据。
- `wcs_offsets_epoch` 只在全表刷新成功后递增。

### 2.3 写入路径：Work Zero

写入只保留一条路：

```text
UI Work Zero
  -> UI C 组帧
  -> UDS native command gate
  -> 单 writer/FIFO native/linuxcncrsh gate
  -> native/LinuxCNC/HAL owner 校验 fresh 状态 / WCS / axis / native idle / RTCP OFF / 同源刀补模态并执行 LinuxCNC G10/等价接口
  -> LinuxCNC/微内核 WCS 持久 owner 把所选轴偏移写入 `PARAMETER_FILE` 并更新 runtime
  -> native/interpreter owner 从源位置回读对应 WCS 行，刷新 resident WCS memory/API 并递增 generation
  -> UI/provider 只消费 fresh resident memory/API，重新计算并刷新该轴加工坐标
```

UI 不得直接写参数文件、V3 私有持久文件或临时 JSON 来伪造成功。
这里的 `idle` 只能由 native/LinuxCNC gate 基于原生状态判定；不得由 V3 速度阈值、SHM、JSON、Broker result、`/run` 文件或 UI 状态拼出。

主页面 `归零` 的唯一语义是单轴 Work Zero：操作员从机械坐标列或加工坐标列点选一个轴时，列别只用于确定 UI 选中位置，写入源始终是该轴的 fresh native/机械坐标。点击 `归零` 后，native/LinuxCNC owner 必须把当前机械位置按 LinuxCNC 原生 `G10 L20 P<当前 active WCS> <轴>0` 或严格等价接口写入当前 active G54-G59.3 的对应轴偏移；该偏移必须进入 LinuxCNC/微内核 WCS 持久参数文件 `PARAMETER_FILE`，随后由 native/interpreter owner 从源位置回读并刷新 resident WCS 内存/API，UI 再消费新的 generation 重新计算该轴加工坐标并显示为 `0`。本质上是把当前选定轴的机械坐标关系记录到当前加工坐标系偏移，不是修改机械坐标、G92、Home、设置页设0或驱动零点。UI 不得在命令发送或文件写入后直接本地赋值为 0，不得运行期自行读取参数文件，也不得把加工坐标显示值、SHM 投影、旧 offset 或按钮缓存作为写入值；未点选单轴、选择全轴、active WCS 不明、机械坐标/offset readback 不 fresh、参数文件源位置写入未确认、resident generation 未更新或写入后该轴 WCS 未回读为 0 时必须 fail-closed。

## 3. 字段 owner 卡

| 字段/动作 | 禁止的第二真源 | 目标 owner | native/API 回读 | 退休条件 |
| --- | --- | --- | --- | --- |
| active WCS | UI 按钮态、Broker result、私有 active WCS cache、SHM 字段 | LinuxCNC interpreter runtime | `stat.g5x_index`、`stat.g5x_offset` 同源回读 | 产品路径不读写 active WCS 私有文件 |
| G54-G59.3 offset | V3 WCS JSON、UI dirty、`/run` snapshot、SHM config cache | LinuxCNC `PARAMETER_FILE` 与 resident WCS table/native API | 当前 active WCS 用 `stat.g5x_offset`；G54-G59.3 全表必须由 resident WCS table/native API 回读，参数文件 key/epoch 只作持久化冷证据 | 旧写入 owner、读取入口、allowlist 和示例删除 |
| Work Zero 结果 | MDI/G10 accepted、Broker JSON、参数文件 mtime 单点变化、SHM 显示投影 | 所选单轴 fresh 机械坐标关系写入当前 active WCS 对应轴 offset，LinuxCNC runtime 与参数文件一致 | active WCS、所选轴 offset、所选轴当前 WCS=`0`、tool-tip/WCS 语义同源连续确认 | 未回读前不得清 dirty 或显示成功 |

## 4. UI 消费

UI 消费分两类：坐标显示只消费允许的 SHM 显示投影；WCS/offset 消费 native/interpreter owner 维护的 resident WCS snapshot/API：

- 坐标面板、偏置页机械坐标、设置页机械坐标、刀路绿色机械点、实际刀尖/刀柄反馈点和 MCS 基础投影使用 `mcs`。
- 主页面加工坐标列使用当前同源 `mcs - active_wcs_offset`；它必须始终与机械坐标相差当前 active WCS offset。Work Zero 后 offset 改变，加工坐标应随 native/interpreter 回读更新到当前点为 0；普通轴运动或手转反馈变化时 offset 不变，加工坐标应随机械坐标等量变化。
- WCS 名称和 offset 使用 fresh native/interpreter WCS snapshot，不从 SHM 字段消费；WCS snapshot 必须带 `wcs_generation`、freshness 和 active WCS 标识。
- 主页面启动后不得只做一次 WCS/native readback；继电器断电重启或发布器晚于 UI 就绪时，WCS provider 必须继续低频读取 native WCS/offset 并刷新 resident snapshot，UI 主循环只合并 fresh snapshot 到当前 scene，再用 `mcs - active_wcs_offset` 显示加工坐标。
- UI G-code 黄色轨迹线使用程序文本自身解析出的 G54-G59.3 模态选择 WCS offset：程序点 XYZ 先加上该点所属 WCS 的 native/interpreter offset，得到机械坐标显示点后再投影。若只有当前 active WCS offset 可用，则仅当程序点所属 WCS 与当前 active WCS 相同才允许显示黄色线；如果程序使用其它 WCS 且没有全表 offset 回读，必须 fail-closed 隐藏/等待 offset，而不能按当前 WCS 或默认 G54 代画。
- 非活动 WCS 缺全表时不得用 0、旧 offset 或 active offset 猜位置，应显示 `wcs-ofs--` 或 degraded。
- 显示层不做抗抖、死区保持、迟滞、连续样本确认或运动窗口缓存。
- 坐标面板显示不做数值滤波，但必须 fail-hold：`snapshot_stale`、`status_epoch=0`、非有限数或占位坐标帧不得覆盖上一帧有效 MCS 显示值；WCS 名称、offset 和 Work Zero 结果仍必须来自 native/interpreter readback，不能由 SHM `status_epoch` 或 stale 坐标帧证明。
- 主页面坐标数字层必须按变化单元格重画；不得在 30Hz 热路径中整块清空坐标 canvas，避免单帧无效数据或无变化刷新造成坐标区域全量闪烁。
- 小于 `0.0005` 的显示归零只属于三位小数格式化，不是稳定滤波。

## 5. WCS 切换

WCS 切换只改变 LinuxCNC runtime active WCS，不修改 G54/G55 偏移值。

要求：

- 通过 Command Broker UDS/native dispatcher/native gate 发起。
- 等 native/interpreter API 回读 `stat.g5x_index` 和 `stat.g5x_offset`。
- 不另存 active WCS 文件。
- 不把按钮态、UI dirty、Broker result 或旧诊断当真源。
- Python/脚本中介不得直接 `import linuxcnc`、创建 `linuxcnc.command()` 或 `stat().poll()`；若有兼容 adapter，只能薄转发到登记 native gate。

## 6. Work Zero 门禁

Work Zero 必须 fail-closed：

- 缺 fresh native/microkernel 状态。
- 机器非 idle、MDI 忙、Jog/Home/Start 活跃。
- 未选择 WCS、轴或目标字段。
- RTCP actual enabled。
- 同帧刀补/刀尖语义缺失。
- native gate/linuxcncrsh gate 不可用。
- G10/等价写入后 native/interpreter/API 未回读确认。

RTCP 开启时必须拒绝 Work Zero，不能在 RTCP 几何、G53、刀长、WCS 混合状态下写偏移。

gate 执行后必须恢复/确认 MANUAL 或原约定模式；模式恢复失败时不得把 Work Zero 标为成功。

推荐响应字段：

```text
ok
code
active_wcs
axis
target_offset
wcs_offsets_epoch
readback_confirmed
pending_readback
details
```

| 层级 | owner |
| --- | --- |
| UI 按钮和显示 | 源码真源中的当前 v5 UI command/action dispatch 与 WCS/坐标显示 owner；旧 v3 UI command 路径不得恢复为产品入口 |
| C UI native gate | 当前 v5 UI 到 UDS native command gate 的唯一薄适配 owner；旧 v3 native-controls 路径不得作为新实现入口 |
| LinuxCNC 写入 gate | 登记 native command gate 或单连接 linuxcncrsh gate |
| State Publisher | 当前 State Publisher 仅作为 UI 坐标、转速、进给速度和倍率显示投影同步，禁止将 WCS 状态或偏移量写入 SHM |
| SHM ABI | 当前 `/dev/shm/v3_status_shm` typed ABI 与 C reader；字段白名单按 `功能/0-4共享内存.md`，不得新增 WCS/offset/Work Zero 字段 |

## 8. 验收

本地至少证明：

- active WCS 来自 LinuxCNC interpreter/native/API。
- WCS 全表冷路径不进 UI 热循环。
- Work Zero 写入后必须等 runtime/native/API 回读。
- RTCP ON 时 Work Zero 拒绝。
- WCS 切换不修改 offset。
- 旧 JSON/私有文件/Broker result 不作为 WCS 真源。
- SHM ABI 字段和 C reader/writer 对齐，且静态检查必须证明不存在 WCS/offset/Work Zero 字段、mask 或 payload。
- 重启/继电器断电重启后，常驻 WCS/position publisher 必须在 LinuxCNC native 可读后自动恢复 fresh active WCS 与 offset readback；UI 主刷新循环必须继续合并 fresh native WCS/offset readback，不能因启动首帧缺 WCS 而长期显示 `加工 --` 或 `--.---`；短暂 `stat.poll()` 异常不得立即覆盖上一帧 fresh WCS/position 显示，超过 freshness 窗口必须 fail-closed 为 unavailable，UI 不得长期显示旧值。

板端声明通过时必须有：

- 原始 UI/operator 触发路径。
- UDS request/response。
- LinuxCNC runtime/native/API 回读；参数文件 mtime/hash/key/epoch 只能作为持久化冷证据。
- native/interpreter/API 连续确认 active WCS、offset、generation 和 Work Zero 结果；SHM 只可作为坐标显示截图证据。
- UI 截图或状态证据。
- 若影响运动，按 `REQ-AUTO-STATE-DRIVEN` 跑 native active model 对应的 `cc-ac.ngc` 或 `cc-bc.ngc` 黄金运动闭环。

## 9. 禁止事项

- UI 热路径直接读参数文件或私有 JSON。
- Python/脚本中介直接拥有 LinuxCNC command/stat owner。
- G10 已发送就显示成功。
- 只看文件 mtime/hash 就清 UI dirty。
- RTCP 开启时写 Work Zero。
- 用 WCS 偏移修 MCS/编码器/设零显示问题。
- 把 Work Zero 当机器回零、设0、rebase、G92 或 RTCP 几何修改。

- 将 WCS 偏移、活动 WCS 状态或 Work Zero 结果同步到共享内存（SHM）中；UI 消费必须直连登记的 native/interpreter provider、API 或 UDS 接口，核心要求是不走 SHM、私有 JSON、`/run` result 或旧缓存。
## 10. 关闭条件

关闭 WCS/Work Zero 问题时必须说明：

- active WCS、offset、全表、Work Zero 分别使用哪个 owner。
- 写入路径是否通过 native/linuxcncrsh gate。
- native/interpreter/API 回读字段和 generation；如引用 SHM，只能说明坐标显示投影证据。
- 本地测试、板端证据、截图或黄金运动结果。
- 未闭合事项只写真实缺口，不写过程日志。
