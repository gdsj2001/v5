# 微内核错误提示完整映射表

需求真源：`REQ-NATIVE-OPERATOR-ERROR-MAP`。

## 1. 产品口径

- State Publisher 是微内核原生错误通道的唯一读取 owner；UI、Broker、设置页和按钮分支不得各自读取或翻译原生错误。
- 当前版本基线来自板端使用的 2.9.7 v5 补丁源码真实错误发射点，共 555 项：197 个解释器固定错误，358 个任务层、实时运动层和动态解释器格式错误。
- 每一项必须有稳定源 ID、原生格式模式、通俗中文原因、处理组和统一显示分级；源码升级后必须重新提取，映射覆盖率不是 100% 时禁止部署。
- UI 只显示中文标题、中文原因和中文下一步。底层控制框架名称、原始英文、格式串、内部进程名、原生源 ID、adapter 别名码和诊断路径只允许进入工程日志。
- 映射 owner 必须为每个错误事件输出 `popup`、`top_status` 或 `log_only`，UI 只消费该结果，不得由页面按钮、动作种类或 UI 私有表重新判断同一错误的显示方式。
- `popup` 用于已经阻止当前操作、影响运动或加工结果、涉及安全/限位/驱动/程序执行，或者明确需要操作员处理的错误。`top_status` 用于不阻断正常使用、无需立即人工处置且允许继续操作的短暂或提示性错误。`log_only` 只用于已经自动恢复、对当前操作和结果无影响、没有操作员处置动作的工程诊断事件。
- 不得仅凭“来自微内核”就一律跳窗，也不得仅凭“主页面普通动作”就一律降为顶部提示；显示分级必须以本次事件对当前操作、安全、运动、加工结果及人工处置需求的实际影响为准。安全、运动、加工结果或当前动作已经失败/受阻时不得降级为 `top_status` 或 `log_only`。
- 原生已有等价错误时，删除 UI/Broker 自造的重复可见文案；必要的前置 gate 可以保留，但内部结果码只能作为别名指向同一原生项，不能拥有第二套显示文字。
- 没有原生等价的本地输入错误、owner 通道不可用、网络/授权/资源缺失可以保留，但也必须进入同一集中格式化入口。
- 未识别的新原生错误显示“控制系统返回未识别错误，请停止当前操作并联系维护人员”，可附不透明指纹；不得把未识别原文直接显示给操作员。
- 原生错误是事件，不同步到通用状态 SHM；唯一 owner 把最新错误事件写入独立 resident native memory block，UI 按 generation 只显示一次并在超时后恢复常态状态。
- Windows 唯一映射表源码是 `board/config/ui/v5_native_operator_error_map.tsv`，提取/校验代码在 `board/services/state_publisher/v5_native_operator_error_map.py`、`v5_native_operator_error_map_test.py`，事件块实现及 smoke 在同目录 `v5_native_operator_error_status.*`、`v5_native_operator_error_status_smoke.c`。
- 板端安装映射表固定为 `/opt/8ax/v5/config/ui/v5_native_operator_error_map.tsv`，State Publisher 启动时一次性完整读入并校验数量/哈希，运行热路径不得重新读盘；事件块固定为 `/dev/shm/v5_native_operator_error_status.bin`，只承载映射后的中文和工程指纹，不承载原始英文。板端安装路径和事件块都是构建/运行产物，不是第二份源码 owner。

## 3. 处理组

| 处理组 | UI 标题 | UI 下一步 |
| --- | --- | --- |
| `PROGRAM_SYNTAX` | 程序格式错误 | 检查当前高亮程序行的指令、参数和值，修正后重新打开并运行程序 |
| `PROGRAM_GEOMETRY` | 程序几何错误 | 检查当前高亮程序行的平面、圆弧、循环和终点参数，修正后重试 |
| `PROGRAM_FLOW` | 程序流程错误 | 检查子程序、O字、调用、返回和重映射关系，修正后重新打开程序 |
| `PROGRAM_FILE` | 程序文件错误 | 检查程序文件是否存在、可读且有正确结束标记，修正后重新打开 |
| `MOTION_HOME` | 需要回零 | 选择机械全轴并完成本次开机回零，再重新执行当前动作 |
| `MOTION_LIMIT` | 轴限位阻止运动 | 停止当前动作，检查目标位置、软限位和限位开关，确认后再运动 |
| `MOTION_JOG` | 点动未执行 | 松开点动按钮，检查轴选择、锁定和运行状态后重新点动 |
| `MOTION_KINEMATICS` | 运动学状态错误 | 停止运行并检查运动模型、旋转中心和运动学配置，修正并重启后再试 |
| `MOTION_PROBE` | 探测条件错误 | 检查探针状态、探测方向和目标距离，恢复正确状态后重新探测 |
| `MOTION_PLANNER` | 运动轨迹无法执行 | 检查当前程序段的目标、速度和轨迹连续性，修正后重新运行 |
| `SAFETY_ENABLE` | 机器状态不允许动作 | 先排除现场风险，取消急停并确认机器已使能，再重新操作 |
| `MACHINE_MODE` | 运行模式不允许动作 | 停止或暂停当前任务，切换到动作要求的手动、自动、MDI、关节或坐标模式后重试 |
| `DRIVE_FEEDBACK` | 轴反馈故障 | 停止运动，检查驱动使能、编码器反馈和跟随误差，排除故障后重新使能 |
| `SPINDLE` | 主轴动作失败 | 停止程序，检查主轴使能、定向、速度和反馈后重新执行 |
| `TOOL` | 刀具或刀补错误 | 检查刀号、刀具参数、刀补模式和当前平面，修正后重新运行 |
| `CONFIG` | 系统配置错误 | 保持机器停止，由维护人员检查对应配置并完成重启和回读确认 |
| `INTERNAL` | 控制系统内部错误 | 停止当前运行并保留程序名和行号，联系维护人员处理 |
| `USER_PROGRAM` | 用户程序报告错误 | 按用户程序给出的原因修改程序或现场条件后重新运行 |
| `PASSTHROUGH` | 控制系统报告错误 | 停止当前动作并重新执行一次；若再次出现，联系维护人员 |

## 4. 内部重复别名

下表中的代码可继续作为 gate/result/event 的内部关联值，但禁止直接显示；它们必须进入统一映射。

| 内部别名 | 统一映射语义 |
| --- | --- |
| `POWER_ON_HOME_REQUIRED`、`WORK_ZERO_NOT_HOMED`、`NOT_HOMED`、`UNHOMED` | 原生“动作要求先回零” / `MOTION_HOME` |
| `HOME_PRECONDITION_ESTOP`、`ESTOP` | 原生“机器处于急停或未上使能” / `SAFETY_ENABLE` |
| `HOME_PRECONDITION_DISABLED`、`DISABLED` | 原生“机器处于急停或未上使能” / `SAFETY_ENABLE` |
| `HOME_PRECONDITION_BUSY`、`INTERPRETER_NOT_IDLE`、`NOT_IDLE` | 原生“当前运行模式/解释器状态不允许动作” |
| `JOG_PRECONDITION` 的未回零、急停、未使能和运行中分支 | 分别映射到 `MOTION_HOME`、`SAFETY_ENABLE` 或原生模式错误 |
| `POWER_ON_HOME_STATUS_UNAVAILABLE`、`HOME_STATUS_UNAVAILABLE` | 无原生等价；保留为 owner 状态通道不可用，但只显示中文，不显示内部码 |

## 6. 显示分级与形式

统一映射 owner 输出以下三种显示分级：

| 显示分级 | 使用条件 | UI 行为 |
| --- | --- | --- |
| `popup` | 当前操作已失败或受阻；影响安全、运动、加工结果、限位、驱动、程序执行；或必须由操作员确认/处理 | 显示三段式中文跳窗，并保留到操作员关闭或按对应流程处理 |
| `top_status` | 不阻断正常使用、无需立即人工处理、允许继续操作的短暂或提示性错误 | 顶部短提示一行中文原因，按统一超时恢复，不弹窗 |
| `log_only` | 已自动恢复且对当前操作和结果无影响、没有操作员可执行的处理动作 | 不产生操作员可见提示，只写工程日志 |

同一原生源 ID 可以有默认分级，但本次事件关联的动作失败、运行阶段和安全影响可以把它升级为 `popup`；不得把安全、运动、加工结果或已阻止当前动作的错误降级。重复的同一事件按 `generation` 只提示一次，持续状态不得循环跳窗；只有新事件或影响级别发生实质变化后才允许再次提示。

`top_status` 只显示一行中文原因，不显示源 ID、内部码或原始英文。`popup` 固定为：

```text
提示: <处理组对应的中文标题>

原因: <映射后的中文原因，保留轴号、行号、限位值等安全现场量>

下一步: <处理组对应的中文下一步>
```

内部错误和未知错误不得把进程名、函数名、文件路径、回溯路径或底层框架名称带入原因；这些内容只写工程日志。用户程序主动报告的文本可以显示，但仍须过滤底层框架品牌。

未知原生错误无法证明为无影响事件，因此默认按 `popup` 处理，只显示通俗中文未知错误和不透明指纹；原始文本仍只进入工程日志。

## 7. 验收

- 源码提取测试必须得到 197 / 358 / 555 和本文件记录的清单哈希。
- 每个源 ID 必须唯一，每个原生模式必须恰好映射一次；通配透传项最后匹配，不能吞掉更具体错误。
- 对 555 项逐项执行格式匹配与中文渲染测试；结果不得为空，不得出现底层框架品牌、原始英文格式串或内部 adapter 别名。
- 覆盖测试必须证明每项错误都得到 `popup`、`top_status` 或 `log_only`，并证明影响安全、运动、加工结果或已阻止当前动作的事件不能被降级；无影响、可继续使用的事件不得无条件弹窗。
- 覆盖测试必须证明同一 `generation` 不重复跳窗，未知原生错误默认进入中文三段式跳窗且不暴露原始文本。
- 静态门禁确认 UI 按钮分支不再拼接未回零、急停、未使能、模式不允许、限位、探测和程序解释类重复错误。
- 上板从原始 UI 路径至少触发未回零动作和一条无效 MDI/G-code，证明原生错误事件、resident block、中文映射和顶部/弹窗显示闭环；最终恢复急停安全状态。
