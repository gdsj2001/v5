# 设置页设置驱动按钮作用和目的

引用需求真源：`REQ-BUS-PULSE-MODE-DIFFERENCE`、`REQ-DOC-SINGLE-SOURCE`、`REQ-PARAM-MEMORY-LIGHTWEIGHT-SAVE`、`REQ-NATIVE-OWNER-FIRST`、`REQ-SETTINGS-RUNTIME-DRIVE-ONLY`、`REQ-SCALE-CHAIN-HISTORY-POLLUTION`、`REQ-LINUXCNC-COMMAND-GATE`、`REQ-G92-NATIVE-RUNTIME`、`REQ-WCS-NATIVE-OWNER`、`REQ-ROTARY-UNWRAP-ABSOLUTE-PROOF`、`REQ-ROTARY-REBASE-NATIVE-GATE`、`REQ-RTCP-G53-NATIVE-ACTUAL`、`REQ-MOTION-STILLNESS-TOLERANCE`。

## AI 阅读入口

- 本文同样遵守 `功能/0开机参数入内存.md` 的最高启动内存架构：LinuxCNC/微内核、UI 和产品自写运行代码闭包开机全量入 RAM；运行期热路径不得临时读盘、懒导入、扫描目录、读取旧 JSON/result 或用 SHM 显示投影冒充 owner 真源。
-  本文定位：设置页“设置驱动”相关动作、BUS 驱动写入/读回和退役动作边界。
- 先读 `REQ-BUS-PULSE-MODE-DIFFERENCE`、`REQ-SCALE-CHAIN-HISTORY-POLLUTION` 和驱动 profile 文档，再改任何驱动动作。
- 设置驱动动作不得写成通用参数真源、运动安全真源或 Pulse HOME owner；缺 profile/readback/模式证明时 fail-closed。

本文只定义 BUS/EtherCAT 模式下设置页 `设置驱动` / `settings_set_drive` 的动作。`计算bit` / `settings_calc_bit` 是禁止产品路径：产品 UI、Broker 和板端脚本不得再通过微动、样本或 `log2(count)` 计算编码器 bit。

Pulse/StepDir 模式下，设置页驱动动作按钮中的 `服务器下载`、`扫描从站`、`复位驱动`、`读取驱动`、`清除故障`、`设置驱动` 按 `REQ-BUS-PULSE-MODE-DIFFERENCE` / `功能/总线脉冲区别.md` 三层拒绝：Broker/product action 拒绝、C dispatch 拒绝、UI 灰色不可点击。本文后续规则只适用于 BUS/EtherCAT 可用状态。

设置页轴参数表显示 `bit` 列，位置在电子齿轮 `分子` 列左侧。默认 bit 只从驱动映射表或本机驱动参数表读取；用户可通过三角下拉选择 bit 位，人工选择写入本机参数表并优先于映射表默认值。`驱动count/圈`、`多圈周期` 仍不在轴参数表显示，只作为后端直接参数用于电子齿轮计算、绝对多圈诊断、旋转 count-domain 和 fail-closed 校验。
`bit` 列不得提供或显示“自动”选项；UI 只显示准确数字 bit，缺有效 bit 时显示 `NAT`。人工选择数字 bit 后，`settings_runtime.axes[]` 必须长期保存 `encoder_resolution_bits_source=ui_manual_override` / `encoder_bits_source=ui_manual_override`，`设置驱动`、开机加载和状态发布都必须优先使用该人工值，不得被 profile 默认 `encoder_bits` 覆盖。

## 当前规则

最新闭环修正规则（2026-07-03）：`设置驱动` 写入电子齿轮和 `drive.write_mode` 后仍必须读回一致才算成功；若同一设置驱动事务内的恢复出厂/必读 SDO 读回已经证明 `mode_of_operation` 等于目标模式且驱动无 fault，后端必须把 mode 写入步骤记录为 `pre_write_readback_already_at_target` 并跳过重复 `drive.write_mode` SDO 下载。若执行 `drive.write_mode` 时出现 `Input/output error` 或同类下载失败，只允许在同一事务内执行 OP recovery/必读读回证明；只有目标 mode、电子齿轮和 fault-free 状态全部读回一致时，才可把该步骤记录为 `download_sdo_error_but_readback_matched`。读回缺失、mode 不一致或 fault 未清时必须返回 `DRIVE_MODE_READBACK_MISMATCH` / `DRIVE_SET_PARTIAL`，不能无条件吞掉写失败，也不能增加备用脚本、旧路径或兜底成功分支。如果电子齿轮分子/分母已经读回一致、驱动无 fault、EtherCAT 已恢复 OP，而单次 `mode_of_operation`/`drive.read_mode` 读回为空或瞬时不一致，后端必须在同一设置驱动事务内做有限短间隔重读/必要时重写 mode 后再判定。有限重试仍读不到目标模式时才返回 `DRIVE_MODE_READBACK_MISMATCH`，不能用一次 SDO upload 瞬时失败直接中断整个闭环。

1. 驱动映射表和本机驱动参数表必须按轴显式提供以下参数：
   - `profile_id` / `drive_profile_id`
   - `encoder_bits`
   - `encoder_resolution_bits`
   - `drive_internal_counts_per_motor_rev`（由最终 bit 派生）
   - `encoder_single_turn_counts`（由最终 bit 派生）
   - `encoder_multiturn_limit`
   - `encoder_multiturn_raw_modulus`
2. 代码只能直接读取本机 `settings_runtime.axes[]` / 运行内存的最终参数。启动、保存或准备 `设置驱动` 时，后端先按轴配置的 `profile_id` 从 `driver_profile_map.json` 选择命令 profile；profile 只提供默认 `encoder_bits`。当前汇川 SV630N 默认 `encoder_bits=18`，对应 `drive_internal_counts_per_motor_rev=262144`；不得沿用旧 23 位 / 8388608 作为默认。若轴参数表已保存人工 bit 覆盖（`encoder_resolution_bits_source=ui_manual_override`），必须长期使用人工 bit，不得回退映射表默认；否则使用 profile 默认 bit。`设置驱动` 按钮引用的驱动内部一圈 count/raw count 必须由这个最终 bit 派生，禁止引用已删除的 `驱动count/圈` 表列、旧 profile count、旧参数表 count、UI 默认 count，也禁止通过旧 `计算bit` 微动流程生成 bit。没有人工覆盖且所选 profile 缺少 `encoder_bits`，或人工覆盖标记存在但 bit 值无效时，后端必须清空/忽略本轴旧的 bit/count 派生字段并 fail-closed，不能沿用上一次 `设置驱动` 或旧映射表残留。
3. 电子齿轮目标只由轴参数和驱动参数表计算：

```text
ratio = motor_rev / load_rev
target_command_counts_per_motor_rev = pitch / ratio / target_precision
egear = drive_internal_counts_per_motor_rev / round(target_command_counts_per_motor_rev)
```

目标精度输入归一化：
- `target_precision` 是优先输入；如果轴级字段为空，后端必须使用 `electronic_gear.target_precision`。
- 如果 `复位驱动` / `读取驱动` 后只保留了 `electronic_gear.target_unit_per_count` 或 `electronic_gear.actual_unit_per_count`，`设置驱动` 必须把该值投影为本次事务的 `target_precision` 后再计算电子齿轮，不得因轴级字段为空直接失败。
- `target_precision`、螺距/旋转每转角度、`motor_rev/load_rev` / `motor_revs_per_load_rev` 三类输入必须来自当前 profile、驱动读回、人工标定确认、runtime/native owner 回读或同 generation owner；不得从旧 `settings_runtime.json`、旧 `target_counts/egear`、`pulse_equivalent_counts_per_unit` / `cia402_pos_scale_counts_per_unit` 投影、UI 默认值或缺省 `1/1` ratio 补出本次电子齿轮目标。
- 任一输入缺少可信来源时，`设置驱动` 必须 fail-closed，并在错误详情列出缺失字段；不得在内存轴对象中写入默认 `0.0001/0.001`、`4.0/360.0` 或 `motor_rev/load_rev=1/1` 后继续计算。`target_precision`、螺距/旋转每转角度、`motor_rev/load_rev`、`motor_revs_per_load_rev`、电子齿轮分子和分母都必须大于 0；任何 0、负数、NaN、无穷或非数字输入必须返回结构化业务错误，不能触发未捕获的 Python 除零或异常闪退。该规则归属 `REQ-SCALE-CHAIN-HISTORY-POLLUTION`，本文只定义设置驱动本地需求边界。

4. 电子齿轮写入并读回一致后，运行参数表同步保存：
   - `egear_numerator`
   - `egear_denominator`
   - `feedback_counts_per_motor_rev = drive_internal_counts_per_motor_rev * egear_denominator / egear_numerator`
   - `actual_precision`
   - `rotary_load_counts_per_rev`
5. A/C 旋转轴的启动、设零、等效角、unwrap、checkpoint 和 Rebase 只读取参数表里的这些字段，不再等待 `计算bit`。
6. 当前源码的 `设置驱动` 只支持普通 CSP 驱动设置路径。Broker canonical request 会为每个目标轴生成 `mode_int8=8` / `mode_of_operation=8`，后端请求过滤按电子齿轮目标和 CSP `8` 一起校验；请求里模式字段缺失、非数字、负数或不等于当前 canonical CSP 目标时 fail-closed。主轴、速度环轴、特殊同步轴或非 CSP 轴不得复用当前普通设置驱动路径，必须先新增对应 profile/轴 owner 和模式目标规则后再接入。

## 设置驱动动作

`设置驱动` 用于把当前轴参数、电子齿轮和模式参数写入驱动，并读回验证。

执行顺序：

1. 保存当前 dirty 轴参数，并先按轴配置 `profile_id` 选择映射表 profile，把默认 bit 或人工覆盖 bit 刷新到 `settings_runtime.axes[]` / 运行内存；内部一圈 count、raw count 由最终 bit 派生。用户可编辑的螺距、减速比、目标精度和人工 bit 选择也必须已落盘。bit 默认来自映射表，人工选择长期优先；内部一圈 count、多圈周期不在轴参数表中显示或编辑。
2. 自动执行安全前置：确认机器停稳、下使能或急停状态符合驱动写入要求。
3. 扫描/匹配驱动 profile，确认所需标准命令存在。
4. 计算电子齿轮分子/分母。
5. 写入电子齿轮、模式等驱动参数。
6. 恢复/轮询 EtherCAT OP。
7. 读回分子/分母、模式和关键参数，一致后保存运行参数表并刷新内存。

等待与超时：

- `设置驱动` 是逐轴恢复出厂、写电子齿轮、恢复 OP、读回校验和刷新参数表的长流程；UI 不得沿用普通驱动写动作的 70 秒等待上限。
- 写电子齿轮、写 mode 的 SDO 通信超时只约束单条 SDO 报文；写入后的 EtherCAT OP 状态机恢复必须使用独立的 OP recovery 超时，且默认最小值不得低于多轴总线恢复所需窗口。不得把 OP 恢复超时绑定为 `max(8s, egear_timeout, mode_timeout)` 这类随 SDO timeout 缩短的动态值；OP 未恢复时返回 `DRIVE_POST_WRITE_OP_RECOVERY_FAILED`，但该错误必须建立在独立 OP 恢复窗口耗尽后。
- UI 等待期间必须持续刷新 `drive_set_progress.json` 的阶段进度，只有 Broker `job_status` 明确返回失败或超过 `drive_set_parameters` 专用长流程上限后，才允许弹失败窗口。
- 后台最终返回 `DRIVE_SET_OK` 时，UI 必须按成功处理并刷新轴参数表，不得因为普通 job 等待上限先到而把已成功动作显示成 `DRIVE_WRITE_JOB_TIMEOUT`。
- 后台最终返回 `DRIVE_SET_OK` 且表格显示 `已写入` 后，UI 必须立即标记 backend restart pending；右上角按钮必须从 `返回` / `保存` 切换为 `保存并重启`，等待操作员统一保存并重启后端。

失败规则：

- 缺最终 `encoder_bits`、由 bit 派生出的 `drive_internal_counts_per_motor_rev`、`encoder_multiturn_limit` 或 `encoder_multiturn_raw_modulus` 时 fail-closed。
- 当前普通设置驱动路径只接受 canonical CSP `8`；请求模式字段缺失、非法或不等于 canonical 目标时 fail-closed。
- 电子齿轮派生输入为 0、负数、NaN、无穷或非数字时 fail-closed，并返回结构化错误；不得出现未捕获 `ZeroDivisionError` 或后台 job 无 JSON 闪退。
- 写入后 EtherCAT OP 独立恢复窗口耗尽仍未 OP 时 fail-closed；不得因短 SDO timeout 误缩短 OP 恢复窗口。
- 恢复出厂或清除故障后的成功判定必须同时满足必读 SDO 读回完整、EtherCAT OP、`error_code=0`、statusword 无 fault、`drive.read_egear` 读回可用；`statusword`、`drive.read_egear` 等必读项读回失败时，只能在同一事务内有限重试/OP recovery，不能把缺失读回当作 `no_fault` 或成功。
- 恢复出厂、清除故障或设置驱动中的 OP recovery 不能只检查 `ethercat slaves` 显示 OP；当必读 SDO 读回或写命令出现 `Input/output error`、读回缺失或邮箱不可用时，必须在 canonical EtherCAT recovery 内对目标从站执行有限 PREOP/OP 状态循环，并用必读 SDO readback probe 证明邮箱恢复。探针未通过只能继续有限恢复或 fail-closed，不能把 OP 状态本身当作成功。
- `读取驱动` 按钮的必读项读回失败时，后端必须对失败目标轴执行有限 PREOP/OP 状态循环，并以同一个 `read_drive_for_target` 探针重新证明读回完整；若读回显示真实 fault，才允许执行标准 `drive.reset_fault` 后再读一次。有限恢复后仍缺失 `statusword`、电子齿轮或其他必读项时，结果只能保持 `DRIVE_READ_PARTIAL` 并保留失败证据，不得补默认值、跳过该轴或把后续 UI 等待改成兜底成功。
- `设置驱动` 写入电子齿轮和 `drive.write_mode` 后，写后校验和最终刷新 `settings_runtime` 的汇总读回都必须使用必读 SDO readback 作为 OP recovery 探针；`statusword`、电子齿轮分子/分母或 mode 等必读项缺失时，必须对目标从站执行有限 PREOP/OP 状态循环后再用同一个 `read_drive_for_target` 证明读回完整。若有限恢复后仍缺必读项或无法证明 mode 与目标一致，顶层结果必须转为 `DRIVE_SET_PARTIAL`。不得出现 `DRIVE_SET_OK` 已返回但 `settings_runtime.axes[].drive_parameters_valid=false`、`drive_mode_readback_mismatch` 或必读 SDO 缺失仍残留的分裂状态。
- `复位驱动` / `读取驱动` 写回设置表时，驱动实际读回的出厂 `1/1` 只能进入 readback 证据字段和 `readback_mismatch` 状态，不能覆盖已配置的目标电子齿轮，也不能让空的顶层 `egear_numerator` / `egear_denominator` 遮蔽 nested `electronic_gear` 中的 canonical 目标值后变成 `0/0`。
- 缺 `drive.set_egear`、`drive.read_egear` 或读回不一致时 fail-closed。
- 任何弹窗必须显示具体原因，不允许提示“查看日志”。
- 成功窗口不自动关闭。

- `settings_axis_zero` / 设置页 BUS `设0` 的安全窗口必须由 microkernel/native gate 判定，并在同一个 LinuxCNC/native raw joint 坐标域提供必要 readback。禁止把 UI/SHM display/projection zero 后的 `mcs`、`position`、`actual_position`、`joint_actual_position`、投影坐标或 V3 私有阈值与 native raw joint 值相减来判断移动；上一轴设零导致的 display zero/projection refresh 不得触发 `SETTINGS_ZERO_AXIS_MOTION_ACTIVE`。只有 native gate 返回机器不在允许窗口、原生状态不可用或 raw readback 不可用时，才允许 fail-closed。

- `settings_calc_bit`
- `drive_calculate_bit`
- `--action calculate-bit`
- `confirm_drive_calculate_bit`

旧测试或诊断脚本若仍需要微动采样，只能放在 `repo_ignored/<task>/scratch/` 或板端 `/tmp/v3_test_tools`，不得作为产品路径、Broker 路径或 UI 按钮路径。
