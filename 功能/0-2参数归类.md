## 归类原则

设置页参数只按三类 owner 归类：

1. 微内核参数：LinuxCNC runtime INI、HAL/native、kinematics/native、WCS/native、已登记 native memory/API。凡是微内核已有或应该下沉到微内核的参数，必须使用微内核/native readback，不得进自建参数表。
2. 自建参数：微内核参数和驱动映射表参数都没有承载的设置页参数。自建参数必须开机进入 RAM，UI 自动读取，不得显示 `NAT`、空占位、UI 默认值、旧 JSON、`/run` result 或 SHM 投影。
3. 驱动映射表参数：驱动参数映射表/drive provider，只服务驱动 profile、电子齿轮/模式等已登记动作的真实读写证据和授权下载链路。H01/H05/H08 等驱动整定 SDO 保持厂家默认值，不进入参数表、profile commissioning 或普通写路径；唯一写入例外是显式 `复位驱动` 使用当前型号 profile 的 `drive.restore_factory_defaults` 恢复厂家默认。PDO 是固定 ESI/XML/profile 实时布局，不进入用户可编辑参数表。驱动映射表参数不能反向成为 LinuxCNC 运动参数、坐标基、RTCP/G53、WCS/G92、刀补、模态或安全真源。

读取来源和写入目标必须分开记录：读取来源决定 UI 初始显示和默认值来源；写入目标必须落到对应硬盘持久 owner。双写不得形成双真值；对 runtime/native 类字段，单字段确认只允许证明持久 owner 写入并从同 owner/resident 表回读一致，运行态生效必须等受控重载/运行态写入后，再由微内核/native 或 drive provider readback 一致确认。`写入状态` 只表达驱动写入/readback 动作结果，不得作为普通参数提交成功状态。

UI 中任何可修改参数控件发生改变时，UI 只能提交字段保存请求给唯一 settings/native apply owner；由该 owner 串行写入字段对应硬盘/native 持久 owner，并从同一 owner/readback 链路读回后把结构化结果返回 UI 刷新单元格。UI 进程不得直接调用参数表写盘函数、runtime INI 写盘函数或用 LVGL 控件文本/RAM 临时变量/屏幕显示冒充保存成功。只要 owner 写入或 readback 不一致，就必须保持/刷新为 owner 读回值；普通参数提交不得改写 `写入状态`。

字段已经写入唯一 owner 并从同一 owner/readback 链路回读一致后，该字段不再属于“修改未保存”；UI 只能显示已保存、待重启生效或等价 restart pending。软件盘尚未确认的编辑态只属于当前控件输入态，不是稳定参数真源；不得把编辑态、dirty 标志或 restart pending 扩展为 Start/Resume 前置 gate。

设置页右上角是同一按钮的两态合同：进入设置页、本次会话尚无有效参数变更且无 restart pending 时显示 `返回主页`，只执行本地页面导航，不发 settings action、不触发 restart；任意字段在软键盘 `OK` 或下拉提交后写入自己的唯一 owner 并完成源位置 readback，或设置动作成功回读 `restart_required/restart_deferred` 后，立即单向切换为 `保存并重启`。点击 `保存并重启` 时，常驻设置动作 owner 必须先按这些已提交的最终字段自动执行 `设置驱动`，逐物理从站写入并 fresh readback 电子齿轮及 profile 必需参数；成功后才准备开发板断电重启等价的 clean restart handoff，失败保持设置页且不重启。不得把未写入硬盘参数表的 UI 临时值当成已保存参数；不得把 `保存并重启` 扩成无关运动准入、安全互锁或 Start/Resume 拦截，也不得实现成只重启 UI/Broker/State Publisher 的局部服务重载。

设置页参数 UI 的视觉布局必须直接继承 v3 设置页源码坐标，不得另造一套布局。v3 基准坐标为：顶部三块从 `y=42` 开始，左侧运动模型面板 `16,42,220,176`，G53 面板 `247,42,449,176`，右上机械坐标外框 `696,42,312,176`；G53 标题 `264,55`，G53 表头 `X/Y/Z=395/479/563,y=86`，原 v3 行为 `A中心/C中心/对刀仪/5方向监测仪`，单元格列锚点固定为 `366/450/534`；驱动动作按钮行为 `x=410/494/.../914,y=236`；轴参数大面板为 `21,225,982,343`，轴表标题 `y=230`，横向滚动区 `95,274,908,300`，轴名行从 `y=318` 起按 31px 行距排列。轴参数表必须在红框所示左标题区和右端保留图形化左右翻页按钮，用于横向翻页查看轴参数列；按钮只控制现有横向滚动区按页左/右滚动，不改变轴名固定列、列锚点、字段 owner、保存/readback 语义，也不得用第二套表格或隐藏列替代现有横向滚动区。右上机械坐标小面板必须按 v3 `create_settings_machine_coordinate_widget(page, 696, 42)`：外层保留 `696,42,312,176` 背景，内部坐标面板只显示 X/Y/Z 加 fresh Active Model descriptor 对应的两根旋转轴和 `MCS`，AC 显示 A/C、BC 显示 B/C，后续模型按 descriptor 显示自己的轴组；未知模型不得回退 A/C。数值区保持 v3 固定宽度/数码字视觉；龙门、刀库是下方轴参数表第 7/8 轴，不得塞进右上机械坐标小面板形成自造 8 行普通 label 表。新增 `B中心` 行必须插在 `A中心` 与 `C中心` 中间，但不得因此把驱动动作按钮行、轴参数大面板、轴表标题、横向滚动区、轴名行或轴参数行整体改成另一套布局；为容纳第 5 行，只允许在 G53 小表内部做最小竖向压缩，保持 v3 的列锚点、颜色、表头、单元格风格和视觉层级。轴参数标题文案沿用 v3 `轴参数(总线)`，不得把当前新增轴数写成标题视觉元素。轴参数表新增 `方向` 列时，只允许压缩 v3 原 `轴模式`/`从站` 段内部宽度来插入，`目标精度` 及其右侧所有列必须继续使用 v3 原始 x 坐标锚点（`272/390/472/...`），不得因为新增列把后续字段整体右移。

轴参数表控件类型固定：`轴模式`、`方向`、`从站`、`回零次序`、`回零方向`、`bit` 使用下拉；`分子`、`分母`、`写入状态` 为自动计算/动作结果只读显示；除此之外的轴参数单元格都必须点击弹出软件盘输入。所有下拉列的触发命中区必须覆盖整个单元格，不得只在三角箭头或文字局部触发。软件盘必须保持 v3 设置页同款数字键盘布局和视觉风格，用 LVGL 面板/按键/标签自绘实现，不得用 LVGL 默认 `lv_keyboard` 成品键盘替代。软件盘打开/返回必须遵守 `REQ-UI-FIRST-FRAME-CACHE`：打开前必须通过公共首帧 guard/helper（当前源码为 `V5UiFirstFrameGuard` 或等价 owner 入口）保留当前设置页帧，再创建全屏不透明遮罩快速盖住后部设置页内容，键盘区域作为独立命中区拦截内部点击；点击软件盘外的遮罩区域必须立即退出键盘。退出或取消时必须恢复打开前保留帧，不触发后部设置页整页重绘；确认后立即写入该字段对应硬盘参数表，并按 owner/readback 回读，只刷新发生变更的参数单元格。

轴参数表数字显示和输入只允许 `目标精度` 使用小数；除 `目标精度` 外，所有数字参数在表格中必须显示为整数，软件盘提交值也必须是整数。读取 owner 得到的历史小数、LinuxCNC 秒单位换算值或比例链重算值，只能在设置页显示层按机械输出侧整数归一化；其它数字字段带小数点、分数、乘号或非整数格式时必须拒绝提交。

G53 机床坐标目标位置表中，只有图示红线 owner 格 `A中心 X`、`B中心 Y`、`C中心 Z` 保持对应 WCS/native owner 只读，不从设置页软件盘写入。除此之外的 G53 区单元格，包括 `A中心 Y/Z`、`B中心 X/Z`、`C中心 X/Y`、`对刀仪 X/Y/Z`、`5方向监测仪 X/Y/Z`，点击必须弹出同一套 v3 样式 LVGL 自绘软件盘输入；确认后按字段归属写入对应硬盘参数表并回读刷新。

`A中心 Y/Z`、`B中心 X/Z`、`C中心 X/Y` 属于 G53 native geometry/runtime INI readback。设置页软件盘确认后只能向 settings/native apply owner 发保存请求，由该 owner 写入 active runtime INI `[RTCP]` 的唯一 owner，并从同一 `[RTCP]` 字段源位置回读一致后返回 UI 清 dirty、关闭软件盘并刷新当前单元格；UI 进程不得直接写 runtime INI，不得再写 `microkernel_parameter_table.tsv`、`settings_runtime.json`、自建参数表或任何 V3 shadow owner。该刷新只表示持久 owner 已写入/待受控重载，不表示当前 native actual 已改变；受控重载/运行态写入完成后，最终 actual 仍必须从微内核/native 或 runtime INI/native readback 读回一致。加载设置页时不得把旧 UI 默认值、SHM、`/run` result 或旧 cache 当成 G53 几何真源；当前运行态 actual 以 native/readback 为准。`对刀仪` 和 `5方向监测仪` 仍按自建参数表 owner 读写刷新。

运动模型提交必须是一个完整、可回滚的 settings/native owner 事务，不能只改 `[RTCP]`、`[KINS]` 或 `[TRAJ]` 文本。descriptor 选中的每个轴必须把对应 `[AXIS_*]` 参数装载到实际 `JOINT_<status_slot>`，并对速度、加速度、软限位、比例链/SCALE、回零参数和反向间隙做源位置回读；离开 active set 的轴保留自己的 `[AXIS_*]` 持久参数，不得污染 active `JOINT_n`。同一模型事务还必须按 target descriptor 生成完整默认 axis→slave：active axis 写其 `status_slot` 对应从站编号，离开 active set 的转换轴写 `NAT`；MODEL/KINS/TRAJ、AXIS↔JOINT 和 self axis→slave 任一写入或回读失败时全部回滚。事务成功后 UI 立即从 self owner 回读刷新表格并弹出“必须设置轴参数和驱动”的提示；提示窗原关闭按钮显示 `设置驱动`，释放后自动发起正式设置驱动动作。轴参数每行 `从站` 下拉仍可在保存重启前人工覆盖默认值；因此保存并重启还必须按最终字段再次完成设置驱动和电子齿轮 fresh readback。clean restart 后，INI active AXIS/JOINT/KINS/TRAJ 与 HAL/native router 读取最终 axis→slave 表并以同一 mapping generation 一次性 commit；映射缺失、重复或与 active set 不一致时保持去使能并结构化报告。禁止模型外代码按扫描顺序、驱动名称、旧 joint 或表格行号推断映射。

这里必须分开两类 owner：脉冲当量、软限位距离、速度、加速度等属于逻辑轴 `[AXIS_X..C]`，模型改变时只按 descriptor 把目标逻辑轴自身 `[AXIS_*]` 参数装入对应 JOINT；从站映射改变时不得从 donor 从站或 donor 逻辑轴复制。BUS count-domain 0 点属于物理从站驱动，以 `slave_position` 为唯一身份。轴参数 `从站` 下拉把某个物理从站改绑到另一逻辑轴后，新逻辑轴的 active JOINT `SCALE` 必须取自身 `[AXIS_*]` owner，而零点继续使用该物理从站原有证据；不得读取新逻辑轴历史旧从站零点。运动模型提交本身不得触发这种改绑。

轴参数表的速度、加速度和软限位按机械输出侧记录和显示，不按电机侧 RPM 记录。速度单位为 X/Y/Z 直线轴 `mm/min`、active-model descriptor 登记的旋转轴 `deg/min`；runtime INI `MAX_VELOCITY` 按 LinuxCNC 秒单位保存，设置页写入时将每分钟单位除以 60，读取显示时乘以 60。换算必须考虑螺距/每转角度、`motor_rev/load_rev` 和减速比；当前默认值为 X/Y/Z `速度=10000`、`加速=500`、软限位 `-500/+500`，登记旋转轴 `速度=50000`、`加速=2000`，旋转轴软限位默认沿用 owner 的 no-soft-limit 语义。

## 从站/bit 受控重启生效机制

从站和 bit 的物理生效路径固定为“字段 owner 保存 -> 保存并重启 -> 开发板断电重启等价 clean restart handoff -> 开机链路重新入 RAM -> 物理总线/native readback”。从站持久表可由模型切换事务写入 descriptor 默认值，也可随后由轴参数下拉人工覆盖；两者都必须由 resident self owner 源位置回读。bit 只写驱动参数表/驱动映射表并由 resident drive owner 或 drive provider 回读。字段保存阶段不得直接改写微内核运行态内存、HAL 映射、EtherCAT master 或 live 绑定；clean restart 后 HAL/native router、Home 与普通运动目标必须消费最终从站表并通过同代 readback 证明物理输出已跟随。

点击 `保存并重启` 后，由 `v5-settings-actiond` 先在同一个后台 job 中执行 `drive_set_parameters`，以最终持久轴参数和 axis→slave 映射完成电子齿轮及 CSP 运行模式的既有写入/fresh readback；该动作不得附带 H01/H05/H08 整定 SDO 写入。只有动作成功才允许 canonical clean restart owner 挂起交互并拉起系统级干净重启流程，失败时不得生成或提交 reboot handoff。重启流程必须达到开发板断电后重新上电的同等干净度：停止 UI、Broker、Action Dispatcher、State Publisher、LinuxCNC/微内核运行服务、HAL/EtherCAT/drive provider 等运行闭包，清理或重建 `/run` 运行态、UDS socket、SHM 显示投影、native 状态块、resident snapshot、命令队列和进程内 cache，再按开机链路从唯一持久 owner 重新装载到 RAM。可使用受控 clean restart 脚本，但脚本职责不能只是 restart 若干服务；如果软件 clean restart 不能证明与开发板断电重启等价，必须走板端系统级 reboot 或断电重启等价路径。

clean restart 期间必须让相关驱动进入 Disabled / Safe-Op 或等价安全去使能状态，再重新初始化 HAL、EtherCAT 主站、固定 PDO 实时布局、轴到从站绑定、encoder bit 和轴当量换算。调参 SDO 不属于重启初始化输入；只有电子齿轮/CSP 必要动作按型号 profile 的固定命令执行并 fresh readback。重启后的 UI/Broker/Publisher/provider 必须消费新开机链路装载出的 resident owner/readback；重启完成前不得宣称运行态已生效。

clean restart 成功证明必须来自真实 owner readback：硬盘持久 owner 与新装载 resident snapshot 一致，EtherCAT/驱动物理总线读回的从站 evidence、电子齿轮/CSP 模式 readback、固定 PDO 布局 identity、encoder bit readback 和 native/runtime 轴绑定 generation 一致。只要任一从站握手、必要电子齿轮/CSP 动作、固定 PDO 布局、驱动 readback、native/runtime generation、resident snapshot 签名或清理后的运行态重建不一致，必须终止重启完成声明并 fail-closed，保持去使能并报告结构化错误；严禁带半更新参数、脏绑定、旧 bit、旧 `/run` 快照、旧 SHM/native 状态块或旧进程 cache 上线。

动态 Native Gate 参数注入只作为未来优化保留。启用前必须由底层 native owner 提供 P0 原子事务：微内核确认运动完全静止且处于安全窗口，驱动安全去使能或可原子重映射，HAL/EtherCAT/native runtime 一次性提交并可失败回滚，随后由物理总线和 native memory readback 证明一致。在该完整事务存在前，UI/Broker/设置页不得直接修改微内核运行内存来让从站或 bit live 生效。

## 禁止项

- `轴模式`、`方向`、`从站`、`回零次序`、`回零方向`、`bit` 必须渲染为实际下拉控件；其中 `从站` 下拉候选只能是固定 `NAT` 加 `扫描从站` 写入硬盘自建参数表 `SETTINGS/slave_options` 的未占用真实候选。真实候选显示为 `轴号 驱动名型号`，提交落盘值仍为纯轴号；`NAT` 提交落盘值为字面量 `NAT`，只表示该轴显式不绑定物理从站，不得作为缺失候选或旧值 fallback。
- 除下拉列以及自动计算/只读的 `分子`、`分母`、`写入状态` 外，轴参数表其他参数不得做成静态文本；点击必须弹出 v3 同款 LVGL 自绘数字软件盘，并按硬盘参数表写入/回读链路保存。
- 不得把“读一个地方、写两个地方”误写成双真值；`从站` 改变后写硬盘自建参数表并由 resident self owner 回读，`bit` 改变后写硬盘驱动参数表并由 resident drive owner/drive provider 回读。二者都不得恢复 `microkernel_parameter_table.tsv`，最终运行态生效以 clean restart 后的真实 owner readback 与硬盘持久 owner 一致为准。
- 比例链字段、电子齿轮、runtime/native `SCALE` 和 drive provider readback 的一致性审计只能由 settings_apply、drive provider 或已登记 native owner 执行，容差和单位必须来自 owner；不得在 UI/Broker 中写固定阈值或自造当量真值，不得恢复上层 `motion_stillness_gate`、`velocity == 0` 或 `Interpreter == IDLE` 私有停稳判据。
- 右上角必须按本次设置会话状态切换：无有效参数变更/restart pending 时显示并执行 `返回主页` 本地导航；字段 owner 写入/readback 或动作 owner 的 restart pending 一旦成立，同一按钮必须单向切换并执行 `保存并重启`。`保存并重启` 不能做成普通返回、只切主页面、只改按钮文字或只 restart 若干 daemon；它必须先自动完成最终设置驱动与电子齿轮 fresh readback，再进入参数落盘确认后的开发板断电重启等价 clean restart，任一步失败都不重启。
- 不得把微内核/native 已有或应该拥有的参数放进自建参数。
- 不得为了消除 `NAT` 把 UI 默认值、旧 `settings_runtime.json`、`config_cache`、SHM、`/run` result 或驱动映射表中不存在的伪字段当真值。
- 不得把驱动参数映射表扩展成运动参数、G53/RTCP、WCS/G92、刀补、模态或安全 owner。
- 龙门和刀库是第 7、8 轴，不是占位行；有 owner 的字段必须和其他轴一样显示，不得灰色禁用。
