## 1. 总线模式边界

BUS 模式指底层轴驱动适配器为 EtherCAT/CiA402 伺服链路。设置页只负责显示运行内存投影、提交字段请求和触发受控动作，不是运动真源。

| 类型 | 作用 | 生效边界 |
| --- | --- | --- |
| UI 编辑值 | 用户当前输入或下拉选择 | 只是 dirty/request，不是运行态真值 |
| 参数 owner | runtime INI、`self_parameter_table.tsv` 的 `SETTINGS/bus_pulse_setting`、已登记 native/settings apply owner/gate、Settings Drive 私域 | 写入唯一 owner 并源回读后才算字段保存成功；helper 只能是执行适配层，不能成为第二真源 |
| 运行态真值 | 微内核/native owner/API 的 fresh 状态；SHM 只投影坐标、转速、进给速度和主轴/进给倍率显示；当前模态、当前刀号和刀长直连 native modal/tool 状态块或等价 API | native actual 用于运动准入和动作证明；SHM 仅用于 UI 显示 |
| 驱动动作 | 通过 Broker 调用 EtherCAT 扫描、读取、复位或写 SDO | 必须有动作结果和读回证据，不能靠 UI 成功态代替 |

硬边界：

- UI 不直接读写 `/opt/8ax/phase0_bus5/settings_runtime.json`、runtime INI 或 HAL pin；`/run/*.json` 只允许作为本次弹窗/action evidence、阶段诊断或兼容结果输入，不得作为控制依据、保存成功、稳定显示、运动准入或参数真源。
- 每个可编辑字段提交时只提交本字段，写本字段唯一 owner，并回读本字段源位置；成功后清本字段 dirty，保留 restart pending。
- 参数页任一可编辑字段完成唯一 owner 写入和源位置回读，或 backend owner 返回 restart pending 后，右上角按钮必须立即从 `返回` 改为 `保存并重启` 并在本次会话保持；字段仍在编辑、写入失败或源回读尚未完成时由该字段提交动作继续处理，不能把未落盘 dirty 留给保存并重启按钮补救。只有本次会话尚无有效 owner 变更且没有 backend restart pending 时，按钮才允许显示 `返回`。
- `保存并重启` 的完成边界必须对齐 `功能/0-1开机参数入内存.md` 和 `功能/0-2参数归类.md`：字段 owner 写入/源回读后进入 canonical clean restart handoff，停止并重建 UI、Broker、Action Dispatcher、State Publisher、LinuxCNC/微内核、HAL/EtherCAT/drive provider 等运行闭包，清理或重建 `/run`、UDS socket、SHM/native 状态块、resident snapshot、命令队列和进程内 cache，再按开机链路重新装载到 RAM，并用重启后的 resident owner/native/drive readback 一致证明完成。只发起 backend + UI restart handoff、只返回主页面、只恢复若干服务或只看到 machine-on/Operation Enabled 都不能单独代表完成。
- `从站` 和 `bit` 的物理生效按 `功能/0-2参数归类.md`：字段保存阶段只写对应持久 owner 并回读，不能 live 改微内核运行内存；点击 `保存并重启` 后由 canonical clean restart owner 让驱动进入 Disabled / Safe-Op 或等价安全去使能，重新初始化 HAL、EtherCAT 主站、SDO/PDO 映射、轴绑定和 bit/当量换算，并以物理总线和 native/runtime readback 一致作为成功证明，任一环节失败必须 fail-closed。
- 运动、设0、Home、Jog、Start、设置驱动等执行动作必须在执行时读取 fresh microkernel/native 安全状态并 fail-closed；不得把 SHM 显示投影当安全证明。
- 授权/DNA 校验必须前置到后台非实时路径：开机、backend/UI restart 后，或 `estop_or_reset` 上使能前由授权 owner 完成 live DNA 多次读取、授权验签和 hash 比对，并生成只读 latch 状态。运动、Home、Jog、Start、设0、设置驱动等热路径只消费 latch，不得每次动作重新读硬件 DNA、验签授权文件或访问 VPS。
- 设置页触摸热路径不能被后台维护刷新阻塞；普通单元格按下只打开键盘，`设0/zero` 只能在明确点击完成事件后执行。
- 设置页触摸闭环验收的推荐人工顺序为：`下载授权 -> 服务器下载 -> 扫描从站 -> 复位驱动 -> 读取驱动 -> 清除故障 -> 设置驱动 -> X/Y/Z/A/C 每个轴逐个设零/设0 -> 设置 A/C 轴坐标 -> 保存并重启`。该顺序只用于设置页链路验收和人工排查，不得变成后台动作前置门禁；每个按钮动作必须在本次执行时读取 fresh owner/readback 并自行闭合。该顺序也不得替换成主页 `Home -> Open Program -> Start` 运动链路；主页运动链路只能在设置页链路闭合后另行验收。
- 可编辑单元格按下到软键盘出现并显示可输入数字不得超过 `0.2s`；数字键按下到输入框显示该数字也不得超过 `0.2s`。软键盘 `OK` 或下拉提交后，UI 本地提交反馈、dirty/pending 标记或结构化拒绝必须在 `0.5s` 内出现；字段 owner 写入和源位置回读若可在本地内存/短路径完成，也应在 `0.5s` 内清 dirty。涉及 native、文件 flush、网络、驱动硬件动作、后台重启或长流程 readback 的提交，不得为了满足 `0.5s` 伪造成功，必须在 `0.2s` 内出现进度窗口或明确 pending/fail-closed 状态，并持续显示阶段进度和耗时。超过时限必须定位并修复，不能继续后续步骤掩盖慢点。

## 4. 字段 Owner 总览

| 字段族 | 前台保存 owner | 运行态真值 |
| --- | --- | --- |
| `motion_driver_mode` | `self_parameter_table.tsv` 中唯一的 `SETTINGS/bus_pulse_setting` 保存 requested/持久 owner | 冷启动先读该参数决定 BUS/EtherCAT 或 Pulse 后端；真实 active mode 必须由 mode-switch/native provider 按四项条件回读：requested mode 等于 SelectLink 选中 INI、LinuxCNC fresh/running 且使用该 INI、期望侧 RT/HAL module 已加载、非期望侧 module/adapter inert 或 disabled；不得从 UI 文案、dirty、旧 JSON、`settings_runtime.json` 或 SHM 推断 |
| `axis_mode`、`precision`、`pitch`、`motor_rev/load_rev` | `axis_mode`、`precision`、`pitch`、`motor_rev/load_rev` 属于 runtime INI/native 比例链 owner；必须写入同 owner 并从 runtime/native readback 读取。只有 microkernel/native 和驱动映射表都未承载的其它设置页轴参数才进入自建参数表 | runtime INI/native `SCALE` 或对应 owner fresh 回读；SHM 不承载 SCALE 真源 |
| `max_velocity`、`max_acceleration`、`backlash`、`FERROR`、`MIN_FERROR` | runtime INI `[AXIS_*]` / `[JOINT_*]` | LinuxCNC/native runtime 回读；不得新增 V3 私有运行/停稳容差字段 |
| `home_order/home_sequence` | runtime INI/native `HOME_SEQUENCE` | Home 预检和 native/API readback |
| `soft_minus/soft_plus` | runtime INI/native limit owner | LinuxCNC/native runtime limit readback |
| `encoder_bits`、内部一圈 count、多圈周期、电子齿轮读回状态 | Settings Drive 私域、profile、EtherCAT/SDO 读回证据 | 驱动读回证据 + 运动准入 gate |
| `zero/设0` | BUS count-domain 零点或 Pulse `HOME_OFFSET` owner | `功能/2-5轴设0软件偏置方案.md` 定义的 native/owner 证明；SHM 只显示坐标投影 |
| `motion_model` / `[RTCP].MODEL` | runtime INI `[RTCP]` / kinematics/native model owner | 设置页选择值必须提交给微内核/native owner 并回读；AC/`XYZAC_TRT` 选择 A/C G53 几何，BC/`XYZBC_TRT` 选择 B/C G53 几何，后续模型按 active model 选择对应几何组 |
| `G53_A_Y/G53_A_Z/G53_B_X/G53_B_Z/G53_C_X/G53_C_Y` | active runtime INI `[RTCP]` owner | RTCP/G53 native readback；B 中心 X/Z 是 G53 几何坐标，B 中心 Y 来自当前模态加工坐标系 Y 偏移值 |

字段规则：

- `soft_minus/soft_plus` 的 UI 值为 `0` 表示该方向软限位不生效；保存时必须由 owner 换算为可执行限位语义，不能把 UI 值当 LinuxCNC 真值。
- 非回零运动触发正向软限位时，native motion owner 必须停止当前动作、保持机器与驱动使能、拒绝继续正向运动并允许负向点动退出；负向软限位按相反方向处理。单个正向或负向物理限位开关触发时使用完全相同的方向性处理，不得因为单方向限位直接 Machine Off、撤销驱动 Operation Enabled 或要求重新上使能。自动程序、MDI、探测和攻丝不得在限位保持有效时自动恢复，必须结束当前命令并由操作员反向点动退出后重新发起。
- 软限位和物理限位只共享“单方向停止并允许反向退出”语义，不得混淆 owner：软限位来自 native `MIN_LIMIT/MAX_LIMIT` 与关节坐标，物理限位来自 fresh `joint.N.neg-lim-sw-in/pos-lim-sw-in`。同一关节正负物理限位同时生效、限位方向无法判定，或同时存在急停、驱动故障、跟随误差等独立硬故障时必须按对应安全 owner fail-closed；本条不得削弱这些故障原有的下使能规则。回零期间限位/回零开关的处理继续由原生 homing 配置和 Home owner 裁决。
- A/C 旋转轴默认 `0/0` 表示 no-soft-limit；不得生成误导性的 `±360` 默认限制。
- `max_velocity`、`max_acceleration` 和 `soft_minus/soft_plus` 在轴参数表中表达机械输出侧参数，不是电机侧 RPM、转子速度或电机角加速度。速度保存时必须结合螺距/每转角度、`motor_rev/load_rev` 和减速比所在比例链 owner 做运行态换算；UI 和 owner readback 中 X/Y/Z 直线轴速度单位为 `mm/min`，A/C 旋转轴速度单位为 `deg/min`，不得把 UI 值当电机 RPM 写入或显示。
- 设置页轴参数默认值：X/Y/Z 直线轴 `max_velocity=10000`、`max_acceleration=500`、`soft_minus=-500`、`soft_plus=500`；A/C 旋转轴 `max_velocity=50000`、`max_acceleration=2000`，软限位默认仍按 no-soft-limit `0/0`，除非 owner 已有明确限位 readback。龙门、刀库按其实际轴类型和 owner readback 显示，不得用电机侧默认值兜底。
- A 中心 X 显示当前模态加工坐标系 X 偏移值，B 中心 Y 显示当前模态加工坐标系 Y 偏移值，C 中心 Z 显示当前模态加工坐标系 Z 偏移值；`G53_A_X` / `G53_B_Y` / `G53_C_Z` 不能进入 dirty 或保存链路。B 中心 X/Z 参考 A/C 中心坐标体系，使用 `G53_B_X/G53_B_Z` 或等价 native geometry owner。
- `运动模型/模型选择` 的下拉文本只能作为一次 UI 请求输入；保存成功必须以 `motion_model` owner 写入和源位置 readback 为准。主页面黄色轨迹、active model 旋转轴显示、RTCP ON 当前加工坐标系投影和微内核运动学都消费同一个 active model readback，不得由设置页文本、UI 缓存或默认 AC 分支各跑一套。
- 编码器 bit、内部一圈 count、多圈周期来自 profile/本机参数表和驱动证据；不再通过三点微动或 `log2(count)` 计算。

## 5. 保存与重启

单字段即时保存：

1. 软键盘 `OK` 或下拉选择变化后，UI 只提交本字段。
2. Broker 调用 `settings_apply`，由字段 owner 写入唯一源。
3. owner 源回读一致后清本字段 dirty；失败时保留 dirty 并弹出结构化原因。
4. 需要后端重启才进入运行态的字段，保留已源回读确认的显示值和 `backend_restart_pending`。

右上角 `保存并重启`：

1. 任一字段完成唯一 owner 写入和源回读，或 backend owner 返回 restart pending 后，按钮从 `返回` 改为 `保存并重启`；不得保留 `保存`、`保存并返回` 或只对部分 restart-required 字段切换的旧分支。
2. 字段持久化和源回读由此前字段提交 owner 完成；点击按钮时不得再读取、确认、迁移、补写或修正任何参数 owner。
3. 点击后不执行 Crev/SCALE、active model、轴/从站绑定、零点证据、profile、驱动/Machine/Home/idle/stillness、runtime/SHM freshness 或其它业务 precheck，直接准备 canonical clean restart handoff；该 handoff 必须达到开发板断电重启等价的干净度，不能降级为只重启 UI/Broker/State Publisher 或只刷新 provider。
4. 重启后的 resident owner、native/runtime、drive provider 和必要物理总线 readback 一致后，本次按钮流程才可宣称完成；UI 可以在等待期间显示锁屏/进度/重启中状态。恢复主页面只是显示结果，不是完成证明。machine-on 和驱动 Operation Enabled 属于后续恢复诊断，不能替代 owner/readback 完成证明。

不得出现的行为：

- 用 UI dirty 值、`settings_runtime.json`、`/run` result、Broker 快照或默认值冒充运行态真值。
- 字段保存失败后静默清 dirty。
- 因驱动参数变化在 `保存并重启` 前弹出“先设置驱动参数”阻断；真正写驱动由 `设置驱动` 动作负责。
- 未确认源 owner 写入就触发 backend restart。
- 把“已发起 handoff”“已返回主页面”“若干服务已重启”“machine-on 恢复”或 `Operation Enabled` 当成 clean restart 完成证明。

## 6. 按钮说明

| 按钮 | native action | 页面语义 | 关闭条件 |
| --- | --- | --- | --- |
| `登记本机码` | `settings_dna_register` / `device_dna_register` | 现读本机 live PL Device DNA 和设备公钥并登记到 VPS | `DNA_REGISTER_OK` 或 `DNA_REGISTER_ALREADY_PRESENT`，VPS 返回并保存 canonical `vpsDistributionId`；板端本地不保存 DNA/hash，不从 `device_id`、`vps_distribution_id`、旧状态文件或授权文件回退推断显示 ID |
| `下载授权` | `settings_auth_download` / `device_authorization_download` | 现读本机 live PL Device DNA，带本地 ID 让 VPS 校验后下载工厂端签发授权并用板端公钥验签 | `DEVICE_AUTH_DOWNLOAD_OK` 或 `DEVICE_AUTH_ALREADY_VALID`，DNA/hash 只在本次校验内存使用 |
| `服务器下载` | `settings_server_download` | 下载 public/private profile 映射表 | 结果能区分成功、缺失、失败和 not_modified；schema 细节看映射表 owner |
| `扫描从站` | `settings_scan` | 扫描 EtherCAT 从站并匹配 profile | 当前从站、OP 状态和 profile 唯一匹配可被后续动作使用 |
| `读取驱动` | `settings_read` | 读取驱动/运行参数 readback | 读动作不写参数；结果只作为本次动作证据或诊断输入，页面稳定刷新必须消费 drive/native provider fresh readback |
| `清除故障` | `settings_fault_reset` | 清驱动报警/fault | 不改变轴参数，不证明电子齿轮已写入 |
| `复位驱动` | `settings_drive_reset` / `drive_factory_reset` | 通过 profile 的 restore-defaults 命令恢复驱动厂家默认参数 | 高风险写驱动动作；实时扫描、匹配 profile、写入并回读；成功后旧齿轮/设0证据失效 |
| `设置驱动` | `settings_set_drive` / `drive_set_parameters` | 写电子齿轮和驱动私域参数并 SDO 读回 | 每轴目标值、对象、写入结果、读回值和一致性记录完整 |
| `返回 / 保存并重启` | `settings_save_return` | 动态返回或保存重启入口 | 字段 owner 源回读和 canonical clean restart handoff 按第 5 节闭合 |

上表中的 `服务器下载`、`扫描从站`、`复位驱动`、`读取驱动`、`清除故障`、`设置驱动` 主要描述 BUS/EtherCAT 目标可用状态；Pulse/StepDir 或现场无 BUS/EtherCAT 目标时按 `功能/2-1总线脉冲区别.md` 的能力边界处理。代码修复方向是补齐真实动作、profile/readback 和结构化结果，不把静态禁用、空跑分支或长期 fail-closed 分支当作功能闭合。

## 7. 运动准入依赖

以下共有条件未满足时，Start/Jog/MDI/G-code/rotary 等运动入口必须 fail-closed：

- 授权/DNA latch 已由后台非实时 owner 建立，且状态为 valid；latch 只包含 hash、授权范围、生成时间、boot/restart 代次和验签结果，不包含 raw PL DNA、token、私钥或授权原文。
- latch 缺失、过期、三次 live DNA 读取不一致、授权 DNA hash 与本机 hash 不匹配、授权文件验签失败、授权过期、系统时间不可用或状态 owner 不可用时，运动入口和 `estop_or_reset` 上使能前置必须返回 `MOTION_GATE_DNA_NOT_VERIFIED` 或对应授权失败码。
- active mode、soft limit、`HOME_SEQUENCE`、runtime INI/native owner 和 native/API readback 一致。
- 执行动作现场必须直接读取 microkernel/native safety owner、generation/readback token 或登记 native gate 的 fresh 事实；不得用 SHM fresh/stale 帧、`status_epoch`、UI 显示值、`/run` result 或旧缓存作为准入、动作 proof 或安全证明。

BUS/EtherCAT 运动入口还必须满足：

- 当前从站、profile、标准命令和驱动对象匹配唯一有效。
- X/Y/Z/A/C 的电子齿轮写入和 SDO 读回一致。
- BUS `设0/zero` 的 count-domain 零点证据满足对应 owner 要求。
- `pitch/motor_rev/load_rev/precision/axis_mode` 改变 `SCALE` 时，已设0轴的硬盘 raw soft limit 必须按新 `zero_counts / SCALE + UI限位距离` 同步重算，UI soft limit 仍显示相对 0 位距离。

Pulse/StepDir 运动入口还必须满足：

- Pulse `HOME_OFFSET`、`home_direction`、`HOME_SEARCH_VEL/HOME_LATCH_VEL`、soft limit 和 `HOME_SEQUENCE` 来自 runtime INI/native owner fresh readback。
- Pulse Home 只按 LinuxCNC/HAL 原生 homing、homed/all_homed 和 native/API readback 闭合；不得要求 EtherCAT 从站、profile、SDO、电子齿轮或 BUS `zero_model` 证据。

BUS 主页面 `回零` 的准入以授权/DNA、设置页回零次序和 `设0` count-domain 目标证明为核心；它不消费回零方向、HOME_SEARCH 或限位开关语义。Pulse Home 的回零速度和开关语义按 Pulse owner 处理。

## 8. 人工操作顺序

1. 确认 active mode 是 BUS/EtherCAT，并进入设置页确认 LinuxCNC/native 字段来自 native/受控 readback；unknown/degraded/mismatch 先处理。
2. 点击 `下载授权`，确认授权下载、验签和 DNA 匹配；若因本机码或授权状态缺失失败，停在授权 owner 修复，不通过改顺序绕过。
3. 点击 `服务器下载`，确认 public/private profile 映射表下载、校验和板端缓存结果。
4. 点击 `扫描从站`，确认当前从站、OP 状态和 profile 唯一匹配。
5. 点击 `复位驱动`，确认恢复厂家默认参数、必要 fault 清除、自动读回和最终读回校验；复位后旧电子齿轮、设零和 count-domain trust 证据失效。
6. 点击 `读取驱动`，确认必读 SDO、电子齿轮、statusword 和驱动私域读回完整。
7. 点击 `清除故障`，确认驱动报警/fault 已清除；该步不改变轴参数，也不证明电子齿轮已写入。
8. 点击 `设置驱动`，确认逐轴电子齿轮、模式写入、OP 恢复、必读 SDO 读回和 `DRIVE_SET_OK`，并确认右上角进入 `保存并重启` 状态。
9. 按 X、Y、Z、A、C 每个轴逐个执行设置页单轴 `设零/设0`；BUS/EtherCAT X/Y/Z/A/C 单轴 `设0` 都不弹危险确认，Pulse `设零` 按键盘提交处理。当前轴未取得请求、raw/count-domain 或 Pulse owner 回读、硬盘 raw 限位写入（BUS）和结果码证据前，不进入下一轴。
10. 设置 `运动模型/模型选择` 和 A/B/C 中心坐标；模型选择必须写 `motion_model` owner 并源位置回读，只允许编辑 owner 明确的 `G53_A_Y`、`G53_A_Z`、`G53_B_X`、`G53_B_Z`、`G53_C_X`、`G53_C_Y`；A 中心 X、B 中心 Y、C 中心 Z 是当前模态加工坐标系偏移值，不进入 dirty 或保存链路。每个字段提交后确认字段保存成功或结构化失败。
11. 点击 `保存并重启`，按钮不再读取或确认字段 owner，也不执行任何参数/模型/驱动/runtime/Machine 业务 precheck，直接准备 canonical clean restart handoff；只有重启后的 resident owner/native/runtime/drive provider 与必要物理总线 readback 一致，才能确认本次保存重启完成。恢复主页面只是 UI 展示恢复，不是完成证明；若 clean restart 不能证明达到开发板断电重启同等干净度，必须走板端系统级 reboot 或断电重启等价路径并 fail-closed。
12. 真实运动前先做急停、软限位、回零次序、设0投影和低速 jog 的单项验证。

## 9. 最小验收清单

- UI 列名、字段 key 和 `settings_apply` 支持字段一致。
- 进入设置页时，坐标、转速、进给速度和主轴/进给倍率显示可来自 fresh SHM 显示投影；当前模态、当前刀号、刀长、active mode、授权状态和其它稳定显示来自开机已加载的 microkernel/native 常驻内存、参数 owner 映射或受控 readback；缺失时显示 unavailable/degraded。设置页不得在显示刷新、字段切换或保存成功判断时直接读硬盘配置、旧 JSON、`/run` result、脚本快照或 profile 文件兜底。
- 单字段保存只写本字段 owner，只回读本字段源位置；失败不清 dirty。
- 可编辑单元格和软键盘数字输入的本地响应均不超过 `0.2s`；软键盘 `OK` 或下拉提交后的 UI 本地提交反馈、dirty/pending 标记或结构化拒绝不超过 `0.5s`。字段 owner 写入和源位置回读若进入 native、文件 flush、网络、驱动硬件动作、后台重启或长流程 readback，必须显示 pending/进度并等待真实回读，不得为了满足 `0.5s` 伪造成功。
- 修改任意可编辑参数后，右上角显示 `保存并重启`；字段已即时保存但尚未统一重启生效时仍保持 `保存并重启`。
- `保存并重启` 触发 canonical clean restart 时不重新检查字段 owner 或任何业务状态；字段 owner 写入/回读由此前字段提交动作负责。
- 驱动参数不因 `settings_apply OK` 自动视为已进驱动；必须由 `设置驱动` 的 SDO 读回证明。
- `复位驱动` 后，旧电子齿轮、设0、rotary count checkpoint 和 count-domain trust 证据全部失效。
- `settings_dna_register` 只登记 DNA；`settings_auth_download` 只下载验签授权；板端不签发授权。
- 上使能本身不是危险确认动作，但只能在无运动意图、无未完成 MDI/程序队列、无冲突后台写类 job、急停/硬故障/限位状态允许，并且 microkernel/native enable gate 通过后由后端自动执行；结果必须用 fresh microkernel/native 回读证明，不得要求操作员手动补上使能。设置页 BUS 单轴 `设0` 不作为危险确认动作；A/C 轴坐标写入、会导致旋转轴实际运动或相位重定的动作才需要二次危险确认。
- 开机、backend/UI restart 后或 `estop_or_reset` 上使能前必须能看到授权/DNA latch valid；运动热路径只读 latch，不重复读硬件 DNA、验签文件或访问 VPS；latch invalid/degraded 时设置页显示原因，运动和上使能 fail-closed。
- `settings_server_download` / `settings_scan` 的 schema、profile 选择和新增品牌流程只按映射表 owner 验收。
- Pulse/StepDir 或现场无 BUS/EtherCAT 目标时，`服务器下载` 按授权/VPS owner 继续执行；`扫描从站`、`复位驱动`、`读取驱动`、`清除故障`、`设置驱动` 由后台按真实硬件能力、目标从站、profile、授权和安全状态处理。能跑通真实 action/profile/readback 的路径必须优先跑通；确实不适用时也要返回结构化原因并在进度弹窗显示。
- `motion_model` / `[RTCP].MODEL` 和 `G53_A_Y/G53_A_Z/G53_B_X/G53_B_Z/G53_C_X/G53_C_Y` 只写 active runtime INI `[RTCP]` / native model/geometry owner，不写 `settings_runtime.json`。
- `soft_minus/soft_plus=0` 代表该方向软限位不生效，并由 native/runtime owner 表达。
- 真实板端低速验收必须分别触发一个正向软限位、一个负向软限位、一个正向物理限位和一个负向物理限位：每次都证明当前运动停止但 `machine_enabled=true`、触发方向的后续点动被拒绝、相反方向点动能够退出；同一关节正负物理限位同时生效仍须 fail-closed。限位测试不得用改坐标、伪造 UI 状态或直接后端命令替代原始页面点动路径。
- `S/从轴/刀库` 当前不得作为 X/Y/Z/A/C 运动轴的通过条件或可误触入口。

## 10. 当前硬边界

- G53/RTCP 几何和运动模型：按 `REQ-RTCP-G53-NATIVE-ACTUAL`，native/runtime INI `[RTCP]`、`motion_model` owner 和微内核/native actual 是真源；UI dirty、设置页下拉文本、Broker 快照、SHM 显示投影、`/run` JSON、`settings_runtime.json` 都不是几何或模型真值。
- `settings_runtime.json`：按 `REQ-SETTINGS-RUNTIME-DRIVE-ONLY`，只保留 Settings Drive 私域和驱动/count-domain 证据；速度、加速度、HOME、HOME_OFFSET、HOME_SEQUENCE、软限位、RTCP/G53、WCS/G92/tool/HAL/modal 不得写回该文件当产品参数源。
- 字段 owner：按 `REQ-PARAM-MEMORY-LIGHTWEIGHT-SAVE`，每个表格字段必须接回唯一 owner，再由 owner/native fresh 回读；UI dirty 值只是一笔请求输入，SHM 只显示允许的 UI 投影字段。
- 脉冲切换限制：因脉冲/StepDir 硬件及固件暂时未就绪，当前 `step_ip_v1_5.contract.json` 仍为 `cold_staged_not_runtime_selectable` 时，运动模式下拉中的 Pulse 目标必须 disabled，任何强制提交必须由 C/Broker dispatch fail-closed，不能进入真实模式切换流程。该限制属于当前硬件准入例外，只能由硬件/固件未就绪这一真实原因触发；代码应优先补齐真实 Pulse 切换与回零闭环，不能把 UI 置灰、Broker/C dispatch 拦截或长期拦截当作最终方案。硬件/固件就绪、板端证据齐备并由 `REQ-BUS-PULSE-MODE-DIFFERENCE` owner 更新为 `runtime_selectable` 后，Pulse 字段和驱动动作能力边界按 owner 调整。
- native 优先：按 `REQ-NATIVE-OWNER-FIRST` 和 `REQ-LINUXCNC-COMMAND-GATE`，LinuxCNC/HAL/EtherCAT/native helper 已有 owner 的能力，不在 V3 里扩展第二真源。
