# 触摸校准与输入链路说明

引用需求真源：`REQ-DOC-SINGLE-SOURCE`、`REQ-PARAM-MEMORY-LIGHTWEIGHT-SAVE`、`REQ-TOUCH-CALIBRATION-INPUT`。

## AI 阅读入口

- 启动内存/热路径通用规则：见 `REQ-PARAM-MEMORY-LIGHTWEIGHT-SAVE` / `功能/0-1开机参数入内存.md`，本文只保留本功能特有边界。
- 本文定位：触摸校准文件 owner、真实手指输入链路和旧路径退役边界。
- 先读 `REQ-TOUCH-CALIBRATION-INPUT`；本文不定义 UI 按钮语义、远程输入协议、运动门禁或后端控制链。
- 产品 UI 只以 safe_ui 校准文件为主 owner；旧 `/opt/8ax/ui/re_touch_calibration.json` 不得作为读取 fallback、写入镜像或验收对象。

本文是触摸校准文件 owner、真实手指输入链路和校准验收边界的 owner。本文不定义 UI 按钮语义、远程输入协议、运动门禁或后端控制链；这些仍按对应 `REQ-*` owner。

## P0 产品要求

产品 UI 的触摸坐标必须使用现场真实手指校准后的 `raw-evdev-cal-v2` 仿射矩阵。产品生效文件固定为：

- v5 产品 UI 实际加载：`/opt/8ax/safe_ui/re_touch_calibration.json`
- v5 触摸诊断日志：`/run/8ax_v5_product_ui/touch_events.jsonl`
- 禁止路径：`/opt/8ax/ui/re_touch_calibration.json`

校准脚本完成后只以 v5 产品 UI 主文件 `/opt/8ax/safe_ui/re_touch_calibration.json` 为生效 owner。旧 `/home/cnc/z20_touch_calibration.json` 只能作历史现场文件参考，不作为 v5 验收真源；旧 `/opt/8ax/ui/re_touch_calibration.json` 不得作为产品 UI 读取 fallback、写入镜像或验收对象。

生效参数：

```json
{
  "mode": "raw-evdev-cal-v2",
  "profile": "screen-fit-topleft-1024x600-v1",
  "anchor": "topleft-v1",
  "display": [1024, 600],
  "sx": 1.0145462880571579,
  "sxy": 0.01046522451551646,
  "syx": -0.011917978012422892,
  "sy": 1.013616478583013,
  "ox": -13.746023173761872,
  "oy": -9.09696616291931
}
```

## P0 禁止误判

触摸验收必须基于真实手指输入和现场校准矩阵。模拟点击、远程输入或猜偏移量不能替代真实触摸校准；按钮可见但手点偏移时，优先处理触摸校准链路，不得把偏差直接归因于按钮区域、UI 逻辑或触摸门禁。

## P0 输入链路

`re` UI 读取触摸输入：

- 输入设备：`/dev/input/event0`
- udev 链接：`/dev/input/by-path/z20-touchscreen`
- 产品触摸诊断：`/run/8ax_v5_product_ui/touch_events.jsonl`
- 远程输入状态：当前 v5 remote input/relay 状态按 `/root/Desktop/v5/app/src/v5_remote_*` 和 `/run/8ax_v5_product_ui/` 运行事实记录；旧 v3 状态文件不作为验收真源。
- UI 校准主文件：`/opt/8ax/safe_ui/re_touch_calibration.json`
- 禁止读取/镜像写入旧路径：`/opt/8ax/ui/re_touch_calibration.json`

v5 产品 UI 触摸诊断链路：

- 输入设备仍以 `/dev/input/by-path/z20-touchscreen` 为首选，实际指向 `/dev/input/event0`。
- v5 触摸诊断日志固定为 `/run/8ax_v5_product_ui/touch_events.jsonl`。
- v5 UI action 诊断日志固定为 `/run/8ax_v5_product_ui/ui_events.jsonl`，只记录真实 LVGL button click 后的 action/command 诊断证据，不作为产品状态真源、fallback 或命令结果来源。
- v5 触摸诊断启用 marker 固定为 `/run/8ax_v5_product_ui/enable_touch_diagnostics`。
- v5 不得把 `/run/8ax_v3_product_ui/touch_events.jsonl` 作为验收真源、fallback 或镜像写入目标；旧 v3 运行目录只能作为旧系统诊断事实存在。
- v5 触摸诊断只能读取真实 evdev 事件并套用 `safe_ui` 校准矩阵，不得用远程输入、模拟点击、固定坐标、扩大热区或旧 UI 日志替代真实手指证据。
- v5 新增校准工具、脚本、日志、部署入口和运行目录命名不得再使用 `z20`；旧硬件 udev 路径 `/dev/input/by-path/z20-touchscreen` 只作为外部既成设备路径保留，不作为 v5 新文件/目录/脚本命名依据。
- v5 校准工具只允许 raw evdev 输入域：可以在 Python `evdev` 包可用时使用该包读取同一 evdev 设备，也可以在包缺失时直接解析 Linux `input_event`；不得保留 Qt pointer、鼠标、远程输入、测试直调或屏幕坐标替代链路。
- v5 校准文件只接受 `raw-evdev-cal-v2`、`screen-fit-topleft-1024x600-v1`、`topleft-v1`；旧 `raw-evdev-cal-v1` 只能判为不支持并要求重新校准，不得作为兼容迁移读取。

触摸映射公式：

```text
screen_x = raw_x * sx  + raw_y * sxy + ox
screen_y = raw_x * syx + raw_y * sy  + oy
```

相关仓库内文件：

- `safe_ui_lvgl/re_touch_calibration.json`
- v3 只读参考：`D:/v3/tools/z20_touch_calibration/**`
- v5 当前工具入口：`tools/v5_touch_calibration/v5_touch_calibration.py`
- v5 当前板端入口：`/opt/8ax/tools/v5_touch_calibration/v5_run_touch_calibration.sh`

## v5 触摸校准工具移植

旧 Z20 原校准程序只能作为 `D:/v3` 只读参考；移植到 v5 后必须改成 v5 自有命名和路径：

- VM 真源：`tools/v5_touch_calibration`
- 板端安装：`/opt/8ax/tools/v5_touch_calibration`

v5 工具文件：

- `tools/v5_touch_calibration/v5_touch_calibration.py`
- `tools/v5_touch_calibration/v5_touch_core.py`
- `tools/v5_touch_calibration/v5_touch_evdev.py`
- `tools/v5_touch_calibration/v5_touch_window*.py`
- `tools/v5_touch_calibration/v5_apply_touch_calibration.py`
- `tools/v5_touch_calibration/v5_run_touch_calibration.sh`

v5 适配包装：

- `tools/v5_touch_calibration/v5_restart_after_touch_calibration.sh`

适配原因：

- `re` 板端没有 `cnc` 用户。
- `re` 不使用 Z20 原 `linuxcnc-lcd.service`。
- 旧程序默认写旧中间文件；v5 校准完成后只允许把最终 `raw-evdev-cal-v2` 写入 `/opt/8ax/safe_ui/re_touch_calibration.json`。
- v5 校准工具的 apply/restart helper 固定为 `/opt/8ax/tools/v5_touch_calibration/` 同目录入口；不得扫描 `/opt/z20`、`/home/cnc`、`/usr/local/sbin` 或旧工程 helper 候选路径。

包装脚本负责：

1. 停止 `re` UI。
2. 设置 `linuxfb` / 触摸输入环境。
3. 启动 v5 PyQt5 触摸校准窗口。
4. 校准完成后只同步 JSON 到 `/opt/8ax/safe_ui/re_touch_calibration.json`，不得再生成 `/opt/8ax/ui/re_touch_calibration.json` 镜像。
5. 重启 v5 UI。
6. 如发现旧 `/opt/8ax/ui/re_touch_calibration.json`，只能删除该退休文件；不得读取、复制、迁移或把它当作 fallback。

## 重新校准步骤

在 Windows 上执行：

```powershell
ssh re-board "pkill -f '[v]5_touch_calibration.py' 2>/dev/null || true; nohup /opt/8ax/tools/v5_touch_calibration/v5_run_touch_calibration.sh >/tmp/v5_touch_calibration.log 2>&1 &"
```

然后在屏幕上按顺序点击 5 个黄色十字中心。

校准完成后检查：

```powershell
ssh re-board "cat /opt/8ax/safe_ui/re_touch_calibration.json; sha256sum /opt/8ax/safe_ui/re_touch_calibration.json"
```

确认 `re` UI 是否加载新参数：

```powershell
ssh re-board "touch /run/8ax_v5_product_ui/enable_touch_diagnostics; tail -n 20 /run/8ax_v5_product_ui/touch_events.jsonl"
```

## 注意事项

- 触摸校准必须用真实手指点屏幕，不能用模拟点击替代。
- 如果用户反馈“点不准”，先看 `touch_events.jsonl` 中同一轮事件的 `calibration_path`、`calibration_enabled`、`raw_x/raw_y`、`x/y` 和 UI action 日志，不要先改按钮坐标。
- 校准窗口如果显示不完整，应优先修校准窗口布局，不要修改校准算法。
- v3/旧工程中的 Z20 校准程序只能作为只读参考；v5 新增工具、脚本和部署入口必须使用 `v5_*` 命名。
- 触摸校准不影响 EtherCAT、8ax 后端、驱动参数和运动控制。

## 验收标准

触摸修复闭环必须同时满足：

1. `/opt/8ax/safe_ui/re_touch_calibration.json` 存在且内容合法；产品脚本和 UI 不得读取或写入 `/opt/8ax/ui/re_touch_calibration.json`。
2. `/run/8ax_v5_product_ui/touch_events.jsonl` 中能看到 `calibration_path=/opt/8ax/safe_ui/re_touch_calibration.json` 且 `calibration_enabled=true`。
3. 真实手指点击右侧按钮、顶部网络按钮、弹窗关闭按钮均能稳定命中。
4. `touch_events.jsonl` 与 UI 事件日志能关联同一轮 `raw/screen -> press/release -> clicked object/action`。
5. 不能把触摸偏差通过扩大所有按钮热区来掩盖。

v5 补充验收标准：

1. `/run/8ax_v5_product_ui/touch_events.jsonl` 必须出现真实手指事件，且 `calibration_path=/opt/8ax/safe_ui/re_touch_calibration.json`、`calibration_enabled=true`。
2. `/run/8ax_v5_product_ui/ui_events.jsonl` 必须出现同一轮真实手指触发的 `button_clicked` 事件，并记录 v5 action、prepared/local_only、command name 和 command owner。
3. v5 UI action 诊断日志只能由 LVGL 真实输入回调产生；不得用远程输入、模拟点击、测试直调 `v5_main_page_trigger_action()` 或手写日志替代。
4. v5 校准工具源码和板端安装目录不得命中 Qt pointer 替代输入、旧 `raw-evdev-cal-v1` 兼容、旧 `/opt/z20` helper、旧 Z20 工具名或旧 `/opt/8ax/ui/re_touch_calibration.json` 读取路径；旧 `/opt/8ax/ui/re_touch_calibration.json` 只允许作为删除对象出现。
