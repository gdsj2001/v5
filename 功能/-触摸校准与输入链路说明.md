## P0 产品要求

产品 UI 的触摸坐标必须使用现场真实手指校准后的 `raw-evdev-cal-v2` 仿射矩阵。产品生效文件固定为：

- 板端 Z20 原校准程序输出：`/home/cnc/z20_touch_calibration.json`
- v3 产品 UI 实际加载：`/opt/8ax/safe_ui/re_touch_calibration.json`
- 禁止路径：`/opt/8ax/ui/re_touch_calibration.json`

校准脚本完成后必须让 Z20 输出和 v3 产品 UI 主文件内容一致。旧 `/opt/8ax/ui/re_touch_calibration.json` 不得作为产品 UI 读取 fallback、写入镜像或验收对象。

生效参数：

```json
{
  "mode": "raw-evdev-cal-v2",
  "profile": "screen-fit-topleft-1024x600-v1",
  "anchor": "topleft-v1",
  "display": [1024, 600],
  "sx": 1.0145462880571579,
  "sxy": 0.01046522451551646,
  "syx": -0.011917978012422892,
  "sy": 1.013616478583013,
  "ox": -13.746023173761872,
  "oy": -9.09696616291931
}
```

## P0 禁止误判

触摸验收必须基于真实手指输入和现场校准矩阵。模拟点击、远程输入或猜偏移量不能替代真实触摸校准；按钮可见但手点偏移时，优先处理触摸校准链路，不得把偏差直接归因于按钮区域、UI 逻辑或触摸门禁。

## P0 输入链路

`re` UI 读取触摸输入：

- 输入设备：`/dev/input/event0`
- udev 链接：`/dev/input/by-path/z20-touchscreen`
- 产品触摸诊断：`/run/8ax_v3_product_ui/touch_events.jsonl`
- 远程输入状态：`/run/8ax_v3_product_ui/remote_input_status.json`
- UI 校准主文件：`/opt/8ax/safe_ui/re_touch_calibration.json`
- 禁止读取/镜像写入旧路径：`/opt/8ax/ui/re_touch_calibration.json`

触摸映射公式：

```text
screen_x = raw_x * sx  + raw_y * sxy + ox
screen_y = raw_x * syx + raw_y * sy  + oy
```

## Z20 原校准程序移植

Z20 原校准程序已迁入 `v3` 仓库，正式流程只能使用仓库内副本：

- Windows：`tools/z20_touch_calibration`
- 板端：`/opt/8ax/tools/z20_touch_calibration`

仓库内工具文件：

- `tools/z20_touch_calibration/z20_touch_test.py`
- `tools/z20_touch_calibration/z20_apply_touch_calibration.py`
- `tools/z20_touch_calibration/z20_run_touch_test.sh`
- `tools/z20_touch_calibration/z20_launch_touch_calibration.sh`

`re` 适配包装：

- `tools/z20_touch_calibration/re_run_z20_touch_calibration.sh`
- `tools/z20_touch_calibration/re_restart_after_z20_touch_calibration.sh`

适配原因：

- `re` 板端没有 `cnc` 用户。
- `re` 不使用 Z20 原 `linuxcnc-lcd.service`。
- Z20 原程序默认写 `/home/cnc/z20_touch_calibration.json`，而 v3 产品 UI 加载 `/opt/8ax/safe_ui/re_touch_calibration.json`。

包装脚本负责：

1. 停止 `re` UI。
2. 设置 `linuxfb` / 触摸输入环境。
3. 启动 Z20 原 PyQt5 触摸校准窗口。
4. 校准完成后只同步 JSON 到 `/opt/8ax/safe_ui/re_touch_calibration.json`，不得再生成 `/opt/8ax/ui/re_touch_calibration.json` 镜像。
5. 重启 `re` UI。

## 重新校准步骤

在 Windows 上执行：

```powershell
ssh re-board "pkill -f '[z]20_touch_test.py' 2>/dev/null || true; nohup /opt/8ax/tools/z20_touch_calibration/re_run_z20_touch_calibration.sh >/tmp/re_z20_touch_calibration.log 2>&1 &"
```

然后在屏幕上按顺序点击 5 个黄色十字中心。

校准完成后检查：

```powershell
ssh re-board "cat /opt/8ax/safe_ui/re_touch_calibration.json; sha256sum /home/cnc/z20_touch_calibration.json /opt/8ax/safe_ui/re_touch_calibration.json"
```

确认 `re` UI 是否加载新参数：

```powershell
ssh re-board "touch /run/8ax_v3_product_ui/enable_touch_diagnostics; tail -n 20 /run/8ax_v3_product_ui/touch_events.jsonl"
```

## 注意事项

- 触摸校准必须用真实手指点屏幕，不能用模拟点击替代。
- 如果用户反馈“点不准”，先看 `touch_events.jsonl` 中同一轮事件的 `calibration_path`、`calibration_enabled`、`raw_x/raw_y`、`x/y` 和 UI action 日志，不要先改按钮坐标。
- 校准窗口如果显示不完整，应优先修校准窗口布局，不要修改校准算法。
- `re` 里的 Z20 校准程序是工具移植，允许保留 Z20 文件名；产品 UI 仍使用 `re` 自己的运行链路。
- 触摸校准不影响 EtherCAT、8ax 后端、驱动参数和运动控制。

## 验收标准

触摸修复闭环必须同时满足：

1. `/opt/8ax/safe_ui/re_touch_calibration.json` 存在且内容合法；产品脚本和 UI 不得读取或写入 `/opt/8ax/ui/re_touch_calibration.json`。
2. `/run/8ax_v3_product_ui/touch_events.jsonl` 中能看到 `calibration_path=/opt/8ax/safe_ui/re_touch_calibration.json` 且 `calibration_enabled=true`。
3. 真实手指点击右侧按钮、顶部网络按钮、弹窗关闭按钮均能稳定命中。
4. `touch_events.jsonl` 与 UI 事件日志能关联同一轮 `raw/screen -> press/release -> clicked object/action`。
5. 不能把触摸偏差通过扩大所有按钮热区来掩盖。
