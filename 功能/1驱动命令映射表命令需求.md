## 1. 目标

驱动映射表的目标是让新增驱动品牌、型号或固件时优先更新 VPS/profile，而不是修改板端 UI、Broker、后台 action/native owner 或运动业务代码。

标准闭环：

```text
用户/经销商上传驱动资料
  -> VPS/工厂端保存原始资料并解析成标准 driver_profile_map.json
  -> public/private profile 通过审核、实测、canonical JSON、sha256 后发布
  -> 板端授权下载 public/private 缓存
  -> 按真实 EtherCAT 从站、轴绑定和 DNA 选择唯一 profile
  -> 业务层只调用 profile 中的 drive.* 标准命令
  -> 写后/读后按 profile 声明读回验证
```

上传资料不能直接成为板端运行输入。任一环节缺资料、缺命令、未授权、SHA/schema 失败、真实从站不匹配、多匹配或读回不一致，必须 fail-closed。

从站绑定和 encoder bit 的运行态生效按 `功能/0-2参数归类.md`：profile 和驱动命令只提供驱动私域证据、默认 bit、SDO/PDO 对象和读写能力；设置页字段保存不直接 live 改微内核运行内存。`保存并重启` 后的 backend reload 必须重新初始化 HAL/EtherCAT/SDO/PDO/轴绑定并从物理总线读回一致，否则保持去使能并 fail-closed。

## 2. 真源和缓存

| 位置 | 角色 | 规则 |
| --- | --- | --- |
| VPS public | `/opt/8ax-auth/storage/drive-profiles/public/driver_profile_map.json` | 公共发布真源；同目录必须有 `.sha256` |
| VPS private | `/opt/8ax-auth/storage/private/<vpsDistributionId>-<pl_dna_hash>/driver_profile_map.json` | 当前设备私有真源；路径使用 VPS 返回并保存的 ID-DNA folder，不得退回 hash-only 或旧 ID-only；该 folder 只在 VPS 服务端内部解析，板端下载 URL 只携带 6 位 ID 并用临时 DNA/DNA hash 校验；同 schema，只服务授权范围 |
| 源码 public 编辑源 | `board/config/drive-profiles/public/driver_profile_map.json` | Windows 唯一维护输入，不是板端运行 fallback |
| 源码 private 编辑源 | `board/config/drive-profiles/private/<pl_dna_hash>_driver_profile_map.json` | Windows 唯一维护输入；上传 VPS 时去掉文件名前缀 |
| 板端 public 缓存 | `/opt/8ax/drive-profiles/public/driver_profile_map.json` | 下载校验通过后原子替换 |
| 板端 private 缓存 | `/opt/8ax/drive-profiles/private/driver_profile_map.json` | VPS private 明确不存在时必须删除本地缓存 |

硬规则：

- VPS 上的 `driver_profile_map.json` 就是板端可执行的标准映射表；下载后不得转义成第二套结构。
- 板端只允许使用下载后的 public/private 缓存表作为命令来源；源码表、旧 `effective` 合成表、诊断快照和人工临时文件都不能成为产品 fallback。
- public/private 不得字段级合并。选择规则是：当前 DNA private 唯一匹配优先；private 无匹配时 public 唯一匹配；private 已选中后不得从 public 补缺命令。
- public 未更新时不重写本地文件；private 远端删除时必须删除本地 private 缓存，并让后续扫描/设置/读取/复位只能走 public。
- 下载写盘必须字节保真：远端字节先验 SHA、再 JSON/schema/scope/profile 校验，最后临时文件 + fsync + atomic rename。

## 3. 发布物要求

`driver_profile_map.json` 必须是合法 JSON，不允许注释。发布物必须包含：

| 字段 | 要求 |
| --- | --- |
| `schema` / `schema_version` | 标识当前 profile schema |
| `map_version` | 非空，可用于版本判断和回滚 |
| `release_contract` | 至少声明固件范围、对象字典范围、授权功能范围和回滚能力 |
| `profiles` | 非空数组 |
| `profiles[].profile_id` | 唯一、可审计 |
| `vendor_id/product_code/revision/model` | 用于真实从站匹配，类型稳定 |
| `profiles[].commands` | 非空对象，key 必须是 `drive.*` 标准命令 |

命令对象通用要求：

| 字段 | 要求 |
| --- | --- |
| `standard_command` | 若存在，必须与命令 key 一致 |
| `supported` | 必须是布尔值 |
| `object` | 必须是对象；Adapter 派生命令也要用 `object.name` 标明来源，不能用 `null` |
| `access` / `data_type` | 必须能确定读写方向和数据类型 |
| `timeout_ms` / `retry` | 写命令和高风险命令必须声明 |
| `verify_object` / `verify_expected` / `verify_tolerance` | 会写入或改变驱动状态的命令至少声明一个读回验证字段 |
| `failure_code` / `failure_message_cn` | 必须给 UI/Broker 返回结构化失败 |

canonical JSON 以 `board/services/auth_download/drive_profile_download_core.py::canonical_json_bytes()` 或等价同名函数为准：UTF-8 无 BOM、LF 换行、稳定 key 顺序、文件末尾一个 LF，字符串里的路径反斜杠在上传前递归归一为 `/`。历史 v3 canonical-JSON 工具已删除，只可作为算法来源记录，不是 v5 产品路径。

## 4. 运行期选择

产品运行期只能消费一条已装载到 RAM 的闭环：

```text
VPS public/private 真源
  -> 板端下载缓存 public/private
  -> 开机装载或受控刷新为驱动映射内存快照
  -> 真实 EtherCAT 扫描身份
  -> 设置页轴-从站绑定
  -> private 优先的整 profile 唯一选择
  -> 调用选中 profile 的原始 drive.* 标准命令对象
  -> 写后/读后验证
```

扫描选择必须记录在内存快照和动作证据中：

| 字段 | 作用 |
| --- | --- |
| `selected_map_path` | 选中 public 或 private 缓存路径 |
| `selected_map_sha256` | 选中文件 SHA |
| `map_source` | `private` 或 `public` |
| `profile_id` | 选中的 profile |
| `axis` / `slave_index` | 当前轴和真实从站位置 |
| `vendor_id/product_code/revision/model` | 真实从站身份 |
| `commands` 摘要 | 实际可调用的 `drive.*` 命令集合 |

`slave_index` 只能用于现场拓扑中的 `axis -> slave_index -> selected profile` 绑定，不能作为 VPS profile 匹配字段。

运行期驱动动作不得直接打开 `driver_profile_map.json`、源码 public/private 编辑源、旧 effective 合成表或测试夹具文件。profile 文件读取只允许发生在开机装载、授权下载成功后的受控刷新、设置保存后的 owner reload 或显式诊断；热路径以已校验的 `selected_map_sha256` 和内存命令对象为准。

## 5. P0 必须命令

缺 P0 命令时，关联动作必须 fail-closed。

| 标准命令 | 作用 | 缺失影响 |
| --- | --- | --- |
| `drive.identify` | 读取 `vendor_id/product_code/revision/model/serial` | 禁止 profile 选择和设置驱动 |
| `drive.read_statusword` | 读取 CiA402 状态字 | 禁止设置驱动和运动准入 |
| `drive.write_controlword` | 写 shutdown/switch on/enable/disable/quick stop | 禁止 enable/disable/fault reset |
| `drive.reset_fault` | 清除 fault bit/error_code | 清除故障按钮和自动清故障 blocked |
| `drive.restore_factory_defaults` | 恢复驱动出厂参数 | 复位驱动按钮 blocked；需要默认基线的设置驱动 blocked |
| `drive.read_mode` | 读取当前模式 | 模式相关动作 blocked |
| `drive.write_mode` | 写目标模式 | 模式切换 blocked |
| `drive.read_actual_position` | 读取 CiA402 整位置反馈 | 普通设 0、位置诊断和同源审计 blocked |
| `drive.read_abs_multi_turn` | 读取绝对多圈圈数 | A/C 绝对多圈 basis blocked |
| `drive.read_abs_single_turn` | 读取当前圈内绝对位置 | A/C split basis blocked |
| `drive.read_abs_position64` | 读取或 Adapter 合成完整多圈绝对 count | A/C 完整绝对 basis 和断电恢复 blocked |
| `drive.read_actual_velocity` | 读取实际速度 | 运动 gate 不能确认静止 |
| `drive.read_encoder_resolution` | 动态读取原始编码器单圈分辨率 | 只用于审计交叉校验，不能替代本机参数表 |
| `drive.set_egear` | 写电子齿轮比分子/分母 | 设置驱动 blocked |
| `drive.read_egear` | 读回电子齿轮比 | 不能清 `drive_param_dirty` |
| `drive.save_parameters` | 保存参数到驱动非易失存储 | profile 要求持久化时 blocked |

绝对多圈 A/C 轴必须同时具备四个读取能力：`drive.read_actual_position`、`drive.read_abs_multi_turn`、`drive.read_abs_single_turn`、`drive.read_abs_position64`。其中 `drive.read_actual_position` 仍是 CiA402 整位置反馈，用于普通设 0 和同源审计；后三个用于绝对多圈 split/position64 basis。

`drive.read_abs_position64` 可以是厂家直接 64 位对象，也可以由映射表声明 `compose_from=[drive.read_abs_multi_turn, drive.read_abs_single_turn]` 后由 Adapter 合成。Adapter 合成必须防撕裂：`turn1 -> single_turn -> turn2`，只有 `turn1 == turn2` 才直接接受；跨边界不一致时重读，耗尽重试返回 `DRIVE_ABS_SPLIT_READ_INCOHERENT`。

## 6. P1 和 P2 命令

P1 用于增强诊断、回零、停止和异常处理；缺失时不一定阻断所有配置，但关联动作必须 degraded 或 blocked。

| 标准命令 | 作用 |
| --- | --- |
| `drive.read_error_code` | UI/日志显示可诊断故障 |
| `drive.read_io_status` | 回零前检查限位、home、alarm 输入 |
| `drive.homing_start` / `drive.homing_status` | 驱动侧 homing 闭环 |
| `drive.abort_motion` / `drive.quick_stop` | 软件停止路径；不替代硬急停 |
| `drive.software_reset` | 显式驱动软件重启；不得被普通复位驱动默认触发 |
| `drive.read_following_error` | 运动质量和失控诊断 |
| `drive.read_bus_state` | EtherCAT/CiA402 通信健康 |

P2 用于维护、诊断和高风险位置基准动作。没有 P2 不应阻断基础配置，但不能伪装成已支持。

| 标准命令 | 作用 | 额外边界 |
| --- | --- | --- |
| `drive.read_torque/current/temperature/dc_link_voltage` | 诊断 | 只读 |
| `drive.write_limit_config` / `drive.read_limit_config` | 驱动侧限位配置 | 必须读回验证 |
| `drive.backup_parameters` / `drive.restore_parameters` | 备份/恢复驱动参数 | 高风险，需显式确认 |
| `drive.preset_position` | 改写驱动内部当前位置基准 | 必须走 position basis transaction |
| `drive.clear_multi_turn` | 清除或重置多圈计数 | 不得用于普通回零、设 0、清故障 |
| `drive.write_position_offset` | 写厂家位置偏置对象 | 不得替代软件设 0 |
| `drive.verify_position_basis` | 读回位置基准确认位或状态 | 高风险命令成功闭环必需 |
| `drive.read_encoder_multiturn_limit` | 读取厂家明确声明的多圈物理报警/安全上限 | ESI 位宽和 raw modulus 不能映射为该命令 |
| `drive.calibration_move_relative` / `drive.calibration_jog` | 旧微动校准命令 | 产品路径禁止，必须 `supported=false` |

## 7. 编码器和 count-domain 参数

`driver_profile_map.json` 只描述型号级默认和命令能力；本机最终采用值必须写入 Settings Drive 私域 owner，并由 State Publisher/Broker 冷启动加载到内存投影。不得把板端运行读回值写回 VPS public/private map。

| 字段 | 角色 | 规则 |
| --- | --- | --- |
| `profile_id` / `drive_profile_id` | 轴绑定的命令 profile | Settings Drive 私域证据；用于选择命令对象和默认 `encoder_bits` |
| `encoder_bits` / `encoder_resolution_bits` | 编码器物理单圈 bit | profile 只给默认；本机人工 override 必须长期优先 |
| `drive_internal_counts_per_motor_rev` | 电子齿轮前驱动内部一圈 count | 由最终 bit 计算 `2^encoder_bits`，不是映射表运行输入 |
| `feedback_counts_per_motor_rev` | 当前 0x6064 反馈域电机一圈 count | 由电子齿轮读回计算并保存；count-domain 运行准入依赖它 |
| `encoder_single_turn_counts` | 原始单圈 count 审计值 | 用于交叉校验，不能替代 `feedback_counts_per_motor_rev` |
| `encoder_multiturn_raw_bits/raw_modulus/raw_value_type` | 多圈 raw 计数器解析信息 | 只能表示回绕周期，不是物理安全上限 |
| `encoder_multiturn_limit` | 多圈物理/配置上限 | 只有厂家对象或本机参数明确给出时可信 |
| `rotary_load_counts_per_rev` | 负载一圈对应反馈 count `Crev` | 由反馈 count、电子齿轮和减速比确认 |

运行期涉及 LinuxCNC 比例时，最终真源仍是 runtime INI/native `SCALE` 和 native/API readback。不得用持久化倒数比例、旧浮点零位、UI 默认值或旧 profile count 替代。

## 8. 复位驱动和恢复出厂

`复位驱动` 的业务含义是恢复驱动出厂默认参数，不是清报警。`清除故障` 只能调用 `drive.reset_fault`。

复位驱动前置条件：

- 当前 action 明确请求，或当前 profile 声明 `require_factory_reset_before_set_egear=true`。
- 目标轴全部有真实从站、OP 状态、唯一 profile，且 `drive.restore_factory_defaults.supported == true`。
- microkernel/native drive-write gate 判定当前处于允许写驱动的安全窗口；V3 不用速度容差、SHM、JSON 或私有阈值自行证明空闲。
- 目标驱动处于 profile 要求的安全状态；SV630N 03713 类写对象已证明需要 `enabled=false` 或 `estop=true` 加速度 0。
- UI/Broker 必须携带二次确认和一次性写门禁。

完成边界：

| 步骤 | 要求 |
| --- | --- |
| 扫描和匹配 | 进入脚本后实时扫描从站并匹配目标轴，不用旧扫描缓存 |
| 备份 | profile 支持时先 `drive.backup_parameters`；不支持时结果必须写明风险 |
| 恢复 | 只调用 `drive.restore_factory_defaults`，不得用 `drive.reset_fault`、`drive.software_reset` 或硬编码 SDO 冒充 |
| OP 恢复 | 写入返回后先恢复/轮询目标从站 OP，再做 SDO 读回 |
| fault 处理 | 恢复出厂后若有 fault/error，只能通过同一 profile 的 `drive.reset_fault` 有限重试 |
| 参数重建 | 重新确认 bit、内部 count、多圈周期、电子齿轮读回、模式和反馈 count |
| 证据失效 | 旧 `zero_model`、count checkpoint、电子齿轮 verified、rotary count trust 必须重新验证 |

复位驱动不得触发 LinuxCNC/backend 重启，也不得把设置页右上角语义改成“保存并重启”。它只写驱动私域读回/失效证据；后续重新写电子齿轮、设 0、保存并重启属于后续设置驱动或设置保存流程。

## 9. 高风险 position basis 命令

凡是 profile 命令声明 `mutates_position_basis=true`，执行器必须把它纳入 crash-consistent 事务：

1. 持久化 `position_basis_reset_pending=true`，同时将旧 checkpoint 标记 invalid。
2. 执行 profile 的 `write_sequence`、`verify_sequence` 和 `post_sequence`。
3. 读回验证成功后写入新的 `position_basis_generation`、checkpoint、低 32 位投影、profile/config hash 和 basis commit 记录。
4. fsync + atomic rename 后清 pending。
5. 任一步失败、断电或 pending 未清除时，开机必须禁止使用旧 checkpoint 投影恢复，并要求维护确认或重新设 0。

`drive.preset_position`、`drive.clear_multi_turn`、`drive.write_position_offset` 若缺 `requires_position_basis_transaction=true` 或 `checkpoint_policy=invalidate_then_rebuild`，必须视为不完整命令并 fail-closed。

业务层、Broker、UI 和后台 action/native owner 不得硬编码厂家 SDO index/subindex。新增品牌只能新增或修正 profile，不能新增厂家 if-else。

| `board/app/src/v5_settings_actions.*`、`v5_settings_action_ipc.*`、`v5_settings_page_actions.c`、`v5_settings_page_status.c` | UI C 代码只发送动作请求和展示结果；不得解析厂家对象、不得读写映射表 |

## 11. 关闭条件

实现或审查适配层时必须同时满足：

1. 板端只使用下载后的 public/private 缓存表；源码 `board/config/drive-profiles/` 和旧 `effective` 不作为产品 fallback。
2. 下载文件通过 SHA、schema、`map_version`、`release_contract`、命令完整性和 profile 唯一性校验后才原子替换。
3. private 远端删除会删除本地 private 缓存；后续动作不得继续使用旧 private。
4. 运行期选择可追溯 `selected_map_path/selected_map_sha256/map_source/profile_id/axis/slave_index`。
5. `设置驱动`、`读取驱动`、`故障复位`、`复位驱动`、`设 0` 只调用 `drive.*` 标准命令。
6. `复位驱动` 只走 `drive.restore_factory_defaults`；`清除故障` 只走 `drive.reset_fault`。
7. supported 写命令都有读回验证；缺时 fail-closed。
8. A/C 绝对多圈轴具备四个位置读取能力；split 合成必须防撕裂。
9. `drive.read_encoder_multiturn_limit` 只表达厂家已验证物理报警/安全上限；ESI 位宽和 raw modulus 不得冒充。
10. count-domain 旋转轴使用本机 `feedback_counts_per_motor_rev`、电子齿轮读回和 `SCALE`/native readback，不用旧倒数比例或 UI 默认值。且该比例与回读数据必须通过 UDS 接口直连微内核内存或 INI 回读，严禁从共享内存（SHM）或 display 投影中读取。
11. `mutates_position_basis=true` 的命令必须有 position basis transaction，否则 fail-closed。
12. 日志和结果包含 profile/source、axis/slave、标准命令、目标值、读回值、执行序列摘要、zero/position basis 结果和结构化失败码。
13. 比例链联动：凡是通过 `drive.set_egear` 修改了电子齿轮比，或通过 `drive.*` 改变了影响反馈分辨率的驱动参数，执行器在读回验证成功后，必须同事务触发 `REQ-SCALE-CHAIN-HISTORY-POLLUTION` 规定的重算流程：根据最新读回的值更新 INI 中的 `SCALE`，并使用 count 换算新的 raw 软限位（`MIN_LIMIT/MAX_LIMIT`）重写 INI，绝对禁止保留旧 V3 坐标或比例缓存。
14. 驱动读回数据收敛：所有通过 `drive.*` 标准命令读写的驱动私域参数与证据（如：SDO 读回分子分母、encoder 字节、多圈上限、A/C count-domain 设零锚点），在板端持久化时只允许存入符合 `REQ-SETTINGS-RUNTIME-DRIVE-ONLY` 规范的 drive-only `settings_runtime.json` 中，严禁将 LinuxCNC 运动字段、坐标系、TLO、RTCP 几何或运行模态作为快照混入其中。

## 12. 新增品牌和提升公共表

新增或修正驱动映射固定流程：

```text
厂家资料进入 VPS 私有区
  -> 解析为标准 schema 条目
  -> 先写当前 DNA private 表
  -> 板端服务器下载 public/private
  -> 板端真实从站唯一选择 profile
  -> 设置驱动/读取/复位/清故障/设 0 只调用 drive.* 命令
  -> 私有实测通过后人工提升到 VPS public
  -> 其他板端下次服务器下载获得支持
```

private 提升 public 前必须具备：

- 原始上传文件、解析工具版本、生成时间、测试设备 DNA 和 hash 留档。
- P0 命令完整，`drive.identify` 能唯一匹配真实驱动。
- `drive.set_egear` / `drive.read_egear` 写后读回一致。
- A/C 绝对多圈所需四个位置读取能力齐全，split/position64 审计通过。
- 缺对象、类型不匹配、读回不一致、OP 恢复失败和 fault 未清都能 fail-closed。
- 证据包包含板端日志、Broker 返回、驱动读回、映射表 hash 和回滚版本。

禁止：

- 为新增品牌修改板端业务代码里的 SDO index/subindex。
- 把新品牌表写进源码端或板端固定包作为产品 fallback。
- 下载后再转义成第二套 `objects`/散字段结构。
- private 未实测直接合入 public。
- 手工编辑板端下载缓存作为交付方式。
- public/private 合并成第三张产品真源。
- private 远端已删除时继续使用本地旧 private 缓存。
