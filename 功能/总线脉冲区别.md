## 1. 一句话区别

BUS / EtherCAT 模式的代码围绕驱动、从站、profile、SDO/PDO、电子齿轮读回、绝对 count-domain 和微内核/native 回读证明工作。它关心的是“当前驱动真实反馈 count 是多少、这个 count 怎样通过 runtime SCALE 进入 LinuxCNC/native 控制空间、驱动证据是否可信”。

Pulse / StepDir 模式的代码围绕 LinuxCNC/HAL 原生 homing 和 runtime INI 的 `HOME_*` 字段工作。设置页轴参数表里按当前轴 `设零` 时弹出键盘，输入坐标值作为当前轴 0 位，并写入当前轴 `HOME_OFFSET`；同一轴参数表里有独立 `回零方向` 列，可用下拉选择回零方向并写 `home_direction` owner。

所以两者不能互相复用：BUS 的 count-domain 零点证据不能生成 Pulse `HOME_OFFSET`；Pulse 的 `HOME_*` 参数也不能当作 BUS 驱动零点、电子齿轮或绝对多圈证据。

## 2. 模式真源

| 项目 | BUS / EtherCAT | Pulse / StepDir |
| --- | --- | --- |
| 模式识别 | `ethercat` / `bus` 归一为 BUS | `pulse` / `stepdir` / `step_dir` / `step/dir` 归一为 Pulse |
| 当前模式真源 | `motion_driver.conf` 只保存 requested/持久 owner；真实 active mode 必须由后端/mode-switch/微内核或 native 常驻内存链路在四项条件同时成立后直接回读：requested mode 等于 SelectLink 选中 INI、LinuxCNC fresh/running 且使用该 INI、期望侧 RT/HAL module 已加载、非期望侧 module/adapter inert 或 disabled | `motion_driver.conf` 只保存 requested/持久 owner；真实 active mode 必须由后端/mode-switch/微内核或 native 常驻内存链路在四项条件同时成立后直接回读：requested mode 等于 SelectLink 选中 INI、LinuxCNC fresh/running 且使用该 INI、期望侧 RT/HAL module 已加载、非期望侧 module/adapter inert 或 disabled |
| 不能作为真源 | UI 文案、dirty 值、旧 `run_mode_bus_pulse`、旧 JSON/cache | UI 文案、dirty 值、旧 `run_mode_bus_pulse`、旧 JSON/cache |
| 模式不明时 | fail-closed，不执行运动或写入 | fail-closed，不执行运动或写入 |

`settings_runtime.json` 只允许保存 Settings Drive 私域和深度清洗后的驱动/count-domain 证据。运动参数、HOME、软限位、WCS、RTCP/G53、tool、modal 和安全 actual 都不得被写回该文件当产品真源。

### 2.1 模式切换闭环

总线/脉冲模式切换必须走受控闭环：

```text
C/LVGL UI
  -> Command Broker UDS settings_apply 请求，携带目标 motion_driver_mode
  -> Product Action
  -> 物理写入 motion_driver.conf
  -> 源位置回读确认 == 目标模式
  -> 返回 RESTART_PENDING，前台锁屏/进度等待
  -> Product Action 或受控 helper 触发 mode-switch.service
  -> mode-switch.service 停止 LinuxCNC 并尝试卸载当前实时模块/HAL 设备
  -> 若卸载不干净或资源残留，fail-closed 到受控 systemd reboot 目标
  -> 若卸载干净，SelectLink 按 motion_driver.conf 选择启动链接
  -> ethercat: v3_ethercat.ini + lcec 实时模块
  -> pulse: v3_pulse.ini + owner 签核的 Pulse HAL/RT adapter
  -> 重新加载 LinuxCNC 和模式状态 provider
  -> mode-switch/微内核或 native 状态按四项并联条件验证真实模式
  -> native active_driver_mode 直接回读
  -> UI 只有读到 native active_driver_mode == 请求目标模式时才关闭锁屏/进度
```

切换状态机固定为：

```text
REQUESTED_WRITTEN
  -> SWITCH_PENDING
  -> REBOOT_REQUIRED | RELOAD_RUNNING
  -> ACTIVE_VERIFIED | DEGRADED
```

硬边界：

- UI 只发 Broker 请求，不得直接触发 systemd、shell、reboot 或切换脚本。
- Product Action 只能写 `motion_driver.conf` 这个模式 owner，并源位置回读确认；不得写 `settings_runtime.json`、旧 JSON、UI 默认值或临时文件作为模式真源。
- `mode-switch.service` 或现有受控 backend restart 等价入口是唯一受控系统切换入口；等价入口必须输出 `mode_switch_state` / `mode_switch_state_machine` / `mode_switch_owner` 字段，按 `REQUESTED_WRITTEN -> SWITCH_PENDING -> RELOAD_RUNNING|REBOOT_REQUIRED -> ACTIVE_VERIFIED|DEGRADED` 表达进度。不得在产品代码里调用 `os.system("sudo reboot")` 或等价自由 shell。
- 软重载只在 LinuxCNC、实时模块和 HAL 设备完全卸载干净时继续；卸载失败、资源占用、模块残留或状态不可证明时，必须进入受控整机重启，不得硬切。
- SelectLink 是唯一启动选择器，只能按 `motion_driver.conf` 选择真实存在的 `v3_ethercat.ini` 或 `v3_pulse.ini`；两个 INI、配套 HAL 和签核的 Pulse adapter 不存在时不得实现空选择器或文档式选择器。其它脚本、JSON、UI dirty 值和默认模式不得参与选择。
- 当前 Pulse 冷态预配置来源固定为 `new-vivado/` 的 8 轴 `step_ip` ABI：`step_ip/s_axi` base/range、`ID=0x53544550`、`VERSION=0x00030500`、`N_AXES=8`、per-axis stride 和寄存器语义只从 `new-vivado/z20_v1_5_hw_project/board_inputs/v3_hardware_abi.h` 与 `new-vivado/z20_v1_5_hw_project/ip_repo/axi_stepdir_enc_v2/step_ip_register_map.md` 消费。`D:\Z20_ZYNQ` 只能作为历史对照，不再作为当前 v3 Pulse SelectLink 目标或 adapter 来源。当前 Pulse 只能作为冷态预配置资产存在，`step_ip_v1_5.contract.json` 必须保持 `cold_staged_not_runtime_selectable`，软件运行层不得启动 Pulse；只有 bitstream、device-tree/UIO、生产 HAL/RT adapter、HAL/INI、FPGA `ID/VERSION/N_AXES=8` 只读探针和板端波形/安全证明全部成立后，才能按 owner 更新 contract 为 `runtime_selectable` 并进入 SelectLink 可启动路径。
- mode-switch/状态 provider 不得只相信 `motion_driver.conf`，也不得在 steady-state 通过硬盘文件兜底 active mode。发布/回读 `active_driver_mode` 前必须同时证明 requested mode 等于 SelectLink 选中 INI、LinuxCNC fresh/running 且使用该 INI、期望侧 RT/HAL module 已加载、非期望侧 module/adapter inert 或 disabled；任一条件缺失、过期或矛盾时返回 degraded/unknown，UI 继续锁定。
- UI 解锁条件只能是 native active_driver_mode == requested_mode；`RESTART_PENDING`、服务已启动、reboot 已触发、SHM 字段或命令已发送都不能代表切换完成。

### 2.2 关闭口径

模式切换 software gate 关闭只证明 UI pending/锁定、State Publisher active mode、三层 BUS 动作拒绝和 BUS 回归不退化已经闭合。它不能证明 Pulse/StepDir 硬件可运行。

要声明 Pulse/StepDir 可运行，必须另有板端硬件闭环证据：Step/Dir 波形方向、频率、脉宽/间隔正确；enable 在启动、重启、崩溃、服务停止和切换窗口保持 disabled 或无有效脉冲；急停、限位、home 输入有效；watchdog、underrun、overrun 或通信中断时 fail-closed；reboot/crash/no-service 无误脉冲、无错误方向、无自动 enable；Pulse `回零` 按 `回零次序` 和 `回零方向` 建立 homed/all_homed 并经 LinuxCNC/HAL/native 回读。缺这些证据时只能称为 `software_gate_only` 或 `local_verified_only`。

## 3. 代码链路差异

| 层级 | BUS / EtherCAT | Pulse / StepDir |
| --- | --- | --- |
| UI 设置页轴参数表设0 | `execute_settings_axis_zero_gate_command()`：先确认，再提交当前行单轴 `settings_axis_zero`，不输入数值 | `execute_settings_axis_zero_pulse_home_offset_command()`：打开软键盘，输入坐标值作为当前行当前轴 0 位；`回零方向` 列用下拉选择 |
| Broker | 同一个 `settings_axis_zero` action，但校验 `target_scope=bus_count_domain_zero`、`apply_mode=count_domain_zero` | 同一个 `settings_axis_zero` action，但校验 `target_scope=pulse_mechanical_home_offset`、`apply_mode=persist_home_offset` |
| product action | `actions_axis_zero_bus_runtime.py` 读驱动 actual/split absolute，生成 count-domain 零点证据 | `actions_axis_zero_pulse_runtime.py` 构造 `HOME_*` apply 请求 |
| settings apply | BUS 不生成 Pulse HOME 参数，不恢复旧浮点零位字段 | `settings_apply_pulse_home_offset.py` 和 INI generation 写当前轴 `HOME_OFFSET` |
| 驱动命令 | 通过 profile 的 `drive.*` 标准命令映射到实际 EtherCAT SDO/PDO | 不走 EtherCAT SDO，不读 BUS zero_model，不写驱动 profile |
| 状态回读 | native/microkernel provider 回读 active mode、驱动/count-domain、MCS、degraded reason；SHM 只显示坐标/模态/速度倍率投影 | native/microkernel provider 回读 active mode、runtime INI/native HOME 和 homed/all_homed；SHM 只显示坐标/模态/速度倍率投影 |

### 3.1 UI 共用边界

BUS / EtherCAT 和 Pulse / StepDir 共享同一套设置页、按钮框架、弹窗、顶部状态和命令发送；不得为了 Pulse 另建一套设置页、第二套 UI 树、第二套命令 shell 或第二套状态显示逻辑。UI 可用 SHM 显示坐标/模态/速度倍率，但 active mode、homed 和安全状态必须直连 native/microkernel 状态 provider。

Pulse 模式在 UI 上只允许做必要的局部差异：

- 轴参数卡：`设零` 弹键盘输入当前轴 0 位，写当前轴 `HOME_OFFSET`。
- 轴参数卡：显示独立 `回零方向` 列，下拉选择方向并写 `home_direction` owner。
- 设置驱动动作行：6 个 BUS 驱动动作灰色且不可按；这只是同一套 UI 的 mode-aware 禁用，不是 Pulse 专用 UI。

其它设置页布局、导航、保存流程、提示弹窗和状态显示继续复用 BUS UI。UI 只能按 native `active_driver_mode` 调整这些局部控件状态，不得用 SHM 字段、UI 文案、dirty 值或旧 JSON 自己推断模式。

## 4. 设置页设0差异

| 项目 | BUS / EtherCAT | Pulse / StepDir |
| --- | --- | --- |
| 用户看到的语义 | 轴参数表单轴 `设0` / 当前驱动位置作为逻辑零点证据 | 轴参数表单轴 `设零` / 当前轴 0 位 |
| 用户输入 | 不输入数值 | `设零` 弹出键盘输入当前轴 0 位；`回零方向` 列下拉选择方向 |
| 后端动作 | 读取驱动 count-domain 位置，保存 `zero_counts`、A/C 绝对多圈或 split count、profile/hash/readback 证据 | 写 runtime INI/native owner 的当前轴 `HOME_OFFSET`；`回零方向` 列写 `home_direction` owner |
| 当前坐标是否立即清零 | 不能只靠 UI 减法声称已生效；按钮保存通过只证明零点证据/owner 写入，live 坐标归零需后续 native/UI 显示闭环 | 不能只靠 UI 立即改坐标；必须经 owner 写入和后续 LinuxCNC/HAL native 回零证明 |
| 禁止行为 | 普通设0不得写 `0x607C`、`0x1010`、drive preset、drive rebase 或厂家位置偏置 | 不发送 EtherCAT SDO，不读取 BUS `zero_model`，不写 `motion_driver.conf` |

BUS 轴参数表单轴设0保存的是驱动/count-domain 证据，不是机器回零动作。Pulse 轴参数表单轴设零保存的是当前轴 0 位坐标值对应的 `HOME_OFFSET`，不是驱动零点证据。

## 5. 主页面回零差异

| 项目 | BUS / EtherCAT | Pulse / StepDir |
| --- | --- | --- |
| 主页面 `回零` 目标 | 回到设置页 BUS `设0` 的 count-domain 零点证据对应位置 | 执行 LinuxCNC/HAL 原生 Home |
| 运动依据 | native count-domain 零点证据 + active runtime INI/native `SCALE` + native fresh MCS/readback | 轴参数表 `回零次序` 决定先后轴，`回零方向` 决定找限位方向，触碰限位/home 输入后按 `HOME_OFFSET` 建立坐标 |
| 成功边界 | 驱动/adapter 到位证明 + LinuxCNC/native homed 标记 + homed/all_homed native 回读 | LinuxCNC/HAL `homed/all_homed` 经 native/API 回读 |
| 速度/加速度 | 只从 runtime INI/native `max_velocity_units_per_s` 和 `max_acceleration_units_per_s2` 取，不用 SHM、UI dirty、旧 JSON 或固定低速 | Pulse `HOME_SEARCH_VEL/HOME_LATCH_VEL` 与设置页最大速度同幅值，并由 runtime INI/native owner 生效 |
| 不能跳过 | 不能因为当前位置接近 0、已 homed 或目标位移为 0 跳过标准 Home 链路 | 不能用 BUS 零点证据、当前位置接近 0 或 UI 显示值代替触碰限位/home 输入的 LinuxCNC/HAL homing |

`回零`、`归零` 和设置页 `设0` 是三件事：`回零` 改 homed 状态，`归零` 写 WCS offset，设置页 `设0` 只写对应模式的零点或 HOME 参数。

## 6. 设置驱动只属于 BUS 驱动域

`设置驱动`、`扫描从站`、`读取驱动`、`复位驱动` 和 `清除故障` 都是 EtherCAT/驱动 profile 域动作。代码通过下载后的 public/private `driver_profile_map.json` 选择唯一 profile，再调用 `drive.*` 标准命令；板端业务代码不得硬编码厂家 SDO index/subindex。

设置页驱动动作行里的 `服务器下载`、`扫描从站`、`复位驱动`、`读取驱动`、`清除故障`、`设置驱动` 只属于 BUS / EtherCAT 模式。切到 Pulse / StepDir 模式时，这些按钮功能上不存在，必须三层拒绝：Broker/product action 层拒绝，C dispatch 层拒绝，UI 视觉灰色且不可点击。缺任一层都不算完成；`下载授权`、`登记本机码` 不属于这 6 个 BUS 驱动动作，不能被一起禁用。

BUS 代码在这里会处理：

- 真实从站扫描、OP 状态、profile 唯一匹配。
- `drive.set_egear` / `drive.read_egear` 写入和读回电子齿轮。
- `drive.read_actual_position`、`drive.read_abs_multi_turn`、`drive.read_abs_single_turn`、`drive.read_abs_position64` 等 count-domain 位置证据。
- `drive.restore_factory_defaults` 复位驱动，并使旧电子齿轮、设0、rotary checkpoint 和 count-domain trust 证据失效。

Pulse 模式不从这些驱动动作里获得 HOME 真源。Pulse 的当前轴 0 位和 HOME 参数属于 LinuxCNC runtime INI/native owner；驱动 profile、SDO readback、电子齿轮证据不能替代 Pulse homing 配置。

## 7. 旋转轴差异

A/C 旋转轴在 BUS 场景下强依赖整数 count-domain：`Crev`、电子齿轮读回、减速比、runtime `SCALE`、绝对多圈 direct64/split/checkpoint 证明和 native readback 必须一致。等效角显示、等效回零、unwrap、checkpoint 恢复和 Rebase gate 都不能只靠浮点角度、SHM 显示投影或 UI 显示值。

Pulse 场景下 A/C 仍显示等效角，但 Home 本身走 Pulse 的 LinuxCNC/HAL homing 链；不能把 Pulse `HOME_OFFSET` 当作 BUS 绝对多圈零点，也不能把 BUS `zero_counts` 当作 Pulse 回零参数。

普通等效回零是物理运动到最近等效零，不是 Work Zero、G92、drive preset 或 Rebase。任何 position-basis mutation 都必须走独立 native gate 和同周期证明。

## 8. 运行热路径差异

BUS 运行前需要把驱动 profile、电子齿轮读回、count-domain 零点、runtime SCALE、授权/DNA latch、RTCP/G53 几何等慢证据前置并锁存。G-code 正在运行时，不允许再读 SDO、重算 profile、重建 settings projection 或写周期性诊断 JSON 来补证据。

Pulse 运行前需要把当前轴 0 位 `HOME_OFFSET`、`home_direction`、runtime INI/native HOME、软限位、SCALE、速度/加速度和 homed 状态等 native 投影准备好。G-code 正在运行时，同样只消费 microkernel/native actual；SHM 仅供 UI 坐标/模态/速度倍率显示，不从 UI dirty、旧文件或 `/run` 结果补参数。

两种模式共享的热路径原则是：Start/Jog/Home/运动动作只读取 fresh microkernel/native actual 和已前置信标，不能在按钮热路径里临时读盘、fork helper、跑 `halcmd`、重建大 JSON、访问旧结果文件或把 SHM 显示投影当控制证明。

## 9. 不是总线/脉冲差异的内容

下列文档定义的是两种模式共同遵守的边界，不应被理解成两套业务实现：

- WCS / Work Zero：真源是 LinuxCNC `PARAMETER_FILE` 和解释器状态；不能替代机器回零、BUS 设0、Pulse HOME、Rebase 或 RTCP 几何。
- 跳窗提示：只定义结果码、中文原因、处理方式和确认继续规则；不能因为 BUS/Pulse 分支不同就新增第二套弹窗协议。
- UI 刀路：只显示同一帧 SHM/内存投影，不计算真实 RTCP 运动学，不写驱动、HAL、INI 或 G-code。
- VPS/授权：只提供驱动 profile、授权/DNA、远程输入和下载链路；不改变 BUS/Pulse 的运动真源。
- 触摸校准：只影响输入坐标链路，不影响 EtherCAT、8ax 后端、驱动参数或运动控制。

## 10. 不能混用的红线

- 不得用 BUS `zero_model`、`zero_counts` 或驱动 actual 生成 Pulse `HOME_OFFSET`。
- 不得把 Pulse 当前轴 0 位、`home_offset`、`pulse_home_model`、`HOME_SEQUENCE` 写入 `settings_runtime.json`。
- 不得用 UI 模式文字、dirty 值、旧 `run_mode_bus_pulse` 或旧 JSON 判断当前 active mode。
- 不得把设置页 BUS `设0` 当作主页面 `回零`。
