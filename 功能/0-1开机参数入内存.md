# 开机参数入内存与微内核参数真源要求

最高口径：LinuxCNC/微内核/native 到 UI 的轻量数据，以及 LinuxCNC/微内核自身运行所需闭包，必须在开机阶段一次性进入常驻 RAM。运行期 UI、Broker、动作脚本和显示 provider 不得反复读取硬盘、JSON、INI、var、tool 文件、脚本快照或 result 文件来补当前状态。

这里的“全读进内存”指 LinuxCNC/微内核运行闭包、UI 运行闭包和所有产品自写运行代码闭包，不是把 SHM 扩展成通用状态池，也不是把 UI 变成控制真源。

```text
硬盘/闪存持久 owner
-> 开机一次性读取 LinuxCNC/微内核运行闭包、UI 运行闭包、产品自写运行闭包和参数 owner 映射到 RAM
-> LinuxCNC/微内核/native 常驻内存与原生 readback
-> State Publisher 30Hz 写 UI 显示 SHM 投影
-> UI 只读显示
```

硬盘/闪存只允许作为开机装载、设置保存落盘、重启/重载前的持久 owner；运行期控制、运动准入、状态 readback、UI 显示 provider 和动作证明必须来自 LinuxCNC/微内核/native 常驻内存或原生 API。

开机常驻内存闭包必须至少包含四类：

1. LinuxCNC/微内核运行闭包：executable、共享库、RTAPI/实时模块、HAL component、kinematics component、INI/HAL 配置、`PARAMETER_FILE`、`tool.tbl`、G53/RTCP 几何、WCS/G92 初始 readback、驱动映射和已登记 native gate 表。
2. 参数 owner 映射闭包：参数表字段、唯一 owner、唯一 writer、源位置回读字段、generation/readback token、drive-only 例外字段和删除旧路径条件。
3. UI 显示投影闭包：只允许坐标/轨迹、主轴转速、进给速度、主轴倍率、进给倍率进入 `/dev/shm/v3_status_shm`；当前模态文字直连微内核/native modal/tool 状态块或等价原生 API；历史 `runtime_modal_text` 和 `V5_STATUS_VALID_MODAL` 必须从 SHM ABI 删除，不得作为 padding、诊断文本、兼容字段或 fallback 保留。
4. 产品自写运行闭包：C UI、Command Broker、非控制/诊断/设置域 action、provider、native gate wrapper、linuxcncrsh 常驻客户端状态、单 FIFO 命令队列、连接健康状态、运行期需要的资源索引、Python 字节码和动作登记表必须在启动阶段加载到 RAM。Home、Work Zero、Jog、Start、RTCP/WCS 等控制动作不得经 Python product action 中转，按 `功能/0-3native_helper白名单与验收说明.md` 直达 native gate。运行期 UDS 调用不得临时从硬盘发现脚本、动态导入 action、扫描目录补入口、读取 UI 资源文件补状态、按命令重建 linuxcncrsh 短连接或靠 fallback wrapper 恢复旧路径。

设置页、设置驱动、从站扫描和参数编辑属于受控设置阶段，不属于运动控制、状态显示或 Start/Resume 热路径。受控设置阶段允许按字段唯一 owner 读写硬盘/闪存持久文件，例如字段保存写回、扫描结果落地、候选列表刷新、drive-only 证据保存和受控重启/重载准备；写入后必须刷新或重载对应 RAM/native/provider 快照，并以同 owner 源位置回读或受控重载后的 native/provider readback 作为结果依据。凡修改 LinuxCNC runtime INI、`PARAMETER_FILE`、`tool.tbl`、自建/驱动参数表、drive profile map 或等价参数文件，必须先从 Windows source truth 单独上传这些参数文件到板端对应持久 owner，并用板端 hash/源位置回读确认一致后，才允许继续整包部署、受控重启、UI/operator 验证或运动闭环；不得把参数文件夹带在后续整包部署里当作已完成参数同步。

运行期允许触发硬盘/闪存的场景只有：受控设置阶段读写唯一持久 owner、程序文件选择/打开并生成运行内存副本、事件触发且限频的日志/诊断写入、受控重启或重载。日志/诊断不得进入 Start/Resume、State Publisher steady-state、UI 稳定显示或正常空闲循环的周期落盘路径。除此之外，UI 稳定显示、控制动作、运动准入、保存成功、参数稳定显示和安全判断都不得读硬盘兜底；Start/Resume 等热路径不得再读程序文件、脚本、配置或资源。

G-code 打开和投喂边界：用户在 Program Open/Load 打开的原始 G-code 文件是什么内容，Start/Resume 就必须把同一个原始文件身份投喂给 LinuxCNC/微内核；`Set Open` 路径、resident `source_path` 和 `source_sha256` 必须对应操作员打开的原始文件，不得改成过滤后的临时文件、sidecar、运行副本、等效角改写文件或 UI 预览解析产物。Program Open/Load 允许建立只读 resident runtime identity、hash、line count、预览点和 first-point metadata，但这些 metadata 只能服务显示、身份校验和 native gate 请求，不能改变投喂给微内核的 G-code 文本、行顺序、A/C word、M-code 或模态。

G-code A/C 一圈内等效角分类属于 `功能/3-3旋转轴等效角目标与解决方案.md` 的微内核/native rotary owner。Program Open/Load 可以在 resident runtime metadata 中记录 native parser 需要的只读身份和上下文；Start/Resume 热路径不得由 UI、Broker、Python action、刀路显示层或运行副本重写器读取/重写源 G-code 来实现 `A0`/`C0` 等效零。`G90/G53 A0 C0` 这类已由 native 判定为非连续累计段的一圈内相位/回零目标，必须由 native/motmod/microkernel 走最近等效角或最近 `k*360` 等效零；连续加工段、多圈累计目标和语义不明目标仍交由 native `continuous_delta` / `ambiguous_block` 判定。

`保存并重启` 是开发板断电后重新上电等价的干净重启需求，不是普通页面返回、局部服务 restart、provider reload 或只重启 UI/Broker。执行时必须在参数已落入唯一持久 owner 并回读一致后，进入 canonical clean restart handoff：停止 UI、Broker、Action Dispatcher、State Publisher、LinuxCNC/微内核运行服务、HAL/EtherCAT/drive provider 等运行闭包，清理或重建 `/run` 运行态、UDS socket、SHM 显示投影、native 状态块、resident snapshot、命令队列和进程内 cache，然后按开机链路从唯一持久 owner 重新装载到 RAM，重新初始化 LinuxCNC/HAL/EtherCAT/SDO/PDO/轴绑定/比例链/raw limit，并用 native/drive owner readback 证明一致。若软件 clean restart 不能证明达到断电重启同等干净度，必须走板端系统级 reboot 或开发板断电重启等价路径；不得把“已发起若干服务重启”当成完成。

## P0 CPU 热路径要求：慢投影只能缓存后发布

CPU 调度最高要求：微内核/native 运动精度链路默认独占 CPU0；当前 LinuxCNC/RTAPI realtime/servo 线程必须由 canonical LinuxCNC 启动脚本固定到 CPU0，并以 `/proc/<pid>/task/<tid>/status` 的 `Cpus_allowed_list=0` 作为启动与板端验收读回证据。UI、Broker、State Publisher、provider/action 脚本、诊断、部署/构建辅助和日志等非微内核任务不得主动绑定、抢占或逃逸到 CPU0。若未来 CPU0 在真实运动负载下接近满负荷，需要让微内核/native realtime/servo owner 扩展使用 CPU1，必须先更新本文 owner 并提供连续运动和实时确定性证据；当前产品路径不得让 LinuxCNC 实时线程漂移到 CPU1，也不得把 CPU1 拆成第二套状态 owner、第二套运动判断或割裂的运行链路。UI 和其它普通进程必须继续让道给微内核；运行平稳、运动连续和实时确定性优先于 UI 刷新、吞吐、诊断、自动化和便利性。

CPU 诊断和后续优化必须使用同一时间窗口内的 `/proc/stat`、线程 CPU ticks、`/proc/<pid>/task/<tid>/status` affinity readback 和对应服务 diagnostics 增量；不得用单次 `ps %CPU`、`top` 瞬时排序、SSH/dropbear、verify、one-shot publisher、`halcmd`、测试 sampler 或重启过程中的临时负载判断常驻基线。任何 CPU 优化不得以停止 LinuxCNC servo、降低 native safety 实时性、关闭 EtherCAT 周期、断开远程画面或删除 readback 证据为代价。

### P0.1 执行口径

- 参数表里的全部参数都必须走同一条闭环，不按字段另开显示/保存捷径。开机链路是：硬盘参数或原生 owner -> 对应 owner/backend apply 读入微内核 native 运行内存 -> UI/provider 直连显示。参数表不得再通过 SHM 稳定显示。
- UI 参数表稳定显示只能来自 fresh 微内核 native 运行内存或登记只读 API。SHM 投影、Broker/read-only 结果、`/run` JSON、`config_cache`、诊断快照、UI 本地缓存及 dirty 值都不能作为稳定显示兜底；它们只能作为一次请求输入、动作证据或离线诊断。
- 微内核 native 常驻内存是运行态事实和运动安全判断的 owner；除编码器反馈机械坐标和运动学理论坐标外，不得把这些事实投影为 typed SHM 或诊断状态块用于稳定显示。普通产品路径不得通过 SHM、JSON 或上层缓存访问替代下层内存事实，也不得重建自造速度门禁。唯一 owner 的读取必须是启动加载、保存事务、重启/重载、generation/签名变化触发或显式诊断，不能成为每帧同步机制。
- UI 更改链路是：UI dirty/request（保存请求构建，提交瞬间从当前微内核/native 状态 provider 取最新 generation/status，坐标 SHM epoch 不能作为保存准入证明）-> Broker UDS -> 已登记 settings/native apply owner 或受控 `settings_apply` -> 写入唯一硬盘/native 参数 owner -> 源位置回读确认 -> UI 可清该字段 dirty、显示“已保存/待重启”并进入下一步。需要 backend reload 的字段保留 restart pending；右上角 `保存并重启` 的边界是开发板断电重启等价 clean restart handoff，必须重新启动并重建 UI/Broker/Action Dispatcher/State Publisher/LinuxCNC/微内核/HAL/EtherCAT/drive provider 运行闭包，清掉旧 `/run`/socket/SHM/native 状态块/resident snapshot/cache/队列后按开机链路全量入 RAM。重启后 machine-on、驱动 Operation Enabled、native/runtime generation、从站/bit/比例链/raw limit 和必要 drive readback 只作为 fresh 运行态诊断与证明；旧运行态中的 SHM、result JSON 或 resident cache 不能作为完成条件。
- UI 首个可见帧之前，启动脚本或等价 supervisor 必须已经完成参数 owner 闭包装载与 fresh 坐标显示投影就位：self/drive 参数表、drive profile resident snapshot、State Publisher `/dev/shm/v3_status_shm` fresh dynamic 帧必须可读并通过完整性校验后，才允许清屏和启动/显示 UI。首屏不能先显示空坐标、`--` 或 degraded/unavailable 再等待下一帧补齐；若超时，应阻止 UI 出屏并记录启动阻塞，而不是让用户看到不完整坐标界面。
- 字段已经写入唯一 owner 并从同一 owner 回读一致后，UI 文案只能表达“已保存/待重启生效”或等价 restart pending；不得继续显示成长期“修改未保存”。软件盘尚未确认的输入态只属于当前控件编辑态，不是参数真源，也不得扩展成 Start/Resume 前置 gate。
- 设置页数字软件盘 `OK` 是字段级保存确认：按下后必须关闭软件盘，写入该字段对应的唯一硬盘/持久 owner，从同一 owner 源位置回读，并只刷新被修改的 UI 单元格为回读值；对需要重启/重载才进入 native 运行态的字段，该单元格显示的是已落盘/待重启值，运行态最终生效仍由右上角 `保存并重启` 后的 native/runtime readback 证明，不能因为运行态尚未重载而让 `OK` 停留在软件盘或不刷新该格。
- “写到参数表”只能写到该字段的唯一 owner：微内核/native 字段如 runtime INI、`PARAMETER_FILE`、`tool.tbl`、HAL/EtherCAT/native owner/gate 或登记 owner；`设置驱动` 私域字段才允许写 `settings_runtime.json` drive-only/驱动证据。任何字段都不能因为 UI 需要显示而新建 V3 第二套落盘参数表。
- 全量编译/部署不得覆盖板端已有设置，也不得恢复 `microkernel_parameter_table.tsv` 或任何第二套微内核参数表。`self_parameter_table.tsv` 是唯一使用“本地 schema/template + 板端同 key value 合并”的自建参数表：输出的 `axis`、`field`、行顺序和目标文件位置只能来自本地表；仅当板端行是合法三列 TSV，且 `(axis, field)` 能对应本地表同一个 key 时，才把板端 value 写入本地模板；板端缺失、格式错误、重复污染、未知 key 或本地已删除字段一律不带入，新版本本地新增 key 使用本地默认值。`drive_parameter_table.tsv`、LinuxCNC runtime INI（当前 `linuxcnc/ini/v5_bus.ini`）、LinuxCNC `PARAMETER_FILE`（当前 `linuxcnc/runtime/var/linuxcnc.var`）和 `tool.tbl` 这类非自建表参数 owner，在 Windows 到 VM 同步、推送或部署打包前必须先从板端真实 owner 复制回本地 source truth 覆盖同路径，再用覆盖后的本地文件同步、打包和推送；板端缺失时才保留本地 seed 并明确提示。任何本地参数 owner 文件被合并或覆盖前，都必须先在项目顶层 `bak/` 目录生成当前本地文件备份，Windows 本地为 `D:\v5\bak`；写完后必须重新读回并验证格式/内容未被破坏，失败必须恢复备份并失败退出。合并输出必须按本地表顺序重写为每个 key 一行，防止设置文件随保存/部署逐步追加膨胀；宁可弃用无法对应的数据，也不能写错 key、写错位置或破坏本地表格式。
- 文件写入、UI 输入值、`config_cache`、Broker/result JSON、`/run` 快照、诊断 JSON、SHM 投影或默认模型值都不是保存成功。字段级“已保存/待重启”必须有唯一 owner 写入/持久化结果和源位置回读；需要跨重启恢复的字段还必须保留 restart pending，运行态最终生效和刀路显示仍必须以后续微内核/native 内存或同 owner 运行态直接回读为准。
- 运行期 actual 与持久参数分域：G-code/MDI/按钮造成的解释器模态、G92 actual、RTCP/switchkins ON/OFF、倍率、HAL/IO actual、运动状态等只改当前微内核 native 运行内存/actual，并由 UI/provider 直连显示，不反写硬盘默认值、runtime INI 默认、`settings_runtime.json`、SHM 或 V3 私有参数文件。
- UI 机械坐标、UI 轨迹动态当前位置、绿色机械点、实际刀尖/刀柄反馈点和所有以 MCS 为基础的显示值，源头必须是编码器反馈或 LinuxCNC/HAL/EtherCAT/native owner 提供的关节实际反馈；该反馈先进入后端运行内存，再由 State Publisher 写入同一个 typed SHM/受控内存投影，UI 只显示这一帧内存值。UI 不得从参数表、`settings_runtime.json`、`/run` JSON、Broker result、旧 SHM 快照、模型默认值、命令坐标 `cmd_mcs` 或本地缓存推算，兜底机械坐标。
- SHM freshness 与 `status_epoch` 异常不得静默重置：`age_ms < 0`、状态时间戳向前跳、`status_epoch` 回退、monotonic 基准异常或系统时钟突变，必须标记 degraded/time anomaly；该异常只影响 SHM 显示投影 freshness、UI 降级和诊断。控制请求、保存请求、运动准入和动作 proof 不得用 `status_epoch` 放行或拒绝，必须读微内核/native owner generation/readback token 或登记 gate。UI 显示层最多可用本地 monotonic 时间做短暂视图 grace，但不得 reset epoch、改写系统时钟、伪造 fresh、复用异常 epoch 放行，或把时间轴自动归零当成链路恢复。
### P0.2 核心口径
- 除 `设置驱动` 的驱动私有域外，能由 LinuxCNC runtime INI、`PARAMETER_FILE`、`tool.tbl`、HAL/EtherCAT actual 或已登记 native owner/gate 承载的参数，必须进入对应 native owner。

## 0. 最终口径：进微内核与废弃文档

### 0.2 唯一长期例外：设置驱动域

## 1. 开机进入内存的参数

### 设置页轴参数

旧入口删除对象：`/opt/8ax/phase0_bus5/settings_runtime.json` 中的微内核字段。目标上它只能保存 `设置驱动` drive-only 私域和驱动证据，不得继续作为微内核字段的落盘参数表或配置生成输入。
目标真源：轴/关节映射、比例、速度、加速度、限位、回零、RTCP/G53 几何等字段最终进入 runtime INI、LinuxCNC/native owner 或已登记 native owner/gate；`settings_runtime.json` 只保留设置驱动私域和驱动证据。
历史不合规过渡路径（不得恢复）：
- `lvgl_app/scripts/re-v3-lvgl-ui.init` 在 UI 拉起前调用 `v3_settings_read_only.py --inject-shm /dev/shm/v3_status_shm`。- `v3_settings_read_only.py` 只读一下 `settings_runtime.json`，生成 SHM JSON payload 里的 `config_cache`。- `v3_state_publisher.py` 每帧通过 `_settings_runtime_config_cache()` 把同套 `config_cache` 发布到 `/dev/shm/v3_status_shm`，UI/运动/安全再从该字段 fallback。
上述路径不满足本文最高规则：它绕过唯一 owner -> 微内核 native 运行内存/API 直接回读的闭环，且会导致 `config_cache` 和 `settings_projection` 成为 UI/运动/安全 fallback 的第二真源。产品路径必须保持删除启动启用 `--inject-shm` 链路；State Publisher 不发布通用 `config_cache`、`settings_projection` 或任何白名单的 JSON payload；参数表 UI 需要显示时必须直连微内核 native 运行内存、runtime INI readback 或登记 API provider。`settings_read_only`、`settings_read_result.json` 和历史 `config_cache` 必须从产品路径删除，不能作为兼容证据。
历史 `settings_projection` 不再作为过渡 SHM payload。若仍有调用面，必须按待删迁移债务处理：先接入微内核 native 直连 provider，再删除对应 SHM 字段、测试口径和文案；不得恢复开机一次性注入、不得让 UI 直接读硬盘，也不得在 provider 缺失时用默认值或 `/run` result 文件兜底。
历史 `config_cache.axes[]` 曾包含的主要每轴设置页参数：
- 轴标识 and 映射：`axis`、`axis_mode`、`slave_index`。- 精度 and 比例：目标精度只允许作为一次请求或 Settings Drive 电子齿轮私域输入；实际运动比例以 active runtime INI/native `SCALE` 和微内核/native 内存或原生 API 直接回读为准，不得进入 SHM。历史倒数比例字段不得恢复到 `settings_runtime.json` 持久字段或配置生成输入。- 编码器/驱动计数：`profile_id`、`drive_profile_id`、`encoder_bits`、`raw_encoder_bits`、`drive_internal_counts_per_motor_rev`、`encoder_multiturn_limit`、`feedback_counts_per_motor_rev`、`rotary_load_counts_per_rev`。- 运行限制 and 回零：`mechanical_max_velocity_rpm`、`max_velocity_units_per_s`、`max_acceleration_units_per_s2`、`backlash`、`min_limit`、`max_limit`、`home_offset`、`home_direction`、`home_sequence`；BUS 轴只保存 drive count-domain 证据，显示运行位置从 active runtime INI/native owner 直接回读推导，不得进入 SHM。
这些字段必须按语义拆分后进入各自的 owner，不得继续依赖 `config_cache` 聚合模型。- 运动配置字段迁入 runtime INI/native owner，并用微内核/native 内存或原生 API 直接回读和重启回读证明；不得进入 SHM。- 驱动 profile、编码器 raw/multiturn、电子齿轮、count-domain 校准和驱动读回证据留在设置驱动私域。- UI 显示必须由 owner 拆分为直连微内核/native 和 typed display provider；`settings_projection` 不得继续承载参数 SHM 投影，不能反向成为参数 owner；`config_cache` 不得重新定义一套参数模型，不得作为显示兜底、保存成功、board live、运行安全 fallback 或参数真源证据。
- 设置页显示的整数运动字段必须在直连 provider 返回 UI 前按字段精度归一化；例如速度/加速度回读从 runtime INI 秒单位换算产生 `10000.000000004999` 这类浮点尾差时，显示 provider 值必须收敛为 UI 等价的 `10000`，不得只在 UI 文本层遮蔽。
### 显示/刀路辅助参数
来源：当前 runtime INI 的 `[DISPLAY]` / `[RTCP]` 和微内核/native 内存或原生 API 直接回读；不得同步到 SHM。`settings_runtime.json` 中的旧显示几何字段不得作为 `settings_apply` 的持久配置生成输入、显示长度最终真源、待迁移来源或离线诊断兜底。
UI 状态 provider / 微内核直连读取：
- runtime INI `[DISPLAY]` 下的 `T0_TOOL_HOLDER_LENGTH_MM = 15`，作为 T0 显示刀柄长度直接回读并在启动时进入 native/tool 状态发布内存；若 `tool_display` / `toolpath_display` / `tool` 下的 `t0_tool_holder_length_mm` 必须从产品路径删除，不得作为输入、兜底、过渡来源或离线诊断真源。
- runtime INI 的 `[RTCP]`、kinematics 选择和 G53/A-C 几何，作为 `rtcp_ac_geometry` 及其有效标志直接回读。
- HAL pin/native component、常规 native daemon 或登记 native memory block 读回 RTCP/switchkins actual 状态；开机策略目标是 RTCP OFF，但 UI 状态 provider 从同套 native 内存/API fresh 回读前只能返回 unknown/degraded，不得返回有效 OFF，也不能把按钮默认文案、旧 SHM 帧、一次性 helper 输出或旧 `/run` 结果当作已关闭。
- 启动自动 enable/backend ready 后必须执行一次 RTCP OFF 归一化：先通过 native 状态源取 `motion.switchkins-type`，若实际为 ON，只能通过已登记固定范围 native owner/gate 将当前运行期 actual 设 OFF，并等待微内核 native 内存 fresh 直接回读确认；失败时记录启动归一化失败并保持 unknown/fail-closed。
- 运行期 RTCP 开/关只改变当前微内核 native 运行内存/actual，不写回 runtime INI、`settings_runtime.json`、V3 JSON/cache 或其他硬盘参数；`stat.kinematics_type` 只能作能力诊断，不能代表 actual active。State Publisher 不得保留 `RTCP_STATUS_HELPER_FALLBACK`、`RE_V3_RTCP_STATUS_HELPER_FALLBACK` 或等价环境变量重新打开短生命周期 helper 查询。
- LinuxCNC `PARAMETER_FILE` / `linuxcnc.var` 的 G54-G59.3 全表偏置，结合 `stat.g5x_index` / `stat.g5x_offset` 直接提供 `active_wcs`、`wcs_offsets` 和当前活动 offset。
- LinuxCNC 解释器模态和 G92 原生状态，直接提供当前模态文本、活动几何平面/进给/刀具路径模式、`g92_offset[5]`/valid mask 和显示 G92 策略状态；常规开机默认模态为 `G90`、`G17`、`G94`、`G40`、`G49`、`G54`、`G64`、`G80`，RTCP 策略目标为 OFF 但必须由 native 内存/API fresh 回读确认，G92 默认 inactive/zero 或已明确采用的 native G92 策略。运行期 MDI/程序/按钮改变这些模态时，只改变当前 interpreter/native 内存状态；State Publisher 不得把 `runtime_modal_text` 作为主 UI 真源写入 SHM；历史 `runtime_modal_text` 和 `V5_STATUS_VALID_MODAL` 必须从 SHM ABI 删除，不得作为 padding、诊断文本、兼容字段或 fallback 保留，也不得把 SHM modal 当动作证明、门禁或回写来源。
- 模态和 G92 状态不得写回 `RS274NGC_STARTUP_CODE`、runtime INI、`settings_runtime.json`、`linuxcnc_modal_defaults.var` 或任何 V3 默认模态文件。G92 未登记时只显示诊断并作为 Work Zero 等动作的 fail-closed 门禁，不作为 V3 坐标补丁。缺少 fresh native `g92_offset` readback 时，Work Zero 等坐标写入动作必须拒绝，不能把 0 偏移默认放行。
- LinuxCNC `stat.tool_offset`、`stat.tool_in_spindle`，必要时由 `tool.tbl` / native tool owner 的当前刀长、刀半径和 `[DISPLAY] T0_TOOL_HOLDER_LENGTH_MM` 显示刀柄长度兜底，直接提供 `tool_offset`、`active_tlo_z_mm`、`display_tool_length_mm`；当前刀号为 T0 时发布 `15.000mm`，不得发布默认 `0.000mm` 冒充刀长。
- LinuxCNC/HAL/EtherCAT 运行事实，例如 `estop`、enabled、task/interp state、homed/all_homed、限位和 IO 互锁状态，只作为当前运行事实直接提供，不从 V3 文件恢复。
除 `spindle_speed_rpm`、`linear_velocity_mm_per_min`、`feedrate_override`、`spindle_override` 等 UI 显示投影外，这些值不得由 `services/state_publisher/v5_state_publisher*` 写入 `/dev/shm/v3_status_shm`。历史 `runtime_modal_text` 和 `V5_STATUS_VALID_MODAL` 必须从 SHM ABI 删除，不得作为 padding、诊断文本、兼容字段、fallback 或主 UI 模态真源保留。UI 消费微内核 native 状态 provider 或登记 API，不走 UI 热路径；SHM 只保留 30Hz UI 显示投影。
### 驱动映射
来源：`/opt/8ax/drive-profiles/public/driver_profile_map.json` 和 `/opt/8ax/drive-profiles/private/driver_profile_map.json`。
UI 不直接读取驱动映射表；驱动映射属于开机常驻内存闭包，不是 UI 运行期热路径文件。State Publisher、Broker 或产品 action 在启动阶段必须加载并校验映射表内存快照；运行期驱动动作和旋转轴 count-domain 判断只消费该快照或刷新后的 native/drive provider。- `services/drive_profile/v5_drive_bus_action.py` 和对应常驻 drive provider 只能作为启动装载或受控重载入口读取 public/private 映射表，匹配 `settings_runtime` 中每轴的 `profile_id` / `drive_profile_id`。- State Publisher 的旋转轴计数域也按映射表签名与 `settings_runtime` 签名缓存结果。
映射表提供 SDO 命令对象、编码器分辨率、多圈限制、旋转负载单圈计数等驱动能力信息。`feedback_counts_per_motor_rev`、`rotary_load_counts_per_rev`、`zero_abs_counts64` 和 split raw basis 属于设置驱动/count-domain 例外快照；可以开机入内存供等效角、unwrap、checkpoint、Rebase 启动检查使用，但不能反向成为 LinuxCNC 运动参数、WCS、RTCP、刀补、运行模态或坐标基真源。倒数比例只能由 active runtime INI/native `SCALE` 在内存中即时反算用于诊断显示，不得作为 `settings_runtime.json` 持久字段或配置生成输入恢复。
## 2. 后续如何调用

### 设置页显示
设置页字段读取顺序是：
`正在编辑的 UI dirty 值（仅编辑请求态） -> fresh 微内核/native 常驻内存或登记只读 API readback -> 自建参数表中唯一 owner 的轴参数 RAM 快照/readback -> 驱动参数映射表或 drive provider readback -> unavailable/degraded`

对应代码：- `app/src/v5_settings_page.c`、`app/src/v5_settings_axis_table.c` 和 `v5_settings_axis_table_value()` 只能把 dirty 值作为当前编辑待提交输入；字段级“已保存/待重启”不等于运行态已生效，保存后的稳定显示必须在重启或运行内存刷新后回到 fresh native/provider readback。- `app/src/v5_settings_parameter_store.c` 只服务自建参数表和 drive-only 持久 owner；参数表字段不得从 `/dev/shm/v3_status_shm` 读取，除非该字段本身属于 SHM 显示白名单。历史 `config_cache.axes[]` 不得作为参数表稳定显示来源；旧入口必须从产品路径删除。- `/run/8ax_v5_product_ui/settings_action_events.jsonl` 和动作响应快照只允许作为一次动作证据或离线诊断；不得作为显示刷新辅助、首屏主路径、SHM 缺失 fallback 或硬盘兜底。
UI C 不允许直接读 `/opt/8ax/phase0_bus5/settings_runtime.json`、runtime INI、`linuxcnc.var`、`tool.tbl` 或其他硬盘参数文件。内存 readback 缺失时必须显示 unavailable/degraded 或 blocked，不能用静态默认值、旧参数表、旧 `/run` 结果或模型默认值冒充当前生效参数。
### 设置保存

设置页修改后：
- UI 组装 `settings_save_return` / `settings_apply` 请求时，必须在保存请求构建，提交瞬间从当前微内核/native UI 状态 provider 取最新 owner generation/readback token，不得使用进入设置页或开始编辑时缓存的旧 token；SHM `status_epoch` 只用于 UI 显示 freshness 和诊断，不参与保存准入。请求再通过 Command Broker UDS 发送。
- Broker 调用 `services/command_gate/v5_settings_apply.c` 对应的 registered settings/native apply owner。settings/native apply owner 可以在一次事务中使用完整内存请求 payload 来生成 runtime INI、校验源输入和执行 readback，但这个 payload 不能从旧 V3 落盘参数表恢复，也不能原样保存成新的 V3 参数表。
- 属于微内核 native 的字段必须写入其唯一硬盘/native owner，再刷新或重载运行内存并直接回读；属于 `设置驱动` 私域的字段可以写入 drive-only `settings_runtime.json`，但仍必须完成驱动 owner 写入/读回 UI provider 直连显示刷新，不得依赖 SHM。
- 写入 `settings_runtime.json` 时必须经过 drive-only 提取器，只保留设置驱动私域字段和深度清洗后的驱动证据；运动、坐标、刀具、模态、RTCP/G53 几何、显示默认、安全互锁和倍率字段不得落入该文件，也不得藏在 `zero_model`、`drive_position`、readback 或诊断子对象中。
- UI 本地值或 `immediate_persist` 只能表示编辑态请求态已更新；State Publisher 发布 generic `config_cache` 和 `/run` result 更新不能替代微内核 runtime fresh typed 回读证明。
- 需要 LinuxCNC 生效的字段必须走后端重启、受控 apply 或已登记 native owner/gate；字段级“已保存/待重启”以 native owner 持久化结果和源位置回读为准，右上角 `保存并重启` 必须执行开发板断电重启等价的 clean restart handoff，不得降级为只重启部分服务、只返回主页面或只刷新 provider；运行态最终生效以重启后微内核 native 运行内存直接回读和必要的 drive/native readback 为准，不得进入 SHM。
### 运行状态和刀路显示
LinuxCNC 运行状态分两条读取链路：
- UI 坐标值、UI 轨迹变化、转速、进给速度和主轴/进给倍率的统一显示链路固定为：微内核 native 对应 owner -> 后端运行内存 -> State Publisher -> `/dev/shm/v3_status_shm` typed SHM -> UI SHM reader -> `V5UiStatusView` -> 坐标面板、刀路动态点和主页面显示。缺 fresh 显示投影时只能显示 degraded/unavailable，不能用参数表、文件快照或本地旧值假装当前显示有效。
- 当前模态文字、WCS 偏置、刀具长度、RTCP/G53 几何、旋转轴计数域、急停/使能、安全、行号、诊断和系统指标必须由 `app/src/v5_ui_status_view.c`、`app/src/v5_ui_model.c`、`app/src/v5_main_page.c`、`app/src/v5_toolpath_display.c` 和主页面/辅助页面通过微内核 native 状态 provider 或登记 API 直连读取，不进入 SHM。进给速度、转速和主轴/进给倍率允许作为 SHM UI 显示投影，但不能作为动作证明或门禁。
- Broker、后台 action/native dispatcher 和 owner 响应结果不得读取 typed SHM 作为动作依据、准入、保存证明或安全证明；需要状态时必须读微内核/native 内存/API 或登记 native memory block。
### 驱动动作

设置驱动、驱动扫描、驱动读回、驱动复位等 设置驱动 域动作不走 UI 缓存直接推断驱动能力。 - 动作入口只消费启动阶段已加载的 drive-only 轴到从站/profile 绑定、驱动证据和驱动映射内存快照；`settings_runtime.json` 和 profile 映射表只能在启动装载、设置保存后刷新或受控重载时读取。 - EtherCAT 从站扫描结果进入驱动/native provider 内存，动作匹配只消费该内存视图。 - 结果写入 Broker/action 响应和状态文件只能作为动作证据；后续 UI 稳定显示来自驱动/native provider readback，SHM 只显示坐标/轨迹、转速、进给速度和倍率白名单字段；当前模态文字直连微内核/native modal/tool 状态。运行判断和成功证明必须来自微内核 native 内存或驱动 owner 直接回读。总线设零使用这些驱动读回作为证据，但零点/坐标基的长期 owner 应迁入微内核/已登记 native owner/gate 或受控 runtime apply 路径，不再扩展 V3 参数文件真源。
## 3. `settings_runtime.json` drive-only 收口要求

`/opt/8ax/phase0_bus5/settings_runtime.json` 必须收敛为 `schema=re.v3.settings_runtime.drive_only.v1`。该文件只允许持久化设置驱动私域和深度清洗后的最小驱动 count-domain 证据： - 轴到从站/profile 绑定。 - 电子齿轮输入和读回。 - encoder raw/multiturn 参数。 - drive SDO 写入/读回证据。 - A/C count-domain 校准。 - BUS 设零 `zero_model` 证据。
`zero_model`、`drive_position`、readback 或诊断子对象只能保存驱动对象、电子齿轮、encoder raw/multiturn、count-domain 零点锚点、驱动读回和必要验收时间状态码；不得保存 `pre_status`、`linuxcnc_snapshot_before`、`runtime_scale_check_before`、WCS/G92/TLO/tool/RTCP/modal/status 帧或 runtime INI 回读行。这些运行事实必须留在微内核/native 内存 owner，并由 UI provider 直连显示；离线诊断证据只作诊断；Broker `/run` 结果只能作动作证据，不能落进 `settings_runtime.json`，也不能作为参数表稳定显示或保存成功真源。
该文件禁止持久化： - runtime INI 运动约束：HOME/HOME_OFFSET/HOME_SEQUENCE、软限位、速度、加速度、FERROR、BACKLASH、`[TRAJ]`、`[AXIS_*]`、`[JOINT_*]`。 - `[DISPLAY]` 显示默认、tool/TLO/T0 显示默认。 - WCS、G92、RTCP 运行期开关、G53/kinematics 几何。 - HAL/IO 安全互锁、feed/rapid/spindle override truth。
`settings_apply` 可以继续在内存中使用一次完整请求 payload 来生成 runtime INI 并在同一事务里做源读回，但该 payload 不能从落入 V3 自造参数表恢复，也不能原样写回任何 V3 参数文件。写入 `settings_runtime.json` 的内容必须经过 drive-only 提取器裁剪。UI 参数稳定显示必须来自 native/provider readback；SHM 投影只来自 State Publisher 拆分后的 typed fresh display 白名单字段。真正成功证明来自 owner/native 直接回读。`config_cache` 必须从产品路径删除，不能合并 active runtime INI 和 drive-only 字段后继续充当 UI/SHM 投影或参数真源。
`settings_runtime.json` 不得再由 `machine_settings.json` 回填或恢复。drive-only 提取器必须删除运行时标注型字段（例如 `drive_axis_role`）和非必要派生显示字段；schema、role、truth-boundary 等说明只能作为结果诊断字段或字段卡分类，不得成为新的参数数据模型。
## 4. 设置页字段接口 owner 规则

设置页每个数据必须按“唯一 owner -> 微内核 native 运行内存/API -> UI 直连 provider”显示，UI 修改必须走 “Broker -> settings_apply/native owner -> owner 写入 -> 微内核 native 运行内存刷新 -> native/API fresh
直接回读 -> UI 可清该字段 dirty”闭环。下列字段不得再由 `settings_runtime.json`、`machine_settings.json`、`config_cache`、`/run` result 或 UI 默认值充当真源：
- 轴显示语义元数据：`axis_mode`、目标精度 precision、`mechanical_max_velocity_rpm/max_velocity`、`soft_minus/soft_plus`、`zero`
  可以写入 active runtime INI `[AXIS_*]`/`[JOINT_*]` 及等价 v5 设置元数据，用于 UI 从运行内存读回。`pitch`、`motor_rev`、`load_rev` 属于已由微内核/native 承载的比例链字段，必须写入 active runtime INI/native owner 并从同 owner 运行内存/readback 读取；`slave` 这类微内核/native 和驱动参数映射表都没有承载的设置页轴参数，进入自建参数表并由 UI 自动读取，只作为设置页参数和后续受控 apply 的输入 owner，不得冒充 native 已生效 readback。 - 运动实际真源：`SCALE`、`MAX_VELOCITY`、`MAX_ACCELERATION`、`BACKLASH`、`FERROR`、`MIN_FERROR`、`HOME_SEQUENCE`、Pulse `HOME/HOME_OFFSET/HOME_SEARCH_VEL/HOME_LATCH_VEL`
  必须写入 LinuxCNC runtime INI `[JOINT_*]`/`[AXIS_*]` 原生字段并从 owner 回读。- BUS/Pulse 零位边界：模式差异、切换闭环、Pulse 设零/回零和驱动动作能力边界以 `REQ-BUS-PULSE-MODE-DIFFERENCE` / `功能/2-1总线脉冲区别.md` 为准；本文只维护字段 owner。BUS 设零保留 drive-only `zero_model` 证据并通过 native/runtime 直接回读显示；Pulse 设置 `zero` 显示和保存机械零点到 `HOME_OFFSET`；两者都不能把旧 V3 JSON 当长期 owner。- 驱动私域：`profile_id/slave_index`、电子齿轮、encoder raw/multiturn、SDO 写入/读回证据，A/C count-domain 校准仍属于 Settings Drive private domain；只允许在 drive-only
  `settings_runtime.json` 或驱动映射 profile owner 中持久化。- `settings_projection` 是历史 SHM 设置页投影债务；设置页必须改为 active runtime INI、drive-only owner 和微内核/native 直连 provider 显示，不得把它作为保存成功、运行安全门禁、board live 或配置生成真源。

## P0 启动预加载与实时常驻内存锁定规则

本章节是顶部最高口径的执行拆解。目标是消除运行时闪存/磁盘 I/O 抖动，确保微内核与上层控制系统的实时响应性能。

### 1. UI 与产品自写代码启动预加载（自写代码常驻）
- **C 语言交互层（C UI）**：
  - 前端进程编译为静态/预链接的单体二进制文件；运行期必须使用启动阶段已加载的页面、字体、图片、布局、按键表、命令表和资源索引。
  - 开机启动后立即调用 `mlockall(MCL_CURRENT | MCL_FUTURE)`，锁定前端交互所有代码页及资源页在 RAM 中，杜绝运行时内存页面置换。
  - UI 运行期不得为了按钮动作、页面刷新、参数显示、图标/字体、状态解释或动作结果再读取硬盘资源兜底；需要可变内容时必须来自微内核/native 常驻内存、SHM 显示白名单或受控程序文件选择。
- **Python 业务层（Command Broker 与 Actions）**：
  - 后台守护进程和 command/action gate 在引导阶段通过固定登记表或等价 resident registry 初始化。
  - 自动遍历并导入（`importlib.import_module`）全部业务动作处理器（包括 `settings_apply`、`settings_axis_zero`、`drive_bus` 等），完成 Python 字节码在 RAM 中的常驻编译，使后续 UDS 指令调用全部在内存中进行，避免运行态临时读取 Python 脚本。
  - Broker、Actions、provider、native gate wrapper 和 action registry 必须在启动时形成固定内存登记表；运行期不得按请求再扫描脚本目录、懒导入模块、按文件名执行未登记脚本或从旧路径 wrapper 兜底。
  - 产品自写守护进程同样必须预加载业务模块、配置 schema、协议表和最小资源索引；运行期只消费内存快照或受控刷新后的 provider。

### 2. 微内核与实时层物理锁定（LCNC / RTAPI 强实时）
- **微内核运行闭包全量入 RAM**：
  - LinuxCNC/微内核启动所需的 executable、共享库、RTAPI/实时模块、HAL component、kinematics component、INI/HAL 配置、`PARAMETER_FILE`、`tool.tbl`、G53/RTCP 几何、WCS/G92 初始 readback、驱动映射和已登记 native gate 表，必须在开机阶段一次性读入 RAM 或由 initramfs/RAM disk 提供。
  - 开机装载完成后，运行期控制、运动准入、状态 readback、UI 显示 provider 和动作证明必须来自 LinuxCNC/微内核/native 常驻内存或原生 API，不得再临时读取硬盘文件作为当前真源。
  - 设置保存允许写回唯一持久 owner；写入后必须刷新或重载对应 native 常驻内存，并用该 owner 的内存/API readback 证明结果。单纯落盘成功不等于运行态已生效。
- **物理内存锁死（Memory Locking）**：
  - 底层实时装载器（`rtapi_app` 及相关的伺服/总线线程）必须无条件启用物理内存锁死。在内核空间与用户空间强制锁定所有实时内存页面（`mlockall`），杜绝页面换出（Page-Out）。
  - 使用 `mlockall` 的 UI、provider、State Publisher 和产品自写常驻进程必须有资源预算边界：G-code 运行内存副本、轨迹缓冲、页面资源、字体/图片缓存和临时解析结构不得无限增长。具体预算由板端内存、运行闭包大小和 owner 验证确定，不能在基石规则中写死固定 MB 或某个 LVGL 私有 API。超出预算时必须 fail-closed 或 degraded/unavailable，不得用 swap、临时读盘或旧缓存兜底。
  - UI 轨迹渲染、G-code parser、运行预览和其它频繁动态变化的大块内存必须优先进入独立资源域管理：可使用 Memory Pool、owner 专用 arena、整体生命周期对象池，或通过 `mmap`/`munmap` 申请和释放独立映射。设计目标是让文件切换、页面销毁和受控重载能整体归还物理页；不得依赖普通 `malloc/free` 在 glibc heap 中留下的空洞页自然回收。
  - 加工文件关闭、`program_close`、程序切换、页面销毁和受控参数 reload 等非实时受控边界，拥有资源的 parser/UI/provider owner 必须释放可回收堆对象、清理 resident 引用，并在主循环空闲期执行 `malloc_trim(0)` 或平台等价 trim 作为碎片收缩动作。`malloc_trim(0)` 不是成功证明，只是 glibc 环境下允许并要求的受控回收手段。
  - Start/Resume、实时插补、急停响应、servo 周期、LinuxCNC realtime/HAL/EtherCAT 周期和任何微内核运动热路径中，严禁新增大块动态分配、释放扫描、heap trim、`malloc_trim(0)` 或等价内存整理动作；热路径只能消费已经装载和锁定的 resident 资源。
  - 内存回收验收以板端实际 RSS/Locked 水位为准，不以“代码调用了某个函数”为准。至少通过 `watch -n 1 'grep -E "VmRSS|VmLck|Locked" /proc/$(pgrep v5_lvgl_shell)/status'` 或等价板端自检，在 `N>=10` 次加载/卸载大 G-code 文件、程序切换和页面切换循环后，确认 `VmRSS`/`VmLck`/`Locked` 回落到同一稳定基线；曲线不得出现单调增长的阶梯状泄露趋势。
- **动态链接符号立即绑定（LD_BIND_NOW）**：
  - 守护进程或实时进程的拉起环境变量中，必须显式定义 `LD_BIND_NOW=1`。
  - 指示动态链接器在程序启动的第一时间解析所有关联共享库（`.so`）的动态符号并全量载入 RAM，消除由于懒加载（Lazy Loading）导致运行态突发闪存读取的隐患。
- **物理共享内存限制（tmpfs）**：
  - 所有的进程间同步（SHM，例如 `/dev/shm/v3_status_shm`）必须在 RAM 内存文件系统中创建，完全避免物理闪存写入。
  - SHM 只保留 UI 显示投影：坐标/轨迹、主轴转速、进给速度、主轴倍率、进给倍率。当前模态文字直连微内核/native modal/tool 状态块或等价原生 API；RTCP actual、WCS/G92、tool、驱动模式、安全、停稳、动作成功和参数真源不进入 SHM。
- **RAM 盘根文件系统镜像（RAM Disk / initramfs）**：
  - 板端最终运行形态必须在引导阶段将 LinuxCNC/微内核运行闭包和业务运行闭包打包为只读镜像或等价 manifest，并一次性释放到 `initramfs`/RAM disk 或等价 RAM 驻留路径中。
  - 闪存介质在运行期应只读挂载或从热路径隔离；除明确的设置保存、程序文件选择、事件触发且限频的日志/诊断写入和受控重载外，控制与显示热路径不得触发闪存读写。
