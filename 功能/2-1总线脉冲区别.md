## 1. 一句话区别

BUS / EtherCAT 模式的代码围绕驱动、从站、profile、SDO/PDO、电子齿轮读回、绝对 count-domain 和微内核/native 回读证明工作。它关心的是“当前驱动真实反馈 count 是多少、这个 count 怎样通过 runtime SCALE 进入 LinuxCNC/native 控制空间、驱动证据是否可信”。BUS 主页面在选择 `机械全轴` 后按 `回零` 是真实 Home 运动请求，不是把当前坐标、MCS/SHM 显示值、`zero_model` 或 homed 标记改成 0；MCS/WCS 单轴选择后的 `回零` 是登记 native positioning owner 执行的真实回机械 0 / 回加工 0 运动，详细分支以 `功能/3-2回零按钮开机强制回零与双模式真实回零方案.md` 为准。

Pulse / StepDir 模式的代码围绕 LinuxCNC/HAL 原生 homing 和 runtime INI 的 `HOME_*` 字段工作。设置页轴参数表里按当前轴 `设零` 时弹出键盘，输入坐标值作为当前轴 0 位，并写入当前轴 `HOME_OFFSET`；同一轴参数表里有独立 `回零方向` 列，可用下拉选择回零方向并写 `home_direction` owner。

所以两者不能互相复用：BUS 的 count-domain 零点证据不能生成 Pulse `HOME_OFFSET`；Pulse 的 `HOME_*` 参数也不能当作 BUS 驱动零点、电子齿轮或绝对多圈证据。

### 2.1 模式切换闭环

总线/脉冲模式切换必须走受控闭环。当前阶段 Pulse/StepDir 只允许作为冷态预配置资产存在，contract 仍为 `cold_staged_not_runtime_selectable` 时，UI 可以显示 Pulse 选项但必须明确 disabled；任何强制提交都必须在 C/Broker dispatch 层 fail-closed，不能进入下面的运行切换流程。下面流程只适用于当前可运行的 BUS/EtherCAT，或未来 Pulse contract 被 owner 更新为 `runtime_selectable` 之后的 Pulse 切换。

```text
C/LVGL UI
  -> Command Broker UDS settings_apply 请求，携带目标 motion_driver_mode
  -> 已登记 settings/native mode-switch owner
  -> 物理写入 motion_driver.conf
  -> 源位置回读确认 == 目标模式
  -> 返回 RESTART_PENDING，前台锁屏/进度等待
  -> canonical clean restart handoff
  -> 停止并重建 UI/Broker/Action Dispatcher/State Publisher/LinuxCNC/微内核/HAL/EtherCAT/drive provider 等运行闭包
  -> 清理旧 /run、UDS socket、SHM/native 状态块、resident snapshot、cache 和命令队列
  -> 若软件 clean restart 不能证明达到开发板断电重启同等干净度，fail-closed 到受控 systemd reboot / 断电重启等价路径
  -> SelectLink 按 motion_driver.conf 选择 owner 签核的真实启动链接
  -> ethercat: owner 签核的 EtherCAT INI + lcec 实时模块
  -> pulse: owner 签核且 runtime_selectable 的 Pulse INI + Pulse HAL/RT adapter
  -> 重新初始化 LinuxCNC/HAL/EtherCAT 或 Pulse adapter 和模式状态 provider
  -> mode-switch/微内核或 native 状态按四项并联条件验证真实模式
  -> native active_driver_mode 直接回读
  -> UI 只有读到 native active_driver_mode == 请求目标模式时才关闭锁屏/进度
```

切换状态机固定为：

```text
REQUESTED_WRITTEN
  -> SWITCH_PENDING
  -> CLEAN_RESTART_RUNNING | REBOOT_REQUIRED
  -> ACTIVE_VERIFIED | DEGRADED
```

硬边界：

- 设置阶段 UI 只发 Command Broker UDS/settings_apply 请求，不得直接触发 systemd、shell、reboot 或切换脚本。
- 已登记 settings/native mode-switch owner 只能写 `motion_driver.conf` 这个模式 owner，并源位置回读确认；不得写 `settings_runtime.json`、旧 JSON、UI 默认值或临时文件作为模式真源。
- `mode-switch.service` 或现有受控 backend restart 入口只有在被纳入 `功能/0-1开机参数入内存.md` 要求的 canonical clean restart handoff 后才可作为模式切换入口；等价入口必须输出 `mode_switch_state` / `mode_switch_state_machine` / `mode_switch_owner` 字段，按 `REQUESTED_WRITTEN -> SWITCH_PENDING -> CLEAN_RESTART_RUNNING|REBOOT_REQUIRED -> ACTIVE_VERIFIED|DEGRADED` 表达进度。不得在产品代码里调用 `os.system("sudo reboot")` 或等价自由 shell。
- 软件 clean restart 只有在 LinuxCNC、实时模块、HAL 设备、运行闭包、`/run`、UDS socket、SHM/native 状态块、resident snapshot、cache 和命令队列都清理并重新装载，且能证明达到开发板断电重启同等干净度时才允许继续；卸载失败、资源占用、模块残留、旧运行态残留或状态不可证明时，必须进入受控整机重启，不得硬切。
- SelectLink 是唯一启动选择器，只能按 `motion_driver.conf` 选择真实存在、owner 签核的 EtherCAT INI 或 Pulse INI；Pulse INI、配套 HAL/RT adapter 和板端硬件证据不存在或 contract 不是 `runtime_selectable` 时，不得实现空选择器、文档式选择器或隐藏 fallback。其它脚本、JSON、UI dirty 值和默认模式不得参与选择。
- 当前 Pulse 冷态预配置的硬件源码 owner 已移动到 `vivado_hw_project/`：AXI base/range 以 `vivado_hw_project/vivado_hw_project.srcs/sources_1/bd/system/system.bd` 为准，`ID=0x53544550`、`VERSION=0x00030500`、per-axis stride 和寄存器语义以 `vivado_hw_project/ip_repo/axi_stepdir_enc_v2/step_ip_register_map.md` 为准，当前实例轴数以 `vivado_hw_project/vivado_hw_project.srcs/sources_1/bd/system/ip/system_axi_stepdir_enc_v2_0_0/system_axi_stepdir_enc_v2_0_0.xci` 的 `N_AXES=6` 和板端只读探针一致性为准。软件 contract 在 `board/linuxcnc/components/step_ip_v1_5.contract.json`，PetaLinux RT adapter 源码在 `board/petalinux/project-spec/meta-user/recipes-apps/v5-stepgen-module/files/zynq_stepgen_hw.c`；`new-vivado/`、`D:\Z20_ZYNQ` 和旧 ABI header 只允许作历史对照，不再是当前 Pulse SelectLink 目标或 adapter 来源。当前 Pulse 只能作为冷态预配置资产存在，contract 必须保持 `cold_staged_not_runtime_selectable`，软件运行层不得启动 Pulse；只有 bitstream、device-tree/UIO、生产 HAL/RT adapter、HAL/INI、FPGA `ID/VERSION/N_AXES` 只读探针和板端波形/安全证明全部成立后，才能按 owner 更新 contract 为 `runtime_selectable` 并进入 SelectLink 可启动路径。
- mode-switch/状态 provider 不得只相信 `motion_driver.conf`，也不得在 steady-state 通过硬盘文件兜底 active mode。发布/回读 `active_driver_mode` 前必须同时证明 requested mode 等于 SelectLink 选中 INI、LinuxCNC fresh/running 且使用该 INI、期望侧 RT/HAL module 已加载、非期望侧 module/adapter inert 或 disabled；任一条件缺失、过期或矛盾时返回 degraded/unknown，UI 继续锁定。
- UI 解锁条件只能是 native active_driver_mode == requested_mode；`RESTART_PENDING`、服务已启动、reboot 已触发、SHM 字段或命令已发送都不能代表切换完成。

### 2.2 关闭口径

模式切换 software gate 关闭只证明 UI pending/锁定、native active-mode provider、动作能力按真实 owner 处理和 BUS 回归不退化已经闭合。它不能证明 Pulse/StepDir 硬件可运行。

要声明 Pulse/StepDir 可运行，必须另有板端硬件闭环证据：Step/Dir 波形方向、频率、脉宽/间隔正确；enable 在启动、重启、崩溃、服务停止和切换窗口保持 disabled 或无有效脉冲；急停、限位、home 输入有效；watchdog、underrun、overrun 或通信中断时 fail-closed；reboot/crash/no-service 无误脉冲、无错误方向、无自动 enable；Pulse `回零` 按 `回零次序` 和 `回零方向` 建立 homed/all_homed 并经 LinuxCNC/HAL/native 回读。缺这些证据时只能称为 `software_gate_only` 或 `local_verified_only`。

## 3. 代码链路差异

| 层级 | BUS / EtherCAT | Pulse / StepDir |
| --- | --- | --- |
| UI 设置页轴参数表设0 | `execute_settings_axis_zero_gate_command()`：直接提交当前行单轴 `settings_axis_zero`，不输入数值 | `execute_settings_axis_zero_pulse_home_offset_command()`：打开软键盘，输入坐标值作为当前行当前轴 0 位；`回零方向` 列用下拉选择 |
| Command Broker UDS/native dispatcher | 同一个 `settings_axis_zero` 请求，但校验 `target_scope=bus_count_domain_zero`、`apply_mode=count_domain_zero` | 同一个 `settings_axis_zero` 请求，但校验 `target_scope=pulse_mechanical_home_offset`、`apply_mode=persist_home_offset` |
| settings/native apply owner | BUS 读驱动 actual/split absolute，生成 count-domain 零点证据 | Pulse 构造 `HOME_*` apply 请求 |
| settings apply | BUS 不生成 Pulse HOME 参数，不恢复旧浮点零位字段 | settings/native apply owner 写 runtime INI/native `HOME_OFFSET`；脚本名只能作为当前实现落点，不能成为第二 owner |
| 驱动命令 | 通过 profile 的 `drive.*` 标准命令映射到实际 EtherCAT SDO/PDO | 不走 EtherCAT SDO，不读 BUS zero_model，不写驱动 profile |
| 状态回读 | native/microkernel provider 回读 active mode、驱动/count-domain、MCS、degraded reason；SHM 只显示坐标/轨迹、转速、进给速度和主轴/进给倍率投影，当前模态直连 native modal/tool 状态 | native/microkernel provider 回读 active mode、runtime INI/native HOME 和 homed/all_homed；SHM 只显示坐标/轨迹、转速、进给速度和主轴/进给倍率投影，当前模态直连 native modal/tool 状态 |

### 3.1 UI 共用边界

BUS / EtherCAT 和 Pulse / StepDir 共享同一套设置页、按钮框架、弹窗、顶部状态和命令发送；不得为了 Pulse 另建一套设置页、第二套 UI 树、第二套命令 shell 或第二套状态显示逻辑。UI 可用 SHM 显示坐标/轨迹、转速、进给速度和主轴/进给倍率，但当前模态、active mode、homed 和安全状态必须直连 native/microkernel 状态 provider。

Pulse 模式在 UI 上只允许做必要的局部差异：

- 轴参数卡：`设零` 弹键盘输入当前轴 0 位，写当前轴 `HOME_OFFSET`。
- 轴参数卡：显示独立 `回零方向` 列，下拉选择方向并写 `home_direction` owner。
- 设置驱动动作行：继续复用同一套 UI；不得靠静态灰禁或预检拦截闭合。Pulse/StepDir 模式下点击驱动动作时，由后台按 fresh `active_driver_mode`、目标从站、profile 内存快照、授权和安全 owner 执行真实动作或返回结构化 `not_applicable` / `resource_missing` / `failed_closed`；UI 只显示后台结果和下一步，不用 SHM 字段、UI 文案、dirty 值或旧 JSON 自己判断模式或伪造完成。

这里的“不得靠静态灰禁闭合”只针对设置驱动动作、授权下载、服务器下载、扫描/读取/复位/清故障等按钮。它不取消当前阶段 Pulse 模式切换入口的硬禁用要求：Pulse contract 仍为 `cold_staged_not_runtime_selectable` 时，运动模式下拉中的 Pulse 目标必须 disabled，强制提交必须由 Broker/C dispatch fail-closed。

其它设置页布局、导航、保存流程、提示弹窗和状态显示继续复用 BUS UI。UI 只能按 native `active_driver_mode` 调整这些局部控件状态，不得用 SHM 字段、UI 文案、dirty 值或旧 JSON 自己推断模式。

## 4. 设置页设0差异

| 项目 | BUS / EtherCAT | Pulse / StepDir |
| --- | --- | --- |
| 用户看到的语义 | 轴参数表单轴 `设0` / 当前驱动位置作为逻辑零点证据 | 轴参数表单轴 `设零` / 当前轴 0 位 |
| 用户输入 | 不输入数值 | `设零` 弹出键盘输入当前轴 0 位；`回零方向` 列下拉选择方向 |
| 后端动作 | 读取驱动 count-domain 位置，保存 `zero_counts`、A/C 绝对多圈或 split count、profile/hash/readback 证据 | 写 runtime INI/native owner 的当前轴 `HOME_OFFSET`；`回零方向` 列写 `home_direction` owner |
| 当前坐标是否立即清零 | 不能只靠 UI 减法声称已生效；按钮保存通过只证明零点证据/owner 写入，live 坐标归零需后续 native/UI 显示闭环 | 不能只靠 UI 立即改坐标；必须经 owner 写入和后续 LinuxCNC/HAL native 回零证明 |
| 禁止行为 | 普通设0不得写 `0x607C`、`0x1010`、drive preset、drive rebase 或厂家位置偏置 | 不发送 EtherCAT SDO，不读取 BUS `zero_model`，不写 `motion_driver.conf` |

BUS 轴参数表单轴设0保存的是驱动/count-domain 证据，不是机器回零动作。Pulse 轴参数表单轴设零保存的是当前轴 0 位坐标值对应的 `HOME_OFFSET`，不是驱动零点证据。

## 5. 主页面回零差异

本节 BUS/Pulse 模式差异只作用于当前 UI 选择为 `机械全轴` 的正常 Home 分支。选择 MCS 单轴时，两种模式都由登记 native positioning owner 让所选轴真实移动到机械坐标 0；选择 WCS 单轴时，两种模式都让所选轴真实移动到当前 active 加工坐标 0，且不修改 WCS offset。这两个单轴分支不改变 `homed/all_homed`，不得因能力缺失回退为全轴 Home。

| 项目 | BUS / EtherCAT | Pulse / StepDir |
| --- | --- | --- |
| `机械全轴` + 主页面 `回零` 目标 | 每次按键都触发 native/EtherCAT Home owner 的真实回零运动循环；运动参考可来自设置页 BUS `设0` 的 count-domain 零点证据或 owner 签核的 home reference，但不能只改坐标为 0 | 执行 LinuxCNC/HAL 原生 Home |
| 运动依据 | native count-domain 零点证据 + active runtime INI/native `SCALE` + native fresh MCS/readback | 轴参数表 `回零次序` 决定先后轴，`回零方向` 决定找限位方向，触碰限位/home 输入后按 `HOME_OFFSET` 建立坐标 |
| 成功边界 | 本次 Home transaction 的真实运动/到位/readback token + 驱动/adapter 到位证明或 LinuxCNC/native 机器坐标到位证明 + LinuxCNC/native homed 标记 + homed/all_homed native 回读 | LinuxCNC/HAL `homed/all_homed` 经 native/API 回读 |
| 速度/加速度 | 只从 runtime INI/native `max_velocity_units_per_s` 和 `max_acceleration_units_per_s2` 取，不用 SHM、UI dirty、旧 JSON 或固定低速 | Pulse `HOME_SEARCH_VEL/HOME_LATCH_VEL` 与设置页最大速度同幅值，并由 runtime INI/native owner 生效 |
| 不能跳过 | 不能因为当前位置接近 0、已 homed 或目标位移为 0 跳过标准 Home 链路 | 不能用 BUS 零点证据、当前位置接近 0 或 UI 显示值代替触碰限位/home 输入的 LinuxCNC/HAL homing |

BUS `机械全轴` 回零必须由 microkernel/native/EtherCAT Home owner 执行真实运动。若当前物理位置已经在 home reference，native owner 也必须执行可证明的 backoff/search/latch 或等价真实 Home cycle；如果底层暂不具备这种能力，只能返回结构化 capability/blocker，不能用坐标清零、`Set Home` 标记、`zero_model` 写入、Work Zero、G92、drive preset 或 SHM/MCS 显示变化冒充完成。

`机械全轴` + `回零`、单轴 + `回零`、`归零` 和设置页 `设0` 必须分开：机械全轴回零触发正常 Home 并在完成后改变 homed 状态；MCS/WCS 单轴回零只执行真实到 0 定位且不改变 homed；归零写当前 active WCS offset 且不运动；设置页设0只写对应模式的零点或 HOME 参数。

## 6. 设置驱动只属于 BUS 驱动域

`设置驱动`、`扫描从站`、`读取驱动`、`复位驱动` 和 `清除故障` 都是 EtherCAT/驱动 profile 域动作。public/private `driver_profile_map.json` 只能在下载完成后的受控刷新、启动装载或受控重载时进入 drive/native provider 内存快照；按钮动作只消费该内存快照、目标从站 fresh readback 和 profile 唯一匹配结果，再调用 `drive.*` 标准命令。板端业务代码不得在按钮热路径读取 profile 文件，也不得硬编码厂家 SDO index/subindex。

设置页驱动动作行里的按钮不得默认用静态禁用/预检拦截组合作为闭合方式。代码修复方向是优先跑通真实 action/profile/readback 路径：`下载授权`、`服务器下载` 走授权/VPS owner，不因 Pulse / StepDir 模式静态禁用；`扫描从站`、`读取驱动`、`复位驱动`、`清除故障`、`设置驱动` 由后台按当前硬件能力、目标从站、profile、授权和安全状态执行。若当前模式或现场没有可用 BUS/EtherCAT 目标，或写类动作缺少必要 profile/readback/安全条件，后台返回结构化 `not_applicable`、`resource_missing` 或 fail-closed 结果；UI 负责显示真实结果和下一步，不把“加一个禁用分支”当作功能闭合。只有硬安全、身份授权、资源确实不存在或目标模式真实不适用时才允许拒绝；拒绝必须来自真实 owner 判断，不能由 UI 文案、旧配置、默认值或为掩盖未实现动作而新增的静态拦截分支决定。

BUS 代码在这里会处理：

- 真实从站扫描、OP 状态、profile 唯一匹配。
- `drive.set_egear` / `drive.read_egear` 写入和读回电子齿轮。
- `drive.read_actual_position`、`drive.read_abs_multi_turn`、`drive.read_abs_single_turn`、`drive.read_abs_position` 及其映射位宽/类型/模数等 count-domain 位置证据。
- `drive.restore_factory_defaults` 复位驱动，并使旧电子齿轮、设0、rotary checkpoint 和 count-domain trust 证据失效。

Pulse 模式不从这些驱动动作里获得 HOME 真源。Pulse 的当前轴 0 位和 HOME 参数属于 LinuxCNC runtime INI/native owner；驱动 profile、SDO readback、电子齿轮证据不能替代 Pulse homing 配置。

## 7. 旋转轴差异

A/C 旋转轴在 BUS 场景下强依赖整数 count-domain：`Crev`、电子齿轮读回、减速比、runtime `SCALE`、绝对多圈 direct64/split/checkpoint 证明和 native readback 必须一致。等效角显示、等效回零、unwrap、checkpoint 恢复和 Rebase gate 都不能只靠浮点角度、SHM 显示投影或 UI 显示值。

Pulse 场景下 A/C 仍显示等效角，但 Home 本身走 Pulse 的 LinuxCNC/HAL homing 链；不能把 Pulse `HOME_OFFSET` 当作 BUS 绝对多圈零点，也不能把 BUS `zero_counts` 当作 Pulse 回零参数。

普通等效回零是物理运动到最近等效零，不是 Work Zero、G92、drive preset 或 Rebase。任何 position-basis mutation 都必须走独立 native gate 和同周期证明。

## 8. 运行热路径差异

BUS 运行前需要把驱动 profile、电子齿轮读回、count-domain 零点、runtime SCALE、授权/DNA latch、RTCP/G53 几何等慢证据前置并锁存。G-code 正在运行时，不允许再读 SDO、重算 profile、重建 settings projection 或写周期性诊断 JSON 来补证据。

Pulse 运行前需要把当前轴 0 位 `HOME_OFFSET`、`home_direction`、runtime INI/native HOME、软限位、SCALE、速度/加速度和 homed 状态等 native 投影准备好。G-code 正在运行时，同样只消费 microkernel/native actual；SHM 仅供 UI 坐标/轨迹、转速、进给速度和主轴/进给倍率显示，当前模态直连 native modal/tool 状态，不从 UI dirty、旧文件或 `/run` 结果补参数。

两种模式共享的热路径原则是：Start/Jog/Home/运动动作只读取 fresh microkernel/native actual 和已前置信标，不能在按钮热路径里临时读盘、fork helper、跑 `halcmd`、重建大 JSON、访问旧结果文件或把 SHM 显示投影当控制证明。

## 9. 不是总线/脉冲差异的内容

下列文档定义的是两种模式共同遵守的边界，不应被理解成两套业务实现：

- WCS / Work Zero：真源是 LinuxCNC `PARAMETER_FILE` 和解释器状态；不能替代机器回零、BUS 设0、Pulse HOME、Rebase 或 RTCP 几何。
- 跳窗提示：只定义结果码、中文原因、处理方式和确认继续规则；不能因为 BUS/Pulse 分支不同就新增第二套弹窗协议。
- UI 刀路：只显示同一帧 SHM/内存投影，不计算真实 RTCP 运动学，不写驱动、HAL、INI 或 G-code。
- VPS/授权：只提供驱动 profile、授权/DNA、远程输入和下载链路；不改变 BUS/Pulse 的运动真源。
- 触摸校准：只影响输入坐标链路，不影响 EtherCAT、8ax 后端、驱动参数或运动控制。

## 10. 验收

本地/源码验收至少证明：

- active mode 没有写入 `/dev/shm/v3_status_shm`，SHM ABI、reader、writer 和 UI state apply 中不存在 active mode、BUS/Pulse proof、HOME、homed、安全或驱动模式字段。
- `motion_driver.conf` 只作为 requested/持久 owner；active mode 必须由 mode-switch/native provider 按四项条件回读，不能由 UI 文案、dirty、旧 JSON、`settings_runtime.json` 或 SHM 推断。
- `settings_runtime.json` 不保存 Pulse `HOME_OFFSET`、`home_direction`、`HOME_SEQUENCE`、Pulse mode、WCS/G92/RTCP/tool/modal 或安全 actual。
- 当前 Pulse contract 为 `cold_staged_not_runtime_selectable` 时，UI 入口 disabled，Broker/C dispatch 强制拒绝 Pulse 切换，且 BUS/EtherCAT 路径不受影响。
- 模式切换进入 canonical clean restart handoff；不能只重启 UI、只重启 provider 或只发起若干 daemon restart 后宣称完成。
- Pulse 轴参数设零只写 runtime INI/native `HOME_OFFSET` owner；BUS 轴参数设0只写驱动/count-domain 零点证据，二者不能互相生成。
- BUS 主页面选择 `机械全轴` 后每次按 `回零` 都必须触发新的 native Home transaction 和真实运动/到位/readback token；选择 MCS/WCS 单轴后每次按 `回零` 必须触发新的 native positioning transaction 并读回对应 MCS/WCS 为 0。已 homed、当前位置为 0、目标 delta 为 0、`zero_model` 存在或命令发送成功都不能单独作为通过条件。
- 设置驱动动作使用 fresh `active_driver_mode`、drive/native provider 内存快照和 profile/readback；拒绝必须来自真实 owner 的 `not_applicable`、`resource_missing` 或 fail-closed，不得靠 UI 文案或静态预检分支冒充闭合。

板端声明 Pulse/StepDir 可运行时还必须证明：

- contract 已由 owner 更新为 `runtime_selectable`，并有 bitstream、device-tree/UIO、生产 HAL/RT adapter、HAL/INI、FPGA `ID/VERSION/N_AXES=6` 只读探针证据；若硬件实例轴数后续改变，必须先同步 Vivado 实例、contract 和本 owner，再按新 actual 验收。
- Step/Dir 波形方向、频率、脉宽/间隔正确，enable 在启动、重启、崩溃、服务停止和切换窗口保持 disabled 或无有效脉冲。
- 急停、限位、home 输入、watchdog、underrun/overrun、通信中断均 fail-closed。
- Pulse `回零` 按 `回零次序` 和 `回零方向` 建立 homed/all_homed，并经 LinuxCNC/HAL/native 回读；涉及运动时按 `REQ-AUTO-STATE-DRIVEN` 跑 native active model 对应的 `cc-ac.ngc` 或 `cc-bc.ngc` 黄金运动闭环。

## 11. 不能混用的红线

- 不得用 BUS `zero_model`、`zero_counts` 或驱动 actual 生成 Pulse `HOME_OFFSET`。
- 不得把 Pulse 当前轴 0 位、`home_offset`、`pulse_home_model`、`HOME_SEQUENCE` 写入 `settings_runtime.json`。
- 不得用 UI 模式文字、dirty 值、旧 `run_mode_bus_pulse` 或旧 JSON 判断当前 active mode。
- 不得把设置页 BUS `设0` 当作主页面 `回零`。
- 不得把 BUS 主页面 `机械全轴` 回零实现为坐标变 0、MCS/SHM 显示归零、写 `zero_model`、单独 `Set Home` 标记、Work Zero、G92、drive preset 或任何不产生真实 Home 运动/到位证明的路径；也不得把 MCS/WCS 单轴回零实现成改坐标或改 offset 而不产生真实定位运动。
- 不得把主页面 `归零` / Work Zero 当作机器 Home、BUS 设0、Pulse HOME、G92、Rebase 或驱动位置改写。
- 不得在普通设0、回零、清故障或自动维护中静默执行 `drive.preset_position`、`drive.clear_multi_turn`、厂家位置偏置或 drive-side zero rewrite。
- 不得在大文件里继续塞入新的 BUS 位移规划、Pulse HOME 状态机、驱动厂家分支、G-code 解析器或跨层决策；新增逻辑必须下沉到对应 owner。
- 不得在物理硬件与底层固件未到位前允许切换到脉冲模式。设置页的运动模式切换必须硬性限制：UI 下拉框应将脉冲模式置灰禁用，若强行提交必须在 Broker/C dispatch 层面直接拦截并明确报错，此为硬件未到位前的最高安全红线。
