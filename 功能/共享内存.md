## AI 阅读入口

- 本文定位：`/dev/shm/v3_status_shm` 的字段清单、允许内容、30ms 常驻同步边界和禁止内容。
- 先读 `REQ-SHM-DISPLAY-PROJECTION`。需要判断字段 owner 时，再读 `功能/0开机参数入内存.md`、`系统代码架构硬边界守则.md`、`功能/微内核.md` 和相关功能 owner。
- SHM 只是 UI 坐标值、UI 轨迹变化、模态文字、转速、进给速度和主轴/进给倍率显示投影，不是运行态真源、保存证明、动作结果、安全证明、Broker readback、参数 owner、诊断总线或兜底通道。
- 本文消费 `功能/0开机参数入内存.md` 的最高启动内存架构：LinuxCNC/微内核、UI 和产品自写代码运行闭包开机全量入 RAM；SHM 发布端只能从微内核/native 常驻内存/API 或登记 native memory block 采样，不能运行期读硬盘、JSON、result、脚本快照或目录扫描来补帧。

## P0 唯一允许的 SHM 内容

/dev/shm/v3_status_shm 只允许承载 UI 显示投影。微内核/native 到 SHM 的同步目标周期为 30ms（适应人眼流畅度极限）。在保障状态与数据保真度（数据不说谎）的前提下，UI 允许 100ms~300ms 之间的短暂视觉延迟（Stale 过渡），这属于合理的人眼感知“视觉大概”范围，无需为此过度消耗 CPU；同步进程必须属于开机已加载的产品自写运行闭包，从微内核/native 常驻内存/API 或登记 native memory block 采样后单向发布，不得在 steady-state 读取硬盘文件、扫描脚本目录、动态导入模块、执行一次性 helper 或读取 `/run` result 补帧。

必须写入 SHM 的 UI 显示字段只有：编码器反馈机械坐标、运动学理论坐标、模态、转速、进给速度、主轴倍率和进给倍率。字段如下：

| 字段/字段族 | 内容 | UI 用途 | 真源 owner |
| --- | --- | --- | --- |
| SHM envelope | `magic`、`version`、`header_size`、`total_size`、`payload_size`、`flags`、`seq`、CRC、dual-page header、page seq/CRC/generation | 只用于 seqlock、版本、长度和帧完整性校验 | SHM ABI |
| `status_epoch` | 当前发布帧 epoch | 只表示这一帧显示投影的新鲜度 | State Publisher 发布帧 |
| `typed_valid_mask` / `typed_flags` | 只允许表达下表显示字段的有效性和帧退化状态 | UI 判断显示项是否可画；不得作为动作准入 | SHM ABI |
| `mcs[5]` | 编码器反馈/关节 actual 收敛后的机械坐标 | UI 坐标值、UI 轨迹当前机械点 | LinuxCNC/HAL/EtherCAT/native 反馈内存 |
| `cmd_mcs[5]` | 运动学/插补器 commanded/theory 收敛后的理论机械坐标；开机初始命令位置和下使能期间的命令起点必须由编码器/驱动 actual 位置经 LinuxCNC/HAL/native 同步计算得到，不能停留在旧命令值或默认 0 | UI 轨迹理论点、跟随误差显示 | LinuxCNC/HAL/native commanded/theory 内存，BUS/Pulse 下使能和开机重基准必须使用同轴 actual feedback 形成的 joint commanded/output |
| `runtime_modal_text` | 历史 ABI padding/旧诊断文本；不得置为主 UI 有效模态 | 不得作为主页面当前模态、RTCP 动作证明、当前刀号或刀长真源 | 微内核/native modal/tool 状态块或等价原生 API |
| `spindle_speed_rpm` | 主轴转速显示值 | UI 转速显示，必须进 SHM | LinuxCNC/HAL runtime actual |
| `linear_velocity_mm_per_min` | 线速度显示值 | UI 进给速度显示，必须进 SHM | LinuxCNC/HAL runtime actual |
| `feedrate_override` | 进给倍率显示值 | UI 进给倍率显示，必须进 SHM | LinuxCNC/HAL runtime actual |
| `spindle_override` | 主轴倍率显示值 | UI 主轴倍率显示，必须进 SHM | LinuxCNC/HAL runtime actual |

这些字段只能从微内核/native 常驻内存/API 或登记 native memory block 采样后发布。缺 fresh 投影时 UI 只能显示 degraded/unavailable；不得用硬盘 JSON、`/run` result、旧 SHM 帧、默认值、Broker 结果或 UI 缓存补成有效。`cmd_mcs` 不得从未重基准的 `stat.position`、默认 0、旧命令缓存或 UI 本地种子值发布；当 LinuxCNC/HAL/native 已提供 joint commanded/output 且该值已在开机或下使能状态跟随 actual feedback 时，State Publisher 必须优先发布该值。
State Publisher/WCS position publisher 必须以常驻进程进入 poll/reconnect 循环；不得在启动后用 `linuxcnc.stat()`、坐标 ready、WCS ready、CMD_MCS ready 或其它 preflight 稳定探测做无限等待而不发布。LinuxCNC/HAL/native 暂不可用时只允许发布 invalid/degraded 帧并继续重连；一旦 native actual 可读，必须立即写入 fresh position block 和 SHM 坐标投影。

## P0 禁止进入 SHM 的内容

下列内容必须直连微内核/native owner 或登记 native memory block，不得写入 `/dev/shm/v3_status_shm`：

| 禁止字段/语义 | 正确 owner | 处理口径 |
| --- | --- | --- |
| WCS/G54-G59.3、`wcs_pos`、`wcs_offset`、`wcs_offsets`、`active_wcs_index` | LinuxCNC `PARAMETER_FILE`、interpreter/native readback | UI/刀路直连读取；不得用 SHM 证明 Work Zero、WCS 切换或坐标基 |
| G92 offset/actual | LinuxCNC interpreter/native G92 actual | 只可按登记 native policy 显示/写入；不得同步到 SHM |
| RTCP/switchkins actual、RTCP/G53/A-C 几何 | HAL/native component、switchkins/native helper、runtime INI `[RTCP]` | `rtcp_active`、RTCP 几何和 `tool_rtcp_geometry` 不进 SHM；已删除 `re_hal_select_helper status-daemon` 和 `/dev/shm/v3_rtcp_status.json` |
| tool/TLO/刀补/当前刀号 | LinuxCNC `tool.tbl`、`stat.tool_offset`、native tool owner | UI 直连显示；不得用 SHM tool/TLO 字段作刀补真源 |
| 急停、使能、安全互锁、限位、all_homed/homed | LinuxCNC/微内核/HAL/EtherCAT realtime/native safety owner | 上层只请求/显示同源回读；不得用 SHM 判定安全、取消急停成功或运动准入 |
| active driver mode、BUS/Pulse proof | mode-switch/backend/微内核 native active-mode readback | 不得用 SHM 或 UI 下拉判断 active mode |
| motion stillness gate、速度门禁容差 | 禁止的 V3 上层门禁 | 不得把门禁参数或判定结果投影到 SHM；停稳和速度安全只属于微内核/native 内部 owner，上层不消费 SHM/JSON 证明 |
| rotary count-domain、unwrap、range guard、wcheckpoint/rebase | 旋转轴 native/window/checkpoint owner | `logical_abs_counts64`、`wcheckpoint_base_counts64`、`runtime_window_counts` 等不进 SHM |
| segment boundary / program boundary proof | 微内核/native segment boundary readback | 不得用 SHM 作为段边界 proof 或 Start 准入 |
| current line / motion line / MDI run line / MDI command | LinuxCNC interpreter/native actual | 可按需直连显示；不得作为 SHM 准入或运行证明 |
| system metrics、CPU usage、PS IP、heartbeat、ready、freshness | OS/UDS/native health 或 supervisor 启动诊断 | 不得用 SHM 表达运行心跳、ready 或性能诊断 |
| settings projection、config cache、保存成功、动作结果、Broker result、diagnostic JSON | 对应 owner/source readback 或动作响应 | 不得作为 SHM payload 或字段；保存成功必须源位置回读 |
| 通用 JSON payload | 无产品 owner | SHM frame 内不得保留通用 JSON payload；typed frame 只保留允许显示投影 |

## P0 禁止保留的 ABI 字段

ABI mask 只能表达产品允许的 UI 显示投影字段。下列字段不得作为产品白名单、兼容字段、诊断字段或 fallback 字段保留；触碰 SHM 打包、reader、layout、测试或 UI state apply 时，必须删除对应 SHM 投影和 ABI 位。

| 禁止 ABI 字段/位 | 状态 | 正确 owner |
| --- | --- | --- |
| `TYPED_VALID_CORE_STATE`、`machine_state`、`program_state`、`task_state`、`interp_state`、`exec_state` | 禁止 | 运行态/解释器状态直连 native provider |
| `TYPED_VALID_ENABLE_STATE`、`TYPED_FLAG_ESTOP`、`TYPED_FLAG_ENABLED`、`TYPED_FLAG_ALL_HOMED`、`homed_mask` | 禁止 | 安全/使能/Home 状态直连 safety/native owner |
| `TYPED_VALID_ACTIVE_WCS`、`TYPED_VALID_WCS_OFFSETS`、`wcs_*` | 禁止 | WCS 直连 `PARAMETER_FILE`/interpreter/native |
| `TYPED_VALID_TOOL_RTCP_OVERLAY`、tool/TLO/holder/tip/RTCP geometry flags | 禁止 | tool/TLO/RTCP/G53 直连 native owner；只允许坐标点本身作为 UI 轨迹显示投影时保留到允许坐标族 |
| `TYPED_VALID_PSIP`、`ps_eth0_ip` | 禁止 | 网络状态走 UI/provider 或诊断，不进 SHM |
| `TYPED_VALID_ROTARY_COUNT_DOMAIN`、`TYPED_VALID_ROTARY_RANGE_GUARD`、`TYPED_VALID_ROTARY_WCHECKPOINT` | 禁止 | 旋转轴 native/window/checkpoint 直连 readback |

| `TYPED_VALID_MOTION_DRIVER`、`active_driver_mode_code` | 禁止 | active mode 直连 mode-switch/native readback |
| `TYPED_VALID_SYSTEM_METRICS`、CPU usage flags | 禁止 | 系统指标走诊断/OS，不进 SHM |
| `TYPED_VALID_SEGMENT_BOUNDARY` | 禁止 | segment boundary 直连 native readback |
| `TYPED_VALID_MOTION_STILLNESS_GATE`、`motion_stillness_gate_packed` | 禁止 | 直接删除；不得改名保留为诊断、兼容字段或上层速度门禁 |
