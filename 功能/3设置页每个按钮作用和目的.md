参数域归属：

最新交互边界（2026-06-27）：设置页每个可编辑字段在软键盘 `OK` 或下拉选择提交时必须立即通过 `settings_apply` 确认保存到该字段唯一 owner，并完成本字段源位置回读后才清该字段 dirty；这包括 `G53_A_Y/G53_A_Z/G53_C_X/G53_C_Y`，它们只写 active runtime INI `[RTCP]` owner，不写 `settings_runtime.json`。右上角 `保存并重启` 不再负责重新保存这些已确认字段，也不在前台等待完整 native 状态恢复、machine-on 或驱动 Operation Enabled 恢复；它只做轻量 restart precheck，发起 backend + UI restart handoff，收到 handoff 成功后关闭进度窗口并返回主页面。settings 页轻量保存重启链路必须在新 UI 恢复/进入主页面后通过正式 `estop_or_reset` 产品动作发起上使能；machine-on 和驱动 Operation Enabled 结果只作为后台恢复诊断，后台重启后的最终运行态真值仍以后续 native/运行内存直接回读为准；SHM 只更新允许的 UI 显示投影。

以下名称只表示 运行内存/UI 显示投影或 UI 分区，不允许落地成 V3 自造参数文件。除 `设置驱动` 的驱动私有域外，可由 LinuxCNC/runtime INI、`PARAMETER_FILE`、`tool.tbl`、HAL/IO 或登记 native gate 承载的字段必须迁入对应 owner；历史上已经造出的 `system_params`、`axis_params`、`tool_params`、`wcs_offset_params`、`io_params` 文件或写入 owner 要删除。

最高参数表闭环：
- 设置页参数表所有字段都遵守 `开机硬盘/native owner -> native/运行内存 -> UI显示；坐标/模态/速度倍率再投影到 SHM`，以及 `UI dirty -> Broker/settings_apply/native owner -> 写唯一 owner -> 刷新 native/运行内存 -> UI回读；允许字段同步 SHM 显示投影`。
- UI dirty 值只表示编辑/请求态；按钮弹窗里的“写入参数表”必须理解为写入该字段唯一 owner，不能理解为新增或继续使用 V3 自造参数文件。
- 保存按钮、即时保存、读取驱动后的表格刷新都必须以 fresh native/运行内存回读或受控 read-only 内存回读快照更新 UI；坐标、模态、速度和倍率可同步刷新允许的 SHM 显示投影。`G53_A_Y/G53_A_Z/G53_C_X/G53_C_Y` 在软键盘 `OK` 或下拉/单元格提交时就是确认保存，必须立即通过 `settings_apply` 单字段写 active runtime INI `[RTCP]` owner 并完成源位置回读；它们不得落入 `settings_runtime.json`，也不得把 UI dirty 值当运行态真值。A 中心没有 X 设置值，C 中心没有 Z 设置值，`G53_A_X` / `G53_C_Z` 不得进入 dirty 或保存链路。旧 `settings_runtime.json`、`machine_settings.json`、默认值、`/run` 结果或 dirty 值不得冒充当前生效参数。

设置页右上角小“机械坐标/MCS”面板是当前状态显示，不是参数表字段。它必须和主页机械坐标大字使用同一个 UI 显示域：基础来自同一帧 SHM `status->mcs`，A/C 在 `rotary_phase` 有效时显示同一等效角。不得在设置页改用未套用 `rotary_phase` 的原始 `mcs`，否则 A/C 会出现设置页和主页读数不一致。数值字体和布局也必须和主页一致：使用同一套七段数码字画布绘制函数和固定宽度数值区域，设置页不得对坐标值使用普通 label 字体，避免数值变化时宽度和位置跳动。

上使能不是危险动作。设置页任一动作需要上使能时，必须由后端自动调用正式 product action，并以 microkernel/native enable gate 的返回和原生回读作为结果；不得要求操作员手动上使能，也不得把上使能做成危险确认弹窗。V3 不再用解释器 idle、速度容差、queued MDI 或 SHM/JSON 状态拼上层门禁。只有转动轴相关动作按危险动作处理：A/C 旋转轴设零、A/C 轴坐标写入、会导致旋转轴实际运动或相位重定的动作必须二次确认；X/Y/Z 直线轴设零、普通保存、读取、下载和上使能本身都不是危险动作。

显式急停按钮不属于上述写驱动安全前置表。急停只能走 Broker 到 native/microkernel 急停 gate/helper 的薄适配，成功以 microkernel/native 回读确认；不得把设置页驱动写入前置、EtherCAT `0x6041` 状态字扫描或 V3 私有 `estop_loop` 逻辑复用成急停成功真源。

`复位驱动` 是长耗时写入动作，等待弹窗不能只显示静态“正在执行”。按钮触发后必须保持弹窗打开，并按阶段刷新进度：提交复位请求、扫描从站并匹配驱动资料、请求 native drive-write safety gate、恢复厂家默认参数、清除恢复后的报警并读回驱动参数、等待后端完成复位与读回校验。复位驱动必须使用专用长流程 job 等待上限，不能和普通驱动写入共用短超时；后台仍在恢复出厂或读回校验时，UI 不得先报 `DRIVE_WRITE_JOB_TIMEOUT`。最后一阶段不能伪装成已经在刷新轴参数表；只有 Broker 返回成功并完成读回写入后，才允许刷新设置页表格。最终成功或失败弹窗仍以 Broker 返回码和读回校验结果为准。

`服务器下载` 和 `读取驱动` 虽然不是写驱动参数的高风险动作，也必须使用和长流程按钮一致的阶段反馈。按钮触发后弹窗保持打开，先显示请求已提交；随后分别从 `drive_profile_server_download_progress.json` 和 `drive_read_progress.json` 刷新真实后端阶段。阶段文本必须显示 `进度 n/m` 和 `耗时 x 秒`；失败时同一弹窗显示 `code/message_cn/failed_stage`，不得长时间停在固定“正在从服务器下载驱动映射表，等待本机授权校验”或只显示最终“读取驱动完成”。

`设置驱动` 也是长耗时写入动作，等待弹窗必须显示后端实际阶段进度：提交请求、扫描/匹配目标轴、逐轴恢复出厂、逐轴计算电子齿轮、逐轴写入电子齿轮和模式、逐轴读回校验、最终读回和运行参数刷新。该进度来自 `drive_set_progress.json`，不是 UI 本地按时间推算；若进度文件暂未出现，弹窗只能显示正在等待后端阶段进度。`drive_set_parameters` 必须使用专用长流程 job 等待上限，避免后台已返回 `DRIVE_SET_OK` 时 UI 先报 `DRIVE_WRITE_JOB_TIMEOUT`。后台最终返回 `DRIVE_SET_OK` 且轴参数表显示 `已写入` 后，右上角按钮必须切换为 `保存并重启`，不能保持 `返回`。

驱动长任务一旦进入 `BROKER_JOB_ACCEPTED` 异步模式，UI 主线程只允许按短周期提交 `job_status` 快查询并刷新弹窗，不能同步等待最终写入结果。`BROKER_JOB_RUNNING` 表示通道正常并清零连续失败计数；`JOB_STATUS_BROKER_SOCKET_UNAVAILABLE`、`JOB_STATUS_BROKER_DIRECT_FAIL`、handler 返回失败、状态结果无法解析或缺失状态码都计入通信失败。连续 5 次失败后，UI 必须结束当前异步任务并显示“后台通道连续不可用”的结构化失败，不得继续等到长超时才让操作员看到失败。若 `job_status` 返回业务最终失败码，则保持原始业务 `code/message/detail`，不要改写成通道丢失。

Broker 异步长任务必须携带 job ownership 元数据：`job_id`、`job_action`、`state`、`started_ns/submitted_ns/completed_ns`、`deadline_ns`、`cancel_requested`、`cancel_reason` 和 `final_result_owner`。取消只能通过正式 `job_cancel` 请求进入，先把任务置为 `cancelling`，由 worker 短周期检查 cancel token/deadline 后进入 `cancelled` 或 `failed_closed`；UI 不得本地伪造取消完成。取消、超时或异常后的最终 `job_status` 仍必须保留原始 `job_id` 和 `final_result_owner`，防止后台结果丢失或串到其它请求。

## 3. 容易写错的点

1. `读取驱动` 走 `drive_parameter_read`；禁止 `settings_read_only` 产品路径作为设置页按钮、Broker route、UI 刷新兜底或 `/run` 读回入口。
2. `下载授权` 当前不在顶部；它在 y=236 的驱动动作行，且该行动作按钮必须保持 82px 宽的单行文字，不允许再拆成两行。
3. `复位驱动`、`清除故障`、`设置驱动` 当前 UI 发起写类请求并带 action confirm 与 `confirm_drive_write_one_shot=true`；Broker 只接受该精确 UI 单次确认或显式全局写门禁。`复位驱动`、`设置驱动` 是长流程，不再把 UI 请求里的旧 `status_epoch` 当成访问驱动前的门禁；安全性由执行时重新读取微内核/native 内存状态、请求身份和映射表对象支持情况保证。`设置驱动` 的写类请求边界只引用 `功能/4设置页设置驱动按钮作用和目的.md`。
4. `设置驱动` 的按钮功能已剥离出本文；详见 `功能/4设置页设置驱动按钮作用和目的.md`。
5. `复位驱动` 就是恢复驱动厂家默认参数，必须走 `drive.restore_factory_defaults`，并在弹窗中写清会清除驱动内部参数、恢复后需要单独重新写电子齿轮和重新设0/验证；不能再把它写成清故障。复位驱动只做驱动复位和回读校验，读回结果只保存为设置驱动私域证据；成功后右上角进入 `保存并返回`，只用于统一保存/返回，不触发 LinuxCNC/backend 重启。
6. `清除故障` 只是清除驱动报警；它和 `设置驱动` 的边界详见 `功能/4设置页设置驱动按钮作用和目的.md`。
7. 轴参数表任意可编辑字段在软键盘 `OK` 或下拉选择变化后会使用 `source=lvgl_v3_settings_axis_param_immediate` 静默调用 `settings_apply`，成功只代表本字段唯一 owner 已按受控路径写入，并只完成本字段源位置回读；需要 backend restart 的运行内存/允许的 SHM 显示投影刷新仍等右上角保存并重启；若字段需要 backend restart，仍保留 restart pending，不等待系统重启即可继续改其它字段。除 `设置驱动` 的驱动私有域外，这不表示 V3 参数表成为长期真源，最终仍以 LinuxCNC/native 内存/API 回读为准。
8. 顶部 `返回/保存并重启` 正常返回路径使用 `source=lvgl_v3_settings_return`；`设置驱动` 前为了计算齿轮比而保存 dirty 字段时才使用 `source=lvgl_v3_settings_before_drive_set`。
9. `保存并重启` 的前台完成边界是字段已逐项源回读确认、restart precheck 通过、backend + UI restart handoff 已发起并能恢复主页面；它不再在按钮响应里同步等待完整 native 状态恢复、machine-on 或驱动 Operation Enabled。后端重启、UI restart、新 UI 进入后上使能和运行态恢复仍由后台流程和 native/运行内存真值链路继续完成。
10. 如果后台重启恢复阶段后续发现 `status_shm_stale`、`STATE_PUBLISHER_RESTARTING`、machine on 或 BUS/EtherCAT Operation Enabled 未恢复，必须保留诊断状态，不能把这些后台状态冒充为已恢复运行态；但它们不再阻断右上角按钮的轻量 handoff 返回主页面。settings 页轻量保存重启必须在新 UI 恢复/进入主页面后通过正式 `estop_or_reset` 产品动作发起上使能；后续运行准入仍以独立 native 真值和运动门禁判断。
10. `settings_save_return` 无 dirty 且无 restart pending 时只是本地返回主页，不发 `settings_apply`。
11. 从轴、S轴、刀库三个按钮当前都是 blocked，不是待验收的真实功能。
12. `settings_save_return` 的进度弹窗是操作员在等待后端期间的可见反馈，必须持续刷新阶段文本；失败必须在弹窗里显示结果，顶部黄色字不能替代弹窗内容；restart handoff 成功按当前代码关闭进度窗口、返回主页面，并用状态消息提示后端和界面正在重启。

## 2026-06-21 轴设0按钮语义修订

设置页每个轴行上的“设0”按钮是单轴按钮，不是全局重启或全轴设零入口。

点击某个轴的“设0”时，UI -> Broker -> product action 只处理该轴：
- 只采集该轴驱动当前位置并写入该轴 `zero_model.zero_counts`、绝对/拆分计数和读写证据；不得恢复或写入顶层 `旧浮点零位字段`、`zero_model.旧浮点零位字段`、`zero_model.旧倒数比例字段` 作为产品输入。
- 不扫描全轴、不构建全轴 target、不改其它轴参数。
- 不在按钮动作中自动重启 backend/LinuxCNC、不执行 runtime rebase；BUS 设0本次动作只证明零点证据和 hard/raw limit owner 写入，UI 等效坐标清零只能作为最后统一保存重启后的显示闭环，不作为本次按钮通过门禁。
- 成功后设置页保持可操作，用户可以继续点其它轴设0或修改其它设置。
- 右上角按钮保持/切换为“保存并重启”，最后一次性执行统一生效和返回。

设置页 `设0` 的 UI 安全包装必须按驱动模式分流，但不能绕过 `功能/4轴设0软件偏置方案.md` 的产品语义：
- BUS/EtherCAT 或绝对总线模式下，X/Y/Z 直线轴点击单轴 `设0` 后直接发送本次单轴 `settings_axis_zero` / `bus_count_domain_zero` 请求，不弹危险确认；若后端需要上使能，必须自动上使能并用 fresh microkernel/native 回读证明。A/C 转动轴点击单轴 `设0` 后必须先显示二次危险确认弹窗，只有操作员在该弹窗内确认后，UI 才能组包发送请求；取消、关闭弹窗或触摸穿透都不得发送 Broker 请求。确认弹窗只授权本次 A/C 单轴请求，不得作为全局写门禁复用到其它按钮或后续点击。
- Pulse/StepDir 模式下，点击轴参数表单轴 `设零` 仍打开 `HOME_OFFSET` 软键盘；键盘不得用 SHM/MCS/`rotary_phase` 显示投影自动种子化输入，只能显示当前字段 owner/readback 初值或空值并等待操作员输入。Pulse 轴参数表另有 `回零方向` 列，通过下拉选择回零方向并写 `home_direction` owner；`设零` 键盘本身不得静默启用或提交 `home_direction=current`，X/Y/Z 也不得提交 `current`。
- A/C 转动轴 `设0` 的确认弹窗和 Pulse 软键盘都必须使用 modal blocked mask 或等效全屏输入吞噬机制；弹窗/键盘关闭时继续使用短输入过渡保护，避免同一次触摸释放落到底层轴表、右上角按钮或 y=236 驱动动作行。

BUS `设0` 成功结果代表当前轴零点证据已落盘、active raw 限位已同步回读；live raw 限位装载和 UI 等效坐标归零要在最后统一保存重启后闭合，不能把 SHM 显示投影写成本次动作成功证明。Pulse `设零` 成功边界按 Pulse owner 的 `HOME_OFFSET` 写入和后续回零/native 证明处理。
