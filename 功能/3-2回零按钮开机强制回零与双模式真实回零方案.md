## 2. 回零模式

主页面 `回零` 不再经 Broker/Python/UI 层的任何位移规划或 jog 适配中介。C UI 只判断当前选择是 `机械全轴` 还是单轴：`机械全轴` 只提交 `ALL_HOME` 意图，不提交模型、轴列表、从站、次序、目标或容差；单轴只提交 `SINGLE_AXIS_HOME`、当前选中轴和选中坐标域。active 运动模型、可执行轴、axis-slave-joint、`HOME_SEQUENCE`、零点和等效旋转角全部由微内核/native resident owner 自己裁决。UI 不计算 delta、不组装替代 G-code、不修改坐标或 offset。

选择分支优先于 BUS/Pulse 模式分支：

| 当前选择 | `回零` 唯一动作 | 成功回读 |
| --- | --- | --- |
| MCS 全轴（`机械全轴`） | 只提交 `ALL_HOME`，由微内核/native Home owner 自行选择并执行 BUS/Pulse 正常 Home | 本次 native Home transaction 和 fresh `homed/all_homed`/结构化结果 |
| MCS 单轴 | 所选轴真实移动到 MCS 0；旋转轴使用 active model 对应轴的最近等效机械 0 | 所选轴 fresh MCS 为 0、实际位置变化/到位 token；不改变 homed |
| WCS 单轴 | 所选轴真实移动到当前 active WCS 0 | 所选轴 fresh WCS 为 0、对应 fresh MCS 实际变化/到位 token、WCS offset 未改变；不改变 homed |

以下 BUS/Pulse 表只适用于 `机械全轴` 分支：

**回零的实际操作完全由微内核根据当前的“脉冲/总线”模式在底层直接分支并控制执行，UI 层与 Broker 绝不参与轨迹计算与多轴时序控制：**

| 模式 | 微内核执行语义 | 成功边界 |
| --- | --- | --- |
| **Pulse/StepDir** | `ALL_HOME` 交给微内核/LinuxCNC/HAL 原生 Home owner；是否可运行、轴集合、`HOME_SEQUENCE`、方向、开关/latch、速度和 `HOME_OFFSET` 都由该 owner 从 resident 配置裁决。 | native owner 返回本次结构化结果与 fresh `homed/all_homed`。 |
| **BUS/EtherCAT** | `ALL_HOME` 交给微内核/EtherCAT Home owner；active model、axis-slave-joint、回零次序、零点、等效角、安全、实际运动和到位判定全部在该 owner 内完成。V5 UI/Command Gate 适配层不得复制这些算法。 | native owner 返回本次结构化结果与 fresh `homed/all_homed`；UI 不用坐标显示或旧结果冒充成功。 |

### 核心硬边界：
1. **机械全轴只有一个上层动作**：UI/IPC/Command Gate 只把 `ALL_HOME` 交给 native Home owner；不得保留 proof move、backoff、remainder、jog、目标/count、容差、到位或 `HOME_SEQUENCE` 执行代码。
2. **单轴定位独立保留**：MCS/WCS 单轴继续由 `native_axis_zero_position` owner 执行，保留轴、坐标域、wcheckpoint 和旋转轴等效 0 所需逻辑；不得与机械全轴实现合并或删除。
3. **回零状态单一源回读**：机械全轴分支的执行进度与最终 `homed/all_homed`，以及 MCS/WCS 单轴分支的实际位置、到位和 offset 未变证明，都只允许从微内核/native 内存/API 直连回读确认，严禁依赖任何外部 `/run` 临时 JSON 文件或共享内存（SHM）的第三方状态写入。
4. **每次按键都是真实运动事务**：`机械全轴` 每次按键都必须产生新的 native Home transaction；MCS/WCS 单轴每次按键都必须产生新的 native positioning transaction。三种分支都要有本次运动/到位证明；`already_homed`、`position_zero`、`target_delta_zero`、`zero_model_present` 只能作为诊断上下文，不能成为 UI 本地空跑成功条件。
5. **微内核是唯一全轴决策 owner**：active mode/model、轴、从站、joint、`HOME_SEQUENCE`、零点、等效角、RTCP、安全、运动和到位均由微内核/native owner 自行读取 resident 状态并裁决；适配层只透传请求和结果。
6. **回零次序只取轴参数**：微内核/native Home owner 在本次 transaction 内直接读取每个 active joint 的 resident `HOME_SEQUENCE`，按数值分阶段、同值成组执行；不按轴名、joint 下标、从站位置或编译时常量排序。UI 和适配层不携带、不缓存、不重排该参数。
7. **完成不能靠改坐标或二次容差门禁**：owner 必须先冻结 exact target 并完整下发对应 trajectory；只有本次 count 确实离开 leg start、native planner 已结束、joint `in-position=true`、fresh actual/count 连续稳定，才允许把该轴计入 `physical_complete_mask`，全轴完成后再原子置本次 `homed`。`target/error` 只作诊断，不在 planner 完成后再次决定成功。Home 结束、失败或取消路径均不得写 `pos_cmd`、`pos_fb`、`free_tp.curr_pos`、`motor_offset`、encoder/raw count、WCS/G92 或显示坐标；未取得运动、planner、in-position、稳定或 generation 证明时必须结构化失败并保留最后 fresh actual。
8. **禁用不是负序号**：负 `HOME_SEQUENCE` 保留 LinuxCNC 同组同步语义；设置页“禁用”统一保存/回读为 `999`，全轴 Home 的本次 run mask 只含 `ABS(H[j].home_sequence) <= 100` 的 active joint。禁用轴仍可存在于机器映射中，但不得被 Home All 启动，也不得令其它轴的 Home 映射校验失败。

## 3. 模式判定

模式判定 owner 不是 UI 下拉框或设置页文字。

BUS/Pulse 模式识别、模式切换闭环和 Pulse 设零/回零总口径以 `REQ-BUS-PULSE-MODE-DIFFERENCE` / `功能/2-1总线脉冲区别.md` 为准；本节只定义主页面 `回零` 如何按 active mode 分支执行。

规则：

- `ethercat` / `bus` 归一为 BUS。
- `pulse` / `stepdir` / `step_dir` / `step/dir` 归一为 Pulse。
- `ALL_HOME` 请求不携带模式；active mode 及其可执行性由微内核/native owner 裁决并返回结果。
- UI/Command Gate 不因模式、模型、轴或从站状态本地吞掉 `ALL_HOME`，也不在适配层切换到其它运动实现。
- 旧 `run_mode_bus_pulse` 字段退役，不能作为 UI dirty、显示入口、兼容字段或当前模式真源。

## 4. 执行入口

`回零` 可能持续数十秒，但 UI 不再把机械全轴 Home 或单轴到 0 定位投递到 Broker `home_precheck` async job。入口只负责按当前选择提交 `ALL_HOME` 或 `SINGLE_AXIS_HOME`；执行进度、全轴 `homed/all_homed`、单轴 MCS/WCS 到 0 readback 和失败原因由对应 native owner 回读并由 UI 显示。机械全轴请求中不得出现 mode/model、轴列表、从站、joint、`HOME_SEQUENCE`、目标、速度、容差或 proof 字段；单轴请求保留轴和坐标域。

执行方式：

```text
C UI 回零选择意图
  -> 机械全轴：ALL_HOME -> LinuxCNC/HAL/EtherCAT/native Home owner
  -> MCS/WCS 单轴：SINGLE_AXIS_HOME + axis + space -> native positioning owner
  -> native/microkernel/HAL/EtherCAT actual readback
  -> UI 状态刷新显示 homed/all_homed、所选轴到 0 或失败原因
```

要求：

- `ALL_HOME` 适配层只提交一次 native 全轴 Home 请求并转发本次结构化结果；不得发 `Set Jog_Incr`、`Set Jog_Stop`、MDI/G-code、分轴 `Set Home`，不得自行执行 proof/backoff/归零/余量追赶。
- 全轴 Home 的模型、映射、次序、等效角、目标、实际运动和完成判定只存在于微内核/native owner；适配层只使用该 owner 的本次 post-trigger `homed/all_homed` 或等价结果作回读，不重算到位。
- 微内核/native Home owner 必须直接使用启动时已进入 resident joint 参数的 `HOME_SEQUENCE` 构造本次阶段；设置页参数只经唯一保存/受控重启链进入该参数，不得在 Home 请求或适配层再次投影顺序。
- native owner 必须先完整下发 frozen exact target 对应的物理位移，再用同一映射的 fresh actual/count 证明 count 已变化、planner 已结束、joint in-position 且连续稳定；驱动未产生真实 movement、planner 未结束、in-position 未成立、fresh count 未稳定或 generation 不一致只能失败。`target/error` 保留为诊断，不得恢复第二 target/tolerance 成功门禁，也不能通过终态坐标同步、当前位置置零、修改 offset 或提前置 `homed` 掩盖。
- 单轴 MCS/WCS positioning owner 继续保留现有目标、wcheckpoint、等效角和到位逻辑；该逻辑不得被全轴 `ALL_HOME` 调用。
- UI 坐标显示必须实时刷新，不等 Home 完成后才刷新。
- E-stop、Jog stop 等紧急通道不得被 Home 请求或状态刷新阻塞。
- 旧 `home_precheck` async job、`job_status` Home 轮询和 Python product action 结果不得恢复为普通 Home 执行路径。
- 命令已发送只用于触发证据；最终状态以 LinuxCNC/HAL/native 回读为准，SHM 只可作为 UI 坐标/轨迹、转速、进给速度和主轴/进给倍率显示证据；当前模态直连 native modal/tool 状态。

## 4.1 当前机械全轴 Home 的 canonical 实现

本节记录当前 Windows canonical worktree 中已经按原始 UI 路径上板验证的 BUS 机械全轴 Home。后续修改必须先对照本节和列出的唯一 owner；不得根据旧聊天、旧截图、旧 adapter、旧 smoke 或某一台机器的轴名顺序重建第二套 Home。

### 4.1.1 适用范围和唯一控制链

当前产品运行态为 BUS/EtherCAT、五个 active logical joint；五是当前板级 ABI 的 joint 数量，不表示轴名、从站号或次序固定。轴名可以随 active model 变化，axis→joint→slave 和 `HOME_SEQUENCE` 都由 resident 参数决定。Pulse 仍是 `cold_staged_not_runtime_selectable`，不属于本节已验证实现。

```text
LVGL 主页面“机械全轴” + “回零”
  -> V5_COMMAND_HOME（无 axis/model/slave/order/target/tolerance）
  -> v5_main_page_home_transaction 异步 run_id + generation
  -> framed UDS EXECUTE
  -> Command Gate begin_home_runtime
  -> 准备 LinuxCNC 上下文且保留已打开程序 identity
  -> linuxcncrsh: Set Home -1
  -> [EMCMOT] HOMEMOD=v5_bus_homecomp
  -> LinuxCNC free_tp native trajectory
  -> v5_bus_axis_router zero-relative count-domain
  -> CIA402 / EtherCAT physical slave
  -> native HAL HOME_STATUS / PROBE_HOME_STATUS
  -> UI 顶部进度、终态和 fresh 坐标显示
```

UI 只做分支选择和异步显示：机械全轴生成 `V5_COMMAND_HOME`；MCS/WCS 单轴生成 `V5_COMMAND_AXIS_ZERO_POSITION + axis + space`。两条请求不能互相 fallback。UI 在 worker 启动前生成 `run_id/generation`，执行线程发送请求，独立进度线程约每 100 ms 读取同一身份的 `PROBE_HOME_STATUS`；旧线程遇到下一 generation 必须退出。终态首次探测失败会有界重试，持续缺失时产生 `HOME_TERMINAL_STATUS_MISSING` 合同错误，不能静默成功。

Home 运行不阻塞 LVGL 线程。fresh `COMPLETE/FAILED/CANCELLED` 到达时，同一轮必须清除 `active/pending/pressed` 和按钮黄色。执行中急停还要立即清按下视觉，但保留匹配 worker 直至消费 native `CANCELLED`，避免旧 `PREPARING` 串入下一事务。

### 4.1.2 启动期 resident 参数和原子映射

“Home 请求不带映射”不等于系统没有读取映射。当前实现只在 Command Gate 启动/受控 reload 时预载一次 canonical 参数，Home 热路径只消费 resident 快照：

- `[TRAJ].COORDINATES` 和 LinuxCNC joint 参数建立 active axis、status slot、`HOME_SEQUENCE`、joint 速度/加速度和限位。
- 设置轴参数 owner 提供每个 axis 的实际 slave position；不得按轴名或 joint 下标推导从站。
- `settings_runtime.zero_model` 只提供 BUS 私域的 `zero_counts`、`counts_per_unit`、`raw_zero_position` 和对应 slave 证据。
- 当前 BUS owner 要求五个 active status slot 与五个 slave position 完整且互不重复；缺项、重复、越界或 `NAT` 均 fail-closed。
- `mapping_generation` 由 status slot、axis code 和 slave position 生成；每个 joint 的 `config_valid/status_slot/axis_code/slave_position/mapping_generation/zero_counts/counts_per_unit` 先 staged，最后一个 record 才推进非零 `commit_seq`。
- native table、`v5_bus_axis_router` 和 `v5_bus_homecomp` 必须回读相同的 `valid/generation/active_mask/commit_seq`，且 router 未拒绝本次 commit，才允许 Home。

`HOME_SEQUENCE` 不复制进 Command Gate 自建表；它由 `v5_bus_homecomp` 在本次事务内直接读取 `H[j].home_sequence`。有效运行值是 `[-100,100]`，分组使用绝对值；负数保留 LinuxCNC 同步组语义。大于 `100` 表示不参加 Home All，设置页 canonical “禁用”值是 `999`。当前某台机器的 Z/X/Y/A/C 顺序只是一组参数实例，不是算法。

### 4.1.3 唯一 count-domain 和零锚公式

`v5_bus_axis_router` 在 servo thread 中持续执行唯一的物理/逻辑转换：

```text
logical_joint_actual = signed32(physical_slave_actual - zero_anchor_counts)
physical_slave_target = signed32(logical_joint_target + zero_anchor_counts)
```

因此 LinuxCNC、Home owner 和 wcheckpoint 看到的 joint count 已是 zero-relative；`zero_anchor_counts` 只能在 router 反馈方向减一次、命令方向加一次。Home 终态、State Publisher、UI、wcheckpoint 或其它 adapter 再减一次都会把物理零锚错误显示成固定角差。

CIA402 将 LinuxCNC position command 转换为 drive count 时必须做对称的 nearest-integer rounding：

```text
scaled = position_command * position_scale
drive_target_count = scaled >= 0 ? int32(scaled + 0.5) : int32(scaled - 0.5)
```

禁止恢复 C 强制转换的向零截断；负数整 count 边界若因浮点尾差落在相邻 ULP，截断会稳定少发一 count。

旋转轴显示直接对 zero-relative wcheckpoint logical count 归一化：

```text
counts_per_rev = abs(counts_per_unit) * 360
phase_counts = positive_mod(logical_counts, counts_per_rev)
display_degrees = phase_counts / abs(counts_per_unit)
```

State Publisher 不再减 `zero_anchor_counts`，也不把 359.99° UI snap 成 0°。一 count 固定残差必须修 router、整数转换或投影根因，不能加显示补偿。

### 4.1.4 Command Gate 薄适配和程序上下文

Command Gate 在 power-on Home 前置检查之前先为本次 `run_id/generation` 建立 runtime state；任何 early return 都要 finish 同一事务。`PROBE_HOME_STATUS` 在 `linuxcncrsh` 长事务互斥之外处理，因此 Home 执行时 UI 仍能读进度，native 急停也不能被 Home 锁阻塞。

发送 `Set Home -1` 前，`v5_linuxcncrsh_prepare_home_entry()` 执行以下有界准备：

1. 读取 task mode、program status 和已打开 program identity。
2. 若程序仍在执行或暂停，发送 `Set Abort` 并等待解释器 idle；这只结束当前执行事务，不清除已打开 G-code。
3. 必要时切 `Manual`。
4. 关闭 Teleop，确认 joint mode。
5. 每一步都复核 program identity 未改变；变化立即失败，禁止 Home 后要求用户重新打开程序来掩盖上下文破坏。

随后只发送一次 `Set Home -1`。`v5_native_home.c` 先记录 native transaction baseline，再以 50 ms 周期读取 native `HOME_STATUS`，最多 2400 次（约 120 s）。它只接受 baseline 之后的新 transaction；第一条匹配事务的 native failure/cancelled 已是终态，不等待 sibling axis 的 aggregate mask 继续收敛。适配层不计算轴、次序、目标、距离、容差或到位。

### 4.1.5 Realtime Home transaction 状态机

当前 BUS 全轴运动 owner 是 `linuxcnc/src/hal/components/v5_bus_homecomp.comp`；`board/linuxcnc/ini/v5_bus.ini` 通过 `[EMCMOT] HOMEMOD = v5_bus_homecomp` 装载。servo period当前为2ms；初始1ms迁移必须同步转换本节登记的cycle-based常量，不能让Home墙钟语义减半。`do_home_joint(-1)` 新建全轴 transaction；直接 `do_home_joint(j)` 在该 HOMEMOD 中明确返回 `V5_SINGLE_HOME_UNSUPPORTED`，因为 MCS/WCS 单轴由独立 positioning owner 执行。

一次全轴 Home 的顺序固定为：

1. **Snapshot**：原子读取 mapping table、run mask、generation 和 commit；run mask 只含 active 且 `HOME_SEQUENCE` 有效的 joint。
2. **PRECHECK**：检查 fresh safety actual、急停已取消、machine enabled、mapping/table/router 一致、axis/slave 唯一、exact integer count 可读、drive 无 fault/error、所有参与轴 `free_tp` 未运行且 `in-position=true`。当前2ms连续3个servo周期、约6ms稳定才继续；初始1ms固定改为连续6周期，仍约6ms。
3. **RTCP force-off**：以本次 native transaction id 提交 `force_off`；必须取得相同 transaction 的 latch ACK、fresh `rtcp_valid=true`、`rtcp_active=0`。等待上限当前2ms为500 servo周期、约1s；初始1ms固定改为1000周期，仍约1s。Home 成功、失败或取消后都保持 OFF，不自动恢复旧 ON。
4. **再次 PRECHECK**：RTCP OFF 后按同一墙钟语义再次复核安全、映射、静止和fresh counts；当前2ms为3周期，初始1ms为6周期，避免关闭RTCP期间状态漂移。
5. **Freeze plan**：一次性冻结本次所有参与轴的 fresh logical start、fixed logical target、delta 和 wcheckpoint generation；冻结计划全部复核成功前不开任何运动副作用。
6. **Arm Home All**：清本次参与轴的旧 `homed`，只在计划已冻结后调用 LinuxCNC base Home-All 建立 `HOME_SEQUENCE` 状态机。
7. **按序分组运动**：每组由 resident `HOME_SEQUENCE` 选择；同绝对值为一组。组内所有成员必须先各取得满足同一墙钟稳定语义的fresh、静止、in-position、同generation样本，随后才整体release；当前2ms为3个样本，初始1ms为6个样本。只启动组内部分轴视为`SEQUENCE_NOT_STARTED`。
8. **每轴真实运动**：使用 LinuxCNC `free_tp` 下发完整 native trajectory；逐周期检查 safety、mapping、RTCP latch、drive、limit、generation 和运动状态。
9. **全轴终态**：所有组结束后，逐轴再次读取fresh actual/generation，要求planner停止、in-position且连续稳定；终态稳定同样从当前2ms的3周期改为初始1ms的6周期。只有`physical_complete_mask == run_mask`、`movement_mask`覆盖全部run mask、全轴终态稳定条件同时满足，才在一个realtime owner中原子设置本次参与轴`H[j].homed=1`并发布completed/terminal mask。

1ms直接常量裁决：`V5_HOME_STABLE_CYCLES`从`3u`改为`6u`，覆盖首次PRECHECK、RTCP后PRECHECK、分组release和terminal稳定；`V5_HOME_WAIT_CYCLES`从`500u`改为`1000u`，覆盖moving precheck、RTCP ACK、sequence start和terminal readback等待。运动距离timeout继续由`ceil(seconds/servo_period)`换算，必须用focused state smoke证明它在1ms下自动得到相同秒数，禁止再手工乘2造成双重放大。

### 4.1.6 线性轴和旋转轴目标公式

所有计算使用 overflow-checked integer count；容差不参与 command delta。

线性轴：

```text
logical_target_counts = 0
delta_counts = logical_target_counts - fresh_logical_start_counts
```

旋转轴：

```text
counts_per_rev = exact_integer(abs(counts_per_unit) * 360)
phase = positive_mod(fresh_logical_start_counts, counts_per_rev)
delta = phase <= counts_per_rev - phase ? -phase : counts_per_rev - phase
logical_target_counts = fresh_logical_start_counts + delta
```

这会选择距离当前 logical count 最近的整圈等效零；恰好半圈时固定选择负方向，不能由浮点尾差随机换向。旋转轴 target 可以是 `0`、`-27000000` 或其它整圈 count，只要等效相位为 0；不要求清除驱动多圈绝对计数。

如果 fresh `delta_counts == 0`，本次 Home 也不能用旧零位或旧 homed 空跑成功。owner 根据软限位选择允许方向，先真实走 `abs(counts_per_unit)` counts（一个轴单位；线性通常 1 mm、旋转轴 1°），再真实返回 frozen zero target。正向可行时优先正向，否则使用负向；两边都不允许则 limit failure。

每条 motion leg 的 LinuxCNC 命令为：

```text
target_position = joint.pos_fb + delta_counts / counts_per_unit
max_velocity = joint.vel_limit
max_acceleration = joint.acc_limit
```

速度和加速度来自每轴 resident LinuxCNC 参数，即设置链进入的最大速度/最大加速度，不使用 UI 常量、旧 `HOME_SEARCH_VEL` 或统一固定速度。动态 timeout 由距离、该轴最大速度、加速度和 servo period 计算：

```text
seconds = distance/max_velocity + 2*max_velocity/max_acceleration + 1s stop margin
timeout_cycles = ceil(seconds/servo_period)
```

软限位仅在 `max_pos_limit > min_pos_limit` 时约束 target；上下限都为 0 表示该轴软限位不生效，不得把 0 误解成只允许坐标 0。

### 4.1.7 wcheckpoint generation 变化

线性轴 current count 直接来自 router logical actual，coordinate generation 为 0。旋转轴每个 fresh sample 使用：

```text
runtime_counts = router zero-relative joint actual
logical_counts = wcheckpoint_base_counts + runtime_counts
```

heartbeat 只表示 freshness，绝不能冒充 wcheckpoint coordinate generation。旋转轴运动中若 fresh wcheckpoint generation/base 增加，owner 先停止当前 `free_tp` leg 并等待 in-position；`logical_target_counts` 保持冻结不变，只计算：

```text
runtime_target_counts = logical_target_counts - new_base_counts
remaining_counts = runtime_target_counts - fresh_runtime_counts
```

随后按剩余量重新启动同一事务。禁止重新选择另一个整圈目标、缩短 logical 距离或把新 base 当新 Home 起点。

### 4.1.8 当前唯一完成条件：没有第二 target/tolerance 门禁

当前全轴 Home 有意删除 planner 完成后的 `abs(actual-target) <= tolerance` 第二成功门禁，也不存在余差补偿 leg。`target_counts/error_counts` 继续发布，但只用于进度、故障定位和证明命令目标，没有成功裁决权。

每个 motion leg 完成必须同时满足：

- 非零 leg 的 runtime count 确实离开本 leg start；“命令已发送”或“进入容差”不算运动证明。
- `joint->free_tp.active == false`，即 native planner 已结束。
- `GET_JOINT_INPOS_FLAG(joint) == true`。
- fresh actual count 连续三个样本相同；运动仍 active 或 actual 仍变化时绝不能完成。

每轴 leg 完成时只写 `physical_complete_mask`，该轴 `homed` 仍保持 0。全轴 sequence 完成后还要进行一次全体 fresh terminal readback，并再次要求 planner off、in-position、同 generation、连续稳定；最后才一次性原子置全部 run axes `homed=1`。

这条规则不允许容差截短完整 command delta、少发最后一 count、提前停止或把微小位移当作没运动。若 A/C 出现固定一 count或固定角差，必须修 integer command、router anchor、wcheckpoint 或显示投影根因；不得恢复 Home target tolerance、补偿循环或 UI snap。

### 4.1.9 失败、急停取消和状态 ABI

运行中每个 servo 周期都重新检查 safety actual、急停、machine enabled、mapping generation/commit、相同 transaction 的 RTCP OFF latch、drive fault/error 和 limit。失败时：

- 禁用本次参与轴 `free_tp`，终止 sequence，清本次 `homing/homed/joint_in_sequence`。
- 保留真实坐标、encoder/raw count、WCS/G92、zero anchor 和 wcheckpoint；不做“回滚到零”或坐标改写。
- 发布 transaction、run/active/completed/terminal mask、failed joint、current axis mask、phase、direct failure、actual/target/error counts、generation、readback-valid 和 motion-active。

初始请求时已经急停属于 `FAILED/HOME_PRECONDITION_ESTOP`；已进入本次 Home 后操作员急停属于 `CANCELLED`。native E-stop 通道位于 `linuxcncrsh` Home 互斥之外，可立即停止运动；UI 只显示“回零已取消”，不弹失败窗。任何 terminal 都必须结束按键 active/黄色，不能用弹窗是否关闭决定事务寿命。

微内核运动、安全、mapping、sequence 错误由 native owner 触发。薄适配层仍可准确报告 IPC、解释器上下文、program identity、事务被替换或终态状态缺失等传输/合同错误，但不得重新计算运动失败或用 adapter 泛化错误覆盖 native 第一条 direct failure。

### 4.1.10 已验证基线和排错顺序

2026-07-16 的 AC active model 原始 UI 实机基线已验证：机械全轴 Home 显示“回零完成”，X/Y/Z/A/C MCS 均为 `0.000`，五轴跟随误差均为 `0.000`，按钮退出黄色，随后恢复急停安全态。该证据只覆盖本次 AC/BUS 配置，不自动扩展为 BC、Pulse 或任意 `HOME_SEQUENCE` 重排已板端通过。

以后出现“轴不动、顺序不对、Home 卡住、旋转轴 359.99°、按钮一直黄色”时，必须按以下顺序读取同一 transaction，禁止先猜 UI 或调容差：

1. UI `run_id/generation`、是否只有一个 active Home、fresh terminal 是否到达。
2. native `transaction/run_active_mask/completed_mask/terminal_mask/current_mask/failure_code/failure_phase`。
3. config/router 的 `valid/generation/active_mask/commit_seq/rejected_commit_seq` 是否完全一致。
4. 本 joint 的 axis code、status slot、实际 slave position、`HOME_SEQUENCE`、zero anchor 和 counts-per-unit。
5. router logical joint target/actual 与对应 physical slave target/actual；不能按轴名猜 slave。
6. 旋转轴 wcheckpoint `valid/generation/base/logical/runtime`，核对 `runtime=logical-base`。
7. `free_tp.active`、joint in-position、movement/physical-complete mask 和连续稳定样本。
8. 最后才看 State Publisher/UI phase；若 native logical 正确而 UI 错，检查是否重复扣 zero anchor。

当前 focused 防回归入口：

- `board/tools/deploy/v5_home_router_bridge_smoke.py`：原子 mapping/router ACK、native `HOME_SEQUENCE`、禁用 999、post-RTCP 单次 freeze、same-sequence barrier、zero-cycle、physical→terminal→atomic commit 顺序、旧 tolerance token 和终态坐标改写为零引用。
- `linuxcnc/src/hal/components/v5_bus_home_state_smoke.c`：真实 movement、planner/in-position/stable、旋转最近等效零、half-turn 负向、wcheckpoint rebase、动态顺序/同步组/禁用、zero-cycle、RTCP transaction ACK 和 all-axis atomic commit。
- `v5_command_gate_smoke`：空参数 `ALL_HOME`、唯一 `Set Home -1`、Abort/Manual/joint-mode 且 program identity 保留、同 run/generation runtime 状态。
- `v5_main_page_actions_smoke`：UI 首返不阻塞、progress、旧 generation 隔离、E-stop cancel、终态缺失 fail-closed 和按钮 active 清理。
- `board/services/state_publisher/v5_machine_status_projection_test.py`：zero-relative rotary logical 不二次扣 zero anchor。

全轴 HOMEMOD/router/HAL owner 变化的最小正式构建是 `linuxcnc-prebuilt` 单次 package DAG，并以 LinuxCNC verified package bundle 重启完整 backend；不得 SCP 单个 Home `.so`。纯 Command Gate 适配变化只构建/部署 `v5_command_gate_server` 的 `command_gate` scope；纯 UI 只构建 `v5_lvgl_shell`；纯 projection 走 `wcs` scope；CIA402 模块走登记的 `backend` scope。以上任何 focused 变更都不要求 rootfs/image，除非用户明确提出 current image 或发布目标。

## 5. 状态 freshness

`回零` 随时可按；`ALL_HOME` 不携带 UI 采样的状态证据，所有准入和运动状态由微内核/native owner 在执行点读取。

UI 不得把 SHM `status_epoch`、坐标显示投影、旧状态帧或本地缓存放入 `ALL_HOME` 请求或作为成功证明。刚重启、State Publisher 尚未写入或 SHM reader 不可用时，不影响 UI 提交 `ALL_HOME`；能否执行以及失败原因由 native owner 的 fresh actual 裁决。

Native owner 可短窗口重试其需要的 fresh resident 状态。机械全轴 owner 读不到安全、模式、模型、映射、次序或 Home actual 时自行 fail-closed 并返回结构化原因；MCS/WCS 单轴 positioning owner 读不到所选轴 fresh MCS、active WCS/offset、active model 或所需证明时自行 fail-closed。UI/Command Gate 不复制这些判断，也不得触发错误分支运动。

机械全轴 `ALL_HOME` 到达后，Command Gate 只建立本次 `run_id/generation` 并准备 LinuxCNC 解释器上下文：必要时 Abort 当前执行事务但保留已打开程序 identity，切到 Manual/joint mode，然后只发一次 `Set Home -1`。fresh 急停、Machine Enable、RTCP、active-axis stillness、驱动故障、模式、模型、映射、目标和实际运动均由 native owner 在任何轨迹下发前再次裁决；分别返回 `HOME_PRECONDITION_ESTOP`、`HOME_PRECONDITION_DISABLED` 或更精确的结构化结果。E-stop/Jog Stop 仍可在 Home 的 `linuxcncrsh` 长事务互斥之外直接抢占。

## 6. 开机回零前置条件

开机、断电重启、受控 clean restart 或 native/LinuxCNC 重新装载后，以下动作必须先取得同一本次运行闭包的 fresh native `all_homed=true`。UI 在下发前必须直读 native modal/Home 状态块或登记 API；command gate 在执行前必须再次读取 LinuxCNC/native actual。两层任一侧读到 `false` 或读不到 fresh 状态都必须 fail-closed，不能发送运动或坐标写入命令。

必须先完成开机全轴回零的当前动作：

- `启动`：程序 AUTO Start 和 MDI 发送/执行。
- 暂停态的 `继续/恢复`；普通 `暂停` 不受该门禁阻断。
- `首点` 定位。
- `点动+/-` 按下即连续点动、松开/触摸丢失即 `Jog Stop`；`Jog Stop` 始终允许。
- 当前选择为 MCS/WCS 单轴时的 `回零`，即真实回机械 0 / 回加工 0 定位。
- `归零` / Work Zero。虽然它不运动，但写入 WCS offset 必须建立在本次开机已经确认的机器坐标基准上。
- 后续新增的探测、换刀定位或其它会产生轴运动/依赖已建立机器坐标基准的动作，必须在登记时归入同一 native 门禁，不得另建第二套判断。

未回零时仍允许的动作：

- `机械全轴` + `回零`，用于建立本次开机的 `all_homed`。
- 普通暂停、Jog Stop、急停、取消急停和其它不产生运动的停止类安全入口；程序继续/恢复仍属于上面的受控动作。
- 程序打开、程序/MDI 编辑、轴选择、WCS 选择、视图、点动步距、进给/主轴倍率、RTCP 开关、G92 清除、页面导航和设置页非运动动作。

UI 提示必须是全屏三段式窗口，且不发送原动作：

```text
提示: POWER_ON_HOME_REQUIRED
原因: 本动作需要先完成本次开机机械全轴回零
下一步: 关闭提示，选择“机械全轴”后按“回零”
```

fresh native Home 状态不可用时使用 `POWER_ON_HOME_STATUS_UNAVAILABLE`，下一步仍是恢复 native 状态后执行机械全轴回零。提示窗只能由 `关闭` 按钮关闭；关闭不执行、排队或重放原动作。

严禁用当前坐标为 0、旧 `homed/all_homed`、旧 result、设置页 `设0`、SHM、UI 缓存、程序状态或磁盘参数替代本次开机回零。command gate 的二次门禁返回同一结构化 code，防止绕过 UI 直接下发；这项门禁是 `REQ-START-RUN-HOT-PATH` 唯一允许的产品前置例外，不得借此恢复其它 V3 自写 Start/Jog/Auto 预检。

## 7. 归零 / Work Zero

`归零` 是 Work Zero：

- Work Zero 的完整门禁、执行和关闭条件以 `功能/1-1加工坐标系偏移值保存与读取实现.md` 为准；本文只维护它和机器 Home 的按钮语义分界。
- 必须由登记 native/linuxcncrsh gate 执行 G10/等价接口，并由 native/interpreter/API 回读 active WCS、offset、generation 和结果；不得用 SHM、Broker result、参数文件 mtime 或 UI 显示值证明成功。
- 机器 idle、WCS 和轴选择明确、fresh native 状态、RTCP OFF、G92 策略、同源刀补模态和模式恢复等门禁全部服从 `1-1` owner。
- 把选中轴当前机械坐标关系写入当前 active WCS 对应轴 offset，使当前加工坐标为 0。
- 不运动、不改变 homed、不替代回零。

## 8. 单轴回机械 0 / 回加工 0

- 选中 MCS X/Y/Z 或 active model 对应旋转轴后按 `回零`，native positioning owner 必须让该轴真实移动到机械坐标 0；不能修改 MCS、清编码器、写 G92、Work Zero、设置页设0或驱动 preset 冒充移动。
- MCS 旋转轴到 0 必须复用 `功能/3-3旋转轴等效角目标与解决方案.md` 的 native 等效角语义，按可信 raw/count-domain 移动到最近等效机械 0；不清驱动多圈、不做 position preset、不写磁盘零偏。
- 选中当前 active WCS 的 X/Y/Z 或 active model 对应旋转轴后按 `回零`，native positioning owner 必须保持当前 WCS offset 不变，让该轴真实移动到加工坐标 0；RTCP ON/OFF 和 active model 运动学补偿只由微内核/LinuxCNC/switchkins 实现，UI 不计算补偿。
- MCS/WCS 单轴分支都不改变 `homed/all_homed`，也不得因实现缺失回退成全轴 Home。缺 fresh native MCS/WCS、active WCS、active model、可信 rotary raw、机器 enabled/idle、软限位或到位证据时结构化 fail-closed。

## 9. 失败提示

主页面普通运动/控制动作失败默认不弹全屏失败窗口；按 `功能/3-1主页面按钮功能.md`，只写事件日志。Home/positioning 的 success、failed、cancelled 任一 fresh terminal 到达时，UI 必须在同轮结束事务并清除 `active/pending/pressed`，回零按钮立即恢复普通色；不得等待弹窗关闭、后续轮询或超时才退出黄色。运动、安全、mapping、sequence 和到位类错误的唯一真源是本次 native direct failure，经统一映射后进入 `popup/top_status/log_only`；UI、Command Gate、Home adapter不得重算目标/容差/proof，也不得用泛化 adapter 错误覆盖 native 第一条原因。IPC 不可用、解释器上下文准备失败、program identity 改变、事务被替换或匹配 terminal 持续缺失属于适配/合同错误，必须以其自己的准确本地化原因显示，不能伪装成微内核运动错误。急停状态按 `回零` 的产品前置提示仍由 `REQ-POWER-ON-HOME-PRECONDITION` owner处理；操作员在执行中主动急停且 native 明确回读 cancelled时只显示顶部“回零已取消”。

UI 必须用统一 `code -> 本地化原因` 对照表生成当前语言顶部短提示和事件日志。Native/UDS/detail 结构中的 `message_cn`、`message`、`reason`、`error` 只能作为详情兜底；不得为了提示恢复 Broker/Python/Home job 中介，也不得用全屏弹窗替代原路径修复。

至少能区分：

- 急停。
- 未上使能。
- 机器不空闲。
- 状态不可用。
- active mode 不可确认。
- 设0目标缺失。
- 驱动读回失败。
- 到零点距离超过安全上限。
- homed 标记未确认。

## 11. 验收

本地必须覆盖：

- 普通 Home 不再投递 `home_precheck` async job，也不依赖 UI `job_status` 轮询证明执行。
- E-stop/Jog stop 可抢占。
- fresh native/provider 状态缺失时不触发运动；SHM `status_epoch` 不得作为 Home 准入证明。
- 机械全轴 UI/IPC 请求只含 `ALL_HOME`，不含 mode/model、轴、从站、joint、`HOME_SEQUENCE`、目标、速度、容差或 proof 字段。
- `v5_native_home.*` 和 Command Gate 全轴分支不得包含 `Set Jog_Incr`/`Set Jog_Stop`、MDI/G-code、分轴 `Set Home`、proof/backoff/remainder、目标/count、等效角、mapping/sequence 或到位算法。
- 机械全轴适配层只提交一次 native Home 请求并转发本次 native result/fresh `all_homed`；模式、模型、映射、次序、零点、安全、运动和完成判定测试属于微内核/native owner。
- 将轴参数 `HOME_SEQUENCE` 改成至少两组不同排列并经受控重启进入 resident 参数后，下一次机械全轴 Home 的实际阶段顺序必须完全随参数变化；同值轴同阶段，且不得出现按 X/Y/Z/A/C、joint 下标或从站位置固定排序的第二实现。
- 将任一轴选择为“禁用”并受控重启后，native readback 必须为 `HOME_SEQUENCE=999`，该轴不进入本次 run/current/completed mask；`-1` 必须保留为同步序号语义，不能再作为禁用值产生误运动。
- 每轴成功前必须证明 frozen exact target 对应的完整 trajectory 已下发、本 leg count 已离开 start、planner 已结束、joint in-position 且 fresh actual/count 连续稳定；任一证明缺失时必须返回准确轴、phase、最后 fresh actual、target/error 诊断和缺失事实，不得置 `homed/all_homed`。验收不得恢复 planner 完成后的 `abs(actual-target)<=tolerance` 第二成功门禁。
- focused 源码审计和 smoke 必须证明 Home owner、辅助头文件及失败/取消路径中不存在对 `pos_cmd`、`pos_fb`、`free_tp.curr_pos`、`motor_offset`、encoder/raw count、WCS/G92 或显示坐标的终态改写，也不存在 coordinate-sync/post-sync/当前位置置零 fallback。持续 count-domain 映射如存在，必须同时作用于目标和反馈并由单一 realtime owner 验证。
- MCS/WCS 单轴请求仍包含轴和坐标域，`native_axis_zero_position` 及其 wcheckpoint/safe-zero/等效角测试必须保留。
- 选择 `机械全轴` 后每次按 `回零` 必须触发新的 native Home transaction，并产生本次真实运动/到位/readback token；选择 MCS/WCS 单轴后每次按键必须触发新的 native positioning transaction，分别证明所选轴 MCS/WCS 到 0，且 WCS 单轴分支 offset 未改变。旧 homed、坐标为 0、目标 delta 为 0、`zero_model` 存在或命令发送成功均不能单独作为通过条件。
- 当前 Pulse contract 仍为 `cold_staged_not_runtime_selectable` 时，Pulse Home 运行请求 fail-closed；未来 contract 为 `runtime_selectable` 后，Pulse 走 LinuxCNC/HAL homed/all_homed。
- `回零` 不改变 active WCS。
- `归零` 不调用 Home。
- 普通运动/控制失败码在顶部短提示和事件日志中本地化显示；Home/positioning 前置拒绝和终态失败按本文件与 `功能/3-1主页面按钮功能.md` 的明确例外复用统一三段式全屏弹窗，执行中由操作员急停且 native 明确 cancelled 的事务不弹失败窗。

板端声明通过必须有：

- 原始 UI/operator 触发路径。
- UDS/native request/response、native Home gate 结构化结果和 readback token；普通 Home 不要求也不得依赖 `job_id/job_status` 作为执行证明。
- LinuxCNC/HAL 或 BUS drive/readback 证据。
- 机械全轴分支提供 LinuxCNC/HAL/native `homed/all_homed` 和 MCS 回读；MCS/WCS 单轴分支提供所选轴实际位置变化、MCS/WCS 到 0、到位 token，WCS 单轴还要证明 offset 未改变；SHM 只可补充 UI 坐标显示截图。
- UI 截图或 remote frame。
- 运动相关闭环按 `REQ-AUTO-STATE-DRIVEN` 跑 native active model 对应的 `cc-ac.ngc` 或 `cc-bc.ngc` 黄金运动测试。
- 当前测试台架电机为空载且未连接机械结构时，主页面 `回零` 的关闭证据必须分别实际执行机械全轴 Home、MCS 单轴到 0 和 WCS 单轴到 0 原路径；该台架状态下机械碰撞/负载安全风险不是默认 blocker，不得只做 direct UDS、构建、截图、坐标/offset 改写或旧 result 替代真实运动。若现场接回机械结构、刀具、工件或负载，先按 `功能/自动闭环测试方式.md` 更新硬件状态和安全门禁。
- 最终安全停机状态。

## 12. 禁止事项

- 恢复 FIFO、临时 response JSON、旧 native daemon、旧 `v3_linuxcnc_*.py` 裸脚本或短生命周期解释器 fallback。
- 用 Work Zero、G92、设置页设0、rebase、驱动 preset 或单轴 WCS 定位替代 `机械全轴` 的正常机器 Home；WCS 单轴定位只能在明确选择 WCS 单轴后执行，且不能改变 homed。
- 旧诊断文件、UI 缓存或私有文件恢复已回零。
- Home 执行阻塞 UI 刷新、远程输入或紧急通道。
- 在机械全轴 UI/Command Gate 适配层读取或固定轴名、从站、joint、扫描序号、模型或 `HOME_SEQUENCE`。
- 在任何层固定机械全轴回零顺序，或用轴名、joint 下标、从站位置、INI 默认顺序覆盖微内核 resident `HOME_SEQUENCE`。
- 在 Home 成功、失败或取消时改写 `pos_cmd`、`pos_fb`、`free_tp.curr_pos`、`motor_offset`、编码器/raw count、WCS/G92 或显示坐标；保留 coordinate-sync、post-sync、当前位置置零、失败回滚或兼容开关等同义支路。
- 未拿到 LinuxCNC/HAL/native 回读就显示“已回零”。

## 13. 关闭条件

关闭回零/归零问题时必须说明：

- 按钮语义：机械全轴正常 Home、MCS 单轴回机械 0、WCS 单轴回加工 0，或归零/Work Zero。
- 当前选择分支；仅机械全轴分支再说明 active mode 是 BUS 还是 Pulse。
- 请求、native/UDS 结构化响应、LinuxCNC/HAL/native 读回和 UI 证据；SHM 只记录显示刷新证据。
- 本地测试、板端截图、运动闭环和最终安全态。
- 未闭合事项只写真实缺口，不写过程日志。
