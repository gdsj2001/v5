## 2. 回零模式

主页面 `回零` 不再经 Broker/Python/UI 层的任何位移规划或 jog 适配中介。C UI 只提交当前选择空间、全轴/单轴和目标轴：`机械全轴` 由登记 native Home gate 下发 LinuxCNC/微内核原生 Home；MCS/WCS 单轴由登记 native positioning gate 执行对应坐标域的真实到 0 运动。UI 不计算 delta、不组装替代 G-code、不修改坐标或 offset。

选择分支优先于 BUS/Pulse 模式分支：

| 当前选择 | `回零` 唯一动作 | 成功回读 |
| --- | --- | --- |
| MCS 全轴（`机械全轴`） | 再按 fresh active mode 进入下表 BUS/Pulse 正常 Home | 本次 Home transaction、真实运动/到位、`homed/all_homed` |
| MCS 单轴 | 所选轴真实移动到 MCS 0；旋转轴使用 active model 对应轴的最近等效机械 0 | 所选轴 fresh MCS 为 0、实际位置变化/到位 token；不改变 homed |
| WCS 单轴 | 所选轴真实移动到当前 active WCS 0 | 所选轴 fresh WCS 为 0、对应 fresh MCS 实际变化/到位 token、WCS offset 未改变；不改变 homed |

以下 BUS/Pulse 表只适用于 `机械全轴` 分支：

**回零的实际操作完全由微内核根据当前的“脉冲/总线”模式在底层直接分支并控制执行，UI 层与 Broker 绝不参与轨迹计算与多轴时序控制：**

| 模式 | 微内核执行语义 | 成功边界 |
| --- | --- | --- |
| **Pulse/StepDir** | 仅当 Pulse contract 已由 owner 更新为 `runtime_selectable` 且 native active mode 已证明为 Pulse 时，微内核控制 HAL 层的原生 Home 流程：根据 INI 中配置的 `HOME_SEQUENCE` 次序进行排队，控制各轴向 `HOME_SEARCH_VEL` 方向运动寻找限位/home 开关，触碰后触发 latch 减速，最终依据 `HOME_OFFSET` 写入绝对坐标系基准。当前 contract 仍为 `cold_staged_not_runtime_selectable` 时，任何 Pulse Home 运行请求都必须 fail-closed。 | LinuxCNC/HAL 物理 homed 标志置位，UI 通过直连微内核内存/API 回读确认。 |
| **BUS/EtherCAT** | 微内核/native Home gate 必须先识别当前实际 BUS 模式，再读取 native/编码器同源 MCS 位置块和设置页 `设0` 所定义 count-domain 零点证据或 owner 签核的 home reference，由已登记 native/EtherCAT/Home owner 执行受控机器零点真实运动。每次按下 `回零` 都必须进入本次 Home 运动循环；即使当前坐标显示为 0、目标位移接近 0 或之前 `all_homed=true`，也不得在 UI/Broker/linuxcncrsh 文本层直接返回成功。若当前物理位置正好处于 home reference，native owner 仍必须执行可证明的 backoff/search/latch 或等价真实 Home cycle；底层不具备这种能力时返回结构化 capability/blocker，不得把坐标改成 0 冒充完成。UI/Broker 不参与轨迹计算、delta 算路、文本 MDI/G-code 组帧或分步 jog；若内部实现临时采用 MDI/G-code 或等价触发，也只能封装在已登记 native Home owner 内，不能暴露成 UI/Broker/linuxcncrsh 文本层规划路线。`Set Home -1` 只可作为本次真实运动后的 homed 状态同步，不得替代实际运动；后续专用 EtherCAT home owner 存在时可由同一 native gate 内部替换执行器。 | 伺服驱动器状态字 OP/到位证明或 LinuxCNC/native 机器坐标到位证明已确认 + 本次真实 Home cycle 的运动/到位/回读 token + 微内核内部 `homed/all_homed` 状态置位，UI 直连回读确认。 |

### 核心硬边界：
1. **彻底取消上层 jog 模拟回零**：物理回零动作必须由微内核在 native 层通过实时周期（realtime cycle）或运动控制器闭环控制，**绝对禁止**在 UI C 代码、Python 脚本或 Broker 中通过计算 delta 距离后用 `jog` 指令模拟回零动作。
2. **微内核统一入参与决策**：不管是 Pulse 还是 BUS 模式，回零运动的最高速度、加速度及超时限制，必须由微内核直接读取 INI 配置或 native 参数执行。UI 只提交选择空间、全轴/单轴和轴标识，不携带任何由 UI 算出的速度、delta、机械目标值或加工目标值。
3. **回零状态单一源回读**：机械全轴分支的执行进度与最终 `homed/all_homed`，以及 MCS/WCS 单轴分支的实际位置、到位和 offset 未变证明，都只允许从微内核/native 内存/API 直连回读确认，严禁依赖任何外部 `/run` 临时 JSON 文件或共享内存（SHM）的第三方状态写入。
4. **每次按键都是真实运动事务**：`机械全轴` 每次按键都必须产生新的 native Home transaction；MCS/WCS 单轴每次按键都必须产生新的 native positioning transaction。三种分支都要有本次运动/到位证明；`already_homed`、`position_zero`、`target_delta_zero`、`zero_model_present` 只能作为诊断上下文，不能成为 UI 本地空跑成功条件。

## 3. 模式判定

模式判定 owner 不是 UI 下拉框或设置页文字。

BUS/Pulse 模式识别、模式切换闭环和 Pulse 设零/回零总口径以 `REQ-BUS-PULSE-MODE-DIFFERENCE` / `功能/2-1总线脉冲区别.md` 为准；本节只定义主页面 `回零` 如何按 active mode 分支执行。

规则：

- `ethercat` / `bus` 归一为 BUS。
- `pulse` / `stepdir` / `step_dir` / `step/dir` 归一为 Pulse。
- 请求显式指定模式时，必须与 active mode 一致；不一致 fail-closed。
- 缺 active mode、native active mode 尚未确认 或运行模式不可确认时，回零不得运动。
- Pulse contract 仍为 `cold_staged_not_runtime_selectable`、Pulse HAL/RT adapter 未被 owner 签核或 native active mode 未证明为 Pulse 时，不得进入 Pulse Home 运行路径；强制提交必须结构化 fail-closed。
- 旧 `run_mode_bus_pulse` 字段退役，不能作为 UI dirty、显示入口、兼容字段或当前模式真源。

## 4. 执行入口

`回零` 可能持续数十秒，但 UI 不再把机械全轴 Home 或单轴到 0 定位投递到 Broker `home_precheck` async job。入口只负责把当前选择提交到登记 native gate；执行进度、全轴分支的 `homed/all_homed`、单轴分支的 MCS/WCS 到 0 readback 和失败原因必须由后续 native/microkernel/HAL/EtherCAT 回读及 UI 状态刷新呈现。`机械全轴` 的实际回零轴集合必须由微内核/native Home owner 消费 fresh active `motion_model` descriptor/readback 后决定：AC/`XYZAC_TRT` 只执行 `X/Y/Z/A/C`，BC/`XYZBC_TRT` 只执行 `X/Y/Z/B/C`，后续登记模型执行自己的 active axis set；UI 不得固定 A/C、不得从下拉文字或旧缓存推导轴集合，NAT、未绑定和非当前模型轴不得进入本次 Home transaction。

执行方式：

```text
C UI 回零选择意图
  -> registered native selection gate
  -> 机械全轴：LinuxCNC/HAL/EtherCAT/native Home owner
  -> MCS/WCS 单轴：native positioning owner
  -> native/microkernel/HAL/EtherCAT actual readback
  -> UI 状态刷新显示 homed/all_homed、所选轴到 0 或失败原因
```

要求：

- BUS `机械全轴` 分支下，`Set Home -1` 只用于同步 native/LinuxCNC homed 状态，不能单独算作回零运动完成；同一 registered native Home gate 必须读取 native MCS、BUS count-domain 零点证据和所需比例链/readback，再交给 native/EtherCAT/Home owner 执行同源受控运动。
- BUS 模式下，禁止用设置页 `设0`、写 `zero_model`、更新 `raw_zero_position`、清 MCS/SHM 显示、Work Zero、G92、drive preset 或单独 `Set Home -1` 来满足机械全轴 Home 或单轴到 0；这些路径只能作为设定零点、坐标系统或状态同步，不是真实运动。
- Pulse 当前仍为冷态预配置时，普通 Home gate 只能返回 Pulse not runtime selectable 的结构化拒绝，不能启动 Pulse HAL/Home 运行层。
- linuxcncrsh/native Home gate 发送 Home 后，不能只看到发送前已经存在的 `all_homed=true` 就在发送前提前成功；必须先向 Home owner 发出本次 Home 触发，再读取触发后的 native/HAL/EtherCAT 回读证据。零速 Home、当前位置已经在 HOME、或驱动/关节已保持 homed 的情况下，LinuxCNC 可能不会把 `all_homed` 先拉低再置高；此时成功边界是本次 Home 命令已由 native gate 发送并得到接收确认，随后 post-trigger `all_homed`/`motion.is-all-homed` 或等价 native 状态稳定为 true。不得把“没有观察到 unhomed 过渡”误判为 Home timeout。
- UI 坐标显示必须实时刷新，不等 Home 完成后才刷新。
- E-stop、Jog stop 等紧急通道不得被 Home 请求或状态刷新阻塞。
- 旧 `home_precheck` async job、`job_status` Home 轮询和 Python product action 结果不得恢复为普通 Home 执行路径。
- 命令已发送只用于触发证据；最终状态以 LinuxCNC/HAL/native 回读为准，SHM 只可作为 UI 坐标/轨迹、转速、进给速度和主轴/进给倍率显示证据；当前模态直连 native modal/tool 状态。

## 5. 状态 freshness

`回零` 随时可按，不等于可以携带空状态证据发运动。

UI 不得把 SHM `status_epoch`、坐标显示投影、旧状态帧或本地缓存作为 Home 运动准入证明；请求里也不得携带 `status_epoch` 作为 native proof。刚重启、State Publisher 尚未写入或 SHM reader 不可用时，UI 只能影响坐标/轨迹、转速、进给速度和主轴/进给倍率显示和提示，不能替代 Home native/owner 对 fresh native/provider 状态的读取；当前模态显示仍直连 native modal/tool 状态。

Native/owner 状态读取可短窗口重试 fresh native/microkernel 状态。机械全轴分支最终读不到 active mode、homed/all_homed、MCS 或 BUS/Pulse Home 证明时 fail-closed；MCS/WCS 单轴分支读不到所选轴 fresh MCS、active WCS/offset、active model 或 positioning owner 要求的证明时 fail-closed；均不得触发错误分支运动。

Home 先读取 fresh microkernel/native 急停与 Machine Enable actual；急停已生效时必须在 stillness、Abort、模式切换和轨迹下发前返回 `HOME_PRECONDITION_ESTOP`，未上使能时返回 `HOME_PRECONDITION_DISABLED`。UI 读取同源 fresh actual 可在下发前直接显示提示，但 command gate 必须再次读取 native safety owner，防止 UI 点击后新发生的急停被漏过；两层都必须复用统一 Home 提示窗，不能把急停混成 `HOME_NATIVE_PARAMETERS_OR_MACHINE_UNAVAILABLE` 或普通顶部短提示。安全允许后，运动态门禁只认 fresh microkernel/native active-axis stillness，不看裸 `exec_state`，也不得在 V3/Broker 里重建速度容差门禁。只要所有 active axes 已静止，急停复位后的残留 `EXEC_WAITING_FOR_MOTION_AND_IO`、Auto/程序上下文、暂停、活动 MDI、行号、运行文件、旧 homed、RTCP actual 或其它非运动状态都不得阻止本次 Home；native gate 可在静止证明之后发送 `Set Abort`，但该 Abort 只用于清理残留解释器上下文，不能把“运动中”转换成“允许自动回零”。若任一 active axis 仍在运动，必须在 Abort、模式切换和 Home 轨迹之前返回 `HOME_AXES_MOVING`；若 fresh stillness 读不到，返回 `HOME_STILLNESS_UNAVAILABLE`。上层只显示 native 结果，不用 SHM、JSON、旧快照或私有阈值自行判定。驱动故障和其它硬安全仍由对应 native owner fail-closed，不属于可被静止条件绕过的非运动门禁。

## 6. 开机回零前置条件

开机、断电重启、受控 clean restart 或 native/LinuxCNC 重新装载后，以下动作必须先取得同一本次运行闭包的 fresh native `all_homed=true`。UI 在下发前必须直读 native modal/Home 状态块或登记 API；command gate 在执行前必须再次读取 LinuxCNC/native actual。两层任一侧读到 `false` 或读不到 fresh 状态都必须 fail-closed，不能发送运动或坐标写入命令。

必须先完成开机全轴回零的当前动作：

- `启动`：程序 AUTO Start 和 MDI 发送/执行。
- 暂停态的 `继续/恢复`；普通 `暂停` 不受该门禁阻断。
- `首点` 定位。
- 增量点动 `点动+/-` 和长按连续点动；`Jog Stop` 始终允许。
- 当前选择为 MCS/WCS 单轴时的 `回零`，即真实回机械 0 / 回加工 0 定位。
- `归零` / Work Zero。虽然它不运动，但写入 WCS offset 必须建立在本次开机已经确认的机器坐标基准上。
- 后续新增的探测、换刀定位或其它会产生轴运动/依赖已建立机器坐标基准的动作，必须在登记时归入同一 native 门禁，不得另建第二套判断。

未回零时仍允许的动作：

- `机械全轴` + `回零`，用于建立本次开机的 `all_homed`。
- 普通暂停、Jog Stop、急停、取消急停和其它不产生运动的停止类安全入口；程序继续/恢复仍属于上面的受控动作。
- 程序打开、程序/MDI 编辑、轴选择、WCS 选择、视图、点动步距、进给/主轴倍率、RTCP 开关、G92 清除、页面导航和设置页非运动动作。

UI 提示必须是全屏三段式窗口，且不发送原动作：

```text
提示: POWER_ON_HOME_REQUIRED
原因: 本动作需要先完成本次开机机械全轴回零
下一步: 关闭提示，选择“机械全轴”后按“回零”
```

fresh native Home 状态不可用时使用 `POWER_ON_HOME_STATUS_UNAVAILABLE`，下一步仍是恢复 native 状态后执行机械全轴回零。提示窗只能由 `关闭` 按钮关闭；关闭不执行、排队或重放原动作。

严禁用当前坐标为 0、旧 `homed/all_homed`、旧 result、设置页 `设0`、SHM、UI 缓存、程序状态或磁盘参数替代本次开机回零。command gate 的二次门禁返回同一结构化 code，防止绕过 UI 直接下发；这项门禁是 `REQ-START-RUN-HOT-PATH` 唯一允许的产品前置例外，不得借此恢复其它 V3 自写 Start/Jog/Auto 预检。

## 7. 归零 / Work Zero

`归零` 是 Work Zero：

- Work Zero 的完整门禁、执行和关闭条件以 `功能/1-1加工坐标系偏移值保存与读取实现.md` 为准；本文只维护它和机器 Home 的按钮语义分界。
- 必须由登记 native/linuxcncrsh gate 执行 G10/等价接口，并由 native/interpreter/API 回读 active WCS、offset、generation 和结果；不得用 SHM、Broker result、参数文件 mtime 或 UI 显示值证明成功。
- 机器 idle、WCS 和轴选择明确、fresh native 状态、RTCP OFF、G92 策略、同源刀补模态和模式恢复等门禁全部服从 `1-1` owner。
- 把选中轴当前机械坐标关系写入当前 active WCS 对应轴 offset，使当前加工坐标为 0。
- 不运动、不改变 homed、不替代回零。

## 8. 单轴回机械 0 / 回加工 0

- 选中 MCS X/Y/Z 或 active model 对应旋转轴后按 `回零`，native positioning owner 必须让该轴真实移动到机械坐标 0；不能修改 MCS、清编码器、写 G92、Work Zero、设置页设0或驱动 preset 冒充移动。
- MCS 旋转轴到 0 必须复用 `功能/3-3旋转轴等效角目标与解决方案.md` 的 native 等效角语义，按可信 raw/count-domain 移动到最近等效机械 0；不清驱动多圈、不做 position preset、不写磁盘零偏。
- 选中当前 active WCS 的 X/Y/Z 或 active model 对应旋转轴后按 `回零`，native positioning owner 必须保持当前 WCS offset 不变，让该轴真实移动到加工坐标 0；RTCP ON/OFF 和 active model 运动学补偿只由微内核/LinuxCNC/switchkins 实现，UI 不计算补偿。
- MCS/WCS 单轴分支都不改变 `homed/all_homed`，也不得因实现缺失回退成全轴 Home。缺 fresh native MCS/WCS、active WCS、active model、可信 rotary raw、机器 enabled/idle、软限位或到位证据时结构化 fail-closed。

## 9. 失败提示

主页面 Home、Work Zero、Jog、Start、RTCP 等运动/控制动作失败默认不弹全屏失败窗口；按 `功能/3-1主页面按钮功能.md`，只写顶部短提示与事件日志。急停状态按 `回零` 是本条新增的明确例外：必须复用统一 Home 三段式全屏提示窗，显示“请先取消急停”、急停导致不能回零的直接原因，以及点击右下角“取消急停”并确认使能后重试的下一步；关闭只关闭提示，不排队、不重放 Home。提示不得只显示“回零失败”或泛化 guarded status。

UI 必须用统一 `code -> 本地化原因` 对照表生成当前语言顶部短提示和事件日志。Native/UDS/detail 结构中的 `message_cn`、`message`、`reason`、`error` 只能作为详情兜底；不得为了提示恢复 Broker/Python/Home job 中介，也不得用全屏弹窗替代原路径修复。

至少能区分：

- 急停。
- 未上使能。
- 机器不空闲。
- 状态不可用。
- active mode 不可确认。
- 设0目标缺失。
- 驱动读回失败。
- 到零点距离超过安全上限。
- homed 标记未确认。

## 11. 验收

本地必须覆盖：

- 普通 Home 不再投递 `home_precheck` async job，也不依赖 UI `job_status` 轮询证明执行。
- E-stop/Jog stop 可抢占。
- fresh native/provider 状态缺失时不触发运动；SHM `status_epoch` 不得作为 Home 准入证明。
- BUS 缺 zero evidence、active `SCALE`、驱动 actual position、profile/readback 时 fail-closed。
- BUS 启动预载必须逐 active 轴证明 `raw_zero_position == zero_counts / counts_per_unit`；缺轴、缺字段、非有限数、比例为 0 或公式不一致时 Command Gate 不得启动 Home owner。
- Pulse contract 精确为 `cold_staged_not_runtime_selectable` 时，即使 Pulse INI 可解析也必须返回 `PULSE_HOME_NOT_RUNTIME_SELECTABLE`，不得发送 `Set Home -1`。
- 选择 `机械全轴` 后每次按 `回零` 必须触发新的 native Home transaction，并产生本次真实运动/到位/readback token；选择 MCS/WCS 单轴后每次按键必须触发新的 native positioning transaction，分别证明所选轴 MCS/WCS 到 0，且 WCS 单轴分支 offset 未改变。旧 homed、坐标为 0、目标 delta 为 0、`zero_model` 存在或命令发送成功均不能单独作为通过条件。
- 当前 Pulse contract 仍为 `cold_staged_not_runtime_selectable` 时，Pulse Home 运行请求 fail-closed；未来 contract 为 `runtime_selectable` 后，Pulse 走 LinuxCNC/HAL homed/all_homed。
- `回零` 不改变 active WCS。
- `归零` 不调用 Home。
- 失败码可在顶部短提示和事件日志中本地化显示，主页面运动/控制动作不恢复全屏失败弹窗。

板端声明通过必须有：

- 原始 UI/operator 触发路径。
- UDS/native request/response、native Home gate 结构化结果和 readback token；普通 Home 不要求也不得依赖 `job_id/job_status` 作为执行证明。
- LinuxCNC/HAL 或 BUS drive/readback 证据。
- 机械全轴分支提供 LinuxCNC/HAL/native `homed/all_homed` 和 MCS 回读；MCS/WCS 单轴分支提供所选轴实际位置变化、MCS/WCS 到 0、到位 token，WCS 单轴还要证明 offset 未改变；SHM 只可补充 UI 坐标显示截图。
- UI 截图或 remote frame。
- 运动相关闭环按 `REQ-AUTO-STATE-DRIVEN` 跑 native active model 对应的 `cc-ac.ngc` 或 `cc-bc.ngc` 黄金运动测试。
- 当前测试台架电机为空载且未连接机械结构时，主页面 `回零` 的关闭证据必须分别实际执行机械全轴 Home、MCS 单轴到 0 和 WCS 单轴到 0 原路径；该台架状态下机械碰撞/负载安全风险不是默认 blocker，不得只做 direct UDS、构建、截图、坐标/offset 改写或旧 result 替代真实运动。若现场接回机械结构、刀具、工件或负载，先按 `功能/自动闭环测试方式.md` 更新硬件状态和安全门禁。
- 最终安全停机状态。

## 12. 禁止事项

- 恢复 FIFO、临时 response JSON、旧 native daemon、旧 `v3_linuxcnc_*.py` 裸脚本或短生命周期解释器 fallback。
- 用 Work Zero、G92、设置页设0、rebase、驱动 preset 或单轴 WCS 定位替代 `机械全轴` 的正常机器 Home；WCS 单轴定位只能在明确选择 WCS 单轴后执行，且不能改变 homed。
- 旧诊断文件、UI 缓存或私有文件恢复已回零。
- Home 执行阻塞 UI 刷新、远程输入或紧急通道。
- 未拿到 LinuxCNC/HAL/native 回读就显示“已回零”。

## 13. 关闭条件

关闭回零/归零问题时必须说明：

- 按钮语义：机械全轴正常 Home、MCS 单轴回机械 0、WCS 单轴回加工 0，或归零/Work Zero。
- 当前选择分支；仅机械全轴分支再说明 active mode 是 BUS 还是 Pulse。
- 请求、native/UDS 结构化响应、LinuxCNC/HAL/native 读回和 UI 证据；SHM 只记录显示刷新证据。
- 本地测试、板端截图、运动闭环和最终安全态。
- 未闭合事项只写真实缺口，不写过程日志。
