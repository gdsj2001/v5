# 制作驱动映射表所需厂家文件清单

用途：给后续 AI/后台制作 `driver_profile_map.json`。核心目标是把不同厂家驱动命令映射成统一板端命令，例如 `drive.set_egear`、`drive.read_egear`、`drive.read_actual_position`、`drive.fault_reset`。

## 1. 最小必须资料

一般只需要这三类资料：

| 资料 | 常见格式 | 用途 |
| --- | --- | --- |
| EtherCAT ESI/XML | `.xml` / `.zip` | 识别驱动型号、Vendor ID、Product Code、Revision、PDO、CoE 对象入口 |
| EtherCAT / CoE / CiA402 技术手册 | `.pdf` / `.docx` | 确认控制字、状态字、模式、SDO 读写流程、报警复位、回零流程 |
| 参数说明书 / 参数表 | `.pdf` / `.xlsx` / `.xls` | 确认厂家私有参数：齿轮比、实际位置读回、保存参数、限位、回零方向、报警码 |

结论：

```text
ESI/XML 负责识别驱动和对象入口。
技术手册负责 EtherCAT/CoE/CiA402 通信控制流程。
参数说明书负责厂家私有参数和齿轮比/实际位置读回/保存/限位/回零。
```

缺哪一类，就不能把对应命令标成 `supported=true`。

## 2. 命令分级

### 2.1 必须命令

这些命令是板端设置页闭环的最低要求。缺少厂家对象、写入值或读回规则时，映射表里对应命令必须 `supported=false`，并且不允许把相关轴运动放开。

| 标准命令 | 必须确认 |
| --- | --- |
| `drive.identify` | Vendor ID、Product Code、Revision、设备名 |
| `drive.read_statusword` | 状态字对象，一般是 `0x6041:00` |
| `drive.write_controlword` | 控制字对象，一般是 `0x6040:00` |
| `drive.read_mode` / `drive.write_mode` | 模式对象，一般是 `0x6061:00` / `0x6060:00` |
| `drive.read_actual_position` | 实际位置对象，一般是 `0x6064:00` |
| `drive.read_actual_velocity` | 实际速度对象，一般是 `0x606C:00` |
| `drive.set_egear` / `drive.read_egear` | 齿轮比分子、分母对象，写入值，写后读回 |
| `drive.fault_reset` | 清除可复位报警的对象/控制字流程，必须和设0严格区分 |
| `drive.save_parameters` | 保存参数对象、签名值、执行条件、读回确认 |

### 2.2 建议命令

这些命令不一定影响最小映射表生成，但建议尽早补齐。缺少时可以先 `supported=false`，不能伪造成已支持。

| 标准命令 | 用途 |
| --- | --- |
| `drive.read_alarm` | 读取当前报警码，便于 UI 和日志显示真实故障 |
| `drive.software_reset` | 驱动软复位；只能用于维护/诊断，不能替代清故障或设0 |
| `drive.read_egear_effective` | 读取实际生效齿轮比，区分“参数已写入”和“已生效” |
| `drive.read_position_latched_state` | 读取厂家位置锁存/状态，辅助诊断；不能替代 `zero_pos` 设0闭环 |
| `drive.read_limit_config` / `drive.write_limit_config` | 驱动侧限位配置；如果限位由 LinuxCNC/HAL 管，必须明确 `supported=false` 和原因 |
| `drive.homing_start` / `drive.homing_status` | 驱动内部回零；如果回零由 LinuxCNC/HAL 管，必须明确 `supported=false` 和原因 |
| `drive.read_digital_inputs` | 读取限位、原点、报警、使能等 IO 状态 |
| `drive.read_temperature` | 读取驱动/电机温度，用于诊断 |
| `drive.read_bus_voltage` | 读取母线电压，用于诊断 |

## 3. 设0依赖实际位置读回，不能猜清零命令

V3 设置页 `设0` 不再要求厂家提供驱动侧零位改写对象。必须从厂家资料或实机读回证明：

```text
index/subindex
数据类型
actual_position 的对象语义
unit/count 换算来源
是否要求实际速度为 0
读回超时如何判断
读回值如何写入 zero_pos
```

`fault reset`、`software reset`、`factory default`、`0x2018:05`、`0x607C`、homing method 35 不能当作 V3 设置页设0。设0动作只做：

```text
actual_counts = drive.read_actual_position()
zero_pos = actual_counts * unit_per_count
machine_pos = actual_counts * unit_per_count - zero_pos
```

## 4. 文件放置

本地厂家资料按品牌型号放：

```text
8ax驱动映射表/<品牌型号>/
  esi/
  docs/
  extracted/
  manifest/
```

用户上传的新驱动资料先放 DNA 私有目录：

```text
private/<vps_distribution_id>-<device_dna_hash>/uploaded_driver_params/
private/<vps_distribution_id>-<device_dna_hash>/driver_profile_map.json
```

源码内映射表位置：

```text
config/drive-profiles/public/driver_profile_map.json
config/drive-profiles/private/<device_dna_hash>_driver_profile_map.json
```

说明：

1. `8ax驱动映射表/` 只放厂家原始资料、解包结果、manifest 和制作证据。
2. `public/driver_profile_map.json` 是公共映射表源码，只有实机验证通过的通用驱动命令才能并入。
3. `private/<vps_distribution_id>-<device_dna_hash>/driver_profile_map.json` 是 VPS 端 DNA 私有映射表目标路径，只服务对应机器；源码文件名仍使用 `config/drive-profiles/private/<device_dna_hash>_driver_profile_map.json`。
4. 板端和 VPS 下载目录不是源码真相，最终必须从以上源码位置生成或同步。

要求：

1. 原始文件保留。
2. 记录 SHA256。
3. 映射表每条命令能追溯到来源文件和对象/页码。
4. 未实机读写读回测试，不得并入公共映射表。
5. 私有映射表只允许对应 DNA 机器下载。

## 5. AI 制作步骤

1. 读 ESI/XML，提取 `vendor_id/product_code/revision/model`。
2. 读技术手册，确认 CiA402 标准命令和 SDO 读写流程。
3. 读参数说明书，确认齿轮比、实际位置读回、保存参数、限位、回零、报警码。
4. 必须命令和建议命令都要填 `supported=true/false`。
5. 写命令必须填 `verify_object` 和读回规则。
6. 缺资料的命令必须 `supported=false`，并写明原因。
7. 先生成 DNA 私有映射表。
8. 实机测试通过后，才允许提升到公共映射表。

## 6. 最小验收

```text
ESI/XML 已保存并记录 SHA256
技术手册已保存并记录 SHA256
参数说明书/参数表已保存并记录 SHA256
身份字段已提取
必须命令逐项标记 supported/unsupported
建议命令逐项标记 supported/unsupported
supported 写命令都有 verify_object
drive.read_actual_position 有厂家对象、数据类型和单位换算来源
drive.save_parameters 有厂家保存流程
未实机测试前没有并入公共映射表
```
