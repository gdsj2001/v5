## 当前代码结论

| 领域 | 当前代码状态 | 结论 |
| :--- | :--- | :--- |
| 生产运动路径 | EtherCAT/BUS 仍是当前生产主路径 | 不能宣称 Pulse 或 Hybrid 已经生产可用 |
| 全局 BUS/Pulse 切换 | 软件门禁已有本地代码闭环，状态为 `local_verified_only` | 还缺 UI/operator 原路径和板端闭环验证 |
| active mode 真源 | `motion_driver.conf` 只是 requested/持久 owner；active mode 由 backend/mode-switch/State Publisher 证明后进入 SHM | 不能用 `motion_driver.conf` 单点读回判断 active mode |
| Pulse/StepDir runtime | `stepdir/pulse` 在硬件 profile manifest 中仍是 `future_not_implemented` | 只能作为冷态资源/未来方向，不能作为 runtime selectable |
| Hybrid profile | `hybrid` 是 EtherCAT 与 StepDir 资源边界并集，状态是 `boundary_union_not_runtime_switch` | 不是 per-axis 混合插补运行模式 |
| 设备授权 | 当前授权链支持 DNA、签名、`not_before`、`expires_at`、`permissions`，当前代码识别 `drive_profile_download` | 尚未实现轴数、反馈数、混合插补等商业 feature 字段 |
| BUS-only 动作门禁 | Broker/product action/C dispatch/UI 路径已有按 active BUS proof 拒绝 Pulse/未知模式的本地门禁 | 仍需板端状态、UI 原路径和 operator 验证 |

## 不能按“已具备”推进的点

1. 不能把 `motion_driver.conf` 写成全机 active 拓扑真源；它只能表达 requested mode。
2. 不能把 `hybrid` profile 当作运行时混合插补开关。
3. 不能把 `allowed_ethercat_axes`、`allowed_pulse_axes`、`allowed_feedback_encoders`、`mixed_interpolation_enabled` 写成已实现授权字段；这些目前只是 future proposal。
4. 不能用 UI 隐藏按钮替代 Broker/product action/HAL/native 安全门禁。
5. 不能让 UI、Python 产品脚本或 Broker 直接写 HAL pin、PL register 或 LinuxCNC/HAL 私有控制真源。
6. 不能声明 Pulse 硬件可运行；还缺 bitstream、device-tree、native driver、HAL adapter、波形、反馈、急停/限位/回零和实机闭环证据。
7. 不能声明 per-axis Hybrid 可运行；还缺每 joint 单一 adapter、单一 feedback owner、混合 INI/HAL 生成、SHM 真源和 golden motion。

## 未来设计切片

### A. 授权 feature schema

状态：`proposal_not_implemented`。

可讨论字段：

```json
{
  "features": {
    "allowed_ethercat_axes": 3,
    "allowed_pulse_axes": 1,
    "allowed_feedback_encoders": 0,
    "mixed_interpolation_enabled": false
  }
}
```

落地前置：

- 先更新 `REQ-DRIVE-PROFILE-AUTH-CHAIN` owner，定义字段、默认值、兼容策略和 fail-closed 语义。
- 字段必须被签名覆盖，缺失、类型错误、超范围、过期、DNA 不匹配都默认拒绝。
- latch/status 只能发布验签后的结果，UI 只显示结果，不成为授权真源。

### B. 当前全局模式授权门禁

状态：未实现。

目标是在 per-axis Hybrid 前，只对现有全局 `motion_driver_mode` requested/apply 路径加授权门禁：

- 未授权 BUS/Pulse 模式时，拒绝写 owner 或拒绝 apply/restart。
- 不改变当前 BUS-only 动作门禁。
- 不引入旧 `run_mode_bus_pulse` 或 UI 私有真源。

验收：本地 contract 测试通过；板端 UI/operator 原路径能证明未授权拒绝、授权后 requested mode 与 active SHM proof 一致。

### C. Pulse/StepDir 硬件 runtime closure

状态：未实现，不能只靠现有冷态资源关闭。

验收：

- bitstream、device-tree、native driver、HAL adapter、PL register map 可追溯。
- Step/Dir 方向、频率、抖动、复位、watchdog、fault、estop、limit、home 可测。
- EtherCAT 模式下 Pulse 输出 inert/disabled；Pulse 模式下 BUS-only 动作仍 fail-closed。
- 有波形、状态 SHM、HAL、LinuxCNC 跟随误差和实机运动证据。

### D. Per-axis Hybrid 混合拓扑

状态：未实现。

验收：

- 每个 joint 只能绑定一个输出 adapter。
- 每个 joint 只有一个 feedback owner。
- 混合 INI/HAL 由 canonical generator 生成，不手写分叉。
- Start/Run/Resume/MDI/Jog/Home 都经过相同授权和 active-mode 门禁。
- `nc/cc.ngc` golden motion、SHM、HAL、日志和实机轨迹证明齐全。

### E. PL/native enable mask 授权锁

状态：未实现。

验收：

- enable mask 只能由固定 native gate/driver 写入，产品 Python/UI/Broker 不直接写。
- 未授权、授权过期、授权减少、watchdog、fault、estop、重启后默认 disabled。
- 写入后必须有 readback；readback 与实际波形/输出不一致时 fail-closed。

## 下一步建议

优先做 A，再决定是否做 B。C/D/E 属于硬件和运动闭环切片，不能只靠文档或本地测试宣布完成。

1. 先在 owner 中定义授权 feature schema 和 fail-closed 语义。
2. 扩展 `device_dna_register_auth.py`、`v3_device_authorization_download.py`、`device_auth_latch.py`，只解析并发布验签后的 feature。
3. 增加缺失字段、非法类型、过期、旧授权包、不含 feature 时默认不解锁 Pulse/Hybrid 的测试。
4. 只有 A 通过后，才考虑给全局 `motion_driver_mode` apply 路径加授权门禁。
