# 设置页触摸闭环测试作业指导书

引用需求真源：`REQ-PARAM-MEMORY-LIGHTWEIGHT-SAVE`、`REQ-SETTINGS-RUNTIME-DRIVE-ONLY`、`REQ-BUS-PULSE-MODE-DIFFERENCE`、`REQ-DRIVE-PROFILE-AUTH-CHAIN`、`REQ-RTCP-G53-NATIVE-ACTUAL`、`REQ-UI-POPUP-RESULT-POLICY`、`REQ-AUTO-STATE-DRIVEN`、`REQ-UI-FIRST-FRAME-CACHE`、`REQ-UI-RUNTIME-REFRESH-RATE`

本文只用于设置页单步 operator path 闭环测试；主页面 model-matched `cc-ac.ngc` / `cc-bc.ngc` 运动黄金链路另按 `待做工作/测试主页.md` 执行。若本文与 `功能/需求真源索引.md` 指向的 owner 冲突，以 owner 为准。

## 当前代码事实

- 源码真源是 `board/`，VM `/root/Desktop/v5` 只是同步副本，不得直接修改。
- 设置页 UI 在 `board/app/src/v5_settings_page.c`；按钮经 `board/app/src/v5_settings_actions.c` 请求 `/run/8ax_v5_product_ui/settings_actiond.sock`。
- `board/services/drive_profile/v5_settings_actiond.py` 当前映射：`扫描从站` -> `scan`，`读取驱动` -> `read`，`复位驱动` -> `factory-reset`，`清除故障` -> `fault-reset`，`设置驱动` -> `set-drive`，轴 `设0` -> `axis-zero`，`保存并重启` -> `settings_save_and_restart`。
- `board/services/drive_profile/v5_drive_bus_action.py` 当前已有真实驱动动作：`read` 返回 `DRIVE_READ_OK/DRIVE_READ_PARTIAL`；`factory-reset` 返回 `DRIVE_RESET_OK/DRIVE_RESET_PARTIAL`；`fault-reset` 返回 `DRIVE_FAULT_RESET_OK/DRIVE_FAULT_RESET_PARTIAL`；`set-drive` 返回 `DRIVE_SET_OK/DRIVE_SET_PARTIAL`。
- `factory-reset` 写 `drive.restore_factory_defaults` 后会把相关轴驱动参数标记为失效：`drive_parameters_valid=false`、`electronic_gear.write_status=invalidated_by_factory_reset`、`restart_required=true`、`restart_deferred=true`。因此执行过复位驱动后，必须再跑 `设置驱动` 和 `读取驱动`，不能把复位后的 1/1 电子齿轮状态留作可加工状态。
- `set-drive` 写电子齿轮和模式，成功标准是 `write_verified_readback`，并且 egear/mode/statusword/error_code 与目标一致；若写入已开始但回读失败，代码会置为 `write_unverified_readback_failed`，测试必须判失败。
- `settings_runtime.json` 是 drive-only schema：`re.v3.settings_runtime.drive_only.v1`。它不能承载 WCS/G92/TLO/tool/RTCP/modal/status 等非驱动真源。
- 设置页 `运动模型` 不是 UI 本地选项或 drive-only 字段：它通过 settings apply 提交给微内核/native `motion_model` owner。提交请求必须有非零 `owner_generation` 和 `readback_token`；设置页返回值、runtime INI 或磁盘文本只能证明保存输入，clean restart 后 fresh native active model 才能证明实际生效。模型切换事务必须同时写入目标 model/KINS/TRAJ、目标逻辑 AXIS→JOINT/status slot 和 descriptor 默认 `axis→slave`：active axis 绑定同号 status slot/slave，离开 active set 的转换轴写 `NAT`。事务成功后立即刷新轴参数表并提示默认映射已自动更新；操作员仍可用轴参数 `从站` 下拉人工覆盖最终 self owner。重启后最终映射不完整、重复或与模型不一致时 UI 仍须启动并再次提示，但使能、Home、Start 保持阻断。
- 轴参数 owner 当前在 `board/app/src/v5_parameter_table.c`：`axis_*_slave` 是 self 参数 owner；`axis_*_encoder_bits` 是 drive-only owner；`G53_A_Y/G53_A_Z/G53_B_X/G53_B_Z/G53_C_X/G53_C_Y` 是 kinematics native/API owner。
- `board/app/src/v5_settings_axis_table.c` 的普通比例链字段通过 `v5_settings_apply_commit_axis_value()` 提交；比例链变更会回填 `raw_min_limit/raw_max_limit` 对应的软限位显示值。
- `bit/encoder_bits` 缺失时显示 `--`，不得提交 `NAT`、`--` 或默认伪值。

## 操作顺序

### A. 身份与下载链

1. 若顶部 `本机码` 不是合法 6 位已登记 ID，先执行 `登记本机码`。
2. 执行 `下载授权`。
3. 执行 `服务器下载`。
4. 确认 resident profile snapshot 已在启动或受控刷新阶段就位。

通过条件：

- `登记本机码` 成功时，顶部只显示 6 位 `vpsDistributionId`；缺少合法 ID 时显示 `未登记`，不得显示 raw DNA、DNA hash 或 id-dna folder。
- `下载授权` 结果必须是 `DEVICE_AUTH_DOWNLOAD_OK`，并证明授权文件写入且签名校验通过。
- `服务器下载` 必须证明 public/private profile 下载、授权校验、hash 校验和板端缓存写入结果；private 缺失只能记录 absent/skipped reason，不能写成 private 成功。
- 弹窗最终正文固定为 `提示 / 原因 / 下一步` 三项；需要操作员确认的成功/失败结果保留到点击关闭。

### B. 驱动只读链

5. 执行 `扫描从站`。
6. 执行 `读取驱动`。

通过条件：

- `扫描从站` 结果为 `DRIVE_SCAN_OK`，`slaves` 非空，并更新 `SETTINGS/slave_options`。
- `从站` 下拉打开时读取当前 resident/self 参数候选；候选显示为 `轴号 驱动名型号`，提交时只写纯轴号。扫描只更新候选，不自动改写现有轴-从站绑定。
- 默认 X/Y/Z/A/B/C/龙门/刀库的 `从站` 列为 `0-7`，不得从 `10` 开始。
- `读取驱动` 结果为 `DRIVE_READ_OK` 才能继续；`DRIVE_READ_PARTIAL` 必须记录缺失的从站、profile 或必读 SDO 后停止。

### C. 驱动写入链

7. 执行 `复位驱动`。
8. 执行 `清除故障`。
9. 执行 `设置驱动`。
10. 再执行一次 `读取驱动`，确认最终驱动参数。

通过条件：

- `复位驱动` 成功结果是 `DRIVE_RESET_OK`，并能证明 `drive.restore_factory_defaults` 执行、目标轴 egear 回读为 1/1、fault/error 清理状态符合 profile 读回、`settings_runtime` 中对应轴被标记为 `invalidated_by_factory_reset`。若结果是 `DRIVE_RESET_PARTIAL`、precheck 失败、写入失败或读回不一致，立即停止。
- `清除故障` 成功结果是 `DRIVE_FAULT_RESET_OK`，并能证明 statusword fault 位清除、error_code 为 0 或 profile 定义的无故障状态。该动作不得伪造参数写入成功。
- `设置驱动` 成功结果是 `DRIVE_SET_OK`，并能证明目标 egear/mode/statusword/error_code 与 profile/readback 一致。当前目标按代码配置检查：X/Y/Z 为 4096/625，A 为 8192/225，C 为 16384/1125；若配置变更，以当前代码和 profile 目标为准。
- `设置驱动` 后 `settings_runtime` 对应轴必须是 `drive_parameters_valid=true`、`electronic_gear.write_status=write_verified_readback`、`drive_readback.error_code=0`，并标记需要 clean restart/reload。
- 复位驱动后若未执行并通过 `设置驱动` 和最终 `读取驱动`，不得离开设置页进入可加工结论。

### D. 字段编辑链

11. 逐类测试一个代表字段：普通输入、下拉字段、`bit` 字段、G53 可编辑字段、轴 `设0`。
12. 每个字段点击前用 MCP 板端抓屏观察并保存当前 fresh frame；点击后重新抓屏确认软键盘、下拉或动作弹窗出现；提交后读取对应 owner readback 与 UI 单元格。

当前字段范围：

- 下拉字段：`轴模式`、`方向`、`从站`、`回零次序`、`回零方向`、`bit`。
- 普通输入字段：`目标精度`、`螺距`、`电机`、`负载`、软限位、速度、加速度、反向间隙。
- `零位/设0` 是动作字段，不是普通键盘写值字段。
- `egear_numerator/egear_denominator/write_status` 是驱动读回显示字段，不是 UI 可编辑参数。
- G53 中 `A中心 X`、`B中心 Y`、`C中心 Z` 是 modal/WCS/native 语义只读；`A中心 Y/Z`、`B中心 X/Z`、`C中心 X/Y` 是 kinematics native/runtime INI owner 可编辑字段；刀库/五向探测等行按 self owner 回读。

通过条件：

- runtime INI owner 字段必须走 settings apply 提交、文件写入/读回、resident/native/runtime 刷新语义和重启挂起状态。比例链字段提交后必须同步重算并显示 `raw_min_limit/raw_max_limit` 对应软限位。
- `从站` 只写 self owner/readback，不再写 microkernel 第二表。
- `bit/encoder_bits` 只写 drive-only owner/drive provider readback，不再写 microkernel 第二表；缺失时显示 `--`，不能提交 `NAT`、`--` 或默认伪值。
- G53 native geometry 写 active runtime INI/native owner；硬盘表或 UI 单元格显示不等于 native 生效，最终成功需要 clean restart/reload 后 native/API readback 一致。
- 事件日志必须出现 `settings_axis_param_commit` 且 `ok=true`；失败事件或 owner 不一致时停止。

### E. 运动模型选择生效链

13. 进入设置页前或点击下拉前，先从微内核/native readback 记录当前 fresh active model；只接受登记模型 `XYZAC_TRT` 或 `XYZBC_TRT`。
14. 在设置页 `运动模型` 下拉中选择与当前 active model 不同的目标：AC 对应 `AC摇篮`/`XYZAC_TRT`，BC 对应 `BC摇篮`/`XYZBC_TRT`。
15. 提交后先确认出现一次三段式“默认映射已自动更新，请核对轴从站映射”提示；关闭只关闭提示。再检查本次 `settings_axis_param_commit`、runtime INI、目标 AXIS→JOINT/status slot 和 descriptor 默认 axis→slave 的同事务 owner readback；此时只能声明“目标模型和默认映射已保存并等待 clean restart”，不得把下拉框文字、磁盘值或 request 当作 active model。随后必须用轴参数 `从站` 下拉人工交换任意两个 active axis，再读取最终 self owner 映射，证明人工覆盖优先于默认表。完成该回读后立即执行第一次 `保存并重启`；只有重启后 fresh native active model/mapping 等于目标表，才允许进入 F。

保存前通过条件：

- 事件必须是 `settings_axis_param_commit`、field 为 `motion_model`、`ok=true`；对应 command-gate/settings apply 请求必须携带非零 `owner_generation` 和 `readback_token`，缺一项即失败。
- 目标必须规范化为登记 canonical model，不允许未知字符串、默认 AC 或 `if AC else BC` 回退。
- 模型下拉只有在目标与当前保存值发生实际切换且 owner 写入/readback 成功时提示一次；提示正文必须说明 descriptor 默认从站映射已自动更新，下一步必须指向“轴参数 → 从站”下拉供人工复核或覆盖，关闭后仍停留在设置页并保留 restart pending。
- AC 目标写后回读至少一致：`[RTCP] MODEL=XYZAC_TRT`、`MOTION_MODEL=AC摇篮`、`KINS_MODULE=xyzac-trt-kins`、`KINS_COORDINATES=XYZAC`、`WRAPPED_ROTARY_MASK=40`、`[TRAJ] COORDINATES=X Y Z A C`。
- BC 目标写后回读至少一致：`[RTCP] MODEL=XYZBC_TRT`、`MOTION_MODEL=BC摇篮`、`KINS_MODULE=xyzbc-trt-kins`、`KINS_COORDINATES=XYZBC`、`WRAPPED_ROTARY_MASK=48`、`[TRAJ] COORDINATES=X Y Z B C`。
- 运动模型提交后 `axis→slave` owner readback 必须精确等于目标 descriptor 默认表：AC 为 `X0/Y1/Z2/A3/B=NAT/C4`，BC 为 `X0/Y1/Z2/A=NAT/B3/C4`。随后操作员通过轴参数 `从站` 下拉作出的人工覆盖必须形成新的最终 owner readback；模型事务不得复制物理从站零点/count-domain/drive actual，人工覆盖也不得迁移逻辑 AXIS 比例链。

### F. 轴设0链

16. 在第一次 clean restart 后重新进入设置页，对 fresh active model 的全部 active axes 逐轴点击 `设0`；随后按最终 axis→slave 映射重跑一次 `设置驱动 -> 读取驱动`。

通过条件：

- UI 不弹普通数字键盘，而是发出 `settings_axis_zero_request`，scope 为 `bus_count_domain_zero`。
- actiond 进入 `settings_axis_zero` -> drive `axis-zero`，结果文件为 `/run/8ax_v5_drive/settings_axis_zero_result.json`。
- 成功结果必须证明当前编码器/驱动 count-domain 位置被写成新的 zero model，同事务重算并写入 raw soft limits，随后用同一 active scale/unit_per_count 回读比较，结果为 `SETTINGS_AXIS_ZERO_ENCODER_MATCH` 或当前代码定义的等价成功码。
- 只显示请求已接收、弹窗有反馈或 UI 值变化，不算设0成功。
- Pulse 模式 HOME_OFFSET 路径另按 Pulse owner 验证，不能和 BUS count-domain 设0混用。
- 旧 active model 下若误点目标模型独有轴，必须以 `RUNTIME_INI_JOINT_MAPPING_INVALID` 或等价 owner 错误 fail-closed，且 `write_executed=false`/`motion_executed=false`；禁止放宽映射校验来省略第一次 restart。

### G. 返回主页 / 保存并重启

17. 进入设置页后、尚未提交任何有效参数变更时，确认右上角显示 `返回主页`；点击后只返回主页面，不调用 settings actiond、不触发 apply/restart。再次进入设置页。
18. 完成一个有效字段提交或运动模型选择，等该字段唯一 owner 写入和源位置回读成功后，确认同一个右上角按钮原位切换为 `保存并重启`，再点击它。模型切换流程中，E 后的第一次用于激活目标 model/mapping；F 及重跑设置/读取驱动完成后必须第二次保存并重启，才可返回主页运动测试。

通过条件：

- 无有效参数变更且无 restart pending 时，按钮必须是 `返回主页`，点击只做本地导航，不得触发 `settings_save_and_restart`。
- 任一字段完成 owner 写入/readback，或设置动作成功回读 `restart_required/restart_deferred` 后，按钮必须立即、单向切换为 `保存并重启`；本次会话不得再退回 `返回主页`。
- 有参数变更后点击 `保存并重启` 必须触发 `settings_save_and_restart` clean restart handoff，不得用本地页面切换绕过重启。
- 合格的 `保存并重启` 等同开发板断电重启：安全去使能、停止 UI/Broker/actiond/State Publisher/LinuxCNC/HAL/EtherCAT/驱动 provider、清运行态、重新加载唯一 owner、重建 HAL/EtherCAT/SDO/PDO/轴绑定/比例链，并做 native/驱动 readback 证明。
- 运动模型只有在该 clean restart 后才能判定生效：native G53 geometry/motion model readback 必须来自本次重启后的新 writer/block，同时满足 monotonic age fresh、CRC/valid 通过，并且 target model/owner readback 精确等于目标 canonical model。当前 ABI 的 `epoch` 是 geometry centers 的 FNV identity，不是 boot generation；几何未变时允许相同，禁止要求 identity 单调增加。LinuxCNC kinematics/coordinates、主页面旋转轴组和黄色轨迹 projector 必须消费同一个 active model readback。AC 显示/消费 A/C 与 AC G53 geometry，BC 显示/消费 B/C 与 BC G53 geometry。
- clean restart 后 active model 与 axis→slave 映射完整、唯一且一致时，UI 正常启动，不出现映射错误提示，并可继续后续安全使能/Home/Start。映射缺失、重复或不一致时，Position/State/UI/relay 仍必须完成启动，坐标和 CPU 不能因映射错误变成 `--` 或长期停在 `booting`；首个完整帧后必须显示一次“运动模型与从站映射不一致”三段式提示，关闭只关闭窗口，使能、Home、Start 仍由 native owner 拒绝。
- 对不一致分支，必须通过设置页轴参数 `从站` 下拉逐轴修正，不能用模型切换、扫描从站、部署默认、直接改文件或 UI 本地值修正；保存重启后再读取 fresh model/mapping generation 和 native/drive actual，匹配后才继续。
- 单轮测试至少完成一次“当前 native model -> 不同目标 model”的真实切换；完整双模型回归分别验证 AC→BC 和 BC→AC。下一步 Golden Motion 必须按本次重启后的 active model 选择 `cc-ac.ngc` 或 `cc-bc.ngc`，不得按设置页选择值预判。
- 若 clean restart 证明不了，必须走板端 reboot/power-cycle 等价路径；只重启少数服务、只返回主页面或只刷新 provider 不能算通过。
- 重启后如果 UI 已出现但坐标/参数/共享内存仍未就位，这是启动顺序 blocker，不能把设置保存闭环写成通过。

## 弹窗、轮询与刷新

- 长任务动作超时以当前 actiond 代码为准：scan 20s、read 45s、fault-reset 45s、factory-reset 90s、set-drive 180s、axis-zero 20s。
- 长任务进行中必须显示进度/阶段或明确等待状态；最终成功/失败结果不得保留倒计时。
- 最终弹窗正文固定三项：`提示:`、`原因:`、`下一步:`。不新增 `详情`、`操作轮`、`现状` 等第四项。
- UI 运行期刷新分层：坐标和轨迹动态层 30Hz；普通按钮轮询 10Hz；急停按键显示/状态 10Hz；其它普通 UI 状态和诊断显示 5Hz。
- 禁止周期性全屏刷新；必须按 dirty 对象、dirty 单元格、dirty 区域或 LVGL invalidated rect 分块按需刷新。
- 软键盘出现和数字输入目标响应按 0.2s 检查；字段提交到 owner 写入/读回按 0.5s 检查。长流程按钮按下后 0.2s 内必须出现进度窗口或明确已提交反馈。

## 停步规则

- MCP 抓屏或 pointer/mouse 能力不可用、页面不对、按钮不可见、目标被遮挡或 fresh frame 不清晰时不点击。
- 点击未命中、事件日志无对应动作、settings_actiond 未接收、结果文件未更新或弹窗长期无阶段推进时，不继续下一步。
- 授权/DNA、服务器下载、profile resident snapshot、从站扫描或读取驱动任一失败，先修该单点。
- 运动模型提交缺少非零 owner generation/readback token、写后配置不完整、默认 axis→slave 不等于目标 descriptor、人工下拉覆盖没有进入最终 owner、clean restart 后 native active model 未切换或主页面/LinuxCNC仍消费旧模型时，立即停止；不得进入 Golden Motion。映射不一致时若 UI 卡在 `booting`、坐标/CPU 因映射错误变 `--`、没有启动提示，或关闭提示后使能/Home/Start 被放行，同样立即判失败。
- `复位驱动`、`清除故障`、`设置驱动` 现在按真实 action/readback 测试；任何 `*_PARTIAL`、precheck failure、write failure、readback mismatch、schema forbidden field 或 `write_unverified_readback_failed` 都必须停步。
- 任何步骤不得用 SHM 显示投影、旧 `/run` 结果、UI dirty、本地默认值或旧 JSON 作为参数保存、驱动写入、运动安全或授权成功真源。
- 本文不记录历史验证流水；每轮测试结果在最终对话或指定证据目录交付。
