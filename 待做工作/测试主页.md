# 主页模拟测试流程

## 失忆/接手必读

若上下文丢失、失忆、换人接手或从压缩摘要恢复，先读 PROJECT_ROOT 下的 `AGENTS.md` 和 `系统代码架构硬边界守则.md`。读完后再继续主页模拟测试。

## 单点迭代与最终上板规则

- 单点诊断阶段可以优先跑 focused gate 和相关模块测试；必要时可用本轮修改文件和直接相关依赖做临时同步/部署来定位问题，但这只能作为诊断或截图前置证据。
- 任何 runtime 源码修复必须先落在 `D:\v5\board` canonical source；Markdown/截图/Windows/Vivado 文件仍落在 `PROJECT_ROOT` 对应 owner。不得只改 VM、板端、staging、deploy_tmp、repo_ignored 或其它临时副本。
- 最终板端闭合必须使用完整当前 PROJECT_ROOT 统一 VM/ARM 编译、打包、部署的新产物，再按原始 UI/operator 路径验证；旧产物、单文件同步或局部部署不能声明黄金链路通过。
- 只有修改跨共享 ABI、CMake/构建系统、公共 UI 框架、SHM/Broker 协议或轻量诊断无法证明时，才提前升级到更大范围编译；但这不降低最终完整构建/部署要求。
- 不要回退、覆盖或清理其他 AI/用户已经落在工作树里的修改；发现冲突时先保留现状，按当前 diff 合并或记录阻塞。

## 目的

用本地抓屏工具配合模拟屏幕点击，按真实主页操作员路径逐步闭合主页黄金链路。主页测试不是一口气跑自动化；每一步都必须先看见屏幕、再点一次、再抓屏确认，有问题就停下修当前问题。

## 开始前置

- 本轮现场确认：电机为空载，且当前没有安全问题；该确认放行本轮 `cc.ngc` 主页运动段单步测试，但不取消逐步截图、单次点击、SHM/Broker 回读、急停可见和异常即停规则。
- 正式开始前，先用继电器重启一次开发板。
- 记录继电器重启证据：断电/上电动作、板端重新连通、SSH/remote frame、remote input、Broker 和 SHM 就绪。
- 重启后先抓第一张屏，确认当前在主页面。
- 第一张屏必须直观看到：急停按钮、启动按钮、回零按钮、打开程序入口、坐标区域、刀路区域。
- 开始前读取 SHM，确认 `stale=false`、`estop=false`、`enabled=true`、`current_vel=0`、机器空闲。
- 若未使能或有弹窗遮挡，必须先按“点击前抓屏 -> 单次点击 -> 点击后抓屏 -> 状态读取”的规则处理该单点，不能直接进入黄金顺序。

## 黄金顺序

1. 回零：除 UI/Broker 成功外，必须观察编码器反馈，确认等效角/关节角确实按回零方向运动并最终稳定。
2. RTCP 开关测试：点击 RTCP ON，抓屏和 fresh SHM 确认 `rtcp_active=true`；再点击 RTCP OFF，抓屏和 fresh SHM 确认回到 OFF。
3. 打开程序
4. 双击 `cc.ngc`
5. 启动
6. 急停
7. 再启动
8. 回零：再次观察编码器反馈，确认等效角/关节角运行与最终稳定状态一致。

## 每步执行规则

- 每个模拟点击前必须抓屏，确认页面、按钮坐标、遮挡状态和急停可见。
- 每次只允许执行一个点击动作；不能把多个点击合并成脚本连续打。
- 点击后必须立刻抓屏，并读取 SHM、Broker 结果或事件日志确认动作结果。
- 抓屏时必须检查直观问题：刀路区是否有应显示的图形、程序名/MDI 显示是否正确、坐标区域是否闪空/乱值、弹窗是否遮挡急停、顶部提示是否误导。
- 回零步骤必须记录回零前、回零中、回零后的编码器反馈或 SHM 等效角/关节角摘要；不能只看按钮变色、弹窗或 Broker 成功。若等效角未运动、方向异常、跳变异常或回零后不稳定，立即停止后续步骤。
- 打开程序前，刀路区已有内存数据的 MCS/WCS/A-C/红绿点等应可见；黄色轨迹只要求在打开 G-code 后出现。
- 打开 `cc.ngc` 后，主页面必须显示程序名 `cc.ngc`，并显示黄色刀路或明确 degraded 原因；不能出现后台成功但 UI 无图形、MDI 无程序显示的状态。
- 启动 `cc.ngc` 后，必须采集运行中状态，确认速度/行号/运动状态变化；运行中至少抓取两张间隔截图，直观看黄色轨迹或当前运动标记是否随 G-code 运行推进，不能只看 SHM/Broker 成功。
- G-code 开头和结尾若出现 `A0`、`C0`、`A0 C0`、`AC0` 这类 A/C 归零目标，必须通过微内核/native 运行链和编码器反馈确认旋转轴 unwrap/连续角闭环；V3 不再检查本地运行副本、filter manifest 或等效角 Python 结果。不能只看源 G-code 文本里的 `0` 就判定已经等效角运行；必须证明显示相位回到 0 附近、raw/编码器实际按规划运动并最终稳定。
- 涉及 RTCP 的 G-code 运行测试还要确认运行到 `M128` 后 native RTCP/switchkins actual 为 ON，程序结束或 `M129` 后回到 OFF；RTCP actual 不以 SHM 字段作为动作证明。

## 停步和修复

- 任何一步抓屏看不清目标、页面不对、按钮被遮挡、急停不可见，禁止点击。
- 任何一步点击后没有对应 UI 变化、Broker 结果、SHM 变化或事件日志，立即停止后续步骤。
- 中途发现问题，先修本地源码或配置，上板部署后从“回零”重新开始，不从失败中间点续跑。
- 未闭合单步前不能进入全量自动化；全量自动化只能作为所有单点通过后的最终回归证据。
- 不需要为了每个小问题全量编译；改 C UI 时可先构建并部署 UI 做单点诊断，改 Python/Broker/脚本/配置时可先同步对应产品路径并重启相关服务定位问题。但最终运动相关结论仍需要完整当前 PROJECT_ROOT 产物部署后的 `cc.ngc` 主页路径证据。
