# 当前代码闭合作业指导书

## 1. 作业目的

按 `功能/` 现有 owner 文档，把当前代码从多入口、脚本中介、热路径读盘、SHM 越界字段和上层自造门禁，收敛为唯一 canonical 运行链路。

最高需求来源：

1. `AGENTS.md`
2. `功能/需求真源索引.md`
3. `功能/` 下登记的唯一 owner 文档

本文是执行指导，不是需求真源。执行中发现需求冲突时，先更新对应 `功能/` owner，再继续改代码。

## 2. 作业范围

本次闭合覆盖：

1. 开机全量读入 RAM 的运行闭包。
2. SHM UI 显示投影边界。
3. Start/Pause/Program Open 热路径。
4. 停稳/速度门禁、WCS、G92、RTCP、旋转轴、刀路过滤等 native owner 回收。
5. BUS/Pulse、驱动 profile、VPS 授权边界。
6. 设置页、弹窗、触摸、网络显示。
7. native helper、CMake、VM 映射和测试假入口清退。

## 3. 环境说明

LinuxCNC、微内核、HAL、EtherCAT、native gate、linuxcncrsh 和板端服务属于 VM/目标板环境，不按 Windows 本地环境判定是否完整。Windows 本地只用于能在本地构建的 UI/C 代码、脚本语法和普通宿主工具；凡是依赖 LinuxCNC 运行库、HAL、实时组件、ARM/板端路径或 VM rootfs 的片段，必须在 VM 部分编译或 VM 全量编译中闭合。

作业中看到 “本地 CMake 编译通过” 时，只表示本地可覆盖部分通过；LinuxCNC 相关 owner 路径还要按本节转入 VM 验证，不能因为 Windows 本地无法运行 LinuxCNC 就补 fallback、mock gate、报错提示或第二套实现。

## 4. 作业原则

1. 先删旧路径，再补 canonical 实现。
2. 不保留 fallback、兼容入口、rename-only wrapper、env-gated 后门、blocked 占位或测试假路径。
3. 删除后用本地编译暴露真实依赖；本地覆盖不了的再用 VM 部分或全量编译。
4. 不用新增报错、弹窗、拒绝提示替代实现。当前路径不能实现时，删除入口，并在 `待做工作/遗留.md` 记录缺的 native/微内核 owner gate 和下一关闭条件。
5. SHM 只给 UI 显示，不做控制、安全、动作成功、参数、WCS、RTCP、tool、active mode、stillness 或诊断真源。
6. Start/Resume 热路径不得读盘、扫脚本、算 SHA、重建 runtime、过滤 G-code、等待 JSON proof 或执行产品层预检门禁。

## 5. 验收口径

代码修改后的默认验收只要求编译通过：

1. 能在 Windows 本地编译的 UI/C、脚本和宿主工具，跑本地 CMake 或对应本地编译。
2. LinuxCNC、微内核、HAL、EtherCAT、native gate、linuxcncrsh、ARM/板端服务和 VM rootfs 相关片段，默认跑 VM 部分编译。
3. 多个 LinuxCNC/VM/ARM 片段互相依赖或无法拆分时，跑 VM 全量编译。
4. 不默认跑 pytest、审计、自动化、板端动作或过程记录，除非编译错误直接指向它们，或用户单独要求。

8. settings 保存只写 owner 允许的持久位置，并由源位置回读更新内存。

完成判定：

1. UI、Broker、product action、State Publisher 热路径无目录扫描、动态脚本选择、临时 JSON 请求/结果文件、ready/heartbeat 文件事实或程序文件重读。
2. 所有动作、字段、资源、profile 和参数 owner 映射都有启动内存入口或明确的受控 refresh 入口。
3. 本地可覆盖部分 CMake 编译通过；LinuxCNC/微内核/native/ARM 片段用 VM 部分编译或 VM 全量编译补证。
