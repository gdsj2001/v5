# Z20 v1.5 Linux / LinuxCNC 侧交付说明

> FPGA owner 边界：本文只描述 `new-vivado/z20_v1_5_hw_project/` 的另一套 FPGA 功能交付，不是当前产品交付入口。当前产品/当前 SD 使用 `vivado_hw_project/`，制卡不得消费本文的 XSA、bitstream 或硬件合同。

文件编码：UTF-8 without BOM。Linux 侧可复制的 DTS/C 片段和本交付文档均不得带 BOM。

## 定位

本文面向该不同 FPGA 功能的后续 Linux、LinuxCNC、HAL、驱动和设备树开发人员，是该功能内部交付索引，不替代寄存器表、manifest、XSA、BD 或端口映射真源。

- 当前交付状态：`local_verified_only`。
- 当前未完成：板端部署、实物接线验证、DO/PWM 负载 off 极性验证、EtherCAT/RGMII Link 保持实测、运动闭环板测。
- 路径基准为 `PROJECT_ROOT`；本文中的项目本地文件均使用 `new-vivado/...` 形式。
- 不得引用、修改或回退到当前产品 owner `vivado_hw_project/`；两个 FPGA 功能必须保持互不依赖，当前制卡只消费 `vivado_hw_project/`。

## 当前关键地址锚点

下表只用于 Linux 侧快速定位。正式驱动、设备树或 HAL 头文件必须从 `new-vivado/z20_v1_5_hw_project/board_inputs/system.xsa` 或当前 BD 地址段重新生成/复核。

| Block | 当前 AXI 地址 | 范围 | ABI 真源 |
| --- | ---: | ---: | --- |
| `step_ip/s_axi` | `0x43CB0000` | `64K` | `new-vivado/z20_v1_5_hw_project/ip_repo/axi_stepdir_enc_v2/step_ip_register_map.md` |
| `pl_estop/S_AXI` | `0x41260000` | `64K` | `new-vivado/z20_v1_5_hw_project/board_inputs/pl_estop_register_map.md` |

驱动初始化必须先读每个自定义 AXI block 的 `MAGIC`/`VERSION` 或等价识别寄存器。识别不匹配时，LinuxCNC/HAL 不得继续绑定输出。

## Linux 设备树接入提示

设备树、驱动匹配名和中断号应从 `new-vivado/z20_v1_5_hw_project/board_inputs/system.xsa` 生成或复核；下表只列当前 Vivado 边界。

| 功能 | 当前硬件边界 | Linux 侧提示 |
| --- | --- | --- |
| 8 轴 step/dir/encoder | `step_ip/s_axi`，`reg = 0x43CB0000 0x10000` | 自定义 AXI-Lite 节点；寄存器语义引用 `new-vivado/z20_v1_5_hw_project/ip_repo/axi_stepdir_enc_v2/step_ip_register_map.md`。 |
| PL 急停 | `pl_estop/S_AXI`，`reg = 0x41260000 0x10000`，IRQ 路由 `pl_estop/estop_irq -> xlconcat_0/In14` | 自定义 AXI-Lite 节点；GIC 数字中断号不要手写，必须由 XSA/device-tree 生成结果确认。 |
| 普通 IO owner | `z20_v15_io_owner/S_AXI`，`reg = 0x41270000 0x10000` | 自定义 AXI-Lite 节点；寄存器语义引用 `new-vivado/z20_v1_5_hw_project/docs/io_owner_register_map.md`。 |
| RS485 | `RS485_FPGA_RX/TX` 通过 PS UART1 EMIO 导出 | 设备树启用 PS UART1；收发器方向/DE 控制仍需板端复核。 |
| ADC | 一路 `XADC_VP/XADC_VN` 专用模拟输入 | 走 Zynq XADC/IIO 路径；不要恢复 MCP3202 SPI ADC。 |
| 触摸 | 触摸 I2C 沿用原工程 I2C1 路径，`TP_INT/TP_RST` 由 IO owner 管理 | I2C 触摸控制器仍按原触摸路径接入；`TP_INT` 从 IO owner status 读，`TP_RST_N` 从 IO owner 写。 |

可直接复制到 PetaLinux `system-user.dtsi` 或 Linux 设备树 overlay 的 AXI-Lite 节点如下；同内容也已生成到 `new-vivado/z20_v1_5_hw_project/board_inputs/z20_v1_5_axi_lite.dtsi`：

```dts
&amba_pl {
    z20_step_ip: step-ip@43cb0000 {
        compatible = "re,z20-v1.5-step-ip", "re,v3-step-ip";
        reg = <0x43cb0000 0x00010000>;
        status = "okay";
    };

    z20_pl_estop: pl-estop@41260000 {
        compatible = "re,z20-v1.5-pl-estop", "re,pl-estop";
        reg = <0x41260000 0x00010000>;
        interrupt-parent = <&intc>;
        interrupts = <0 43 4>; /* IRQ_F2P[14], GIC SPI 75, level-high */
        status = "okay";
    };

    z20_io_owner: io-owner@41270000 {
        compatible = "re,z20-v1.5-io-owner", "re,z20-io-owner";
        reg = <0x41270000 0x00010000>;
        status = "okay";
    };
};
```

`pl_estop` 的中断来自当前 XSA：`pl_estop/estop_irq -> xlconcat_0/In14 -> IRQ_F2P[14]`，DTS 中写为 `interrupts = <0 43 4>`。

## Linux / LinuxCNC 接入顺序

1. 从 `new-vivado/z20_v1_5_hw_project/board_inputs/system.xsa` 生成或更新平台、设备树、clock、IRQ 和 AXI 地址定义。
2. 运行 `new-vivado/z20_v1_5_hw_project/scripts/verify_driver_contract.ps1`，确认 `new-vivado/z20_v1_5_hw_project/board_inputs/z20_v1_5_hardware_abi.h` 和 `new-vivado/z20_v1_5_hw_project/board_inputs/z20_v1_5_axi_lite.dtsi` 与当前 XSA/register-map 一致。
3. 按 `new-vivado/z20_v1_5_hw_project/board_inputs/software_handoff.md` 和 `new-vivado/z20_v1_5_hw_project/board_inputs/README.md` 运行交付前本地检查；至少要确认 manifest hash、hardware ABI、project portability、local closure 仍为当前状态。
4. C 侧驱动或 HAL glue 直接 include `new-vivado/z20_v1_5_hw_project/board_inputs/z20_v1_5_hardware_abi.h`，不要再手工抄 AXI base、offset、MAGIC/VERSION；设备树复制 `new-vivado/z20_v1_5_hw_project/board_inputs/z20_v1_5_axi_lite.dtsi`。
5. 先接只读识别和状态：`step_ip` ID/version、`pl_estop` MAGIC/VERSION/STATUS、`z20_v15_io_owner` MAGIC/VERSION/BUILD_ID。
6. LinuxCNC 运动层对接 `step_ip`，寄存器语义只引用 `new-vivado/z20_v1_5_hw_project/ip_repo/axi_stepdir_enc_v2/step_ip_register_map.md`；8 轴 PULS/DIR 和 8 轴 ABZ 已在 Vivado 边界接通，但板端轴号、极性和闭环行为仍需另行验证。
7. 普通 IO、PWM、轴使能、触摸复位和输入状态对接 `z20_v15_io_owner/S_AXI`；输出和急停相关语义只按单真源边界列出的 Owner 文档读取。
8. 急停状态对接 `pl_estop/S_AXI` 和 IRQ；本文不定义 latch、reset、bus TX、release 或 readiness 语义，也不作为板上安全认证依据。
9. RS485 走 PS UART1 EMIO 导出边界，按 XSA/device tree 接入；收发器方向、板上串口行为和协议仍需软件/板端复核。
10. ADC 当前只有一路 XADC VP/VN；Linux 侧应走 XADC/IIO 或既有 XADC 驱动路径，不得恢复 MCP3202 SPI ADC 方案。

## LinuxCNC/HAL 边界

- LinuxCNC/HAL/EtherCAT 仍是运动控制运行时真源；Vivado 交付只提供 PL 寄存器、pin 边界和 bit/XSA。
- `step_ip` 只提供硬件 step/dir/encoder 执行和计数接口；轨迹规划、坐标系、限位语义、回零策略和安全策略不在本文定义。
- DO/PWM/ENA、PL 急停、bus TX、latch/reset/release 和 readiness 含义不在本文定义；按单真源边界列出的 Owner 文档和当前 manifest/README 输出读取。
- 本文不作为急停安全策略或板端安全证据；不得从本文推导具体安全动作、恢复流程或板端验证结论。

## 禁止项

- 不要在 Linux、设备树、脚本、文档中写入项目盘符绝对路径。
- 不要使用 `vivado_hw_project/` 的 XSA、XDC、BD、wrapper 或旧脚本。
- 不要把 `U10/U9/AA12/AB12` 恢复为 ADC SPI；当前 ADC 是 `XADC_VP/XADC_VN` 一路模拟输入。
- 不要给 `XADC_VP/XADC_VN` 增加普通 PL `PACKAGE_PIN` 约束。
- 不要绕过 `new-vivado/z20_v1_5_hw_project/board_inputs/z20_v1_5_hardware_abi.h`，也不要把 AXI base/offset 手工复制到 C 驱动里。
- 不要手写与 `new-vivado/z20_v1_5_hw_project/board_inputs/z20_v1_5_axi_lite.dtsi` 不一致的 AXI-Lite 设备树节点。
- 不要恢复 HDMI/TMDS；这些管脚已裁决给 MPG/手轮相关输入。
- 不要把 `i2c4`、旧 GPIO0、旧 HDMI、旧 ADC SPI 作为可用产品接口重新接入，除非先更新 Vivado 侧真源、证据和 gate。
- 不要绕过 `new-vivado/z20_v1_5_hw_project/board_inputs/manifest.json` hash 校验直接消费二进制产物。

## 停止条件

遇到以下任一情况，Linux/LinuxCNC 侧应停止接入或拒绝启用输出：

- `new-vivado/z20_v1_5_hw_project/scripts/verify_board_input_manifest.ps1` 失败，或 artifact hash 不匹配。
- `new-vivado/z20_v1_5_hw_project/scripts/verify_driver_contract.ps1` 失败，或 `z20_v1_5_hardware_abi.h` / `z20_v1_5_axi_lite.dtsi` 与当前 XSA/register-map 不一致。
- `new-vivado/z20_v1_5_hw_project/scripts/verify_hardware_abi_header.ps1` 失败，或 `v3_hardware_abi.h` 与当前 XSA/register-map 不一致。
- `new-vivado/z20_v1_5_hw_project/scripts/verify_new_vivado_local_closure.ps1` 不再输出 `new_vivado_local_closure=local_verified_only`。
- `new-vivado/z20_v1_5_hw_project/scripts/verify_project_portability.ps1` 不再输出 `project_portability=ok` / `absolute_path_scan=ok`。
- 任一自定义 AXI block 的 MAGIC/VERSION/BUILD_ID 与引用文件不一致。
- 急停寄存器、IRQ 或状态识别不满足 `pl_estop_register_map.md` 和 Owner 文档要求。
- 软件需要声明板上安全、运动或 IO 行为已验证，但缺少对应实物证据。
