# PL 侧硬件级绝对急停与软硬件协同设计方案

> FPGA owner 边界：本文属于 `new-vivado/z20_v1_5_hw_project/` 所代表的另一套 FPGA 功能，不是当前产品使用的 `vivado_hw_project/` 的副本、升级分支或 fallback。当前产品/当前 SD 的 XSA、bitstream 和默认硬件合同只允许来自 `vivado_hw_project/`；本文方案不得被自动带入当前制卡链。

## 一、 设计背景与目标

在商用数控（CNC）控制系统中，安全（Safety）是系统设计的首要红线。传统的急停（E-Stop）信号通常接入操作系统（PS 侧的 Linux），通过内核驱动、HAL 逻辑及 HMI 界面软件处理后再去封锁伺服。这种软件处理链路存在不可忽视的隐患：**一旦 PS 侧出现系统死机（Kernel Panic）、CPU 满载延迟、或是套接字通信中断，急停响应就会滞后甚至失效。**

为了达到工业级安全标准（如 SIL3 / PLe 级别），本方案提出**在 FPGA（PL 侧）直接硬干预物理信号实现绝对急停，不经过 PS 侧软件中转，同时在 PS 侧实现状态观测和安全互锁复位**的软硬件协同设计。

## 二、 硬件链路与信号接口

### 1. 物理引脚接入（PL 侧）
* **常闭回路（Normally Closed, NC）**：急停物理按钮和限位开关必须采用常闭方式串联接入 PL 侧的 GPIO 引脚。正常工作时引脚保持高电平；一旦急停被按下、线缆折断、或控制箱掉电，引脚瞬间变为低电平，自动触发急停（Fail-Safe）。
* **硬件去抖（De-bounce）**：工业现场电磁干扰（EMC）强，PL 侧必须设计硬件滤波器（如 10ms - 20ms 的积分消抖滑动窗口），滤除高频杂波，防止系统误急停。

### 2. 物理干预输出（PL 强动作）
当 PL 急停状态机触发时，**绕过 PS 直接在纳米级/微秒级时序内**物理拉低以下引脚：
* **PWM 脉冲封锁**：直接将 PL 侧电机控制（或总线主站 IP 内核）输出的所有轴 PWM 脉冲物理拉低，使功率逆变器瞬间失去驱动电流。
* **STO（安全转矩切断）**：直接切断外部伺服驱动器的 STO 光耦输入电源，强制伺服使能硬件断开。
* **通用 IO 输出关闭（DO/PWM）**：通用输出 `DO1` - `DO14`、`PWM1` - `PWM2` 不能绕过急停；急停锁存后必须由 PL 急停门控强制到已确认的 off/safe 电平，不等待 PS、LinuxCNC 或 UI。每路 off 极性必须由原理图、负载接线和板测证据确认后，才能接入真实 DO/PWM pin。
* **总线 TX 发送门控**：接 PL 的总线驱动/控制发送只门控 TX send-enable、driver-enable 或 TX_CTL idle，不断开物理 Link，不复位 PHY/收发器，不关闭链路时钟、电源、复位或 RX/状态观测；解除急停前必须清空或失效急停期间积压的 TX FIFO、控制帧或命令。
* **抱闸控制（Brake Control）**：对于垂直轴（如 Z 轴），优先于伺服脉冲封锁数十微秒触发电磁抱闸，防止主轴因重力坠落。

## 当前 new-vivado 实施裁决（2026-06-30）

* **HDMI 放弃，MPG 使用冲突脚**：旧 HDMI/TMDS 的 9 个共享管脚不再作为 HDMI 使用，全部绑定为手轮 MPG：`AXIS_SEL0` - `AXIS_SEL7` 和 `PULS5_IO`。顶层 `system_top.v` 删除/不连接退役 HDMI wrapper 端口，防止再次把这些脚按 TMDS 输出驱动。
* **急停物理输入**：`EGS_DI/AA19` 已进入 active XDC，并在 `system_top.v` 中作为 `estop_nc_in`。当前顶层按 NC 逻辑处理：高电平健康，低电平或断线触发 `estop_hw_active`。实际机器 NC 串联链、输入保护、断线测试和板级去抖仍需 A11 板测闭环。
* **DO/PWM 顶层硬切断**：`DO1` - `DO14`、`PWM1` - `PWM2` 已通过 `system_top.v` 顶层 `pl_estop_general_output_gate` 后再输出。急停有效时强制 safe level `0`；当前正常输出 owner 仍为 `16'h0000`，所以本地构建是安全关断姿态，不代表普通 DO/PWM 功能已经实现。后续接入软件/PS 正常输出时，只能接在该 gate 前级。
* **网口 TX 发送封锁**：`system_top.v` 把 `estop_nc_in` 传入 wrapper，当前 XSA 的 BD synth netlist 在 `gmii2rgmii` 之前封锁 GMII `TX_EN`，急停时同时把 GMII `TXD` 置 `8'h00` 并清除 `TX_ER`；RGMII `rgmii_tx_ctl`、`rgmii_td`、`rgmii_txc` 仍由 `gmii2rgmii`/ODDR 直接驱动到端口，RX、MDIO、PHY reset、link power 不作为急停动作关闭。这样急停解除后物理 Link 不需要重建，恢复动作只等待硬件/软件互锁放行。同一 generated BD synth 边界已把 `pl_estop/estop_nc_in` 接到顶层 `estop_nc_in`，使 AXI STATUS/IRQ 与硬门控使用同一个物理急停输入。由于该门控和观测对齐落在 Vivado generated wrapper/BD synth 边界，当前新工程用 `scripts/patch_gmii_pre_oddr_estop_gate.ps1` 在综合/实现前可重复恢复该补丁，并由 `verify_pl_estop_safety_boundary.ps1` 检查。
* **当前诚实边界**：以上是本地 RTL/XDC 硬门控实现，不是板级安全认证。还必须测量 `DO/PWM` 实际 off 电平、RGMII TX 是否停止发送、Link/RX 是否保持、释放后是否无 stale TX/control 帧重放，以及 STO/抱闸/轴 gate 是否仍缺真实接线。

## 三、 硬件与电路设计落地细节规范

### 1. 常闭（Normally Closed, NC）回路设计
* **逻辑电平**：正常状态下物理安全回路闭合，PL GPIO 维持高电平（logic 1）；急停按下或发生断线故障时回路断开，引脚通过下拉电阻变为低电平（logic 0）。
* **断线检测能力**：采用常闭接线方式后，一旦外部急停线缆因机床拖链磨损拉断、光耦失效或信号插头脱落，系统能够无需任何额外传感器即自动识别为“急停状态”，实现物理层面的 Fail-Safe（故障安全）。

### 2. PL 侧硬件滤波与电磁消抖（EMC 优化）
* **干扰场景**：数控机床强电柜中变频器起停、主轴加减速及伺服快速运动会引发强烈的电磁辐射，直接影响高阻态 GPIO 信号。
* **滤波算法**：在 FPGA 内实现一个专用的**积分型去抖动滤波器**（如 10ms - 20ms 的积分滑动窗口滤波）。引脚电平必须在预设窗口时间内持续保持为低，状态机才确认为急停状态，从硬件上彻底滤除杂波，规避误动作。

### 3. 垂直轴电磁抱闸的时序控制（防止 Z 轴下坠）
* **重力危害**：垂直轴（如 Z 轴）受重力影响。如果急停瞬间同时切断伺服脉冲和使能，电磁抱闸线圈释放需要数十微秒以上的物理机械衔铁闭合时间。在这段延迟内，Z 轴将处于无控制力的“自由落体”状态，导致主轴头下坠撞伤工件或刀具。
* **微秒级时序规划**：
  * **T0 时刻**：PL 监测到急停触发，立即封锁非重力轴（X/Y 轴等）脉冲，同时**拉低垂直轴电磁抱闸输出信号（刹车线圈断电合闸）**。
  * **T0 + 延迟（数十微秒）**：PL 内部硬件定时器开始计数。等待数十微秒，确保抱闸刹车片物理贴紧并锁死丝杠。
  * **T1 时刻（定时器到期）**：封锁垂直轴（Z轴）的伺服使能与 PWM 脉冲输出，此时机械轴已被抱闸安全托住，确保绝对不发生下坠。

## 四、 软硬件同步与软件状态链路（PL -> PS）

急停的触发虽不经过 PS，但必须实时上报给 PS，以便在 HMI 界面上呈现并停止 G 代码解析任务。

```
[PL 物理急停触发]
       │
       ├──► 物理强制拉低 PWM/STO (无延迟硬保护)
       │
       └──► 触发 PL-to-PS 硬件中断 (IRQ)
                     │
                     ▼
             [Linux 内核驱动捕获]
                     │
                     ▼
             [Command Broker 守护进程]
                     │
                     ▼
             [写入 /dev/shm/v3_status_shm]
                     │
                     ▼
             [lvgl_app (UI界面变红，提示急停异常)]
```

1. **硬件中断机制**：PL 急停触发后，向 ARM 发送硬件中断信号（IRQ）。
2. **内核响应**：Linux 内核驱动响应中断，通知 LinuxCNC 运动内核进入 `ESTOP` 状态。
3. **状态广播**：状态写入双页面共享内存，UI 界面检测到数据改变后，立即红屏报警。

---

## 五、 安全互锁与复位解除链路（PS -> PL）

防止急停旋开后机床突然自行恢复运动，系统必须遵循**“物理旋开 + 软件复位”的双重安全确认逻辑**。

### 1. 硬件互锁逻辑
* 当 PL 急停引脚仍然是低电平时，PL 侧状态机处于 **物理强制锁死** 状态。
* 此时 PS（软件端）通过 AXI 总线发送的任何使能或复位寄存器写入命令，在 PL 硬件层面直接被丢弃忽略，软件无法强行越权恢复。

### 2. 安全复位时序
1. **物理释放**：操作员旋开物理急停按钮，PL 引脚恢复高电平（PL 状态机允许解除但保持锁定）。
2. **软件申请**：操作员在 HMI 界面上点击“复位”按钮，UI 通过 UDS 管道向 `Command Broker` 发送解除急停命令。
3. **总线写入**：`Command Broker` 确认各轴状态就绪，将解除复位信号写入 AXI 映射寄存器。
4. **硬件放行**：PL 接收到软件复位信号，确认物理引脚无异常，解除 PWM 和 STO 封锁，恢复机床正常运转。
